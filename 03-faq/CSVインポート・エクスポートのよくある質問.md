# よくある質問: CSVインポート・エクスポート

---

## Q. CSVインポートのテンプレートはどこで入手できますか？

A. 各カテゴリのインポート一覧画面にある「**テンプレート**」ボタンをクリックすると、Googleスプレッドシートのテンプレートが別タブで開きます。

**アクセス方法**: ホーム画面のクイックリンク「CSVインポート」→ インポートしたいカテゴリをクリック → 一覧画面右上の「テンプレート」ボタン

CSVインポートは2026-06-27時点で9グループ・22カテゴリあります。テンプレートの有無はカテゴリごとに異なります。現行UIでは `原価` は「商品」グループに表示されます。

| グループ | テンプレートあり | テンプレートなし |
|:--|:--|:--|
| 商品 | 商品 / 商品バリエーション / 商品画像 / 商品バリエーション画像 / カタログ | 原価 |
| 価格 | 販売価格 (セール) | 販売価格 (通常) |
| 販売 | — | 販売閾値 |
| 在庫 | 販売可能在庫 / 予約販売 | — |
| メタフィールド | 商品 / 商品バリエーション | — |
| 出荷 | ヤマトB2クラウド | DHL |
| ポイント | 一括加算 / 一括減算 | キャンペーン対象商品 |
| 会員ランク | — | 基準ランク |
| 各種マスター | 管理ユーザー / ロケーション | — |

テンプレートがないカテゴリ（販売価格(通常) / 原価 / 販売閾値 / DHL / キャンペーン対象商品 / 基準ランク）は、テンプレートボタン自体が表示されません。

ヤマトB2クラウド出荷実績CSVは、2026-06-28の検証用CSVで保存後にCSVインポート詳細が作成されるところまで確認済みです。ただし再確認時も `実行ステータス 未実行`、`検証成功 0件`、`検証失敗 0件` のままで、実行・出荷完了遷移は未確認です。

「在庫移動伝票」は在庫グループにあり、一覧画面は `/admin/csv_import/csv_import_operation_inventory_movement_orders`、新規インポート画面は `/admin/csv_import/csv_import_operation_inventory_movement_orders/create` です。空保存時は「ファイルを選択してください」と表示されます。

カタログ商品のテンプレートは `product_code,command` の2列です。`command` は `NEW` でカタログへ追加、`DELETE` でカタログから削除します（2026-06-28実機確認）。

## Q. 「絶対値で反映」と「差分値で反映」の違いは何ですか？

A. **販売可能在庫**のインポートフォームにある選択肢です。

画面内のインフォメーションには以下のように記載されています（原文）:
> 「現在の在庫数が10のとき、CSVで20を指定すると、絶対値で反映した場合の在庫数は20になります。差分値で反映した場合の在庫数は30になります。」

| 選択肢 | 動作 |
|:--|:--|
| 絶対値で反映する（デフォルト） | CSVに記載した数値をそのまま在庫数として上書きする |
| 差分値で反映する | CSVに記載した数値を現在の在庫数に加算して反映する |

インポートフォーム（`/admin/csv_import/csv_import_operation_inventory_logical_available_quantities/create`）でCSVファイルをアップロードする前に選択します。

## Q. インポートが失敗（検証失敗）したときはどうやって確認しますか？

A. インポート実行後に表示される**インポート詳細画面**で確認できます。

1. CSVインポートの一覧画面（例: `/admin/csv_import/csv_import_operation_products`）を開く
2. 実行した履歴行をクリックして詳細画面を開く
3. 「**検証ステータス**」と「**実行ステータス**」のバッジを確認する
4. 「**検証失敗**」の件数がリンクになっているので、クリックすると失敗した行の一覧（`/validation_failure`）を確認できる

詳細画面の主な項目は以下のとおりです。

| 項目 | 内容 |
|:--|:--|
| 作成日 | インポートを実行した日時 |
| 検証ステータス | CSV読み込み・バリデーションの結果（成功 完了 / 失敗） |
| 実行ステータス | 実際のデータ反映の結果（成功 完了 / 失敗） |
| 検証成功 | 検証が通った件数 |
| 検証失敗 | 検証が失敗した件数（クリックで失敗行の一覧へ） |

2026-06-27に販売可能在庫インポートの実行済み履歴で再確認したところ、完了後も「実行する」ボタンは画面上に残りますがdisabledで再実行できません。検証失敗が0件でも `0個の商品` リンクから見出し「検証失敗」の画面へ遷移できます。2026-06-28には商品CSV1件でも、検証成功、不可逆確認ダイアログ、実行ステータス成功完了、商品一覧/詳細反映まで確認しました。

同じく2026-06-28に、カタログ商品CSVの `NEW` / `DELETE` を実行確認しました。どちらも `検証成功 1件の商品`、確認ダイアログ `1件のカタログ商品を登録します。この操作は巻き戻すことができません。`、実行ステータス `成功 完了` になり、`NEW` 後はカタログ詳細に商品が表示、`DELETE` 後は消えることを確認しました。検証用の商品・カタログは削除済みです。

## Q. ポイントの「一括加算」と「一括減算」は何が違いますか？

A. それぞれ別のCSVインポートカテゴリです。

| カテゴリ | 説明（UIの原文） |
|:--|:--|
| 一括加算 | 顧客にポイントを一括で加算することができます。 |
| 一括減算 | 顧客のポイントを一括で減算することができます。 |

一括加算では「テナント」と「顧客IDの種別」（SQが発番した顧客ID / 外部システムが発番した顧客ID）を選択したうえでCSVファイルをアップロードします。

一括減算では「テナント」とCSVファイルのみを指定します。一括加算にある「顧客IDの種別」は表示されません。空保存時は「テナントを選択してください」、テナントだけ選択してファイル未選択の場合は「ファイルを選択してください」と表示されます。

CSVインポートトップ（ホーム > クイックリンク「CSVインポート」）の「ポイント」グループに「一括加算」「一括減算」「キャンペーン対象商品」の3項目として並んでいます。顧客管理一覧の「インポート」ドロップダウン内では「ポイント一括付与」という別表記になる場合がありますが、リンク先は同じ `csv_import_operation_point_pluses` です。

## Q. CSVエクスポートしたファイルはどこで受け取れますか？

A. カテゴリによって受け取り方が異なります。

ヤマトB2クラウドのCSVエクスポートフォーム（出荷管理 > 「条件指定でエクスポート」 > 「ヤマトB2クラウド」）の画面には、以下の通知文が表示されています（原文）:
> 「エクスポート完了後、メールにて以下のダウンロードリンクをお知らせいたします」

2026-06-28再確認では、ヤマトB2クラウドCSVエクスポート履歴画面（`/admin/csv_export/csv_export_operation_inventory_outbound_order_yamato_b2_clouds`）は空状態で、ダウンロードリンクは表示されませんでした。そのため、ヤマトB2については画面文言どおりメールリンクで届く想定ですが、メール本文・リンク有効期限・同梱PDF実ファイルは接続環境で検証待ちです。

一方、売上実績（注文軸）のCSVエクスポートは、実行履歴一覧に「ダウンロード」リンクが表示されることを確認済みです。実行直後は「処理中」になり、完了後に「成功 完了」とダウンロードリンクが表示されます。

エクスポートは非同期処理で行われるため、「実行する」「エクスポートを開始する」を押した直後にはダウンロードできない場合があります。カテゴリによって、履歴一覧にリンクが出るものと、メール通知文言だけが表示されるものがあります。

なお、エクスポートの実行履歴は各カテゴリの一覧画面からも確認できます（例: `/admin/csv_export/csv_export_operation_inventory_logical_quantities`）。

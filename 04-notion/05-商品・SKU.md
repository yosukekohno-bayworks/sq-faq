# 05. 商品・SKU

> このページはWBS-25エリアの第5エリアです。商品・商品コード・商品タイプ・製造元・バリエーション・SKUの関係を理解し、商品を登録できるようになるのが目標です。

## このエリアで学べること

- 「商品」「バリエーション」「SKU」の違いを説明できる
- なぜ在庫や価格がSKU（バリエーション）単位で管理されるのか理解できる
- 商品作成フォーム・バリエーション作成フォームの各項目の意味が分かる
- 商品コード・商品タイプ・製造元・ブランドといった分類情報の役割を把握できる

---

## 機能概要

SQでは「商品」は **商品名・説明文・画像をまとめた管理単位** です。サイズやカラーの違いは「バリエーション」として1つの商品に複数ぶら下がり、在庫数や価格はこのバリエーション（SKU）ごとに持ちます。

商品管理は左メニューの「商品管理」から操作します。

| 画面 | URLパス | 役割 |
|:--|:--|:--|
| 商品一覧 | `/admin/products` | 商品の一覧・検索・タブ切替・作成入口 |
| 商品作成フォーム | `/admin/products/create` | 商品の基本情報・オプション軸・ステータスを登録 |
| 商品詳細・編集 | `/admin/products/{id}` | 商品の編集・ステータス切替・アーカイブ・削除 |
| バリエーション作成 | `/admin/products/{id}/variants/create` | 各バリエーションの価格・SKU・在庫・配送を登録 |
| バリエーション編集 | `/admin/products/{id}/variants/{variant_id}` | バリエーション単位の編集 |

> 商品一覧は左メニューの「商品管理」から開けます。商品には `/admin/products/{id}/edit` という直下の編集ルートは存在せず、編集は詳細画面（`/admin/products/{id}`）で行います。

### 何ができるか

- 商品の一覧表示・ステータス別の絞り込み（タブ切替）
- 商品コードによる検索
- 商品の新規作成・編集・アーカイブ・削除
- バリエーション（サイズ・カラーなど）の追加・編集
- CSVによる商品データの一括インポート

## 商品とSKUの関係（最重要）

EC運営で一番つまずきやすいのは「商品・バリエーション・SKU」の違いです。

| 用語 | 意味 | 例 |
|:--|:--|:--|
| 商品 | 商品名・説明文・画像をまとめた管理単位 | 「エアリズムコットンクルーネックTシャツ」 |
| バリエーション | 商品の色・サイズなどのオプションを組み合わせた1パターン | 「白 / Mサイズ」 |
| SKU（最小管理単位） | バリエーションに紐づく在庫管理コード | `486102-70-S` |

#### 3層の階層図

```
商品: 半袖Tシャツ  …商品名・説明文・画像・分類(タイプ/製造元/ブランド)
├─ バリエーション: 白 / S  →  SKU: 486102-70-S  …在庫数・上代・仕入価格・バーコード・重量
├─ バリエーション: 白 / M  →  SKU: 486102-70-M  …在庫数・上代・仕入価格・バーコード・重量
└─ バリエーション: 黒 / L  →  SKU: 486102-01-L  …在庫数・上代・仕入価格・バーコード・重量
```

商品名・説明文・画像は **商品** に、在庫数・価格・バーコード・重量は最下層の **SKU** にだけ紐付きます。中間のバリエーションは「どの軸の組み合わせか」を表す分類で、実データはその下のSKUが持ちます。

### なぜ在庫や価格はSKU単位なのか

同じ「半袖Tシャツ」でも、SサイズとXLサイズでは倉庫の在庫数も仕入価格も違います。そのため **在庫数・価格・バーコード・重量は商品ではなくバリエーション（SKU）ごとに設定** します。商品単位で持てるのは「商品名・説明文・画像・分類情報」だけです。

> 商品一覧の「在庫」列に表示される数字は **その商品のバリエーション数** です。例: 「20個のバリエーション」= 20種類のSKUが登録済み。実際の在庫数量ではありません。

## 画面・項目の説明

### 商品一覧（/admin/products）

![商品一覧画面。上部に「すべて」「公開中」「下書き」「アーカイブ済み」のタブ、右上に「インポート」「商品を作成する」ボタン](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/02-products-list.png)

#### タブ

| タブ名 | 表示対象 |
|:--|:--|
| すべて | ステータスを問わず全商品 |
| 公開中 | ステータスが公開中の商品 |
| 下書き | ステータスが下書きの商品 |
| アーカイブ済み | アーカイブ済みの商品 |

#### 一覧の列

| 列名 | 内容 |
|:--|:--|
| 商品 | サムネイル画像と商品名 |
| ステータス | 現在のステータス（例: 下書き、公開中） |
| 商品コード | 商品を一意に識別するコード（例: 486125） |
| 在庫 | 登録されているバリエーションの数（例: 20個のバリエーション） |
| カタログ | この商品が属するカタログの数 |

> **カタログとは** — 商品をグループ化し、Shopify等のチャネルごとに「どの商品を連携するか」を決める出品範囲の単位（詳細は **06. カタログ・出品範囲**）。1つの商品が複数のカタログに属けばここの数が増えます。
| 商品タイプ | 商品の種別（例: スウェット、Tシャツ） |
| 製造元 | 製造元名（例: UNIQLO） |

#### 検索

「検索と絞り込みの結果」ボタンをクリックすると「商品コードで検索する」テキストボックスが展開されます。**検索できるのは商品コードのみ**。商品名・ステータス・商品タイプ・タグでの絞り込みUIはありません。

#### インポート

「インポート」ボタンのドロップダウンには以下の5項目があります。

| 項目ラベル | 用途 |
|:--|:--|
| 商品 | 商品データをCSVでインポート |
| 商品画像 | 商品画像の割り当てをCSVでインポート |
| 商品バリエーション | バリエーションデータをCSVでインポート |
| 商品メタフィールド | 商品のメタフィールドをCSVでインポート |
| 商品バリエーションメタフィールド | バリエーションのメタフィールドをCSVでインポート |

> **メタフィールドとは** — 商品やバリエーションに、標準項目とは別に自由に追加できる独自データ項目（例: 素材、保証期間など）。定義済みのメタフィールドに対し値をCSVで一括登録します（詳細は **08. カスタムデータ**）。

#### インポートの流れ（最小手順）

1. 商品一覧右上の「インポート」ボタンをクリックする。
2. ドロップダウンから対象データ種別（商品 / 商品画像 / 商品バリエーション / 商品メタフィールド / 商品バリエーションメタフィールド）を選ぶ。
3. CSVファイルをアップロードして取り込みを実行する。

> 手順の詳細（テンプレート取得方法・必須列・実行後の結果）は **21. CSV・PDF・データ移行** を参照してください。なお商品CSVインポートの実行は未確認です（巻き戻し不可のため検証範囲外）。

### 商品作成フォーム（/admin/products/create）

![商品作成フォーム。基本情報・メディア・バリエーション・検索エンジンリスティングの各セクションと、右サイドバーのステータス・商品分類・タグ](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/03-product-create-form.png)

#### 基本情報

| 項目（UIラベル） | 必須 | 制約・説明 |
|:--|:--|:--|
| 商品コード* | 必須 | 半角英数字・ハイフン・アンダースコアのみ（例: 7835152003XL180）。**作成後に変更不可** |
| 商品名* | 必須 | テキスト（例: 半袖Tシャツ） |
| 説明文 | 任意 | テキストエリア、5,000字上限 |

#### メディア

- 「画像をアップロード」エリア（クリックまたはドラッグ&ドロップでファイル選択）
- 「追加」ボタン（クリックでファイル選択ダイアログを開く）

どちらもローカルファイルを選択してアップロードします。URLからの画像追加はできません。

#### バリエーション設定（オプション軸）

| 項目（UIラベル） | 必須 | 制約・説明 |
|:--|:--|:--|
| オプション名* | 必須 | 軸の名称（例: サイズ） |
| 種別* | 必須 | セレクト: 「サイズ」「カラー」「その他」の3択。**作成時のみ設定可、作成後変更不可** |
| オプション値* | 必須 | 値（例: S）。1行目が初期表示される |
| コード | 任意 | 値に対応するコード（例: 001） |

- 「追加」ボタンでオプション値の行を追加
- 「削除」ボタンで行を削除（1行のみの場合はボタンが無効）
- 「別のオプションを追加する」ボタンでサイズ・カラーなど別の軸を追加

#### 検索エンジンリスティング

| 項目（UIラベル） | 必須 | 説明 |
|:--|:--|:--|
| ページタイトル | 任意 | 検索結果に表示されるタイトル |
| メタディスクリプション | 任意 | 検索結果に表示される説明文（5,000字上限） |

#### サイドバー: ステータス

作成時のステータスは以下の3択（デフォルト: 下書き）。

| 選択肢 | 作成後の状態 |
|:--|:--|
| 下書き | 下書きとして作成 |
| 公開 | 公開中として作成 |
| 非公開 | **アーカイブ済み**として作成（後述の注意参照） |

#### サイドバー: 商品分類

| 項目（UIラベル） | 必須 | 説明 |
|:--|:--|:--|
| 商品タイプ | 任意 | テキスト（例: Tシャツ）。**分類表示用**のメモ扱いで、商品一覧の絞り込みや検索、カタログの自動追加ルールには使われない。運用上の分類用途は限定的 |
| 製造元 | 任意 | テキスト（例: ユニクロ）。**カタログの自動追加ルール「製造元」の完全一致判定**に使われる。大文字小文字・スペースの違いも別物扱いになるため表記を統一すること |
| ブランド | 任意 | 「選択」ボタンでブランドマスターから選択。ブランドに紐づく**ブランドコードが販売閾値ルールの自動追加判定**に使われるため、閾値運用するならマスター登録漏れに注意 |

- 商品タイプは「何に使うか」: 一覧上の列表示や参考情報としての役割のみ。検索・絞り込みの対象にはならない（検索は商品コードのみ）。
- 製造元は「設定するとどうなるか」: 製造元名が一致する商品をカタログへ自動追加するルールの判定材料になる。表記ゆれがあると同じ製造元でも別物として判定される。
- ブランドは「いつ選ぶか」: ブランド単位での販売閾値（自動追加）運用を行う場合に設定する。未選択でも商品作成は可能。

ブランドは事前に「設定 > ブランド」（`/admin/settings/brands`）でマスター登録が必要です。

#### サイドバー: タグ

タグを追加できます。未設定時は「タグは付与されていません」と表示されます。

> タグは**補助的なメモ用**です。商品一覧の絞り込み・検索には使えず（検索は商品コードのみ）、運用上の分類用途は限定的です。

| 項目（UIラベル） | 必須 | 説明 |
|:--|:--|:--|
| 上代* | 必須 | 販売価格（円） |
| 仕入価格 | 任意 | 仕入れ価格（円） |

| 項目（UIラベル） | 必須 | 説明 |

| 項目（UIラベル） | 必須 | 説明 |

### バリエーション編集画面（/admin/products/{id}/variants/{variant_id}）

作成フォームとほぼ同じ構成ですが、以下の点が異なります。

| 差異 | 内容 |
|:--|:--|
| 「在庫管理」リンクあり | 該当バリエーションの在庫管理画面へのリンクが上部に表示 |
| 「仕入価格」なし | 作成フォームにある「仕入価格」欄は表示されない |
| 「原価」セクションあり | 「原価を登録する」ボタンと「原価が設定されていません」が表示 |

> ⚠️ 作成フォームの「仕入価格」と、この「原価」セクションの引継ぎ関係は **実機未確認** です（TODO参照）。
| UPCフィールドあり | 販売セクションにUPC欄が追加 |
| 送信ボタン | ラベルが「更新する」になる |

## 主な操作手順

### 商品を新規作成する（2段階）

商品の作成は **第1部（商品の基本情報）→ 第2部（バリエーション詳細）** の2段階です。

#### 第1部: 商品の基本情報を登録

1. 左メニューの「商品管理」をクリックし、商品一覧画面を開く。
2. 右上の「商品を作成する」ボタンをクリックする。商品作成フォームへ遷移する。
3. 「商品コード*」に半角英数字・ハイフン・アンダースコアでコードを入力する（作成後変更不可）。
4. 「商品名*」に商品名を入力する。
5. 必要に応じて「説明文」欄に商品説明を入力する（5,000字上限）。
6. バリエーションを作る場合、「オプション名*」を入力し、「種別*」で「サイズ / カラー / その他」を選び、「オプション値*」に値を入力する。「追加」で値を行追加できる。
7. 2軸目（サイズ＋カラーなど）が必要な場合は「別のオプションを追加する」をクリックし、同じように入力する。
8. 右サイドバーの「ステータス」で「下書き / 公開 / 非公開」を選ぶ（通常は「下書き」か「公開」）。
9. 必要に応じて「商品タイプ」「製造元」「ブランド」「タグ」を入力する。
10. 「保存する」ボタンをクリックする。商品詳細画面へ遷移する。

#### 第2部: バリエーションの詳細を登録

1. 商品詳細画面の「バリエーション」セクションで「バリエーションを追加する」リンクをクリックする。バリエーション作成フォームへ遷移する。
2. 「オプション」でこのバリエーションに対応するカラー・サイズ等の値を選ぶ。
3. 「上代*」に販売価格を、「SKU (最小管理単位)*」にSKUコードを入力する（どちらも必須）。
4. 必要に応じて仕入価格・バーコード・JAN・重量・関税情報等を入力する。
5. 「作成する」ボタンをクリックする。
6. 残りのバリエーションについても同じ手順を繰り返す。

### 商品をアーカイブ・削除する

| 操作 | 場所 | 取り消し |
|:--|:--|:--|
| 商品をアーカイブする | 商品詳細の「その他の操作」 | 可能（「商品のアーカイブを解除する」で復元。復元後のステータスは「下書き」） |
| 商品を削除する | 商品詳細の「その他の操作」 | **不可（巻き戻せません）** |

> 迷う場合は「アーカイブ」を使ってください。削除確認ダイアログには「この処理は巻き戻すことができません。」と表示されます。

## 注意点・制約

実機確認済みの制約です（根拠: `constraints.md` 該当項目）。

| 制約 | 内容 |
|:--|:--|
| 商品コードは作成後に変更不可 | 半角英数字・ハイフン・アンダースコアのみ使用可。全角・スペース・その他記号は入力できない |
| オプションの「種別」は作成時のみ設定可 | サイズ / カラー / その他 は作成時にのみ設定でき、作成後は変更できない（constraints.md 該当項目） |
| 「非公開」と「アーカイブ済み」は同じ状態 | 商品作成時に「非公開」を選ぶと「アーカイブ済み」として作成される。画面によってラベルが異なる（constraints.md 該当項目） |
| 商品削除は復元不可 | 削除確認ダイアログに「この処理は巻き戻すことができません。」と表示。削除済みURLは「該当するProductが見つかりませんでした。」になる（constraints.md 該当項目） |
| アーカイブ解除後のステータスは常に「下書き」 | 公開中の商品をアーカイブ→解除しても「公開中」には戻らない |
| 検索は商品コードのみ対応 | 商品名・ステータス・商品タイプ・タグでの絞り込みUIはない |
| 商品一覧の「在庫」列はバリエーション数 | 実在庫数ではない。実数は各バリエーション詳細の在庫管理画面で確認 |

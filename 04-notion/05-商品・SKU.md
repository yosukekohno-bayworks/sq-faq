# 05. 商品・SKU

> このページは標準24分類の第5分類です。商品・商品コード・商品タイプ・製造元・バリエーション・SKUの関係を理解し、商品を登録できるようになるのが目標です。

## このエリアで学べること

- 「商品」「バリエーション」「SKU」の違いを説明できる
- なぜ在庫や価格がSKU（バリエーション）単位で管理されるのか理解できる
- 商品作成フォーム・バリエーション作成フォームの各項目の意味が分かる
- 商品コード・商品タイプ・製造元・ブランドといった分類情報の役割を把握できる

---

## 機能概要

SQでは「商品」は **商品名・説明文・画像をまとめた管理単位** です。サイズやカラーの違いは「バリエーション」として1つの商品に複数ぶら下がり、在庫数や価格はこのバリエーション（SKU）ごとに持ちます。

商品管理は左メニューの「商品管理」から操作します。

| 画面 | URLパス | 役割 |
|:--|:--|:--|
| 商品一覧 | `/admin/products` | 商品の一覧・検索・タブ切替・作成入口 |
| 商品作成フォーム | `/admin/products/create` | 商品の基本情報・オプション軸・ステータスを登録 |
| 商品詳細・編集 | `/admin/products/{id}` | 商品の編集・ステータス切替・アーカイブ・削除 |
| バリエーション作成 | `/admin/products/{id}/variants/create` | 各バリエーションの価格・SKU・在庫・配送を登録 |
| バリエーション編集 | `/admin/products/{id}/variants/{variant_id}` | バリエーション単位の編集 |

> 商品一覧は左メニューの「商品管理」から開けます。商品には `/admin/products/{id}/edit` という直下の編集ルートは存在せず、編集は詳細画面（`/admin/products/{id}`）で行います。

### 何ができるか

- 商品の一覧表示・ステータス別の絞り込み（タブ切替）
- 商品コードによる検索
- 商品の新規作成・編集・アーカイブ・削除
- バリエーション（サイズ・カラーなど）の追加・編集
- CSVによる商品データの一括インポート

## 商品とSKUの関係（最重要）

EC運営で一番つまずきやすいのは「商品・バリエーション・SKU」の違いです。

| 用語 | 意味 | 例 |
|:--|:--|:--|
| 商品 | 商品名・説明文・画像をまとめた管理単位 | 「エアリズムコットンクルーネックTシャツ」 |
| バリエーション | 商品の色・サイズなどのオプションを組み合わせた1パターン | 「白 / Mサイズ」 |
| SKU（最小管理単位） | バリエーションに紐づく在庫管理コード | `486102-70-S` |

開発元説明では、`カラー` や `サイズ` のような軸を **商品オプション**、`ネイビー` や `M` のような値を **商品オプション値** と呼びます。バリエーション/SKUは、商品オプション値の掛け合わせで作られます。たとえば「カラー4色 × サイズ5サイズ」なら最大20種類のバリエーションを作れる、という考え方です。

#### 3層の階層図

```
商品: 半袖Tシャツ  …商品名・説明文・画像・分類(タイプ/製造元/ブランド)
├─ バリエーション: 白 / S  →  SKU: 486102-70-S  …在庫数・上代・原価・バーコード・重量
├─ バリエーション: 白 / M  →  SKU: 486102-70-M  …在庫数・上代・原価・バーコード・重量
└─ バリエーション: 黒 / L  →  SKU: 486102-01-L  …在庫数・上代・原価・バーコード・重量
```

商品名・説明文・画像は **商品** に、在庫数・価格・バーコード・重量は最下層の **SKU** にだけ紐付きます。中間のバリエーションは「どの軸の組み合わせか」を表す分類で、実データはその下のSKUが持ちます。

> **ギフト/ラッピングをSKUにしない判断**: 開発元説明では、バリエーションを作ることはSKUが変わることと同義です。ギフトラッピングの有無で在庫管理単位が変わらないなら、カラー/サイズのような商品オプション軸にしない方が自然です。無料の補足情報ならメタフィールドや注文側の属性、有料にするならラッピング用の商品を別途作る、という考え方になります。

### なぜ在庫や価格はSKU単位なのか

同じ「半袖Tシャツ」でも、SサイズとXLサイズでは倉庫の在庫数も原価も違います。そのため **在庫数・価格・バーコード・重量は商品ではなくバリエーション（SKU）ごとに設定** します。商品単位で持てるのは「商品名・説明文・画像・分類情報」だけです。

> 商品一覧の「在庫」列に表示される数字は **その商品のバリエーション数** です。例: 「20個のバリエーション」= 20種類のSKUが登録済み。実際の在庫数量ではありません。

#### 検索

検索パネルは初期表示では出ず、`F` キーまたは検索アイコン操作で開きます。展開後は「商品コードで検索する」テキストボックスが表示されます。**検索できるのは商品コードのみ**。2026-06-28再確認では `486125` で2件、`TEST_E2E` で6件に絞り込まれ、`UNIQLO` や `486125 UNIQLO` は0件でした。商品名・ステータス・商品タイプ・タグでの絞り込みUIはありません。

#### インポート

「インポート」ボタンのドロップダウンには以下の5項目があります。

| 項目ラベル | 用途 |
|:--|:--|
| 商品 | 商品データをCSVでインポート |
| 商品画像 | 商品画像の割り当てをCSVでインポート |
| 商品バリエーション | バリエーションデータをCSVでインポート |
| 商品メタフィールド | 商品のメタフィールドをCSVでインポート |
| 商品バリエーションメタフィールド | バリエーションのメタフィールドをCSVでインポート |

> **メタフィールドとは** — 商品やバリエーションに、標準項目とは別に自由に追加できる独自データ項目（例: 素材、保証期間など）。定義済みのメタフィールドに対し値をCSVで一括登録します（詳細は **08. カスタムデータ**）。

#### インポートの流れ（最小手順）

1. 商品一覧右上の「インポート」ボタンをクリックする。
2. ドロップダウンから対象データ種別（商品 / 商品画像 / 商品バリエーション / 商品メタフィールド / 商品バリエーションメタフィールド）を選ぶ。
3. CSVファイルをアップロードして取り込みを実行する。

> 手順の詳細（テンプレート取得方法・必須列・実行後の結果）は **22. CSV・PDF・データ移行** を参照してください。2026-06-28に検証用商品1件で商品CSVインポートを実行し、検証成功→実行確認ダイアログ→実行ステータス成功完了→商品一覧/詳細反映→商品削除まで確認済みです。

| 項目（UIラベル） | 必須 | 説明 |
|:--|:--|:--|

## 主な操作手順

### 商品を新規作成する（2段階）

商品の作成は **第1部（商品の基本情報）→ 第2部（バリエーション詳細）** の2段階です。

#### 第1部: 商品の基本情報を登録

1. 左メニューの「商品管理」をクリックし、商品一覧画面を開く。
2. 右上の「商品を作成する」ボタンをクリックする。商品作成フォームへ遷移する。
3. 「商品コード*」に半角英数字・ハイフン・アンダースコアでコードを入力する（作成後変更不可）。
4. 「商品名*」に商品名を入力する。
5. 必要に応じて「説明文」欄に商品説明を入力する（5,000字上限）。
6. バリエーションを作る場合、「オプション名*」を入力し、「種別*」で「サイズ / カラー / その他」を選び、「オプション値*」に値を入力する。「追加」で値を行追加できる。
7. 2軸目（サイズ＋カラーなど）が必要な場合は「別のオプションを追加する」をクリックし、同じように入力する。
8. 右サイドバーの「ステータス」で「下書き / 公開 / 非公開」を選ぶ（通常は「下書き」か「公開」）。
9. 必要に応じて「商品タイプ」「製造元」「ブランド」「タグ」を入力する。
10. 「保存する」ボタンをクリックする。商品詳細画面へ遷移する。

#### 第2部: バリエーションの詳細を登録

1. 商品詳細画面の「バリエーション」セクションで「バリエーションを追加する」リンクをクリックする。バリエーション作成フォームへ遷移する。
2. 「オプション」でこのバリエーションに対応するカラー・サイズ等の値を選ぶ。
3. 「上代*」に販売価格を、「SKU (最小管理単位)*」にSKUコードを入力する（どちらも必須）。
4. 必要に応じてバーコード・JAN・重量・関税情報等を入力する。原価は作成後、バリエーション編集画面で登録する。
5. 「作成する」ボタンをクリックする。
6. 残りのバリエーションについても同じ手順を繰り返す。

### 商品をアーカイブ・削除する

| 操作 | 場所 | 取り消し |
|:--|:--|:--|
| 商品をアーカイブする | 商品詳細の「その他の操作」 | 可能（「商品のアーカイブを解除する」で復元。復元後のステータスは「下書き」） |
| 商品を削除する | 商品詳細の「その他の操作」 | **不可（巻き戻せません）** |

> 迷う場合は「アーカイブ」を使ってください。削除確認ダイアログには「この処理は巻き戻すことができません。」と表示されます。

### ステータス変更後の移動先

2026-06-27にテスト商品で、ステータスを動かした後の詳細表示・一覧タブ移動まで確認しました。

| 操作 | 詳細画面の変化 | 一覧での移動 |
|:--|:--|:--|
| 作成時に「非公開」を選択 | 作成後は `アーカイブ済み`。右側ステータス欄に `管理画面では、商品リスト以外は非表示です。` と表示 | `すべて` と `アーカイブ済み` タブに表示。`下書き` タブには出ない |
| `商品のアーカイブを解除する` | ステータスが `下書き` に変わり、非表示説明文は消える | `下書き` タブに表示。`アーカイブ済み` タブから消える |
| `商品をアーカイブする` | ステータスが `アーカイブ済み` に戻り、非表示説明文が再表示される | `下書き` タブから消え、`アーカイブ済み` タブに表示 |
| `商品を削除する` | 削除後は商品一覧へ戻る。削除済み商品URLは `該当するProductが見つかりませんでした。`、削除済みバリエーションURLは `該当するProductVariantが見つかりませんでした。` | 商品コード/SKUで検索しても一覧に出ない |

## 注意点・制約

実機確認済みの制約です（根拠: `constraints.md` 該当項目）。

| 制約 | 内容 |
|:--|:--|
| 商品コードは作成後に変更不可 | 半角英数字・ハイフン・アンダースコアのみ使用可。全角・スペース・その他記号は入力できない |
| オプションの「種別」は作成時のみ設定可 | サイズ / カラー / その他 は作成時にのみ設定でき、作成後は変更できない（constraints.md 該当項目） |
| 「非公開」と「アーカイブ済み」は同じ状態 | 商品作成時に「非公開」を選ぶと「アーカイブ済み」として作成される。画面によってラベルが異なる（constraints.md 該当項目） |
| 商品削除は復元不可 | 削除確認ダイアログに「この処理は巻き戻すことができません。」と表示。削除後は一覧検索から消え、削除済み商品URLは「該当するProductが見つかりませんでした。」、削除済みバリエーションURLは「該当するProductVariantが見つかりませんでした。」になる（constraints.md 該当項目） |
| アーカイブ解除後のステータスは常に「下書き」 | 公開中の商品をアーカイブ→解除しても「公開中」には戻らない |
| 検索は商品コードのみ対応 | 商品名・ステータス・商品タイプ・タグでの絞り込みUIはない |
| 商品一覧の「在庫」列はバリエーション数 | 実在庫数ではない。実数は各バリエーション詳細の在庫管理画面で確認 |
| 商品の直下に `/edit` ルートは存在しない | `/admin/products/{id}/edit` は「このページは存在しないようです」。編集は詳細画面で行う |

> **既存商品へのオプション軸追加**: 開発元説明では、新しい軸を後から追加すると既存バリエーション全体にその軸を持たせる必要があるため、既存商品では許可しない可能性があります。現行UIでも、既存商品詳細で追加できるのは既存軸の値であり、「別のオプションを追加する」は確認できていません。

## 残課題（対象外・データ依存を含む）

完成寄りエリアのため残件は少なめですが、以下は外部連携・巻き戻し不可操作のため残件です。

- [x] **既存商品への新規オプション軸追加**: 2026-06-28実機確認。既存商品詳細では既存軸の `オプション値を追加` は可能だが、作成フォームの「別のオプションを追加する」は表示されず、新しい軸を後から追加する管理画面導線は確認できない
- [ ] **商品の外部チャネル連携時の挙動**: Shopifyは接続済みストアで未連携商品5件まで確認済み。ただし商品一括同期はアクセストークン未設定により `ProductSet 用の認証情報が設定されていません` で停止。実際にShopifyへ商品が作成・更新されるかは認証情報設定後に再検証が必要
- [x] **CSVインポート後の実データ反映**: 2026-06-28実機確認。検証用商品1件のCSVで `検証成功 1個の商品`、確認ダイアログ `1件の商品を登録します。この操作は巻き戻すことができません。`、実行ステータス `成功 完了`、商品一覧/詳細反映、商品削除後の `該当するProductが見つかりませんでした。` まで確認

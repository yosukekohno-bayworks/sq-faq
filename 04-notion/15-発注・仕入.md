# 15. 発注・仕入

> このページは標準24分類の第15分類です。取引先（仕入先）への発注伝票を作成し、ステータスを下書き→発注済み→キャンセル済みと進める手順を理解するのが目標です。2026-06-27の実機確認では、発注済み後に「入荷指示を作成する」導線が表示され、入荷先ロケーションを指定して発注起点の入荷指示を作成できました。発注するだけでは入荷指示や入荷予定は自動作成されず、入荷指示を作成・受領して初めて在庫に反映されます。2026-06-28には、発注起点の入荷指示を作成後、入荷完了前なら発注伝票をキャンセルでき、紐づく入荷指示が `入荷キャンセル` になることを確認しました。

## このエリアで学べること

- 取引先（仕入先）マスタを登録・編集する方法が分かる
- 発注伝票を作成し、商品・数量・単価・税率・通貨を入力できる
- 発注伝票のステータスが「下書き → 発注済み → キャンセル済み」へ遷移することを説明できる
- 発注済み後に入荷指示を手動作成し、入荷予定・入荷完了・在庫反映まで追える
- 発注済み後は伝票の編集ができず、入荷完了前のキャンセルと入荷完了後のキャンセル不可の違いを理解する

---

## 機能概要

このエリアは「オペレーション」グループに属する機能と「設定」グループの取引先マスタから成り立ちます。

| 機能 | 画面URL | 役割 |
|:--|:--|:--|
| 発注管理 | `/admin/inventory_purchase_orders` | 取引先への発注伝票の一覧表示・作成 |
| 発注伝票 作成 | `/admin/inventory_purchase_orders/create` | 取引先・テナント・通貨・商品を指定して発注伝票を下書き保存 |
| 発注起点の入荷指示作成 | `/admin/inventory_purchase_orders/<id>/create_inventory_inbound_order` | 発注済み伝票から入荷先ロケーションを指定し、入荷指示を作成 |
| 取引先マスタ | `/admin/settings/suppliers` | 発注伝票の「取引先」に表示される仕入先の一覧・登録 |

### できること

- **取引先マスタ**: 取引先名・取引先コードを登録し、発注伝票ですぐに選択できる
- **発注伝票の作成**: 取引先・テナント・通貨・商品（バリエーション）・単価・数量・税率を入力して下書き保存
- **ステータス遷移**: 下書き → 発注済み → キャンセル済み まで進められる
- **入荷指示作成**: 発注済み後に入荷先ロケーションを選び、発注数量を引き継いだ入荷指示を作成できる
- **国際仕入れ対応**: 発注伝票ごとに通貨（日本円 / 米ドル / ユーロ / タイ バーツ / シンガポール ドル）を選択可能

### 実機確認範囲のステータス遷移

```
作成する
  ↓
下書き  ──発注する──▶  発注済み  ──キャンセルする──▶  キャンセル済み
                              │
                              ├─入荷指示を作成する──▶ 入荷待ち ──入荷実績登録──▶ 入荷完了
                              └─入荷待ちのまま発注キャンセル──▶ 入荷キャンセル
```

> **重要**: 2026-06-27の発注確認ダイアログは「発注後に入荷指示を作成できます。」という文言です。発注実行だけでは入荷管理一覧に行は増えず、SKU在庫詳細の入荷予定も増えません。発注済み後に表示される「入荷指示を作成する」リンクで入荷先ロケーションを選び、入荷指示を保存すると、入荷管理に発注起点の入荷指示が作成され、対象ロケーションの「入荷予定」が増えます。

## 主な操作手順

### 手順1: 取引先を登録する

1. 左メニューの「設定」を開き、「取引先」をクリックする。取引先一覧（`/admin/settings/suppliers`）が開く。
2. 右上の「取引先を作成」ボタンをクリックする。作成フォーム（`/create`）へ遷移する。
3. 「取引先名」に仕入先の名前を入力する。
4. 「取引先コード」に識別コードを入力する。
5. 「保存する」ボタンをクリックする。
6. 「取引先を作成しました」というメッセージが表示され、取引先一覧に戻ることを確認する。

### 手順2: 発注伝票を作成する（下書き保存まで）

1. 左メニューの「発注管理」をクリックし、一覧画面（`/admin/inventory_purchase_orders`）を開く。
2. 右上の「発注伝票を作成する」ボタンをクリックする。作成フォーム（`/create`）へ遷移する。
3. 「取引先」コンボボックスから、手順1で登録した取引先を選ぶ。
4. 「テナント」コンボボックスから、発注を紐付けるテナントを選ぶ。
5. 必要に応じて「通貨」を選ぶ（デフォルト: 日本円）。
6. 「商品を追加する」のテキスト検索、または「参照」ボタンからバリエーションを選択する。
7. 明細行の「単価」「数量」「税率」を入力する。
8. フォーム下部の「作成する」ボタンをクリックする。
9. 発注伝票の詳細画面に遷移し、ステータスが「下書き」になっていることを確認する。

> **バリエーション選択ダイアログの絞り込み**: 「絞り込みを追加」で選べるのは「商品コード」のみです。SKUコード検索欄に直接入力して探すこともできます。

### 手順3: 発注伝票を発注済みにする

1. 発注伝票の詳細画面（ステータス: 下書き）を開く。
2. 商品・数量・単価・合計金額が正しいことを確認する。
3. 「発注する」ボタンをクリックする。確認ダイアログが開く。
4. 「発注後は伝票を編集できません。発注後に入荷指示を作成できます。」の内容を確認する。
5. 実行ボタンをクリックする。
6. ステータスが「発注済み」に変わり、発注者・発注日時が表示されることを確認する。
7. 商品セクションの見出しが「発注済み商品」に変わることを確認する。
8. 「入荷指示を作成する」リンクが表示されることを確認する。

> 発注実行だけでは入荷管理一覧に発注起点の入荷指示は増えません。2026-06-27の `#IP-1007` では、発注後に対象SKU `TEST_E2E_20260622_GU_1905_NAVY_M` の入荷予定は0のままでした。

## 注意点・制約

### 発注だけでは入荷予定は立たない

発注確認ダイアログの現行文言は「発注後に入荷指示を作成できます。」です。発注済みにしただけでは、入荷管理一覧に発注起点の入荷指示は作成されず、SKU在庫詳細の「入荷予定」も増えません。

`#IP-1007` では、発注直後の入荷管理一覧に新規行は増えず、対象SKU `TEST_E2E_20260622_GU_1905_NAVY_M` の `TEST_E2E_20260622_GU倉庫_ON_1905` は販売可能8 / 手持ち8 / 入荷予定0のままでした。

入荷予定を立てるには、発注済み伝票の「入荷指示を作成する」から、入荷先ロケーションを指定して保存します。

### 発注済み後は編集できない

発注を実行すると確認ダイアログにある通り「伝票の編集ができません」。発注済み状態で編集導線は表示されず、「キャンセルする」か「入荷指示を作成する」のどちらかで進めます。内容を修正したい場合は、入荷完了前に発注伝票をキャンセルし、新しく作り直してください。入荷指示作成済みでも入荷完了前ならキャンセルできますが、作成済み入荷指示は `入荷キャンセル` になります。

2026-06-28追加確認では、発注済み未入荷の `#IP-1013` と入荷完了済みの `#IP-1007` で、詳細画面・`その他の操作`・直接URL `/update` / `/edit` を確認しました。いずれも編集・再編集・差し戻し・下書きに戻す導線はなく、`/update` / `/edit` は「このページは存在しないようです」でした。`#IP-1013` の `その他の操作` は `キャンセルする` が有効、`#IP-1007` は `キャンセルする` がdisabledです。

### キャンセルは巻き戻せない（不可逆）

キャンセル確認ダイアログには「この処理は巻き戻すことができません。」と表示されます。キャンセル済み伝票は再キャンセルできず（メニュー内の「キャンセルする」はdisabled）、キャンセル済み伝票を「発注済み」「下書き」に戻す操作は確認できていません。

2026-06-27の確認では、入荷指示を作っていない `#IP-1008` はキャンセルできました。2026-06-28の確認では、発注起点の入荷指示 `#II-1027` を作成した `#IP-1009` も、入荷完了前であればキャンセルできました。一方、発注起点の入荷指示を入荷完了した `#IP-1007` では、発注伝票側の「キャンセルする」がdisabledになりました。

2026-07-01追加確認では、発注済み未入荷の `#IP-1014` と入荷完了済みの `#IP-1007` を再確認しました。`#IP-1014` は詳細の主要導線が `入荷指示を作成する`、`その他の操作` は `キャンセルする` のみです。`#IP-1007` も `その他の操作` は `キャンセルする` のみですが、入荷完了後のためdisabledです。両方とも、詳細本文・ボタン・その他操作・読み込まれている発注詳細JSにPDF/印刷/メール送信/編集/保存/更新/差し戻しの導線はありませんでした。

### 金額欄の表示タイミングに注意

発注伝票の明細「金額」は、入力中の作成フォームと保存後/発注後の詳細・一覧で見え方が異なります。2026-06-27の `#IP-1007` / `#IP-1008` では、作成フォーム入力中は単価100円・数量1・税率10%でも `￥100`、発注済み詳細・発注管理一覧・キャンセル済み詳細では `￥110` と表示されました。

### 通貨は発注伝票ごとに選択可能

通貨は日本円 / 米ドル / ユーロ / タイ バーツ / シンガポール ドル から選べ、発注伝票ごとに設定できます。国際仕入れに対応するための項目です。

2026-06-28の `#IP-1010` では、下書き保存後の詳細画面でも通貨selectが有効で、`USD` から `EUR` へ変更保存できました。金額表示は `$100.00` から `€100.00` に変わり、為替レートや換算額を入力・確認するUIは見当たりませんでした。`#IP-1013` では発注後に通貨selectと保存ボタンが表示されず、通貨は値表示のみになりました。発注後に通貨を変更する導線は確認できません。

### 取引先一覧の「インポート」は未実装

取引先一覧（`/admin/settings/suppliers`）右上の「インポート」ボタンをクリックしても、画面遷移・ダイアログ・ドロップダウン・トーストは表示されません。2026-06-28再確認でもURLは一覧のままでした。取引先の登録は、現行UIでは1件ずつ手動で行います。

### 取引先は発注伝票作成フォームに即時反映

取引先マスタに登録したデータは、発注伝票作成フォームの「取引先」選択肢にすぐ反映されます。取引先を新規登録してから発注伝票を作成する流れで問題ありません。

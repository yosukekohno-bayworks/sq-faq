# 09. 翻訳

> このページは標準24分類の第9分類です。SQが多言語表示（翻訳）をどこで管理し、翻訳ルールで何を設定するかを理解するのが目標です。採寸ルールは別画面・別データのため [10. 採寸](10-採寸.md) で扱います。

## このエリアで学べること

- 翻訳ルールで多言語表示を管理する場所と、対応する10言語を説明できる
- 翻訳データの自動生成対象（商品・商品オプション・商品オプション値・メタフィールド）が分かる
- 多言語表示の管理場所を他者に説明できる

---

## 機能概要

このエリアが扱う機能は **設定** 配下の翻訳サブページにあります。

| 機能 | 場所 | 役割 |
|:--|:--|:--|
| 翻訳 | `/admin/settings/translation` | 翻訳ルールを選び、対象リソース（商品など）の翻訳状態を確認する |
| 翻訳ルール | `/admin/settings/translation/translation_rules` | 翻訳先言語・自動生成設定・AIプロンプトを管理する |

設定トップページ（`/admin/settings`）の「カスタムデータグループ」から遷移します。

![翻訳トップ画面](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/audit-settings-07-translation-top.png)

### 翻訳の位置づけ（データ全体像）

```
[設定] でルールを定義
   ├─ 翻訳ルール ──▶ 商品 / 商品オプション / 商品オプション値 / メタフィールド を多言語化
   │                       （既存の商品データに自動反映）
   │                       └─▶ チャネル（Shopify等）へ同期（連携環境で検証待ち）
```

翻訳ルールは「設定で定義 → 商品データに反映 → チャネルへ同期」という流れで使いますが、翻訳出力や同期先反映は管理画面単体では確認できていない部分があります（後述「注意点・制約」参照）。

### 翻訳対象となる商品データの種類

翻訳ルールは、以下4種類の商品データを翻訳対象とします。違いを把握しておかないと何を翻訳しているか分からなくなるため、まず定義を押さえてください。

| 用語 | 意味 |
|:--|:--|
| 商品 | 品物の本体（商品マスタ1件） |
| 商品オプション | 商品が持つバリエーションの軸。サイズ・色など（SKU＝バリエーション1つ1つの分類軸） |
| 商品オプション値 | 各オプション軸の個別値。S・M・L、赤・青 等 |
| メタフィールド | 商品に独自追加できる項目のひな形（詳細は08.カスタムデータ） |

> **参考**: SKU（商品バリエーション）は色・サイズ等のバリエーション1つ1つで、在庫や価格はSKU単位で持ちます。商品オプション＝軸、商品オプション値＝その軸の値、の関係です。

### 翻訳でできること

| 機能 | できること |
|:--|:--|
| 翻訳 | 言語別に翻訳ルールを作成し、商品等のデータを多言語化する。翻訳用AIプロンプトも設定できる |

## 画面・項目の説明

### 翻訳（/admin/settings/translation）

管理画面・ストアフロントのデータを多言語に翻訳するルールを設定します。

翻訳トップ（`/admin/settings/translation`）には翻訳ルールの選択ボタンと `ルールを管理する` が表示されます。未選択時は `未選択：翻訳ルールを選択するとリソースを選択できます` と表示され、商品リソースを開けません。既存ルールを選ぶと商品リソースの詳細リンクが有効になり、`/admin/settings/translation/<翻訳ルールID>/products` へ進めます。

商品リソース一覧では `商品コードで検索する` と商品一覧が表示され、商品行を開くと `/admin/settings/translation/<翻訳ルールID>/products/<商品ID>` に遷移します。商品翻訳詳細は、商品・商品オプションごとに `原文` と対象言語列を並べて確認する画面です。2026-06-27実機ではこの商品翻訳詳細画面に保存ボタンや直接編集ボタンは確認できませんでした。

#### 対応言語（10言語）

日本語 / 英語 / 中国語（簡体字） / 中国語（繁体字） / 韓国語 / スペイン語 / フランス語 / ドイツ語 / ヒンディー語 / タイ語

#### 翻訳ルール作成フォーム（/admin/settings/translation/translation_rules/create）

![翻訳ルール作成フォーム](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/kill-settings-01-translation-create-form.png)

| 項目（UIの原文） | 必須 | 補足 |
|:--|:--|:--|
| 名前 * | 必須 | テキストボックス。翻訳ルール一覧で識別するための任意の名称 |
| 言語 * | 必須 | コンボボックス（上記10言語から選択）。**1ルール=1翻訳先言語** |
| 翻訳データを自動で作成する | — | チェックボックス。オン/オフどちらでも保存でき、保存後の詳細画面でも状態が保持される。2026-06-28再確認では、翻訳トップ・一覧・作成・詳細に手動実行/再実行/翻訳/生成ボタンはない。実際の生成タイミング・反映結果は生成済みデータまたは接続環境で検証待ち |

空保存時のエラーは `名前を入力してください` と `言語を選択してください` です。作成フォーム上では、自動生成チェックをオンにしてもプロンプト欄は表示されません。プロンプト欄は保存後の詳細画面で表示されます。

> **運用上の指針**: 名前は言語と用途を含めると一覧で判別しやすい（例: 「英語（標準）」「中国語簡体字（EC用）」）。2026-06-28実機再確認では、英語の翻訳ルールを複数作成でき、同一言語の複数行が一覧に並ぶことを確認しました。ただし、同一言語ルールが複数ある場合の翻訳出力の優先順位・競合・上書き挙動は、実際の翻訳生成結果が必要なため未確認です。

#### 翻訳ルール詳細画面（保存後）

![翻訳ルール詳細画面（上部）](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/kill-settings-02-translation-rule-detail-top.png)
![翻訳ルール詳細画面（下部）](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/kill-settings-03-translation-rule-detail-bottom.png)

- 保存後の詳細画面では「言語」コンボボックスが **変更不可** になります。
- 「翻訳データを自動で作成する」チェックボックスは保存後も表示され、編集可能です。2026-06-28実機再確認では、オンで作成したルールは詳細でもオン、オフで作成したルールは詳細でもオフとして保持されました。
- 詳細画面には「商品 / 商品オプション / 商品オプション値 / メタフィールド」の4セクションが表示され、各セクションで翻訳用AIプロンプト（親プロンプト・カスタムプロンプト）を設定できます。
- カスタムプロンプトを変更すると `保存する` が有効になり、保存後は `翻訳ルールを更新しました` と表示されます。
- **なぜ4つに分けるか**: データ種別ごとに翻訳の性質が異なるため。例えば商品名は短く魅力的に、商品オプション値の「S/M/L」は翻訳しない、メタフィールドは自由記述を自然な文に、のように **種別ごとに最適な指示を書くことで翻訳精度が変わる**。プロンプト設計指針は公式 docs にも記載なし（実機未確認）。

| セクション | 設定内容 |
|:--|:--|
| 商品 | 商品データの翻訳用AIプロンプト（親プロンプト・カスタムプロンプト） |
| 商品オプション | 商品オプションの翻訳用AIプロンプト |
| 商品オプション値 | 商品オプション値の翻訳用AIプロンプト |
| メタフィールド | メタフィールドの翻訳用AIプロンプト |

> **親プロンプト / カスタムプロンプトの運用上の意味**: **親プロンプト** はシステム既定の基本翻訳指示、**カスタムプロンプト** はこのルール固有の追加指示（例: 「ブランド名は英字そのまま」「カジュアルな口調で」）。2026-06-28再確認では、親プロンプトのtextareaは `readonly`、カスタムプロンプトは編集可能でした。詳細画面内に親プロンプトを編集するUIは確認できません。

## 主な操作手順

### 翻訳ルールを作成する

1. 左メニュー **設定** を開き `/admin/settings` へ遷移する
2. 「カスタムデータグループ」の **翻訳** を選び `/admin/settings/translation` へ遷移する
3. **翻訳ルールを作成する** ボタンから作成フォーム（`/admin/settings/translation/translation_rules/create`）へ遷移する
4. 「名前」にルール名を入力する
5. 「言語」コンボボックスから10言語のいずれかを選択する
6. 自動生成が必要な場合は「翻訳データを自動で作成する」にチェックを入れる
7. **保存する** を押す。詳細画面へ遷移し、AIプロンプトが設定可能になる

> 「言語」は保存後に変更できません。別言語が必要な場合は新しいルールを作成してください。

### 翻訳ルールを削除する

1. 翻訳ルール一覧（`/admin/settings/translation/translation_rules`）を開く
2. 削除したいルールの行を選択する
3. `削除する` を押す
4. 確認ダイアログ `翻訳ルールを削除しますか？` で `削除する` を押す

削除後は同じ一覧URLに残り、対象ルールが一覧から消え、トースト `翻訳を削除しました` が表示されます。

2026-06-28実機再確認では、同一言語検証で作成した英語ルール2件をこの手順で削除できました。


### 翻訳用AIプロンプトを設定する

> **親プロンプト / カスタムプロンプトの定義は、上部「翻訳ルール詳細画面」の注記を参照してください。**

1. 翻訳ルール詳細画面を開く（作成後の遷移、または一覧からルールを選択）
2. 対象セクション（商品 / 商品オプション / 商品オプション値 / メタフィールド）を展開する
3. 「親プロンプト」「カスタムプロンプト」を入力する
4. **保存する** を押す

## 注意点・制約

- **翻訳ルールの「言語」は保存後に変更不可** です（実機確認済み）。
- **同一言語の複数ルール作成は可能** です。2026-06-28実機再確認では英語ルールを複数作成できました。ただし、同一言語ルール間の優先順位・競合・上書き結果は、実際の翻訳出力結果が必要なため未確認です。
- **翻訳の実行タイミング**: 2026-06-28再確認では、翻訳トップ・翻訳ルール一覧・作成フォーム・詳細画面の確認範囲に `実行` / `再実行` / `翻訳する` / `生成` などの手動実行ボタンはありませんでした（証跡: `_analysis/complete-live-verification-2026-06-28/09-10-translation-measurement-readonly-recheck-20260628.md`）。実際の生成タイミング・結果は、生成済みデータまたは連携環境が必要な残課題です。
- **商品翻訳詳細は確認中心**: 2026-06-27実機では、商品翻訳詳細に `原文` と対象言語列は表示されますが、直接編集・保存ボタンは確認できませんでした。
- **翻訳出力・チャネル反映は連携待ち** です（下記残課題参照）。2026-06-28再確認では、翻訳トップ・翻訳ルール一覧の本文に未実装プレースホルダー文言はありませんでした。

# 17. 顧客・会社

SQでは、購入者（消費者）と取引先企業を別々に管理します。前者を「顧客管理」、後者を「会社」と呼びます。このエリアでは両者の違いと、それぞれに登録できる情報・行える操作を説明します。

> 対象環境: staging（`https://www.sqstackstaging.com/admin`）。2026-07-16〜17に、Shopifyで新規顧客を作成してSQへの自動登録を確認したうえで、氏名・メールアドレス・電話番号・メモ・タグ・メール購読状態の更新連携、SMS認証の設定/解除、顧客統合、購買サマリー、顧客メタフィールド、ポイント/マイルの加算・減算を実機確認しました。あわせて会社・ロケーション・担当者を新規作成し、必須入力、編集、担当者削除まで再確認しました。2026-07-01には会社ID/会社ロケーションID付きの検証注文を作成し、注文自体には `purchasingCompany` が保存される一方、現行の会社詳細/会社ロケーション詳細UIの「直近の注文」「売上」「注文」には反映されないことを確認しました。
>
> **2026-07-16変更確認:** 会社作成フォームから「会社ID」が削除され、現在の入力項目は「会社名」「コード」の2項目になりました。既存会社の会社IDは詳細画面に引き続き表示されますが、「会社の詳細を編集」ダイアログでは会社名だけを編集でき、会社IDの新規入力・編集導線はありません。
>
> **2026-07-16 Shopify顧客連携確認:** SQ側の「顧客データの同期を有効にする」がONの状態で、Shopify顧客 `9367918117100` を新規作成しました。SQの顧客管理へ即時登録され、会員番号 `9367918117100`、外部ID `gid://shopify/Customer/9367918117100`、氏名、メールアドレス、タグが一致しました。その後Shopifyでタグ `faq-sync-updated-20260716` を追加すると、SQの同一顧客にも反映され、更新日時が `23:06` から `23:08` へ更新されました。
>
> **2026-07-17 全体再確認:** Shopify側で氏名・メールアドレス・電話番号・メモ・タグを変更し、SQの同一顧客へ反映されることを確認しました。タグ削除、メール購読の登録/解除もSQへ反映されました。ポイント10の加算/減算、マイル7の加算/減算、SMS認証の認証済み化/解除/再認証、顧客統合、会社/ロケーション/担当者の新規保存と編集を実行しました。Shopify顧客メタフィールドの表示・マッピングはSQの顧客画面では確認できず、現行のShopifyメタフィールド定義連携画面も「商品」「バリエーション」のみです。

## このエリアで学べること

- 「購入顧客（顧客管理）」と「会社担当者」の違いを説明できる
- 顧客管理の一覧でできること（検索・絞り込み・ポイント一括付与）を把握する
- 会社の新規作成からロケーション・担当者の追加までの一連の手順を実行できる
- 会社のロケーションと住所、請求先住所の関係を理解する
- 顧客・会社に関する制約（手動作成の可否、削除UIの有無など）を把握する

## 機能概要

| 機能 | 画面URL | できること |
|:--|:--|:--|
| 顧客管理 | `/admin/purchasing_customers` | 購入顧客の一覧表示・メールアドレス検索・絞り込み・ポイント一括付与（CSV） |
| 会社 | `/admin/companies` | 法人顧客（会社）の新規作成・一覧管理・会社ごとのロケーション・担当者管理 |
| 会社作成 | `/admin/companies/create` | 会社本体の基本情報（会社名・コード）を登録。ロケーション/住所/担当者は会社詳細から別途追加 |
| 会社詳細 | `/admin/companies/{id}` | 会社名・メモ編集、ロケーション一覧、担当者一覧。現行UIには直近の注文/売上欄があるが、会社注文データは反映されない |
| ロケーション追加 | `/admin/companies/{id}/locations/create` | 会社に複数の納品先（ロケーション）を追加 |
| ロケーション詳細 | `/admin/companies/{id}/locations/{location_id}` | ロケーション名・コード・住所・メモの編集。現行UIには直近の注文欄があるが、会社ロケーション注文データは反映されない |

### 顧客と会社担当者の違い（重要）

| 項目 | 購入顧客（顧客管理） | 会社担当者（会社） |
|:--|:--|:--|
| 登録元 | 販売チャネル経由で自動登録。Shopifyからの新規登録・更新を実機確認済み | 管理画面の会社詳細から手動作成 |
| 用途 | 一般消費者の購入者データ | 発注確認メールの送付先 |
| 作成ボタン | なし（手動作成不可） | あり（会社詳細の「担当者を追加」） |
| 顧客管理一覧への表示 | される | されない（別データ） |
| ディスカウント対象顧客の候補 | される | されない |

> 会社詳細で担当者を作成しても、顧客管理（`/admin/purchasing_customers`）の顧客一覧には表示されません。両者は別のエンティティです。
>
> 用語の補足: 表中の「ディスカウント対象顧客」とは、クーポン（ディスカウント）を適用できる顧客として選択できる候補のことです（詳細は **18. CRM**）。購入顧客データが候補になり、会社担当者は候補に入りません。

### データ相関（会社・ロケーション・担当者・注文）

```
会社（Company）
 ├─ ロケーション（CompanyLocation ＝ 納品先住所）
 │    └─ 注文データの会社ロケーションとして保持できる
 └─ 担当者（Contact ＝ 発注確認メールの宛先メールアドレス）
```

- 検証用注文の作成時に `purchasingCompany.companyID` / `companyLocationID` に会社と会社ロケーションを指定すると、注文データ上の `purchasingCompany.company` / `purchasingCompany.location` に保存されました。
- ただし現行の会社詳細/会社ロケーション詳細UIは、注文一覧や売上を取得するGraphQLを投げていません。会社に紐づく注文を作成しても「直近の注文」は空のまま、ロケーション行の「売上」「注文」も `¥0` / `0` のままでした。したがって、現時点ではこの画面を会社別注文履歴・売上確認画面として案内しないでください。
- **発注伝票**の作成フォームには、会社ロケーション（CompanyLocation）を選ぶ「ロケーション」項目はありません。発注伝票で選ぶ「取引先」は `設定 > 取引先` の仕入先マスタです。発注済み後の「入荷指示を作成する」で選ぶ「入荷先ロケーション」は、在庫を受け入れるSQロケーションであり、会社ロケーションではありません（詳細は **15. 発注・仕入**）。
- 会社ロケーションを選ぶ「ロケーション」項目は、OmnibusCore連携の `連携サイト > 追加する` ダイアログ内で確認できます。これは発注伝票とは別機能です（詳細は **20. 標準販売チャネル連携**）。
- **担当者**のメールアドレスは、発注の確認メール等の送付先として使われます。

### SMS認証

顧客情報右上のメニューから次の往復操作を確認しました。

1. 未認証時に「SMS認証済みにする」を実行すると `成功 / 完了` になる
2. 認証済み時に「SMS認証を解除する」を実行すると `未認証` になる
3. 再度「SMS認証済みにする」を実行すると `成功 / 完了` に戻る

認証済みにする際の確認文は「この顧客のSMS認証を『認証済み』に設定します。電話番号が登録されていない場合は、認証のみが更新されます。」です。解除時は「解除後、顧客は再度SMS認証を行う必要があります。」と表示されます。電話番号なしで認証済みにした後、Shopifyから電話番号を追加しても認証済み状態は維持されました。

> 管理者による状態変更の確認です。顧客の端末へ実SMSを送って認証コードを入力するエンドユーザー側の認証フローは未確認です。

### ポイント・マイル

同じShopify連携顧客で実データを加算・減算し、履歴と残高を確認しました。

| 操作 | 実測結果 |
|:--|:--|
| ポイントを10加算 | 保有ポイントが10。タイトル、取引日時、確定日時、失効日（30日後）、非公開メモが履歴へ表示 |
| ポイントを10減算 | 保有ポイントが0へ戻り、減算履歴が追加 |
| マイルを7加算 | 保有マイル7、累計獲得7。履歴へ表示 |
| マイルを7減算 | 保有マイル0、累計獲得0へ戻り、減算履歴が追加 |

ポイント加算は「加算理由」「ポイント」「有効日数」が必須、ポイント減算は「減算理由」「ポイント」が必須です。マイル加算/減算は理由とマイル数が必須です。マイルダイアログには「累計獲得マイルに反映しない」チェックがあります。ポイント/マイルとも「すべての履歴を表示」から専用履歴画面へ遷移できます。

### 顧客を統合する

統合元 `FAQMergeSource20260716` と統合先 `FAQMergeTarget20260716` をShopifyから作成し、統合元の「その他の操作 > 顧客を統合する」で実行しました。

- 統合先はメールアドレス検索で選択する
- 統合先選択後に「統合する」が有効になる
- 「統合する」を押すと追加の確認ダイアログなしで実行され、統合先詳細へ遷移する
- 統合先には「この顧客には、以下の顧客が統合されています」、統合元には「この顧客は、以下の顧客に統合されています」と相互リンクが表示される
- 一覧では統合元に `(統合済み)` が表示される
- 統合元詳細は引き続き参照でき、統合元/統合先それぞれのメールアドレスと外部IDも表示される

## 主な操作手順

### 会社を新規作成する

1. 左メニューから「会社」（`/admin/companies`）を開く
2. 右上の「会社を作成する」ボタンを押す（`/admin/companies/create` へ遷移）
3. 会社名（必須）・コード（必須）を入力する
4. 「保存する」を押す
5. 会社詳細画面から、必要に応じてロケーションと担当者を追加する

### 会社にロケーション（納品先）を追加する

1. 会社一覧（`/admin/companies`）で対象の会社行をクリックし、会社詳細を開く
2. 「ロケーション」セクションの「ロケーションを追加」ボタンを押す（`/admin/companies/{id}/locations/create` へ遷移）
3. ロケーション名（必須）・ロケーションID（任意）・コード（任意）を入力する
4. 配送先住所・請求先住所を入力する
5. 「保存する」を押す

### 会社に担当者（発注確認メールの宛先）を追加する

1. 会社詳細（`/admin/companies/{id}`）を開く
2. 「担当者」セクションの「担当者を追加」ボタンを押す（ダイアログが開く）
3. 姓（必須）・名（必須）・メールアドレス（必須）・電話番号（任意）を入力する
4. 「保存する」を押す

### 会社名・メモを編集する

1. 会社詳細（`/admin/companies/{id}`）を開く
2. 右サイドバーの「会社名」または「メモ」の右にある編集ボタン（ペンアイコン）を押す
3. 内容を書き換えて保存する

### 顧客をメールアドレスで検索する

1. 左メニューから「顧客管理」（`/admin/purchasing_customers`）を開く
2. 「検索と絞り込みの結果」ボタンを押して検索欄を展開する
3. 「メールアドレスで検索する」テキストボックスにメールアドレスを入力する
4. 検索結果を確認する

### 顧客にポイントを一括付与する

1. 顧客管理（`/admin/purchasing_customers`）を開く
2. 右上の「インポート」ドロップダウンボタンを押す
3. 「ポイント一括付与」を押す（`/admin/csv_import/csv_import_operation_point_pluses` へ遷移）
4. 「テナント」を選択し、「顧客IDの種別」を指定して、指定フォーマットのCSVファイル（列順・ヘッダー名はテンプレートに準拠）をアップロードする
5. プレビューで付与対象の顧客と件数を確認してから「実行」する

> 「顧客IDの種別」の選び方: CSVの顧客識別子がSQ内の顧客IDなら「SQが発番した顧客ID」、Shopify等の外部IDを使うなら「外部システムが発番した顧客ID」を選びます。連携元のID体系に合わせて指定しないと顧客を正しく特定できません（詳細は **21. CSV/PDF・データ移行**）。

> 詳細な手順・CSV仕様は **21. CSV/PDF・データ移行**（CSVインポートの章）を参照してください。

> ポイント一括付与CSVの「実行」は取り消し不可です。実行前に内容を確認してください。

### 担当者を削除する

1. 会社詳細（`/admin/companies/{id}`）の「担当者」セクションで対象の行を選択する
2. 「担当者を削除」を押す
3. 確認ダイアログ「担当者を削除する」で「削除する」を押す
4. 対象の担当者行が一覧から消えることを確認する

> 2026-07-17のstagingでは1行だけを選択しても選択表示が「2を選択済み」となり、確認本文も「選択されている 件の担当者を削除しますか？」と件数が欠けました。削除対象自体は選択した1件のみで、削除後に空表示へ戻ることを確認しました。

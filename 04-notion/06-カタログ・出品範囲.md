# 06. カタログ・出品範囲

> このページはWBS-25エリアの第6エリアです。「どの商品をどの販売チャネルに出すか」を束ねる仕組みである **カタログ** と、商品につける **ブランド** の役割の違いを理解するのが目標です。

## このエリアで学べること

- カタログとブランドの違いを説明できる
- カタログを作成し、商品を手動・自動で追加できる
- 分類条件（製造元・ブランドコード）で商品を自動追加するルールを設定できる
- カタログが「販売チャネルごとの出品範囲」をどう制御するか理解できる

---

## 機能概要

### カタログとは

カタログは **商品をグループ化する入れ物** です。販売チャネル（連携先）ごとに1つのカタログを指定することで、そのチャネルに出す商品を制限できます。例えば「渋谷店カタログ」をOmnibusCore連携に紐付ければ、渋谷店向けの商品だけが同期されます。

カタログ単体では商品を出品しません。**チャネル連携側の設定**でカタログを指定することで、はじめて出品範囲として機能します。例えばOmnibusCore連携なら `/admin/omnibus_core_integrations/{id}` の編集画面（基本設定タブ）の **「商品同期設定 > カタログ」** コンボボックスで当カタログを選択します。

> **チャネル別のカタログ扱い（根拠: データ事典②）**:
> - **Shopify連携**: カタログ指定が**必須**。
> - **OmnibusCore・スマレジ・リテールポータル連携**: カタログ欄の選択肢としてカタログを指定可能（カタログを指定した商品だけがそのチャネルへ連携される）。
> - Shopify・スマレジ・リテールポータル連携での実際の紐付け経路・商品同期挙動は **⚠️ 未確認**（連携未接続）。確定しているのはOmnibusCore連携の編集画面経路のみです。

**データの流れ（全体像）**:

```
商品マスタ(/admin/products)
        │  手動追加 / 自動追加ルール / CSV
        ▼
   カタログ(/admin/catalogs)          ← 「どの商品を束ねるか」
   例: 渋谷店カタログ / 新宿店カタログ / ネット専用カタログ
        │  各チャネル連携の編集画面で「商品同期設定 > カタログ」を選択
        ▼
   各チャネル連携                      ← 「どのチャネルにどのカタログを出すか」
   (OmnibusCore / Shopify / スマレジ / リテールポータル)
        ▼
   同期先（ECサイト / POS / 店舗端末 等）  ← カタログ内の商品だけが連携される
```

> 同じカタログを複数チャネルに割り当てることも、チャネルごとに別カタログを割り当てることも可能です。これで「渋谷店向けはA、ネット店舗はB」といったチャネル別の出品範囲を制御します。

### ブランドとの違い

| | カタログ | ブランド |
|:--|:--|:--|
| 役割 | 商品をグループ化し、チャネルごとの出品範囲を決める | 商品につける属性の1つ（製造元・ブランド名） |
| 登録場所 | `/admin/catalogs` | `/admin/settings/brands` |
| 商品との関係 | 商品を複数カタログに追加できる | 商品マスタでブランドを選択する |
| 用途 | 出品制御・チャネル別の商品集合 | 商品の分類・ラベル表示 |

> **注意**: 商品マスタの「ブランド」（`/admin/settings/brands`）と、設定の「事業部（テナント＝運用ルール分離単位。詳細は03.組織・通知）」で扱われるブランド概念は別データです。本エリアでは商品マスタ側のブランドを扱います。

### 対象画面

| 画面 | URL | 内容 |
|:--|:--|:--|
| カタログ一覧 | `/admin/catalogs` | カタログの一覧・作成入口 |
| カタログ詳細 | `/admin/catalogs/{id}` | 商品一覧・各種操作の起点 |
| 商品追加 | `/admin/catalogs/{id}/create` | 手動で商品をカタログに追加 |
| 自動追加ルール | `/admin/catalogs/{id}/automatic_add_rules` | 分類条件ルールの設定 |
| SKU一覧 | `/admin/catalogs/{id}/catalog_product_variants` | カタログ内SKU一覧 |
| CSVインポート | `/admin/csv_import/csv_import_operation_catalog_products` | CSVで商品を一括追加 |
| ブランド | `/admin/settings/brands` | ブランドマスタ管理 |

## 画面・項目の説明

### カタログ一覧（/admin/catalogs）

| 列名 | 内容 |
|:--|:--|
| 名前 | カタログ名 |
| 商品 | カタログに登録されている商品数（例: 0個の商品） |
| 販売先 | このカタログを参照しているチャネル連携の数（例: 0つの販売先） |

> 「販売先」列は、カタログを参照しているチャネル連携の数です。カタログ詳細画面から直接「販売先」を設定するUIはありません。

![カタログ一覧画面](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/03-catalogs-sort-dropdown.png)

### カタログ詳細画面（/admin/catalogs/{id}）

![カタログ詳細画面](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/03-catalog-detail-created.png)

| ボタン | 機能 |
|:--|:--|
| 自動追加ルール | 自動追加ルールの一覧・設定画面へ遷移 |
| SKU一覧 | このカタログのSKU一覧画面へ遷移 |
| 編集する | カタログ名を変更するダイアログを開く |
| その他の操作 | 「カタログを削除する」「インポート」のメニューを表示 |
| 商品を追加する | 商品を選択してカタログに追加する画面へ遷移 |

詳細画面の本体には **「商品コードで検索する」** テキストボックスがあり、カタログ内の商品を商品コードで絞り込めます。

### カタログ編集ダイアログ

「編集する」ボタンから開く「カタログを編集する」ダイアログで編集できる項目です。

| 項目（UIラベル） | 必須 | 説明 |
|:--|:--|:--|
| タイトル* | 必須 | カタログ名（例: 渋谷店） |

編集できるのはタイトルのみです。チャネル割り当てやロケーションとの紐付けはこのダイアログからは設定できません。

![カタログ編集ダイアログ](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/03-catalog-edit-modal.png)

### 商品追加画面（/admin/catalogs/{id}/create）

「商品を追加する」をクリックすると「カタログに商品を追加する」画面が開きます。

| 要素 | 説明 |
|:--|:--|
| テーブル列 | 商品（サムネイル＋商品名） / 商品コード |
| 商品コードで検索する | テキストボックスで商品を絞り込み |
| 行のチェックボックス | 商品を選択すると「カタログに追加する」ボタンが表示 |
| 一括チェックボックス | 全件を一括選択できる |

![「カタログに商品を追加する」画面](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/03-catalog-add-product.png)

### 自動追加ルール（/admin/catalogs/{id}/automatic_add_rules）

条件に一致する商品を自動的にカタログへ追加するルールを設定できます。「追加する」ボタンで「ルールを追加する」ダイアログが開きます。

![「ルールを追加する」ダイアログ](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/03-catalog-auto-rule-dialog.png)

| 項目 | 選択肢 |
|:--|:--|
| 項目 | 「製造元」「ブランドコード」の2択 |
| 条件 | 「一致する」の1択 |
| 値 | テキストで自由入力 |

設定したルールと完全一致する商品が自動的にカタログに追加されます。

> **用語注記**:
> - **製造元** = 商品マスタ（商品管理 > 商品）で入力する自由テキスト項目。ルールは**完全一致判定**で、大文字小文字・スペース違いも別物扱い。ブランドコードとは別（製造元は商品につける自由テキスト、ブランドコードはブランドマスタ`/admin/settings/brands`のコード）。
> - **ブランドコード** = ブランドマスタ（設定 > ブランド）のコード値。商品が参照しているブランドのコードと突合する。

登録済みルールは「項目」「条件」「値」の一覧で確認できます。行を選択すると「ルールを削除する」ボタンが表示され、確認ダイアログ「カタログのルールを削除しますか？」（本文: 「選択された1件のルールを削除します。この処理は巻き戻すことができません。」）から削除できます。

### CSVインポート画面

「その他の操作 > インポート」からカタログへの商品を一括追加できます。

| 項目 | 必須 | 説明 |
|:--|:--|:--|
| カタログ | 必須 | ドロップダウンで既存カタログを選択 |
| ファイルをアップロード | 必須 | CSVファイルをアップロード（ファイル選択またはドラッグ&ドロップ） |

ページ上部にテンプレートファイル（Googleスプレッドシート）へのリンクが表示されます。

### ブランド作成フォーム（/admin/settings/brands/create）

| 項目（UIラベル） | 必須 | 補足 |
|:--|:--|:--|
| 名前* | 必須 | — |
| 外部ID | 任意 | **作成時のみ設定可。保存後の変更不可** |
| コード* | 必須 | — |

保存後の詳細・編集フォームには「外部ID」フィールドは表示されません。

## 主な操作手順

### カタログを作成して商品を手動で追加する

1. 左メニューから **カタログ**（`/admin/catalogs`）を開く
2. 右上の **カタログを作成する** をクリックする
3. カタログ名を入力し、保存する
4. カタログ詳細画面が開くので、**商品を追加する** をクリックする
5. 「商品コードで検索する」で商品を絞り込むか、一覧から目的の商品を探す
6. 商品行のチェックボックスをオンにする（一括選択も可）
7. **カタログに追加する** をクリックする

### 自動追加ルールを設定する

1. カタログ詳細画面で **自動追加ルール** をクリックする
2. **追加する** をクリックし「ルールを追加する」ダイアログを開く
3. 項目で「製造元」または「ブランドコード」を選ぶ
4. 条件は「一致する」のまま、値に条件テキストを入力する
5. **追加する** をクリックする

> 設定後、条件に一致する商品が自動的にカタログへ追加されます。

### カタログ名を変更する

1. カタログ詳細画面で **編集する** をクリックする
2. 「カタログを編集する」ダイアログで **タイトル*** を変更する
3. 保存する

### カタログを削除する

1. カタログ詳細画面で **その他の操作** を開く
2. **カタログを削除する** をクリックする
3. 確認ダイアログ「カタログを削除しますか？」（本文: 「カタログを削除します。この処理は巻き戻すことができません。」）で **削除する** をクリックする

### CSVで商品を一括追加する

1. カタログ詳細画面で **その他の操作 > インポート** をクリックする
2. インポート画面（`/admin/csv_import/csv_import_operation_catalog_products`）で **カタログ** を選択する
3. テンプレートリンクからフォーマットを確認し、CSVを作成する
4. **ファイルをアップロード** でCSVをアップロードする
5. 保存後、検証を経て **実行する** で反映する

> **注意**: CSVインポートの「実行」は巻き戻し不可です。検証→実行の2段階で進み、誤った場合は修正版CSVを新規インポートしてください。

### 自動追加ルールを削除する

1. カタログ詳細画面で **自動追加ルール** を開く
2. 削除したいルール行を選択する
3. **ルールを削除する** をクリックする
4. 確認ダイアログで **削除する** をクリックする

---

## 注意点・制約

- **カタログの削除は復元不可**: 「その他の操作 > カタログを削除する」から行います。削除前に商品との紐付けを確認してください（確認ダイアログに「この処理は巻き戻すことができません。」と表示）。
- **チャネルとの紐付けは連携側で設定**: カタログとチャネルの紐付けはチャネル連携側の設定から行います。例えばOmnibusCore連携では `/admin/omnibus_core_integrations/{id}` の編集画面「基本設定」タブ > **「商品同期設定 > カタログ」** コンボボックスで当カタログを選択します。カタログ詳細画面から直接「販売先」を設定するUIはありません。
- **CSVインポートの実行は不可逆**: 「実行」は巻き戻し不可。検証→実行の2段階で進めます（参照: 制約台帳「CSVインポートの実行は取り消し不可」）。
- **ブランドの「外部ID」は作成時のみ設定可**: 保存後の変更はできません（参照: 制約台帳「ブランドの外部IDは作成時のみ設定可」）。
- **自動追加ルール削除は不可逆**: 確認ダイアログに「この処理は巻き戻すことができません。」と表示されます。

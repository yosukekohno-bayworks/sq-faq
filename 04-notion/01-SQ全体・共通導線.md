# 01. SQ全体・共通導線

> このページはWBS-25エリアの第1エリアです。SQが何を管理するシステムか、**どのメニューで何ができるか**の全体図を理解するのが目標です。

## このエリアで学べること

- SQが何を管理するシステムか説明できる
- **左メニューの全項目と、それぞれで何ができるか**を把握できる
- 共通操作（一覧・検索・詳細・作成・CSV/PDF入口）が分かる

---

## SQとは

SQは **あらゆるチャネルを横断して在庫・受注・商品を一元管理し、「売り切る力」と「効率化」を両立する次世代の基幹システム** です（[公式ガイド](https://docs.sqstack.com/docs/guide/introduction) より）。

お客様が商品を見て買う「売り場（ECサイト・POS）」ではなく、運営スタッフが在庫・受注・商品を管理する **バックヤード（業務管理の裏方）** として動作します。本マニュアルでは現場目線で分かりやすく「バックヤード」とも呼びます。

### SQの位置づけ（外部連携全体図）

```
  ┌──────────┐    注文・商品     ┌──────────┐
  │ ECサイト  │ ←─────────────→ │          │
  │ Shopify  │                  │          │
  └──────────┘                  │          │
                                │          │
  ┌──────────┐    店舗販売      │    SQ     │  ← このシステム（バックヤードERP）
  │  POS     │ ←─────────────→ │  管理画面  │
  │ スマレジ  │                  │          │
  └──────────┘                  │          │
                                │          │
  ┌──────────┐    在庫作業      │          │
  │  WMS     │ ←─────────────→ │          │
  │ロジザード │                  │          │
  └──────────┘                  └──────────┘
```

| 連携先 | 実機の見出し（原文） | メニューURL | 役割・連携内容（出典） | 実機確認状態 |
|:--|:--|:--|:--|:--|
| **Shopify** | 「Shopify」 | `/admin/shopify_integrations` | EC連携：商品/ロケーション/在庫/顧客/ポイント/会員ランク同期【**公式ガイド**】。実機の空状態説明は「商品や注文データの連携を行います」 | ⚠️ 未接続 |
| **スマレジ** | 「スマレジ連携」 | `/admin/smaregi_integrations` | 実機フォームに商品(カタログ)・在庫(同期方向)・注文(在庫変動)の項目あり。「POS」は**一般知識**【実機フォーム＋一般知識】 | ⚠️ 未接続（公式ガイドは近日連携予定） |
| **ロジザードZERO** | 「ロジザード連携」（※作成フォームは「ロジザードZERO」） | `/admin/logizard_integrations` | 実機画面に役割表記なし。「WMS/倉庫管理」は**一般知識** | ⚠️ 未接続（公式ガイドに記載なし） |
| **Recustomer** | 「Recustomer」 | `/admin/recustomer_integrations` | 実機画面に役割表記なし。「返品管理」は**一般知識** | ⚠️ 未接続 |
| **OmnibusCore** | 「OmnibusCore」 | `/admin/omnibus_core_integrations` | 実機画面に役割表記なし。「最低価格表示」は**一般知識** | ⚠️ 未接続 |
| **リテールポータル** | 「リテールポータル」 | `/admin/retail_portal_integrations` | 店舗スタッフ向け。客注注文/他店在庫確認/注文メモ【**公式ガイド**】 | ⚠️ 未接続 |

> **読み方（優先順位）**: 本マニュアルは **①実機確認済みの事実 ＞ ②公式ガイド ＞ ③一般知識** の順で信頼します。
> - **実機メニューの存在** ＝ stagingの左メニュー・URLで確認済み（2026-06-20）。公式ガイドになくても実機にあれば事実として扱います（ロジザードZERO・スマレジが該当）。
> - **実機の連携画面の見出しは製品名または「〇〇連携」のみ**（2026-06-20実機確認）。**役割名（POS/WMS/返品管理/最低価格表示）は実機画面に表記がなく、すべて一般知識です。**
> - **連携で同期されるデータ** は、staging環境が未接続のため実機で確認できていません。Shopify・リテールポータルは公式ガイドで確定、スマレジは実機フォーム項目ベース、ロジザード/Recustomer/OmnibusCoreの役割は一般知識です。
> - **スマレジ** は実機に連携メニュー・設定フォームが存在し入力可能です。接続後の実際の同期挙動はstaging未接続のため未確認です。なお公式ガイドでは「近日連携予定（Coming Soon）」扱いとなっており、実機の実装と公式の表現に差があります。
> - 接続済み環境での検証が必要です（詳細は [19. 標準販売チャネル連携](19-標準販売チャネル連携.md)・[20. 物流・返品・外部アプリ連携](20-物流・返品・外部アプリ連携.md) のTODO参照）。

## 管理画面のメニューサイトマップ（全体図）

管理画面 `https://www.sqstackstaging.com/admin` の左メニューは、大きく **6つのグループ** で構成されます。[公式ガイド](https://docs.sqstack.com/docs/guide/introduction) は「管理 / CRM / 販売チャネル連携 / 設定」の4区分ですが、実機の左メニューに沿って現場作業単位（オペレーション）を独立させた6区分で示します。各グループ見出しに公式ガイドの対応区分を併記します。

| 公式ガイドの区分 | 本マニュアルの対応グループ |
|:--|:--|
| 管理（商品・顧客・在庫・注文・倉庫） | グループ1（主要業務）・グループ2（オペレーション） |
| CRM（ポイント・会員ランク・ディスカウント） | グループ3 |
| 販売チャネル連携（Shopify・リテールポータル・POS） | グループ4 |
| 設定（データ管理・外部連携） | グループ5 |

### グループ1：主要業務（上段）— 公式ガイド「管理」の主要機能 + 発注/販売設定/会計/分析

| メニュー | URL | 何ができるか |
|:--|:--|:--|
| **ホーム** | `/admin` | ダッシュボード。各機能への導線 |
| **商品管理** | `/admin/products` | 商品・SKU・バリエーション・価格の登録・編集。カタログ・店舗受取の設定 |
| **在庫管理** | `/admin/inventory_items` | 在庫数の確認（9区分）、移動伝票・調整伝票・取置伝票の作成 |
| **注文管理** | `/admin/orders` | 注文の確認・出荷登録（※チャネル接続後に注文が流入） |
| **顧客管理** | `/admin/purchasing_customers` | 購入顧客・会社の確認（※チャネル由来、手動作成不可） |
| **発注管理** | `/admin/inventory_purchase_orders` | 発注伝票の作成・発注・キャンセル（※入荷への自動連動なし） |
| **販売設定** | `/admin/product_price_rules` | 販売価格・予約販売・販売上限・販売閾値のルール設定 |
| **会計** | `/admin/sale_change_line_items` | 売上実績の確認・CSV出力 |
| **分析** | `/admin/analytics` | ※現在「収益」「レポート」はTODO表示（未実装） |

### グループ2：オペレーション（在庫移動の現場作業）— 公式ガイド「管理＞在庫/倉庫」の現場作業

| メニュー | URL | 何ができるか |
|:--|:--|:--|
| **入荷管理** | `/admin/inventory_inbound_orders` | 入荷指示の確認・入荷実績の登録（移動伝票・発注由来） |
| **出荷管理** | `/admin/inventory_outbound_orders` | 出荷指示の確認・出荷実績の登録・ヤマトB2/DHL CSV出力 |
| **在庫依頼** | `/admin/inventory_allocation_requests` | 取り寄せ販売の起点。在庫確保の依頼・引当・移動伝票作成 |
| └ **確保済み** | `/admin/inventory_allocation_request_confirmations` | 在庫依頼の引当済み一覧。ここから移動伝票を一括作成 |

> **バッジ**: 入荷管理・出荷管理・在庫依頼には未完了件数がバッジで表示（0件時は非表示、100件超は「100+」）。

### グループ3：CRM（販売促進）— 公式ガイド「CRM」

| メニュー | URL | 何ができるか |
|:--|:--|:--|
| **ディスカウント** | `/admin/order_price_adjustment_rules` | 割引クーポン・対象顧客・対象店舗・対象商品の設定 |
| **ポイント** | `/admin/point_calculation_rules` | 注文ポイント付与ルール（基本設定・会員ランク倍率・商品倍率） |
| **会員ランク** | `/admin/customer_rank_calculation_rules` | 会員ランク算出ルール（購入金額ベース、Bronze/Silver/Gold等） |

### グループ4：販売チャネル（外部連携）— 公式ガイド「販売チャネル連携」

| メニュー | URL | 何ができるか |
|:--|:--|:--|
| **Shopify** | `/admin/shopify_integrations` | Shopifyストアとの連携設定（※SQコネクターアプリ前提） |
| **OmnibusCore** | `/admin/omnibus_core_integrations` | 最低価格表示（景品表示法対応）の連携 |
| **スマレジ** | `/admin/smaregi_integrations` | スマレジPOSとの連携設定 |
| **リテールポータル** | `/admin/retail_portal_integrations` | 店舗スタッフ向けポータルの連携 |
| **卸売** | `/admin/b2b` | ※現在TODO表示（未実装） |

### グループ5：設定（マスタ・システム設定）— 公式ガイド「設定」

| メニュー | URL | 何ができるか |
|:--|:--|:--|
| **設定** | `/admin/settings` | ロケーション・権限グループ・ブランド・取引先・決済方法・翻訳・採寸・メタフィールド・API/アプリ 等、全マスタ・システム設定 |

> 設定画面はタブ・セクションで細分化されています（ロケーション・ロケーショングループ・管理メンバー・権限グループ・ブランド・取引先・決済方法・販売員・翻訳・採寸ルール・メタフィールド・アプリ 等）。詳細は [04. 基本マスタ](04-基本マスタ.md)・[02. アカウント・権限](02-アカウント・権限.md) を参照。

## 共通操作パターン

### 一覧画面（ほぼ全機能の入口）

| 共通要素 | 説明 |
|:--|:--|
| タブ | ステータス別に表示を切り替え（例：確認待ち／確保済み／終了） |
| 検索・フィルタ | テキスト検索や条件指定で絞り込み |
| テーブル | 行クリックで詳細画面へ遷移 |
| 「〇〇を作成する」ボタン | 右上に配置。新規作成画面へ遷移 |
| 「インポート」「エクスポート」 | 一括操作の入口（機能により有無・実装状態が異なる） |

### 詳細画面

| 共通要素 | 説明 |
|:--|:--|
| ステータスバッジ | 現在のステータスを色付きバッジで表示 |
| 「その他の操作」 | 詳細画面右上のメニュー。キャンセル・クローズ等の操作が集約 |
| 編集制限 | 一部の伝票（外部連携由来等）は商品編集が制限される場合がある |

### 作成フォーム

| 共通パターン | 説明 |
|:--|:--|
| セクション分割 | 「商品」「リクエスト内容」等のセクションに分かれる |
| 「選択」ボタン | ロケーション・商品等の選択はモーダルで行う（テキスト検索＋選択。SKU検索は入力後Enterで発火） |
| 必須項目 | `*` 印付き。空保存でバリデーションエラー |
| 「保存する」 | セクション内またはページ下部。保存後は詳細画面へ遷移 |

### CSV/PDF入口

| 操作 | 場所 |
|:--|:--|
| CSVインポート | 各一覧画面の「インポート」ボタン（22カテゴリ、機能により実装状態が異なる） |
| CSVエクスポート | 各一覧画面の「条件指定でエクスポート」等（9カテゴリ） |
| PDFエクスポート | 一部の伝票で「PDFエクスポート」ボタン |

> **CSVインポートの注意**: 「実行」は巻き戻し不可。検証→実行の2段階。誤った場合は修正版CSVを新規インポートするか、在庫は調整伝票で補正。

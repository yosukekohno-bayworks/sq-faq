# 01. SQ全体・共通導線

> このページはWBS-25エリアの第1エリアです。SQが何を管理するシステムか、**どのメニューで何ができるか**の全体図を理解するのが目標です。

## このエリアで学べること

- SQが何を管理するシステムか説明できる
- **左メニューの全項目と、それぞれで何ができるか**を把握できる
- 共通操作（一覧・検索・詳細・作成・CSV/PDF入口）が分かる

## SQとは

SQは **あらゆるチャネルを横断して在庫・受注・商品を一元管理し、「売り切る力」と「効率化」を両立する次世代の基幹システム** です（[公式ガイド](https://docs.sqstack.com/docs/guide/introduction) より）。

お客様が商品を見て買う「売り場（ECサイト・POS）」ではなく、運営スタッフが在庫・受注・商品を管理する **バックヤード（業務管理の裏方）** として動作します。本マニュアルでは現場目線で分かりやすく「バックヤード」とも呼びます。

### SQの位置づけ（外部連携全体図）

```
  │ スマレジ  │                  │          │

  ┌──────────┐    在庫作業      │          │
  │  WMS     │ ←─────────────→ │          │
  │ロジザード │                  │          │
  └──────────┘                  └──────────┘
```

> - **実機メニューの存在** ＝ stagingの左メニュー・URLで確認済み（2026-06-20）。公式ガイドになくても実機にあれば事実として扱います（ロジザードZERO・スマレジが該当）。
> - **実機の連携画面の見出しは製品名または「〇〇連携」のみ**（2026-06-20実機確認）。**役割名（POS/WMS/返品管理/最低価格表示）は実機画面に表記がなく、すべて一般知識です。**
> - **連携で同期されるデータ** は、staging環境が未接続のため実機で確認できていません。Shopify・リテールポータルは公式ガイドで確定、スマレジは実機フォーム項目ベース、ロジザード/Recustomer/OmnibusCoreの役割は一般知識です。
> - **スマレジ** は実機に連携メニュー・設定フォームが存在し入力可能です。接続後の実際の同期挙動はstaging未接続のため未確認です。なお公式ガイドでは「近日連携予定（Coming Soon）」扱いとなっており、実機の実装と公式の表現に差があります。
> - 接続済み環境での検証が必要です（詳細は [19. 標準販売チャネル連携](19-標準販売チャネル連携.md)・[20. 物流・返品・外部アプリ連携](20-物流・返品・外部アプリ連携.md) のTODO参照）。

## 管理画面のメニューサイトマップ（全体図）

管理画面 `https://www.sqstackstaging.com/admin` の左メニューは、大きく **5つのグループ** で構成されます。[公式ガイド](https://docs.sqstack.com/docs/guide/introduction) は「管理 / CRM / 販売チャネル連携 / 設定」の4区分ですが、実機の左メニューに沿って現場作業単位（オペレーション）を独立させた5区分で示します。各グループ見出しに公式ガイドの対応区分を併記します。

> **【基本概念の前提知識】** 以下の用語はSQ全体で頻出する基本概念です。初めて読む方はまずここを確認してください。
>
> - **テナント**：事業部（運用ルールを分ける管理単位）。（詳細は [03. 組織・通知](03-組織・通知.md)）
> - **ロケーション**：在庫の置き場所。場所種別は **「倉庫」または「店舗」の2種のみ**。倉庫＝在庫保管・発注入荷先・出荷元。店舗＝店舗販売・取置・取り寄せの受け口・リテールポータル対象・店舗受取の対象（倉庫はこれらに出ない）。（詳細は [04. 基本マスタ](04-基本マスタ.md)）
> - **SKU（商品バリエーション）**：商品の色・サイズ等のバリエーション1つ1つ。在庫や価格はSKU単位で持つ。
> - **取り寄せ販売**：倉庫↔店舗間で在庫が足りない時に他拠点の在庫を確保して移動する流れ（在庫依頼→引当→移動伝票→出荷/入荷）。（詳細は [13. 入出荷・在庫依頼](13-入出荷・在庫依頼.md)）

| 公式ガイドの区分 | 本マニュアルの対応グループ |
|:--|:--|
| 管理（商品・顧客・在庫・注文・倉庫） | グループ1（主要業務）・グループ2（オペレーション） |
| CRM（ポイント・会員ランク・ディスカウント） | グループ3 |
| 販売チャネル連携（Shopify・リテールポータル・POS） | グループ4 |
| 設定（データ管理・外部連携） | グループ5 |

### グループ1：主要業務（上段）— 公式ガイド「管理」の主要機能 + 発注/販売設定/会計/分析

| メニュー | URL | 何ができるか |
|:--|:--|:--|
| **ホーム** | `/admin` | ダッシュボード。各機能への導線 |
| **商品管理** | `/admin/products` | 商品・SKU・バリエーション・価格の登録・編集。カタログ・店舗受取の設定 |
| **在庫管理** | `/admin/inventory_items` | 在庫数の確認（9区分）、移動伝票・調整伝票・取置伝票の作成 |

> **【在庫区分（9種）】** SQでは在庫を9つの区分で管理します。（詳細は [11. 在庫状態・在庫数](11-在庫状態・在庫数.md)）
>
> | 区分 | 意味 |
> |:--|:--|
> | 販売可能 | EC・POS等で売れる在庫 |
> | 引当済み | 注文・在庫依頼で確保済み（売りに出ない） |
> | 取置中 | 店舗で個別に取って置いた在庫 |
> | 破損 | 汚損・破損等で売れない在庫 |
> | 検品 | 検品中の在庫 |
> | 予備 | 一時的に確保等の予備枠（検品・破損に該当しない予備用）。手持ちに含まれる。**実機未確認**: SQ管理画面からこの区分へ在庫を移す専用メニュー・調整伝票の理由選択肢は見当たらず、WMS連携等で設定される可能性（要確認） |
> | **手持ち** | 上記6区分の合計（販売可能+引当済み+取置中+破損+検品+予備） |
> | 積送中 | 拠点間移動中の在庫（手持ちには含まない） |
> | 入荷予定 | 発注等でこれから入荷する予定数（手持ちには含まない） |
| **注文管理** | `/admin/orders` | 注文の確認・出荷登録（※チャネル接続後に注文が流入） |
| **顧客管理** | `/admin/purchasing_customers` | 購入顧客・会社の確認（※チャネル由来、手動作成不可） |
| **発注管理** | `/admin/inventory_purchase_orders` | 発注伝票の作成・発注・キャンセル（※入荷への自動連動なし）。発注しても入荷予定や入荷伝票には自動で反映されないため、入荷管理で別途入荷指示・実績登録を行う必要がある |
| **販売設定** | `/admin/product_price_rules` | 販売に関する4つのルールを束ねるメニュー：販売価格（SKU別価格）／予約販売／販売上限／販売閾値（各ルールの意味は下記注記） |
| **会計** | `/admin/sale_change_line_items` | 売上実績の確認・CSV出力。注文・出荷から自動生成される実績を参照する読み取り・出力中心のメニュー（会計伝票の手動作成ではない）。**実機未確認**: 出力単位（注/SKU/日次等）・編集可否は未確認 |
| **分析** | `/admin/analytics` | ※現在「収益」「レポート」はTODO表示（未実装） |

> **【販売設定の4ルール（運用上の意味）】** 販売設定は販売に関する4つのルールを束ねるメニューです。それぞれ何を制御するか：
> - **販売価格**: SKU別の販売価格を設定する。
> - **予約販売**: 在庫が0でも指定した「販売数」の範囲で注文を受け付けるルール。SKU単位・数量は固定値で設定、期間指定は不可（受注生産・取り寄せ等で在庫不足時に受注を維持したい場合に使う）。
> - **販売上限**: 特定の販売チャネル（Shopify等）に対しSKUごとに販売できる数量の上限を設けるルール。チャネル接続済みでないとルールを作成できない。
> - **販売閾値**: 在庫が何個を下回ると販売を止めるかのしきい値。SKU別またはデフォルト値で設定。リテールポータルの販売可否判定にも使われる（閾値割れ時の詳細挙動は要確認）。

### グループ5：設定（マスタ・システム設定）— 公式ガイド「設定」

| メニュー | URL | 何ができるか |
|:--|:--|:--|
| **設定** | `/admin/settings` | ロケーション・権限グループ・ブランド・取引先・決済方法・翻訳・採寸・メタフィールド・API/アプリ 等、全マスタ・システム設定 |

> **【設定画面の役割】** 設定はSQの**全マスタとシステム設定を一元管理**する場所です。拠点（ロケーション/ロケーショングループ）、人（管理メンバー・権限グループ・販売員）、取引（ブランド・取引先・決済方法）、運用ルール（翻訳・採寸ルール・メタフィールド定義）、外部連携（API/アプリ）をここで登録・編集します。ここでマスタが未登録だと、各業務画面の選択肢が空になります。詳細は [04. 基本マスタ](04-基本マスタ.md)・[02. アカウント・権限](02-アカウント・権限.md) を参照。
>
> ※ **メタフィールド定義**：商品等に独自の項目を追加できるひな形。（詳細は [08. カスタムデータ](08-カスタムデータ.md)）

## 共通操作パターン

### 一覧画面（ほぼ全機能の入口）

| 共通要素 | 説明 |
|:--|:--|
| タブ | ステータス別に表示を切り替え（例：確認待ち／確保済み／終了） |
| 検索・フィルタ | テキスト検索や条件指定で絞り込み |
| テーブル | 行クリックで詳細画面へ遷移 |
| 「〇〇を作成する」ボタン | 右上に配置。新規作成画面へ遷移 |
| 「インポート」「エクスポート」 | 一括操作の入口（機能により有無・実装状態が異なる） |

### 詳細画面

| 共通要素 | 説明 |
|:--|:--|
| ステータスバッジ | 現在のステータスを色付きバッジで表示 |
| 「その他の操作」 | 詳細画面右上のメニュー。キャンセル・クローズ等の操作が集約 |
| 編集制限 | 一部の伝票（外部連携由来等）は商品編集が制限される場合がある |

### 作成フォーム

| 共通パターン | 説明 |
|:--|:--|
| セクション分割 | 「商品」「リクエスト内容」等のセクションに分かれる |
| 「選択」ボタン | ロケーション・商品等の選択はモーダルで行う（テキスト検索＋選択。SKU検索は入力後Enterで発火） |
| 必須項目 | `*` 印付き。空保存でバリデーションエラー |
| 「保存する」 | セクション内またはページ下部。保存後は詳細画面へ遷移 |

### CSV/PDF入口

| 操作 | 場所 |
|:--|:--|
| CSVインポート | 各一覧画面の「インポート」ボタン（22カテゴリ、機能により実装状態が異なる） |
| CSVエクスポート | 各一覧画面の「条件指定でエクスポート」等（9カテゴリ） |
| PDFエクスポート | 一部の伝票で「PDFエクスポート」ボタン |

> **CSVインポートの注意**: 「実行」は巻き戻し不可。検証→実行の2段階。誤った場合は修正版CSVを新規インポートするか、在庫は調整伝票で補正。

# 13. 入出荷・在庫依頼

> このページはWBS-25エリアの第13エリアです。在庫依頼（取り寄せ販売）を起点として、出荷管理・入荷管理で在庫を移動させる一連の流れを理解するのが目標です。一部の外部連携項目（Shopify・発注伝票経由の入荷連携など）は未確認です。

## このエリアで学べること

- 在庫依頼（取り寄せ）で他のロケーションから在庫を確保する手順を説明できる
- 出荷管理で出荷実績を登録し、ステータスを出荷完了にする方法が分かる
- 入荷管理で入荷実績を登録し、部分入荷・過入荷の扱いを説明できる
- ヤマトB2クラウド/DHLのCSVを使った出荷実績の一括登録が分かる
- 移動伝票が在庫依頼〜出荷〜入荷をどう連動させるかを説明できる

---

## 機能概要

このエリアは「オペレーション」グループに属する3つの機能から成り立ちます。いずれも在庫移動（取り寄せ販売）の一連の流れの中で連動して使います。

| 機能 | 画面URL | 役割 |
|:--|:--|:--|
| 在庫依頼 | `/admin/inventory_allocation_requests` | 取り寄せの起点。他ロケーションの在庫を確保（引当）する |
| 出荷管理 | `/admin/inventory_outbound_orders` | 移動元ロケーションからの出荷実績を登録する |
| 入荷管理 | `/admin/inventory_inbound_orders` | 移動先ロケーションへの入荷実績を登録する |

### できること

- **在庫依頼**: 商品・希望数・移動先・引当元を指定して在庫確保を依頼。引当後に移動伝票を一括作成
- **出荷管理**: 出荷指示の一覧をステータスタブで確認。出荷実績（配送キャリア・追跡コード）の個別登録、またはCSV（ヤマトB2クラウド・DHL）で一括登録
- **入荷管理**: 入荷指示の一覧をステータスタブで確認。入荷実績を登録して完了にする

### 全体の流れ

```
[在庫依頼を作成]
     ↓ ステータス: 確認待ち
[在庫を引当てる] → ステータス: 確保済み
     ↓ 確保済み一覧から
[移動伝票を作成] → 出荷指示・入荷指示が自動生成
     ↓
[出荷実績を登録] / [入荷実績を登録] （順不同・並行可）
     ↓ 両方完了で
移動伝票が「完了」
```

## 主な操作手順

### 手順1: 在庫依頼を作成する

1. 左メニューの「在庫依頼」をクリックし、一覧画面を開く。
2. 「在庫依頼を作成する」ボタンをクリックする。作成フォームへ遷移する。
3. 「商品バリエーション」欄の「選択」ボタンをクリックし、移動したい商品バリエーションを検索して選ぶ。
4. 「希望数」に移動させたい数量を入力する。
5. 「移動先ロケーション」欄の「選択」ボタンをクリックし、在庫の届け先ロケーション（例: 店舗）を選ぶ。
6. 「リクエスト先ロケーション」欄の「選択」ボタンをクリックし、引当元ロケーション（例: 倉庫）を選ぶ。
7. 「保存する」ボタンをクリックする。
8. 詳細画面に遷移し、ステータスが「確認待ち」になったことを確認する。

### 手順2: 在庫を引当てる（確保する）

1. 在庫依頼の詳細画面（ステータス: 確認待ち）で、画面右上の「在庫を引当てる」ボタンをクリックする。ダイアログが開く。
2. 「ロケーション」でリクエスト先ロケーション（例: 物流倉庫）を確認する。
3. 「数量」に確保する数量を入力する（最大値は希望数まで）。
4. 「引当てる」ボタンをクリックする。
5. ステータスが「確保済み」に変わり、「確保済み数量」が希望数と一致したことを確認する。

> **部分引当の場合**: 引当元在庫が足りないときは、引当可能な分だけ引当てられます。この場合、依頼ステータスは希望数全量を引当てるまで「確認待ち」のままですが、確保済み一覧には引当てた分だけ表示され、その分の移動伝票を作成できます（2026-06-20実機確認）。

### 手順3: 移動伝票を作成する

移動伝票を作成すると、同時に **出荷指示と入荷指示が自動生成** されます。

1. 左メニューの「在庫依頼」→「確保済み」をクリックし、確保済み一覧を開く。
2. 一覧に対象の引当行が表示されていることを確認する。
3. 画面右上の「移動伝票を作成する」ボタンをクリックする。ダイアログが開く。
4. 「移動元」欄のボタンをクリックし、出発元ロケーションを選ぶ。
5. 「実行する」ボタンをクリックする（移動元が未選択の場合はグレーアウトして押せない）。
6. 「移動伝票を一括作成しました」というメッセージが表示され、対象行が一覧から消えることを確認する。
7. 左メニューの「出荷管理」「入荷管理」に未完了件数のバッジが出ていることを確認する。

![「移動伝票を作成する」ダイアログ](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/flow-03-movement-dialog.png)

### 手順4: 出荷実績を登録する

1. 左メニューの「出荷管理」をクリックし、一覧画面を開く。
2. 「出荷待ち」タブに対象の出荷指示があることを確認する。
3. 出荷指示の行をクリックして詳細画面を開く。商品・数量・配送元・配送先ロケーションを確認する。
4. 画面右上の「出荷実績を登録する」ボタンをクリックする。ダイアログが開く。
5. 必要に応じて「配送キャリア」と「追跡コード」を入力する（いずれも任意。空でも登録可）。
6. 「登録する」ボタンをクリックする。
7. 「出荷を完了しました」というメッセージが表示され、ステータスが「出荷完了」になったことを確認する。

![出荷完了後の詳細画面](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/flow-06-outbound-completed.png)

### 手順5: 入荷実績を登録する

1. 左メニューの「入荷管理」をクリックし、一覧画面を開く。
2. 「入荷待ち」タブに対象の入荷指示があることを確認する。
3. 入荷指示の行をクリックして詳細画面を開く。種別が「移動」であることを確認する。商品・数量・ロケーションが正しいことを確認する。
4. 画面右上の「入荷実績を登録する」または「入荷指示を一括受領で完了する」をクリックする。入荷実績登録フォーム、または完了確認画面が開く。
5. 「この処理は巻き戻すことができません。」の説明文を確認する。
6. 数量入力画面の場合は受領数を確認し「登録する」をクリックする。一括受領・完了更新画面の場合は内容を確認して実行する。
7. 「入荷実績を登録しました」というメッセージが表示され、ステータスが「入荷完了」になったことを確認する。
8. 移動伝票の詳細画面を開き、ステータスバッジに「完了」と「入荷完了」が並んで表示されることを確認する。

![入荷完了後の詳細画面](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/flow-05-inbound-completed.png)

> **出荷と入荷は順不同**: ステップ4とステップ5は並行して処理できます。出荷前に入荷を完了しても問題ありません（2026-06-18実機確認）。両方完了してはじめて親移動伝票が完了します。

### 各ステップの確認ポイント

| 確認タイミング | 見るべき箇所 | 期待する状態 |
|:--|:--|:--|
| 手順1完了後 | 在庫依頼 詳細 > ステータス | 確認待ち |
| 手順2完了後 | 在庫依頼 詳細 > ステータス | 確保済み |
| 手順3完了後 | 確保済み一覧 | 対象行が消えている |
| 手順3完了後 | 左メニュー 出荷管理 / 入荷管理 | 未完了件数のバッジ |
| 手順4完了後 | 出荷管理 詳細 > ステータス | 出荷完了 |
| 手順5完了後 | 入荷管理 詳細 > ステータス | 入荷完了 |
| 手順5完了後 | 移動伝票 詳細 > ステータス | 完了 / 入荷完了（2つ並び） |

### 途中で中止したいとき（クローズ操作）

| 在庫依頼のステータス | クローズ操作 |
|:--|:--|
| **確認待ち**（引当て前） | 詳細画面 > 「その他の操作」 > 「在庫依頼をクローズする」で「終了」にできる |
| **確保済み**（引当て後） | クローズ不可。「在庫依頼をクローズする」がグレーアウトして押せない |

## 注意点・制約

### 実績登録は取り消し不可（不可逆）

出荷実績・入荷実績を登録すると**ステータスを巻き戻すことはできません**。入荷実績登録ダイアログにも「この処理は巻き戻すことができません。」と明記されています。登録前に商品・ロケーション・数量を必ず確認してください（実機確認済み）。

- 出荷完了後：「出荷実績を登録する」ボタンが無効化され、画面から再登録・編集・削除は不可
- 入荷完了後：「入荷実績を登録する」ボタンが無効化され、画面から再登録・編集・削除は不可
- 誤って登録した場合は調整伝票で在庫数を修正してください

### 移動伝票のキャンセルは行き止まりになる

移動伝票をキャンセルしても、**元の在庫依頼は「確保済み」のまま残ります**。キャンセル後は移動伝票の再作成も、在庫依頼のクローズもできません。同じ取り寄せをやり直す場合は、在庫依頼を新規作成してください（実機確認済み）。

確認ダイアログ文言: 「移動伝票をキャンセルすると、この伝票に紐づく入荷予定および出荷指示もすべてキャンセルされます。」

### 在庫はマイナスになり得る

配送元ロケーションの表示在庫が0のSKUでも、出荷実績登録はブロックされません。SQは在庫不足を出荷時に止めないため、実在庫を超える出荷登録をすると在庫数がマイナスになります（2026-06-18実機確認: `#IM-1016`）。登録前にSKU・ロケーション・数量を確認してください。

### 出荷指示の引当ステータスは無視してよい

移動伝票経由で生成された出荷指示は、引当ステータスが「引当待ち」のまま推移します（出荷実績登録・出荷完了になっても変わりません）。出荷完了の判断は、出荷実績の登録有無と「出荷完了」ステータスを優先してください（2026-06-20実機確認: `#IO-1018`）。

### リクエスト先ロケーションは引当に実質必須

在庫依頼作成時に「リクエスト先ロケーション」を未指定でも保存は可能ですが、その依頼を引当しようとすると引当ダイアログの「引当元ロケーション」選択肢が空になり引当不可になります（エラー「引当元ロケーションを選択してください」）。引当まで進める場合はリクエスト先の指定が必要です（2026-06-20実機確認）。

### 入荷・出荷の新規作成ボタンはない

入荷管理・出荷管理画面には入荷指示・出荷指示を新規作成するボタンがありません。これらは移動伝票の作成と同時に自動生成されます。

### 確保済みと在庫区分「取置中」は別物

在庫依頼の「確保済み数量」は「在庫を引当てる」を実行した時点で増えますが、この時点ではSKU在庫詳細の販売可能・取置中・手持ちは変化しません。在庫が実際に移動するのは、確保済み一覧から移動伝票を作成し、出荷/入荷の実績登録を進めた後です。

在庫数が変化するタイミング:

| 操作 | 在庫への影響 |
|:--|:--|
| 在庫依頼を作成（確認待ち） | 変化なし |
| 在庫を引当てる（確保済みへ） | 変化なし |
| 移動伝票を作成 | 移動元の「販売可能」が減り「取置中」が同数増える（手持ちは不変。移動先には「入荷予定」が立つ） |
| 出荷実績を登録 | 移動元の手持ちが減る |
| 入荷実績を登録 | 移動先の手持ちが増える |

### 外部連携由来の移動伝票は編集制限あり

確保済み一覧から作成された移動伝票は「外部システムから連携されている移動伝票です」と表示され、商品編集が制限されます。ただし出荷実績登録・入荷一括受領・移動完了までは管理画面で実行できます。

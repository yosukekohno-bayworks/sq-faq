# 06. カタログ

> このページは標準24分類の第6分類です。商品をグループ化する **カタログ** と、商品につける **ブランド** の役割の違いを理解するのが目標です。販売チャネルごとの出品・同期対象、リテールポータル等で扱う商品集合、外部在庫同期での対象商品指定は、カタログを参照する各設定側で扱います。なお、商品管理 > 店舗受取のSKU追加画面にはカタログ選択項目はありません（2026-06-28実機確認）。

## このエリアで学べること

- カタログとブランドの違いを説明できる
- カタログを作成し、商品を手動・自動で追加できる
- 分類条件（製造元・ブランドコード）で商品を自動追加するルールを設定できる
- カタログ単体では出品・同期・店舗受取を実行せず、別の設定画面から参照されることを理解できる

---

## 機能概要

### カタログとは

カタログは **商品を束ねる汎用的な商品グループ** です。実機の作成フォーム（`/admin/catalogs/create`）で入力するのはタイトルのみで、カタログ単体に出品範囲・同期範囲・店舗受取可否・在庫出力条件を設定する項目はありません。

カタログを「何に使うか」は、**カタログを参照する別の設定画面**で決まります。例えばOmnibusCore連携なら `/admin/omnibus_core_integrations/create` の **「商品同期設定 > カタログ」** で、スマレジ連携なら `/admin/smaregi_integrations/create` の **「商品連携設定 > カタログ」** で当カタログを選択します。Shopify / OmnibusCore / スマレジ / リテールポータル側での対象商品指定は [20. 標準販売チャネル連携](20-標準販売チャネル連携.md) と [19. 店舗業務・リテールポータル](19-店舗業務・リテールポータル.md) を参照してください。

> **参照先ごとのカタログ扱い（2026-06-27実機再確認）**:
> - **Shopify連携**: `/admin/shopify_integrations/create` で `カタログ*` が必須。
> - **OmnibusCore連携**: `/admin/omnibus_core_integrations/create` の `商品同期設定 > カタログ` で選択可能。
> - **スマレジ連携**: `/admin/smaregi_integrations/create` の `商品連携設定 > カタログ` で選択可能。同じフォームに `在庫設定 > 在庫同期の方向` があり、`SQからスマレジへ在庫を同期する` を選べる。
> - **リテールポータル連携**: `/admin/retail_portal_integrations/create` で `カタログ*` が必須。
> - 実際にカタログ内の商品だけが各外部システムへ同期されるか、同期頻度・失敗時挙動は **連携接続後の確認待ち** です。

**データの流れ（全体像）**:

```
商品マスタ(/admin/products)
        │  手動追加 / 自動追加ルール / CSV
        ▼
   カタログ(/admin/catalogs)          ← 「どの商品を束ねるか」
   例: EC連携用 / 店舗受取用 / スマレジ在庫同期用
        │  各連携・業務設定画面でカタログを選択
        ▼
   各連携・業務設定                    ← 「どの用途でどのカタログを使うか」
   (Shopify / OmnibusCore / スマレジ / リテールポータル 等)
        ▼
   同期先・利用先（ECサイト / POS / 店舗端末 等）
```

> 実機フォーム上は、用途ごとの連携・業務設定画面でカタログを選択する構成です。同じカタログを複数用途に選べるか、用途ごとに別カタログを使うかの実運用は、接続済み環境での保存・同期結果まで確認が必要です。いずれの場合も、割り当て操作は本ページのカタログ画面ではなく、各連携・業務設定画面で行います。

### カタログの主な利用例

| 利用例 | カタログの役割 | 実機で確認できた設定画面 |
|:--|:--|:--|
| 販売チャネルの商品同期 | どの商品をShopify / OmnibusCore / スマレジへ連携対象にするかを表す商品集合 | Shopify / OmnibusCore / スマレジ連携作成フォーム |
| リテールポータルで扱う商品 | 店舗スタッフ画面で扱う商品集合 | リテールポータル連携作成フォーム |
| 店舗受取できる商品のグルーピング | リテールポータル連携などで扱う商品集合を束ねる利用例。商品管理 > 店舗受取の追加ダイアログにはカタログ選択項目はなく、商品側の受取対象はSKUを直接指定する | カタログ、商品管理 > 店舗受取、リテールポータル > 店舗受取 |
| 外部システムへ在庫数を同期する対象商品 | 外部在庫同期で「どの商品を対象にするか」を表す商品集合。実機上はスマレジ連携で `カタログ` と `SQからスマレジへ在庫を同期する` が同じフォームにある | スマレジ連携作成フォーム |

> **CSV在庫エクスポートとの違い**: CSVエクスポートの「在庫」カテゴリ（`/admin/csv_export/csv_export_operation_inventory_logical_quantities/create`）はロケーション指定のみで、カタログ指定はありません。打ち合わせで出た「外部に在庫数を出力する対象商品一覧」は、CSV出力ではなく外部連携の在庫同期設定として扱うのが実機上は自然です。

### ブランドとの違い

| | カタログ | ブランド |
|:--|:--|:--|
| 役割 | 商品をグループ化する。連携・店舗受取・外部在庫同期などの設定側で参照される商品集合になる | 商品につける属性の1つ（製造元・ブランド名） |
| 登録場所 | `/admin/catalogs` | `/admin/settings/brands` |
| 商品との関係 | 商品を複数カタログに追加できる | 商品マスタでブランドを選択する |
| 用途 | 商品集合の管理。チャネル連携・リテールポータル・外部在庫同期などで対象商品を指定するために使う | 商品の分類・ラベル表示 |

> **注意**: 商品マスタの「ブランド」（`/admin/settings/brands`）と、設定の「事業部（テナント＝運用ルール分離単位。詳細は03.組織・通知）」で扱われるブランド概念は別データです。本エリアでは商品マスタ側のブランドを扱います。

## 画面・項目の説明

### カタログ一覧（/admin/catalogs）

| 列名 | 内容 |
|:--|:--|
| 名前 | カタログ名。連携・業務設定側の「カタログ」コンボボックスに表示される識別名になるため、用途が分かる名前（例: EC連携用、店舗受取用、スマレジ在庫同期用）を付ける。2026-06-28再確認では同名カタログも保存できたため、名前は一意制約ではありません |
| 商品 | カタログに登録されている商品数（例: 0個の商品） |
| 販売先 | このカタログを参照しているチャネル連携の数（例: 0つの販売先） |

### 自動追加ルール（/admin/catalogs/{id}/automatic_add_rules）

条件に一致する商品を自動的にカタログへ追加するルールを設定できます。「追加する」ボタンで「ルールを追加する」ダイアログが開きます。

![「ルールを追加する」ダイアログ](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/03-catalog-auto-rule-dialog.png)

| 項目 | 選択肢 |
|:--|:--|
| 項目 | 「製造元」「ブランドコード」の2択 |
| 条件 | 「一致する」の1択 |
| 値 | テキストで自由入力 |

設定したルールと一致する商品がカタログに追加されます。2026-06-28再確認では、ルール作成後に製造元が一致する商品を新規作成すると、その商品がカタログへ自動追加されました。一方、ルール作成時点で既に存在していた一致商品は、作成直後・10秒後・60秒後の再読み込みでは追加されませんでした。その既存商品を商品詳細で保存し直すとカタログへ追加されたため、自動追加ルールはルール作成時の一括遡及ではなく、商品作成・商品更新のタイミングで評価される挙動として案内します。

> **ルールを機能させる前提**:
> - **製造元**: 商品マスタ（商品管理 > 商品）の「製造元」欄に、ルール値と完全一致する文字列が入力済みであることが前提。未入力の商品は自動追加されない。
> - **ブランドコード**: ブランドマスタ（設定 > ブランド）の「コード」値を入力する（ブランド名ではない）。商品が参照しているブランドのコードと完全一致した商品だけが追加される。
> - **値**: 1ルールにつき単一の文字列。条件選択肢は「一致する」のみです。2026-06-28再確認では、完全一致する新規商品と、ルール作成後に保存し直した既存商品が自動追加されました。同一項目（製造元）の複数ルールはOR条件として動き、3ルール中1件に一致する新規商品は自動追加されました。さらに製造元ルールとブランドコードルールの混在でもOR条件として動き、製造元だけ一致・ブランドコードだけ一致・両方一致の商品はいずれも追加されました。前方一致だけの商品（ルール値 `TEST_VENDOR_PREFIX...` に対して製造元 `TEST_VENDOR_PREFIX..._EXTRA`）は追加されませんでした。カンマは複数値指定ではなく文字列の一部として扱われ、`*` もワイルドカードではなく文字列 `*` そのものとして扱われます。

> **用語注記**:
> - **製造元** = 商品マスタ（商品管理 > 商品）で入力する自由テキスト項目。ルールは**完全一致判定**で、大文字小文字・スペース違いも別物扱い。ブランドコードとは別（製造元は商品につける自由テキスト、ブランドコードはブランドマスタ`/admin/settings/brands`のコード）。
> - **ブランドコード** = ブランドマスタ（設定 > ブランド）のコード値。商品が参照しているブランドのコードと突合する。

登録済みルールは「項目」「条件」「値」の一覧で確認できます。行を選択すると「ルールを削除する」ボタンが表示され、確認ダイアログ「ルールを削除する」（本文: 「選択された1件のルールを削除します。この処理は巻き戻すことができません。」）から削除できます。削除後のトーストは `ルールを削除しました` です。

> **ルール削除後の商品**: 2026-06-28再確認では、ルール削除後も、既に自動追加済みの商品はカタログに残りました。商品本体を削除すると、カタログの商品行も消えます。

### CSVインポート画面

「その他の操作 > インポート」からカタログの商品を一括追加・削除できます。

| 項目 | 必須 | 説明 |
|:--|:--|:--|
| カタログ | 必須 | ドロップダウンで既存カタログを選択 |
| ファイルをアップロード | 必須 | CSVファイルをアップロード（ファイル選択またはドラッグ&ドロップ） |

ページ上部にテンプレートファイル（Googleスプレッドシート）へのリンクが表示されます。2026-06-28実機確認時点のテンプレート列は以下です。

| 列 | 必須 | 内容 |
|:--|:--|:--|
| `product_code` | 必須 | 既存商品の商品コード |
| `command` | 必須 | `NEW` でカタログへ追加、`DELETE` でカタログから削除 |

### ブランド作成フォーム（/admin/settings/brands/create）

| 項目（UIラベル） | 必須 | 補足 |
|:--|:--|:--|
| 名前* | 必須 | — |
| 外部ID | 任意 | **作成時のみ設定可。保存後の変更不可**。外部システム連携用の識別子。自社で外部システムとブランドをコードで突き合わせる場合のみ設定し、使わない場合は空欄でよい。保存後変更不可のため修正時は削除→再作成になる（連携先の具体名は実機未確認） |
| コード* | 必須 | 自動追加ルール「ブランドコード」と販売閾値ルールの完全一致判定に使う値。一意で分かりやすいコード（例: brand-shibuya）を推奨 |

保存後の詳細・編集フォームには「外部ID」フィールドは表示されません。

## 主な操作手順

### カタログを作成して商品を手動で追加する

1. 左メニューから **カタログ**（`/admin/catalogs`）を開く
2. 右上の **カタログを作成する** をクリックする
3. カタログ名を入力し、保存する
4. カタログ詳細画面が開くので、**商品を追加する** をクリックする
5. 「商品コードで検索する」で商品を絞り込むか、一覧から目的の商品を探す
6. 商品行のチェックボックスをオンにする（一括選択も可）
7. **カタログに追加する** をクリックする

### 自動追加ルールを設定する

1. カタログ詳細画面で **自動追加ルール** をクリックする
2. **追加する** をクリックし「ルールを追加する」ダイアログを開く
3. 項目で「製造元」または「ブランドコード」を選ぶ
4. 条件は「一致する」のまま、値に条件テキストを入力する
5. **追加する** をクリックする

> 設定後、条件に一致する新規商品や、商品詳細で保存し直した既存商品がカタログへ追加されます。ルール作成時点の既存商品をまとめて追加する用途では、手動追加またはCSVインポートを使います。

### カタログ名を変更する

1. カタログ詳細画面で **編集する** をクリックする
2. 「カタログを編集する」ダイアログで **タイトル*** を変更する
3. 保存する

### カタログを削除する

1. カタログ詳細画面で **その他の操作** を開く
2. **カタログを削除する** をクリックする
3. 確認ダイアログ「カタログを削除しますか？」（本文: 「カタログを削除します。この処理は巻き戻すことができません。」）で **削除する** をクリックする

### CSVで商品を一括追加・削除する

1. カタログ詳細画面で **その他の操作 > インポート** をクリックする
2. インポート画面（`/admin/csv_import/csv_import_operation_catalog_products`）で **カタログ** を選択する
3. テンプレートリンクからフォーマットを確認し、CSVを作成する
   - 追加: `product_code,command` の `command` に `NEW` を入れる
   - 削除: `product_code,command` の `command` に `DELETE` を入れる
4. **ファイルをアップロード** でCSVをアップロードする
5. 保存後、検証を経て **実行する** で反映する

2026-06-28実機確認では、検証用商品1件を `NEW` でカタログへ追加し、同じ商品を `DELETE` でカタログから削除できました。どちらも詳細画面で `検証ステータス: 成功 完了` / `実行ステータス: 未実行` / `検証成功: 1件の商品` / `検証失敗: 0件の商品` になり、`実行する` 確定後に `実行ステータス: 成功 完了` になりました。確認ダイアログ本文は `1件のカタログ商品を登録します。この操作は巻き戻すことができません。` です。検証用の商品・カタログは削除済みです。

> **注意**: CSVインポートの「実行」は巻き戻し不可です。検証→実行の2段階で進み、誤った場合は反対操作（例: 誤って追加した商品は `DELETE`、誤って削除した商品は `NEW`）の修正版CSVを新規インポートしてください。

### 自動追加ルールを削除する

1. カタログ詳細画面で **自動追加ルール** を開く
2. 削除したいルール行を選択する
3. **ルールを削除する** をクリックする
4. 確認ダイアログで **削除する** をクリックする

## 注意点・制約

- **カタログの削除は復元不可**: 「その他の操作 > カタログを削除する」から行います。削除前に商品との紐付けを確認してください（確認ダイアログに「この処理は巻き戻すことができません。」と表示）。
- **カタログの用途指定は参照先側で設定**: カタログをどのチャネル・業務・外部同期に使うかは、Shopify / OmnibusCore / スマレジ / リテールポータルなどの設定側で決めます。カタログ詳細画面から直接「販売先」や「店舗受取対象」「在庫同期対象」を設定するUIはありません。
- **CSVインポートの実行は不可逆**: 「実行」は巻き戻し不可。検証→実行の2段階で進めます（参照: 制約台帳「CSVインポートの実行は取り消し不可」）。
- **ブランドの「外部ID」は作成時のみ設定可**: 保存後の変更はできません（参照: 制約台帳「ブランドの外部IDは作成時のみ設定可」）。
- **自動追加ルール削除は不可逆**: 確認ダイアログに「この処理は巻き戻すことができません。」と表示されます。

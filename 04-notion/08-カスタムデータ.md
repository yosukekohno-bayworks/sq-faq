# 08. カスタムデータ

> このページは標準24分類の第8分類です。SQの標準項目だけでは足りない「追加情報」を、どこに・どう持たせるかを理解するのが目標です。

## このエリアで学べること

- 標準項目（商品名・価格・在庫数など）以外の「独自の情報」をどこに持たせるか説明できる
- メタフィールド定義で、どのオブジェクトにどんな型の項目を追加できるか把握できる
- ネームスペース（namespace）とキー（key）の組み合わせで、追加項目が一意に識別される仕組みが分かる
- メタフィールドが連携先（Shopifyなど）でどう扱われるかの前提が分かる

---

## 機能概要

### カスタムデータとは

SQには、あらかじめ用意された標準項目（商品名・SKUコード・在庫数・価格など）があります。しかし業務によっては「このSKUはギフト対応可か」「この商品の素材補足」「この顧客の自社会員番号」「この注文の内部メモ」など、**標準項目にはない独自の情報**を持たせたい場合があります。

SQでは、こうした追加情報を **メタフィールド** として任意のオブジェクトに付与できます。管理者は **メタフィールド定義**（追加項目の「型」と「名前」のひな形）を作成し、対象オブジェクトごとに値を持たせます。管理画面での表示・編集導線はオブジェクトによって異なります。

**定義と値の関係（イメージ）**

```
メタフィールド定義（ひな形: 1件）
  ├── SKU A → 値: true（ギフト対応可）
  ├── SKU B → 値: false（ギフト対応不可）
  └── SKU C → 値: （未入力）
```

- 定義は「ひな形」1件。値は各オブジェクトが個別に持つ（定義が値を直接持つわけではない）
- 定義一覧画面（`/admin/settings/metafield_definitions`）では、各定義が **「定義名 / ネームスペース.キー」の2行** で表示される（根拠: SQ完全ガイド.md）

> **用語**: **SKU（商品バリエーション）** は商品の色・サイズ等のバリエーション1つ1つ。在庫や価格はSKU単位で持つ（詳細は12.在庫状態・在庫数）。**ロケーション** は在庫の置き場所で、場所種別は「倉庫」または「店舗」の2種のみ（詳細は04.基本マスタ）。これらはメタフィールドの追加対象オブジェクトにも含まれます。

### 対象画面

| 画面 | URL | 役割 |
|:--|:--|:--|
| メタフィールド定義 一覧 | `/admin/settings/metafield_definitions` | 定義の一覧表示・管理 |
| メタフィールド定義 対象別一覧 | `/admin/settings/metafield_definitions?ownerType=<対象オブジェクト>`（例: `?ownerType=product_variant`） | 対象オブジェクト別の定義一覧。`/admin/settings/metafield_definitions/<対象オブジェクト>` は2026-06-28実機では「このページは存在しないようです」 |
| メタフィールド定義 作成 | `/admin/settings/metafield_definitions/<対象オブジェクト>/create`（例: `/admin/settings/metafield_definitions/organization/create`） | 新しい定義の作成。作成URLは対象オブジェクト別に分岐する。`/admin/settings/metafield_definitions/create`（オブジェクト指定なし）は存在しない（実機で「このページは存在しないようです」。2026-06-24確認） |

> 設定画面（`/admin/settings`）の「カスタムデータグループ」配下に位置づけられています。同グループには「翻訳」「採寸ルール」もありますが、本エリアではメタフィールド定義を主に扱います。

### 何ができるか

- メタフィールド定義画面の対象一覧には **16種のオブジェクト** が表示される
- テキスト・数値・日付・JSONなど **12種の型** から選択できる
- 2026-06-28〜2026-07-01実機確認では、同条件で定義作成できたのは9種（組織・ロケーション・会社・商品・バリエーション・顧客・注文・下書き注文・ディスカウント）
- 詳細画面にメタフィールド欄が出たのは7種（ロケーション・会社・商品・バリエーション・顧客・注文・ディスカウント）。組織は専用詳細画面なし。下書き注文は定義作成・一覧・削除までは確認済みだが、下書き注文本体の作成/詳細画面が開けないため値入力画面は確認できない

## 画面・項目の説明

### メタフィールド定義 作成フォーム

![メタフィールド定義 作成フォーム](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/bfg-G-01-metafield-create-form.png)

フォームのパス: `/admin/settings/metafield_definitions/<対象オブジェクト>/create`（例: 組織は `/admin/settings/metafield_definitions/organization/create`。実機確認 2026-06-24。見出しは「組織メタフィールドの定義を追加する」）。オブジェクト指定なしの `/admin/settings/metafield_definitions/create` は存在しない（「このページは存在しないようです」）。

| 項目（UIの原文） | 必須 | 制約・補足 |
|:--|:--|:--|
| 名前 * | 必須 | 最大50文字 |
| 説明 | 任意 | 最大100文字 |
| ネームスペース * | 必須 | 最大50文字。追加項目を目的で束ねる名前空間（例: 仕入関連は `purchasing`、社内メモは `internal`）。同じネームスペースに複数キーをまとめられ、`ネームスペース.キー` で参照する。2026-06-27実機では3文字の `abc` でも保存できたため、4文字以上必須とは断定しない |
| キー * | 必須 | 最大50文字。項目を識別するID（例: `supplier_code`）。ネームスペース内で一意にし、`ネームスペース.キー` で全体で一意になる。2026-06-27実機では3文字の `def` でも保存できたため、4文字以上必須とは断定しない |
| メタフィールドのタイプ * | 必須 | コンボボックス（下記12種から選択）。入力欄の形式を決める（用途目安は下表） |

空保存時のエラーは `定義名を入力してください` / `ネームスペースを入力してください` / `キーを入力してください` / `タイプを選択してください` です。

| ディスカウント | 表示あり | 作成可 | ディスカウント詳細に `メタフィールド <定義名> 未設定` が出る |

### メタフィールドのタイプ（12種）

| 型（UIの原文） | 用途の例 / いつ選ぶか |
|:--|:--|
| 単一行のテキスト | メモ・コード・短いラベル。短い文字情報を1つ持たせたい場合の既定 |
| 単一行のテキスト(リスト) | タグのような複数値（文字列）。1項目に複数の文字列を持たせたい場合 |
| 複数行のテキスト | 長文の備考・説明。改行を含む長文備考用 |
| 日付と時刻 | 発注日時・対応期限（時刻付き）。時刻まで記録したい場合 |
| 日付 | 関連する年月日。日付だけ記録したい場合 |
| trueまたはfalse | フラグ（オン/オフ）。ギフト対応可否、要対応、確認済など2値の状態を持たせる場合 |
| バーコード(NW7) | NW7形式のバーコード値。バーコード読取値をそのまま保持したい場合 |
| 整数 | 個数・回数など小数のない数値 |
| 小数 | 寸法・割合など小数を含む数値 |
| JSON | 構造化データ。1値では表現できない複数項目をまとめたい場合（例: サイズ表 `{"s":{"w":50,"h":70}}` や連携用パラメータ） |
| ID | 別データの識別子。関連商品のID・外部システムのSKU IDなど、他データを文字列で参照したい場合（検証・入力補助は無く自由入力） |
| リッチテキスト | 書式付きの長文。太字・リスト等の書式を含む説明文用 |

> 作成フォームのDOM上は `ネームスペース` / `キー` に `minlength=4` が付いていますが、2026-06-28再確認では `abc.def` の3文字識別子でも保存できました。保存後は定義の編集導線自体が確認できないため、管理画面UI上で型を変更する導線もありません。

### ネームスペースとキー（識別子）

メタフィールドの識別子は **「ネームスペース.キー」** の形式で管理されます。

- 例: ネームスペース `custom`、キー `note` → 識別子は `custom.note`
- ネームスペース・キーともに **最大50文字**。2026-06-27実機では `abc.def` のように3文字同士でも保存できたため、4文字以上必須という制約は現行UIでは確認できません
- ネームスペースは、複数の関連項目をグループ化する名前空間として使います

## 主な操作手順

### メタフィールド定義を新規作成する

1. 左メニューの「設定」を開き `/admin/settings` へ遷移する
2. 「カスタムデータグループ」の **メタフィールド定義**（`/admin/settings/metafield_definitions`）を開く
3. 一覧画面で対象オブジェクト（組織・ロケーション・会社・仕入れ先ベンダー・商品・バリエーション・顧客・注文・下書き注文・ディスカウント・在庫移動伝票・在庫調整伝票・在庫取置伝票・発注伝票・入荷指示・出荷指示）を選び、**定義を追加する** ボタンから対象オブジェクト別の作成フォーム（例: `/admin/settings/metafield_definitions/organization/create`）へ遷移する
4. 「名前」を入力する（最大50文字・必須）
5. 必要に応じて「説明」を入力する（最大100文字・任意）
6. 「ネームスペース」を入力する（最大50文字・必須）
7. 「キー」を入力する（最大50文字・必須）
8. 「メタフィールドのタイプ」コンボボックスから12種のいずれかを選ぶ（必須）
9. **保存する** ボタンを押して定義を作成する

### 作成後に何が起こるか

定義を保存すると、対応している対象オブジェクトの詳細画面に **「メタフィールド」セクション** と定義名が表示され、値は初期状態で `未設定` になります。ただし、すべての対象で詳細画面に出るわけではありません。2026-06-28実機では、ロケーション・会社・商品・バリエーション・顧客・注文・ディスカウントで表示を確認しました。

| タイミング | 結果 |
|:--|:--|
| 商品メタフィールド定義を保存した直後 | 商品詳細画面に「メタフィールド」セクションが表示される |
| バリエーションメタフィールド定義を保存した直後 | SKU/バリエーション詳細画面に「メタフィールド」セクションが表示され、定義名と `未設定` が出る |
| 組織・下書き注文 | 定義一覧には出る。組織は専用詳細画面なし。下書き注文は2026-07-01再確認で定義作成・一覧表示・削除は成功したが、下書き注文一覧は空、作成ボタンは遷移せず、直URL `/admin/draft_orders/create` は予期せぬエラーのため値入力画面を確認できない |
| 仕入れ先ベンダー・在庫系6対象 | 対象別一覧には出るが、2026-06-28の同条件保存では作成自体がサーバーエラー |

#### 各オブジェクトで値を入力・保存する

定義が作るのは「ひな形」です。実際の値は、各オブジェクト（商品・注文など）の詳細画面で入力します。

1. 対象オブジェクトの詳細画面を開く（確認済み例: 商品詳細）
2. 「メタフィールド」セクションに表示されたフィールドの右側にある **編集ボタン（鉛筆アイコン）** をクリックする
3. フィールド名をタイトルにしたモーダルが開くため、値を入力して **保存する** を押す

2026-06-27実機では、商品メタフィールド定義 `TEST_META_SHORT_20260627` を作成し、商品詳細で値 `TEST_VALUE_20260627` を保存できました。保存後は商品詳細の表示が `未設定` から入力値に変わり、メタフィールド定義一覧の「使用箇所」は `0個の商品` から `1個の商品` に変わりました。未設定状態の値編集モーダルでは `削除する` はdisabledで、`キャンセル` / `保存する` が表示されます。保存後のトーストは `フィールドを更新しました` です。

> 根拠: 2026-06-27 SQ staging実機確認。フィールド右側の編集ボタン＝鉛筆アイコンで値を入力。

![商品詳細のメタフィールドセクション](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/bfg-G-04-product-metafield-section.png)

### 他機能との連携

作成したメタフィールド定義は、本エリア外の機能でも次のように使われます（根拠: SQ完全ガイド.md）。

| 連携先 | 使われ方 |
|:--|:--|
| 顧客一覧の絞り込み | 顧客の定義は、一覧の「絞り込みを追加」→「メタフィールド」で条件として使える |
| CSVインポート（一括投入） | 商品・バリエーションの定義は、CSVインポートの「商品」「商品バリエーション」カテゴリで値を一括登録・更新できる |
| 翻訳ルール | 商品・バリエーションの定義は、翻訳ルールの「翻訳データを自動で作成する」で翻訳対象として扱われる |

## 注意点・制約

- **ネームスペース・キーの文字数制限**: ともに **最大50文字** です。過去資料では4文字以上としていましたが、2026-06-27の実機確認では `abc.def`（3文字同士）で保存できました。最小文字数は断定しません。
- **対象一覧にある=保存できる、ではない**: 対象別一覧には16種が表示されますが、2026-06-28実機では、仕入れ先ベンダー・在庫移動伝票・在庫調整伝票・在庫取置伝票・発注伝票・入荷指示・出荷指示は `metafieldDefinitionCreate` がサーバーエラーとなり定義作成できませんでした。
- **名前の文字数制限**: 最大50文字です。
- **説明の文字数制限**: 最大100文字です。
- **定義と値の関係**: メタフィールド定義は「項目の型と名前のひな形」です。各オブジェクトで入力された値は、そのオブジェクトごとに保持されます（定義が値を直接持つわけではありません）。
- **採寸ルールとの違い**: 同じカスタムデータグループに「採寸ルール」がありますが、採寸ルールは「商品ごとの採寸項目（肩幅・バストなど）と単位を定義する専用機能」であり、メタフィールドとは別物です。採寸ルール作成フォームの項目（ルール名・採寸単位・採寸項目1〜5）の詳細は **10. 採寸** を参照。2026-06-28再確認では、採寸ルール詳細はreadonlyで、商品詳細2件（`486125` / `482787`）にも採寸・測定・measurement系の入力/選択項目はありません。管理画面UIでは、作成したルールを個別商品に適用する導線はありません。API経由・外部連携での割り当て可否は別確認事項です。
- **翻訳への影響**: 翻訳ルールを作成する際、「翻訳データを自動で作成する」をオンにすると、メタフィールドも自動生成対象として扱われます。2026-06-28再確認では、翻訳ルールの自動生成チェックはオン/オフどちらでも保存後の詳細画面で保持されます。翻訳トップ・一覧・作成・詳細に手動実行ボタンはないことも再確認済みです。実際の生成タイミング・メタフィールド翻訳への反映結果は、生成済みデータまたは連携環境が必要な残課題です。
- **定義の削除**: 作成済みのメタフィールド定義は、対象オブジェクトの定義一覧で行を選択すると `定義を削除` が表示されます。確認ダイアログは `メタフィールドの定義を削除しますか？`、本文は「関連するメタフィールドの値は削除されません」と表示されます。削除後は対象定義が一覧から消えます。商品定義では商品詳細側のメタフィールド欄にも表示されなくなりました。下書き注文定義でも2026-07-01に削除後一覧が空へ戻ることを確認済みです。トーストは `メタフィールドの定義を削除しました` です。

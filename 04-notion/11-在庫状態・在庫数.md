# 11. 在庫状態・在庫数

> このページはWBS-25エリアの第11エリアです。SQの在庫データがどのような区分で構成され、注文・取置・移動の各操作でどう変化するかを理解するのが目標です。在庫管理はSQの中核機能で、発注・出荷・販売ルールの判定すべてが在庫数に依存します。

## このエリアで学べること

- 在庫の9区分（販売可能・引当済み・取置中・破損・検品・予備・手持ち・積送中・入荷予定）の意味を説明できる
- 「手持ち」の計算式と、積送中・入荷予定が手持ちに含まれない理由が分かる
- 注文の引当・取置伝票の作成・移動伝票の作成で、それぞれどの区分が増減するか説明できる
- 一覧画面とSKU詳細画面の列構成の違いを把握できる
- ロケーション別に在庫数を確認・編集できる

---

## 機能概要

| 項目 | 内容 |
|:--|:--|
| 機能名 | 在庫管理 |
| 画面URL（一覧） | `/admin/inventory_items` |
| 画面URL（SKU詳細） | `/admin/inventory_items/[id]` |
| 画面URL（調整履歴） | `/admin/inventory_items/[id]/history` |
| 何ができるか | SKU（商品バリエーション）×ロケーション単位で在庫数を確認する。販売可能数の直接編集、在庫振替、調整履歴の確認ができる |

在庫管理画面では、商品ごと・ロケーション（店舗・倉庫）ごとに在庫数を確認します。一覧画面では主要4区分（販売可能・引当済み・取置中・手持ち）を、SKU詳細画面では9区分すべてを確認できます。

### 在庫管理 一覧画面（/admin/inventory_items）

#### ヘッダー

ページ見出しは常に「**在庫管理 :**」です。現在選択中のロケーション名は見出しの外にある独立したボタン（例: 「**物流倉庫 ▼**」）で表示されます。

| 絞り込みを追加 | 絞り込み条件は「**商品コード**」の1種類のみ |
| キャンセル | 検索・絞り込みをリセットして全件表示に戻す |

#### 一覧テーブルの列（主要4区分）

| 列名 | 内容 |
|:--|:--|
| （チェックボックス） | 行選択。全選択しても一括操作ボタンは表示されない |

| 取置中 | 取置・移動用に確保された在庫数（旧「確保済み」） |

### 在庫詳細画面（/admin/inventory_items/[id]）

特定のSKUの在庫数をロケーション別に確認・操作する画面です。

#### ヘッダーボタン

| ボタン（UIラベル） | 動作 |
|:--|:--|
| 調整履歴 | 変更履歴ページ（`/admin/inventory_items/[id]/history`）へ遷移する |
| 商品管理 | バリエーション詳細画面へ遷移する |
| その他のアクション ▼ | 「在庫振替を行う」メニューが展開する |

タブは「**すべて / 店舗 / 倉庫**」の3種で、ロケーション種別を絞り込めます。

#### 在庫数量の9区分

> **2026-06-14更新**: 区分名が「確定済み→引当済み」「確保済み→取置中」に変わり、「積送中」「入荷予定」が新設されました。

| 列名 | 意味 | 手持ちに含まれるか |
|:--|:--|:--|
| ロケーション | ロケーション名 | — |
| 販売可能 | 販売できる在庫数（鉛筆アイコン付き） | 含まれる |
| 引当済み | 注文に引き当て済みの在庫数（旧「確定済み」） | 含まれる |
| 取置中 | 取置・移動用に確保された在庫数（旧「確保済み」） | 含まれる |
| 破損 | 破損品の在庫数 | 含まれる |
| 検品 | 検品中の在庫数 | 含まれる |
| 予備 | 予備在庫の数 | 含まれる |
| 手持ち | 実在庫合計（上記6区分の和） | — |
| 積送中 | 出荷済み・入荷前の輸送中の数（移動先で計上） | 含まれない |
| 入荷予定 | 移動伝票作成〜出荷前の入荷予定数（移動先で計上） | 含まれない |

**手持ちの計算式**（積送中・入荷予定は含まない）

```
手持ち = 販売可能 + 引当済み + 取置中 + 破損 + 検品 + 予備
```

例（物流倉庫、SKU: 486125-09-XL）:
- 手持ち 7 = 販売可能 7（他区分はすべて 0。2026-06-15時点の検証環境データ）

### 「販売可能在庫数を編集する」ダイアログ

在庫詳細の「販売可能」列にある鉛筆アイコン（aria-label: 「販売可能数を編集」）をクリックすると開きます。

| 項目（UIラベル） | 必須/任意 | 内容 |
|:--|:--|:--|
| 理由 | **必須** | テキスト自由入力。空のまま「保存する」を押すと「理由を入力してください」と表示されて保存できない |
| 数量 | — | 数値入力（初期値 0、単位「個」） |
| キャンセル / 保存する | — | 「保存する」で確定 |

**保存したときの挙動（実機確認済み）:**

- 入力した数値が**そのまま新しい「販売可能」数になる**（絶対値の上書き。加算ではない）。例: 販売可能1の状態で数量9を入力 → 販売可能は9になる
- 手持ちも差分だけ増減する（1→9なら手持ちも+8）
- 負数も保存できる（例: 数量 -1 で販売可能・手持ちが -1 になる）。実機確認では `483457-10-L` で 95→1→-1→95 を実施
- 調整伝票は作成されない（調整伝票とは別経路の在庫変更）
- 変更は「調整履歴」に記録される

「**調整履歴**」ボタンを押すと変更履歴ページへ遷移します。ロケーションを選択して履歴を絞り込めます。データがない場合は「**在庫の変更履歴はありません**」と表示されます。

| 列名 | 内容 |

| 日時 | 変更が行われた日時 |
| アクティビティ | 変更の理由・操作名。「販売可能在庫数を編集する」ダイアログで入力した「理由」がここに表示される |
| 販売可能 | 変更後の数量（差分表示付き。例: 5 (+5)） |
| 引当済み | 変更後の数量（差分表示付き。旧「確定済み」） |
| 取置中 | 変更後の数量（差分表示付き。旧「確保済み」） |
| 破損 / 検品 / 予備 | 変更後の数量（差分表示付き） |
| 手持ち | 変更後の数量（差分表示付き） |

## 主な操作手順

### 手順1: ロケーションを切り替えて在庫を確認する

1. 在庫管理一覧（`/admin/inventory_items`）を開く
2. ヘッダーのロケーション名ボタン（例: 「**物流倉庫 ▼**」）をクリックする
3. 「**ロケーションを選択する**」モーダルが開く
4. タブ（すべて / 店舗 / 倉庫）で種別を絞り込む
5. 目的のロケーション行を選択し「**選択する**」を押す
6. 一覧が選択したロケーションの在庫に切り替わる

### 手順2: SKUの在庫内訳（9区分）を確認する

1. 在庫管理一覧で対象SKUの行をクリックする
2. 在庫詳細画面（`/admin/inventory_items/[id]`）へ遷移する
3. タブ（すべて / 店舗 / 倉庫）でロケーション種別を絞り込む
4. 各ロケーション行の販売可能・引当済み・取置中・破損・検品・予備・手持ち・積送中・入荷予定を確認する

### 手順3: 販売可能数を直接編集する

1. 在庫詳細画面で対象ロケーション行の「販売可能」列にある鉛筆アイコンをクリックする
2. 「**販売可能在庫数を編集する**」ダイアログが開く
3. 「理由」に変更理由を入力する（必須）
4. 「数量」に新しい販売可能数を入力する（絶対値。加算ではない）
5. 「**保存する**」を押す
6. 販売可能と手持ちが更新される。変更は「調整履歴」に記録される

### 手順4: 在庫変更の履歴を確認する

1. 在庫詳細画面で「**調整履歴**」ボタンを押す
2. 調整履歴画面（`/admin/inventory_items/[id]/history`）へ遷移する
3. 必要に応じてロケーションを選択して絞り込む
4. 日時・アクティビティ・各区分の差分を確認する

## 操作ごとの在庫の変化（実機検証済み 2026-06-14〜2026-06-20）

各操作が在庫区分に与える影響をまとめた表です。在庫は操作のタイミングで以下のように変化します。

| 操作 | 移動元の変化 | 移動先の変化 |
|:--|:--|:--|
| 在庫依頼を作成する（確認待ち状態） | 変化なし | — |
| 在庫を引当てる（在庫依頼が確保済み状態になる） | 変化なし（数字はまだ動かない） | — |
| 移動伝票を作成する | 販売可能 −n、取置中 +n（手持ちは不変） | 入荷予定 +n |
| 出荷実績を登録する | 取置中 −n、手持ち −n | 入荷予定 −n → 積送中 +n |
| 入荷実績を登録する | — | 積送中または入荷予定 −n → 販売可能 +n・手持ち +n。出荷前でも入荷登録は可能 |
| 移動伝票をキャンセルする | 取置中 −n → 販売可能 +n（元に戻る） | — |
| 取置伝票を作成する | 販売可能 −n、取置中 +n | — |
| 取置を処理済みにマークする | 取置中 −n → 販売可能 +n（解放） | — |
| 調整伝票を実行する | 増減数がそのまま反映 | — |
| 鉛筆で販売可能数を編集する | 「販売可能」を入力値そのものに設定（絶対値）。手持ちは差分だけ増減 | — |

### 区分ごとの増減パターン

| 区分 | 増える操作 | 減る操作 |
|:--|:--|:--|
| 販売可能 | 鉛筆編集／CSV「販売可能在庫」／移動伝票キャンセル（戻り）／入荷実績（移動先に到着）／取置の処理済みマーク | 移動伝票の作成・取置伝票の作成（取置中へ振替） |
| 引当済み | 注文の引き当て | 出荷の進行 |
| 取置中 | 取置伝票の作成／移動伝票の作成（販売可能から振替） | 取置の処理済みマーク／移動の出荷実績／移動伝票キャンセル |
| 破損 / 検品 / 予備 | 2026-06-19の実機確認では、管理画面上にこれらの区分へ直接振り替える操作UIは確認できない | 同左 |
| 手持ち | 入荷実績の登録（移動先）／鉛筆編集 | 出荷実績の登録（移動元） |
| 積送中 | 出荷実績の登録（移動先の入荷予定が積送中へ） | 入荷実績の登録（販売可能・手持ちへ） |
| 入荷予定 | 移動伝票の作成（移動先） | 出荷実績の登録（積送中へ移る） |

## 注意点・制約

- **手持ちはマイナスになり得る**: 在庫が0のロケーションから移動伝票作成・出荷実績登録をしても、SQはブロックしません。実在庫を超えた出荷登録をすると在庫数がマイナスになります（2026-06-18実機確認 `#IM-1016` / `#IO-1016`）。修正は調整伝票や移動伝票で補充してください。<!-- 実機確認済み -->
- **破損・検品・予備区分への振替経路が見当たらない**: 管理画面の「その他のアクション」には「在庫振替を行う」の1メニューのみで、破損・検品・予備専用の操作メニューはありません。調整伝票の「理由」選択肢（廃棄・見本・紛失・棚卸差異・その他）にもこれらの区分は存在しません。WMS（ロジザードZERO）連携経由の可能性があります <!-- TODO: 要確認（WMS連携〈ロジザード等〉経由での設定なのか、開発元への確認が必要） -->
- **「在庫振替を行う」は同一商品内の別バリエーション限定**: 別商品への振替はできません。振替先バリエーションのドロップダウンには同一商品の他バリエーションのみ表示されます。
- **一括操作は提供されていない**: 一覧でチェックボックスを使って全件選択できますが、一括操作ボタンは表示されません。
- **鉛筆編集は調整伝票を作らない**: 販売可能数の直接編集は調整履歴に残りますが、調整伝票（`/admin/inventory_adjustment_orders`）には新しい伝票が作成されません。別経路の在庫変更です。
- **在庫依頼の「確保済み」はステータス名**: 在庫区分の「取置中」（旧「確保済み」）と、在庫依頼ステータスの「確保済み」は別物です。在庫依頼を引当てて「確保済み」になっても、在庫の数字はまだ動きません（動くのは移動伝票作成時）。

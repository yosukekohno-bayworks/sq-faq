# データをCSV・PDFでエクスポートする

> 対象ユーザー: 運営者・管理者　|　所要: 5分（フォーム入力のみ。ファイル受け取りは非同期）　|　最終確認: 2026-07-01

---

## このドキュメントのスコープ

CSVエクスポート画面（`/admin/csv_export`）とPDFエクスポート画面（`/admin/pdf_export`）から各種データをエクスポートする手順を説明する。

**売上実績（注文軸）のエクスポート手順は対象外**。別ドキュメントを参照すること。
→ [売上実績をCSVエクスポートする](./売上実績をCSVエクスポートする.md)

---

## 前提

- CSVエクスポート画面・PDFエクスポート画面を操作する権限が付与されていること
- エクスポート結果は、対象カテゴリの一覧画面に表示される実行履歴から確認できること

---

## エクスポートの共通の流れ

CSVエクスポートの各カテゴリは以下の共通手順で実行する。ヤマトB2クラウド向けCSVは例外で、出荷管理画面の「条件指定でエクスポート」から実行する。

1. 左メニューから「CSVエクスポート」をクリックし、CSVエクスポートのトップ画面（`/admin/csv_export`）を開く。
2. エクスポートしたいカテゴリの行をクリックする。そのカテゴリの一覧画面へ遷移する。
3. 「エクスポート」ボタンをクリックする。エクスポート実行フォーム（`/create` ページ）へ遷移する。
4. 条件を入力する（カテゴリごとに異なる。各カテゴリの手順を参照）。
5. 「エクスポートを開始する」ボタンをクリックする。

![CSVエクスポートのトップ画面（カテゴリ一覧）](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/15-csv-export-top.png)

エクスポートは非同期処理で実行される。完了後は、対象カテゴリの一覧画面に実行履歴が残り、成功した行にはダウンロードリンクが表示される。

メール通知の有無、メール本文、ダウンロードリンクの有効期限は今回の実機確認範囲外です。FAQでは、画面上の実行履歴確認を主導線として案内します。

## 在庫をCSVでエクスポートする

### 手順

1. CSVエクスポートのトップ画面（`/admin/csv_export`）を開く。
2. 「マスターデータ」グループ内の「在庫」の行をクリックする（正式URL: `/admin/csv_export/csv_export_operation_inventory_logical_quantities`）。
3. 「エクスポート」ボタンをクリックする。在庫エクスポートの実行フォームへ遷移する。
4. 「ロケーション」欄の「選択」ボタンをクリックする。「ロケーションを選択する」ダイアログが開く。
5. ダイアログでエクスポートしたいロケーション1件にチェックを入れる。
6. 「選択する」ボタンをクリックしてダイアログを閉じる。
7. 「エクスポートを開始する」ボタンをクリックする。

![「在庫をCSVでエクスポートする」フォーム](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/15-csv-export-inventory-create.png)

### ロケーション選択ダイアログの構成

| 項目 | 内容 |
|:--|:--|
| タブ | 「すべて」「店舗」「倉庫」の3種類 |
| 列構成 | チェックボックス・名前・場所コード |
| 選択数 | 実質1件。2026-07-01実機確認では、1件チェックすると他行のチェックボックスがdisabledになった |
| ボタン | 「キャンセル」「選択する」 |

**注意**: 「ロケーション」フィールドは画面上 * マークがないが、ロケーションを選択しないままでは「エクスポートを開始する」を押しても処理が実行されない。必ず1件選択すること。

2026-07-01実機確認では、`ユニクロ物流倉庫` を指定して実行し、履歴一覧で `成功 完了` と `ダウンロード` リンクを確認しました。履歴の列は `作成日 / 店舗 / ステータス / ダウンロード` で、選択ロケーション名は `店舗` 列に表示されます。CSV本体のヘッダーは `SKU,ProductVariantID,InventoryItemID,Incoming,Available,Committed,Reserved,Damaged,SafetyStock,QualityControl,OnHand` で、ロケーション名・ロケーションコード列は含まれません。

## 商品バリエーションをCSVでエクスポートする

### 手順

1. CSVエクスポートのトップ画面（`/admin/csv_export`）を開く。
2. 「マスターデータ」グループ内の「商品バリエーション」の行をクリックする。
3. 「エクスポート」ボタンをクリックする。商品バリエーションエクスポートの実行フォームへ遷移する。
4. 必要に応じて「商品情報を含める」チェックボックスをONにする。
5. 「エクスポートを開始する」ボタンをクリックする。

### 「商品情報を含める」チェックボックスについて

| 設定 | 説明 |
|:--|:--|
| チェックなし（デフォルト） | 商品バリエーション情報のみをエクスポートする |
| チェックあり | CSVに商品情報（商品コード、商品タイトル、ブランド名など）を含めます |

**画面内補足文（原文）**: 「CSVに商品情報（商品コード、商品タイトル、ブランド名など）を含めます」

2026-07-01実機確認では、チェックなしの履歴は `含めない`、チェックありの履歴は `含める` と表示され、どちらも完了後に一覧の `ダウンロード` リンクからCSVを取得できました。チェックなしは `SKU` から始まるバリエーション列のみです。チェックありはCSV先頭に `ProductCode`, `ProductExternalID`, `ProductTitle`, `ProductStatus`, `ProductType`, `ProductVendor`, `BrandName`, `BrandCode` が追加されます。

親商品のコード・タイトル・ブランドも同じCSVで確認したい場合はON、SKU単位の項目だけで足りる場合はOFFで出力します。

## ロケーションをCSVでエクスポートする

### 手順

1. CSVエクスポートのトップ画面（`/admin/csv_export`）を開く。
2. 「マスターデータ」グループ内の「ロケーション」の行をクリックする（正式URL: `/admin/csv_export/csv_export_operation_location_by_location_group`）。
3. 「エクスポート」ボタンをクリックする。ロケーションエクスポートの実行フォームへ遷移する。
4. 「ロケーショングループ」欄の「選択」ボタンをクリックする。「ロケーショングループを選択する」ダイアログが開く。
5. ダイアログで対象のロケーショングループ1件を選択する。
6. 「選択する」ボタンをクリックしてダイアログを閉じる。
7. 「エクスポートを開始する」ボタンをクリックする。

### フォーム項目

| 項目（UIラベル） | 必須/任意 | 説明 |
|:--|:--|:--|
| ロケーショングループ | 必須（* マーク付き） | 「選択」ボタンからダイアログで1件選択する |

**画面内補足文（原文）**: 「選択したロケーショングループに属するロケーションをCSVでエクスポートします。」

2026-07-01実機確認では、`ユニクログループ` を選ぶと他グループのチェックボックスがdisabledになったため、ロケーショングループは実質1件指定です。履歴一覧の列は `作成日 / ロケーショングループ / ステータス / ダウンロード` で、完了後にCSVをダウンロードできました。CSV本体のヘッダーは `Name,DisplayName,ExternalID,IsListedPublic,IsArchived,LocationCode,Tags,Email,LocationType,PointApplicationType,IsLocalPickupEnabled,IsInventoryAllocationRequestEnabled,MapURL,Phone,CountryCode,PostalCode,Province,City,Address1,Address2` です。

## ディスカウントの利用履歴をCSVでエクスポートする

### 手順

1. CSVエクスポートのトップ画面（`/admin/csv_export`）を開く。
2. 「マスターデータ」グループ内の「ディスカウントの利用履歴」の行をクリックする。
3. 「エクスポート」ボタンをクリックする。ディスカウントの利用履歴エクスポートの実行フォームへ遷移する。
4. 「ディスカウント」欄の「選択」ボタンをクリックする。「ディスカウントを選択する」ダイアログが開く。
5. ダイアログで対象のディスカウント1件を選択する。
6. 「選択する」ボタンをクリックしてダイアログを閉じる。
7. 「エクスポートを開始する」ボタンをクリックする。

### フォーム項目

| 項目（UIラベル） | 必須/任意 | 説明 |
|:--|:--|:--|
| ディスカウント | 必須（* マーク付き） | 「選択」ボタンからダイアログで1件選択する |

**画面内補足文（原文）**: 「選択したディスカウントの使用履歴をCSVでエクスポートします。」

2026-07-01実機確認では、ディスカウント選択ダイアログは `タイトル / ステータス` 列で候補を表示し、1件選択すると他候補がdisabledになりました。`TEST_FAQ_ディスカウント_対象商品テスト` を指定して実行すると、履歴一覧に `作成日 / ディスカウント / ステータス / ダウンロード` 列が表示され、完了後にCSVをダウンロードできました。利用履歴0件の場合でもヘッダーのみ出力され、ヘッダーは `顧客ID,顧客外部ID,店舗名,店舗コード,利用日時` です。

## セール価格をCSVでエクスポートする

### 手順

1. CSVエクスポートのトップ画面（`/admin/csv_export`）を開く。
2. 「価格」グループ内の「セール価格」の行をクリックする（正式URL: `/admin/csv_export/csv_export_operation_product_price_rule_sale_prices`）。
3. 「エクスポート」ボタンをクリックする。セール価格エクスポートの実行フォームへ遷移する。
4. 「販売価格ルール」ドロップダウンから対象の販売価格ルールを選ぶ。
5. 「エクスポートを開始する」ボタンをクリックする。

### フォーム項目

| 項目（UIラベル） | 必須/任意 | 説明 |
|:--|:--|:--|
| 販売価格ルール | 必須（* マーク付き） | ドロップダウンから対象の販売価格ルールを選ぶ |

**画面内補足文（原文）**: 「選択した販売価格ルールのセール価格をCSVでエクスポートします。」

2026-07-01実機確認では、`年末セール価格 (JPY)` を選んで実行し、履歴一覧に `作成日時 / 販売価格ルール / ステータス / ダウンロード` 列が表示されました。完了後にCSVをダウンロードでき、CSVヘッダーは `ProductVariantID,ExternalID,PriceAmount,PriceAmountAfterTaxes,CurrencyCode,StartsAt,EndsAt`、データ行には `PriceAmount=1000`, `PriceAmountAfterTaxes=1100`, `CurrencyCode=JPY`, `StartsAt`, `EndsAt` が出力されました。

## ポイント変動履歴をCSVでエクスポートする

### 手順

1. CSVエクスポートのトップ画面（`/admin/csv_export`）を開く。
2. 「ポイント」グループ内の「ポイント変動履歴」の行をクリックする（正式URL: `/admin/csv_export/csv_export_operation_point_changes`）。
3. 「エクスポート」ボタンをクリックする。ポイント変動履歴エクスポートの実行フォームへ遷移する。
4. 以下の項目を入力する。

| 項目（UIラベル） | 必須/任意 | 説明 |
|:--|:--|:--|
| テナント | 必須（* マーク付き） | ドロップダウンからエクスポート対象のテナントを選ぶ |
| 開始日時 | 必須（* マーク付き） | ポイント変動履歴の絞り込みを開始する日時を入力する |
| 終了日時 | 必須（* マーク付き） | ポイント変動履歴の絞り込みを終了する日時を入力する |

5. 「エクスポートを開始する」ボタンをクリックする。

2026-07-01実機確認では、`ユニクロ`、対象期間 `2026年07月01日 00:00 〜 2026年07月01日 23:59` で実行し、履歴一覧で `注意 処理中` から `成功 完了` に変わること、完了後に `ダウンロード` リンクが出ることを確認しました。ヘッダーは `PointChangeID,PurchasingCustomerID,PurchasingCustomerBarcode,Title,Delta,AvailableAt,ExpiresDate,PointChangeCreatedAt,PointChangeActivityType,Note` です。

同日にポイント一括加算・減算を実行した後のCSVでは、加算行 `FAQ_POINT_PLUS_20260701` が `Delta=5` / `PointChangeActivityType=EARNED`、減算行 `FAQ_POINT_MINUS_20260701` が `Delta=-2` / `PointChangeActivityType=USED` として出力されました。ポイント実績0件の場合でも空ファイルではなくヘッダーのみのCSVが取得できます。

## 納品書PDFの確認状況

PDFエクスポート画面（`/admin/pdf_export`）は、納品書PDFのエクスポートカテゴリを表示する画面です。2026-06-28の実機再確認ではトップに「出荷」グループと「納品書」行が表示され、新規作成ボタンは表示されませんでした。

納品書カテゴリページ（`/admin/pdf_export/pdf_export_operation_packing_slips`）は表示されますが、カテゴリページ自体に新規作成/任意生成ボタンは確認できません。直接 `/admin/pdf_export/pdf_export_operation_packing_slips/create` にアクセスしても「このページは存在しないようです」と表示されます。移動/調整/取置/出荷/入荷/発注の各詳細（`#IM-1027` / `#IA-1013` / `#IR-1011` / `#IO-1027` / `#II-1029` / `#IP-1010`）でも納品書/PDF/印刷ボタンは確認できませんでした。

2026-07-01追加確認では、出荷管理一覧で対象行にチェックを入れると一括操作バーが表示され、`アクション > 納品書（PDF）` からPDFエクスポートを実行できました。確認モーダル後に `/admin/pdf_export/pdf_export_operation_packing_slips` の履歴へ遷移します。ただし `#IO-1047` / `#IO-1046` / `#IO-1048` / `#IO-1020` はいずれも `重大 / 失敗` で、ダウンロードは不可でした。`#IO-1020` は配送・引当済みでも失敗しています。

そのため、現時点では「PDFエクスポート画面から任意に納品書PDFを新規作成できる」とは案内しないでください。案内できる導線は「出荷管理一覧で対象行を選択 → `アクション > 納品書（PDF）`」です。一方、ヤマトB2クラウド条件指定エクスポートフォームには出力物として「同梱する納品書PDF」が表示されます。2026-06-28再確認ではヤマトB2 CSVエクスポート履歴画面も空状態でダウンロードリンクなしでした。実際にPDFファイルがメールリンクで届くか、PDF生成が成功する条件は成功データまたは接続環境で再検証が必要です。

### ヤマトB2クラウドCSVの条件指定エクスポート

1. 左メニューから「出荷管理」をクリックし、出荷管理の一覧画面（`/admin/inventory_outbound_orders`）を開く。
2. 右上の「条件指定でエクスポート」ボタン（ドロップダウン）をクリックする。
3. 展開したメニューから「ヤマトB2クラウド」を選択する。「条件指定でヤマトB2クラウドのCSVをエクスポートする」フォーム画面（`/admin/inventory_outbound_orders/export/yamato_b2_cloud`）へ遷移する。
4. 以下の項目を入力する。

![「条件指定でヤマトB2クラウドのCSVをエクスポートする」フォーム](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/gap-outbound-yamato-export-form.png)

| 項目（UIラベル） | 必須/任意 | 説明 |
|:--|:--|:--|
| 開始日時 | 必須（* マーク付き） | 出荷指示の絞り込みを開始する日時を入力する |
| 終了日時 | 必須（* マーク付き） | 出荷指示の絞り込みを終了する日時を入力する |
| 配送先 (国) | 任意 | マルチ選択可。「すべて選択」あり |
| 決済方法 | 任意 | 「すべての決済」（デフォルト）/「代引きのみ」/「代引き以外の決済」 |
| 出荷作業ステータス | 任意 | 特定のステータスで絞り込みたい場合に選択する |
| 注文タグ（含む） | 任意 | 入力したタグを含む出荷指示のみを対象にする |
| 注文タグ（除外） | 任意 | 入力したタグを含まない出荷指示のみを対象にする |
| CSVの出力後に出荷指示のステータスを出荷作業中に変更する | 任意 | チェックボックス（デフォルト: 未チェック） |

5. 「実行する」ボタンをクリックする。

エクスポートは非同期処理で実行されます。2026-06-28時点では、ヤマトB2クラウドCSVの出力フォームに「同梱する納品書PDF」と表示されること、決済方法が `すべての決済 / 代引きのみ / 代引き以外の決済` の3択であること、ステータス変更チェックがデフォルトOFFであることまで確認済みです。2026-06-28追加確認では、ステータス変更ONで実行しても、移動伝票由来・配送希望日なしの検証用出荷指示 `#IO-1029` は `出荷待ち` のままで、ヤマトB2 CSVエクスポート履歴画面も空状態でした。メール通知・ダウンロードリンク・PDF実ファイルの生成結果は接続環境で検証待ちです。

## うまくいかないとき

**「エクスポートを開始する」を押しても何も起きない（在庫エクスポート）**
- 「ロケーション」欄が未選択の可能性がある。「選択」ボタンからロケーションを1件選択してから実行すること。

**エクスポート完了メールが届かない**
- メール通知の有無は未確認です。まずCSVエクスポート画面の各カテゴリ一覧で実行履歴を確認してください。

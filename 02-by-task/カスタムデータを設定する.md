# カスタムデータを設定する

> 対象ユーザー: 管理者　|　所要: 各セクション5〜10分程度　|　最終確認: 2026-06-19

カスタムデータグループには次の3つの機能があります。本手順書ではそれぞれの作成手順を説明します。

| 機能 | 画面 | 用途 |
|:--|:--|:--|
| メタフィールド定義 | 設定 > メタフィールド定義 | 商品・注文など各オブジェクトにカスタムフィールドを追加する |
| 翻訳 | 設定 > 翻訳 | 商品データなどを多言語に翻訳するルールを作成する |
| 採寸ルール | 設定 > 採寸ルール | 採寸項目と単位を定義したルールを管理する |

---

## 前提

- 3機能すべてに「翻訳ルールの編集権限（`translation_rules:write`）」または「ユーザーの編集権限」が必要です。具体的な権限は以下を参照してください。
  - メタフィールド定義の作成: 対応する権限（例: `products:write`）が必要
  - 翻訳ルールの作成: `translation_rules:write` 権限が必要
  - 採寸ルールの作成: 対応する権限が必要
- 設定画面には左メニュー下部の「設定」からアクセスします。

## 1. メタフィールド定義を作成する

メタフィールド定義（メタフィールドのカスタムフィールド定義）とは、商品・注文・顧客などのオブジェクトに独自のカスタムフィールドを追加するための設定です。定義を作成すると、対象オブジェクトの詳細画面に入力欄が追加されます。

### カスタムフィールドを追加できるオブジェクト（16種）

組織 / 商品 / バリエーション / 顧客 / 注文 / 下書き注文 / 会社 / ロケーション / ディスカウント / 在庫移動伝票 / 在庫調整伝票 / 在庫取置伝票 / 発注伝票 / 入荷指示 / 出荷指示 / 仕入れ先

### 手順

1. 左メニュー下部の「設定」をクリックする。設定トップページが開く。
2. 「メタフィールド定義」をクリックする。オブジェクトの一覧が表示される。
3. カスタムフィールドを追加したいオブジェクト名（例: 「商品」）をクリックする。そのオブジェクトのメタフィールド定義一覧画面に遷移する。
4. 「定義を追加する」ボタンをクリックする。定義作成フォームが開く。
5. 次の項目を入力する。

![「商品メタフィールドの定義を追加する」フォーム](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/bfg-G-01-metafield-create-form.png)

#### 入力項目

| 項目（UIの原文） | 必須/任意 | 制約・補足 |
|:--|:--|:--|
| 名前 * | 必須 | 最大50文字。文字数カウンターが表示される |
| 説明 | 任意 | 最大100文字 |
| ネームスペース * | 必須 | 最大50文字・最低4文字 |
| キー * | 必須 | 最大50文字・最低4文字 |
| メタフィールドのタイプ * | 必須 | コンボボックス（下記12種から選択） |

**「メタフィールドのタイプ」の選択肢（12種）:**

単一行のテキスト / 単一行のテキスト(リスト) / 複数行のテキスト / 日付と時刻 / 日付 / trueまたはfalse / バーコード(NW7) / 整数 / 小数 / JSON / ID / リッチテキスト

6. 「保存する」ボタンをクリックする。

保存が完了すると、対象オブジェクトの詳細画面に「メタフィールド」セクション（h2）が表示され、定義した名前と「未設定」のテキストが即座に出現します。

### メタフィールドに値を設定する

定義を保存した後、各オブジェクトの詳細画面から値を入力できます。

1. 対象オブジェクトの詳細画面（例: 商品詳細）を開く。
2. 「メタフィールド」セクションまでスクロールする。
3. 追加したフィールドの右側にある編集ボタン（鉛筆アイコン）をクリックし、値を入力する。

### 定義の識別子について

定義の識別子は「ネームスペース.キー」形式で管理されます（例: `custom.note`）。定義一覧では「定義名 / ネームスペース.キー」の2行で表示されます。

## 2. 翻訳ルールを作成する

翻訳ルールとは、商品データなどを指定した言語に翻訳するための設定です。ルールごとに翻訳対象の言語と、翻訳に使用するAIプロンプトを管理します。

**対応言語（10言語）:**

日本語 / 英語 / 中国語（簡体字） / 中国語（繁体字） / 韓国語 / スペイン語 / フランス語 / ドイツ語 / ヒンディー語 / タイ語

### 手順

1. 左メニュー下部の「設定」をクリックする。設定トップページが開く。
2. 「翻訳」をクリックする。翻訳一覧画面に遷移する。
   - 翻訳ルールが1件もない場合は「翻訳を始める」という案内と「作成する」ボタンが表示される。
3. 「ルールを管理する」をクリックする。翻訳ルール一覧画面に遷移する。
   - ルールがない初期状態では、翻訳一覧画面の「作成する」ボタンをクリックしてフォームへ遷移することもできる。
4. 翻訳ルール作成フォームが開いたら、次の項目を入力する。2026-06-19の実機確認では、翻訳ルールは `/admin/settings/translation/translation_rules` で一覧確認でき、作成フォームへ遷移できる。

![「翻訳ルールを作成する」フォーム](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/kill-settings-01-translation-create-form.png)

#### 入力項目

| 項目（UIの原文） | 必須/任意 | 補足 |
|:--|:--|:--|
| 名前 * | 必須 | テキストボックス |
| 言語 * | 必須 | コンボボックス（上記10言語から選択） |
| 翻訳データを自動で作成する | 任意 | チェックボックス。オンにすると商品・商品オプション・商品オプション値・メタフィールドの翻訳データが自動生成される |

5. 「保存する」ボタンをクリックする。

2026-06-19の実機確認では、翻訳ルールを保存するとルールを作成でき、削除操作では確認ダイアログ後に `翻訳を削除しました` と表示されました。

### 保存後の詳細画面について

保存後の詳細画面では次の点に注意してください。

- **「言語」コンボボックスは変更不可（disabled）になります。** 別の言語に変更したい場合は、新しいルールを別途作成してください。
- 「翻訳データを自動で作成する」チェックボックスは保存後も変更できます。

詳細画面には翻訳対象ごとに4つのセクション（h2）が表示されます。各セクションで翻訳に使用するAIプロンプトを管理できます。

| セクション（UIの原文） | 翻訳対象 |
|:--|:--|
| 商品 | 商品タイトル・説明文 |
| 商品オプション | 商品オプション名 |
| 商品オプション値 | 商品オプション値 |
| メタフィールド | メタフィールド値 |

各セクションには「親プロンプト」と「カスタムプロンプト」のテキストエリアがあります。カスタムプロンプトを使うと、親プロンプト内の指定箇所にこのルール独自の指示を追加できます。

2026-06-19の実機確認では、翻訳ルール詳細画面に手動実行用の明確なボタンは確認できませんでした。「翻訳データを自動で作成する」をオフにした場合の実行方法は、管理画面単体では断定しません。

## 3. 採寸ルールを作成する

採寸ルールとは、衣料品などの商品に対する採寸項目（肩幅・バストなど）と採寸単位を定義したテンプレートです。

### 手順

1. 左メニュー下部の「設定」をクリックする。設定トップページが開く。
2. 「採寸ルール」をクリックする。採寸ルール一覧画面に遷移する。
3. 「採寸ルールを作成する」ボタンをクリックする。作成フォームが開く。
4. 次の項目を入力する。

#### 基本設定セクションの入力項目

| 項目（UIの原文） | 必須/任意 | 選択肢・補足 |
|:--|:--|:--|
| ルール名 * | 必須 | テキストボックス（プレースホルダー: 「トップス」） |
| 採寸単位 | 任意 | コンボボックス（**なし（デフォルト）/ センチメートル** の2種のみ） |

#### 採寸項目セクションの入力

5. 「採寸項目を追加」ボタン（+アイコン付き）をクリックして、採寸項目の入力欄を追加する。
6. 採寸項目名（例: 「肩幅」「バスト」）を入力する。採寸項目は最大5件まで追加できる。
7. 「保存する」ボタンをクリックする。

保存が完了すると採寸ルールの詳細画面に遷移します。

![保存後の採寸ルール詳細画面](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/kill-settings-04-measurement-rule-detail.png)

### 採寸ルール一覧の表示列

| 列名 | 内容 |
|:--|:--|
| ルール名 | 作成したルール名（クリックで詳細へ遷移） |
| 単位 | 採寸単位（例: センチメートル） |
| 採寸項目 | 設定した採寸項目名（カンマ区切り） |

### 注意: 商品への紐付けについて

現在の管理画面UIでは、採寸ルールを個別の商品に紐づける操作はできません。採寸ルールの詳細画面にも、商品詳細画面にも、採寸ルールを割り当てるUIは存在しません。2026-06-19の実機確認では、採寸項目は最大5件まで追加でき、5件目追加後に追加ボタンが表示されなくなることを確認しました。

商品への割り当て方法が必要な場合は、管理画面外（APIまたは外部連携）の仕様確認事項として扱ってください。

---

## うまくいかないとき

**「定義を追加する」ボタンが見つからない**
- メタフィールド定義一覧画面（設定 > メタフィールド定義 > 対象オブジェクト）を開いているか確認してください。設定トップや他の一覧画面にはこのボタンはありません。

**翻訳ルールの言語を変更したい**
- 保存後は言語を変更できません。別の言語用の新しいルールを作成してください。

**採寸項目が6件以上追加できない**
- 採寸項目は最大5件の制限があります。

**採寸ルールを商品に紐づけたい**
- 現在の管理画面UIでは直接紐づける操作はできません。API経由または外部連携経由での割り当てが必要かどうかは、Stack社確認事項として扱ってください。

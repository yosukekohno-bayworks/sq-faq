# ヤマトB2クラウドで出荷業務を行う

> 対象ユーザー: 在庫管理担当者　|　所要: 15〜30分（件数による）　|　最終確認: 2026-07-03

---

## このドキュメントのスコープ

ヤマトB2クラウドを使った出荷業務の一連の流れを説明します。

1. SQからヤマトB2クラウド用CSVをエクスポートする
2. ヤマトB2クラウドで発送処理を行う
3. 発送後にヤマトB2クラウドの出荷実績CSVをSQへインポートする

DHL を使った出荷実績インポートも同様の手順で行えます（後述）。

---

## 前提

- 出荷管理を操作する権限が付与されていること
- エクスポート完了通知を受け取るメールアドレスが登録済みであること
- ヤマトB2クラウドのアカウントおよび操作環境が別途用意されていること

---

## フロー全体像

| ステップ | 操作場所 | 内容 |
|:--|:--|:--|
| 1 | SQ 出荷管理 | 条件を指定してヤマトB2クラウド用CSVをエクスポート |
| 2 | メール | エクスポート完了通知を受け取り、ファイルをダウンロード |
| 3 | ヤマトB2クラウド | CSVを取り込んで発送処理 |
| 4 | SQ 出荷管理 | ヤマトB2クラウドの出荷実績CSVをインポート |

---

## 手順

### ステップ 1: 出荷指示CSVをエクスポートする

1. 左メニューの「出荷管理」をクリックし、出荷管理の一覧画面を開く。
2. 画面右上の「条件指定でエクスポート」ボタン（右側に下向き矢印がある）をクリックする。ドロップダウンが開く。
3. 「ヤマトB2クラウド」を選択する。エクスポートフォーム画面「条件指定でヤマトB2クラウドのCSVをエクスポートする」へ遷移する。
4. 以下の項目を入力する。

| 項目（UIラベル） | 必須 | 説明・選択肢 |
|:--|:--|:--|
| 開始日時 | 必須（*） | 絞り込みを開始する日時を入力する |
| 終了日時 | 必須（*） | 絞り込みを終了する日時を入力する（デフォルト: 現在日時が入力済み） |
| 配送先 (国) | 任意 | 国を選択して絞り込む（マルチ選択可） |
| 決済方法 | 任意 | すべての決済 / 代引きのみ / 代引き以外の決済 |
| 出荷作業ステータス | 任意 | 指定しない / 出荷待ち / 保留中 / 依頼済み / 作業中 / 欠品・要対応 / キャンセル済み / 出荷完了 |
| 注文タグ（含む） | 任意 | 入力したタグを含む出荷指示のみが対象になる（例: priority） |
| 注文タグ（除外） | 任意 | 入力したタグを含まない出荷指示のみが対象になる（例: test） |
| CSVの出力後に出荷指示のステータスを出荷作業中に変更する | 任意 | チェックボックス（デフォルト: 未チェック）。ONにするとラベル上はCSV出力後に対象出荷指示を出荷作業中へ変更する設定。ただし2026-06-28追加確認では、移動伝票由来・配送希望日なしの検証用出荷指示 `#IO-1029` はON実行後も `出荷待ち` のままでした |

![「条件指定でヤマトB2クラウドのCSVをエクスポートする」フォーム](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/gap-outbound-yamato-export-form.png)

5. 「実行する」ボタンをクリックする。

> ステータスを変えたくない場合は、`CSVの出力後に出荷指示のステータスを出荷作業中に変更する` をOFFのまま実行してください。2026-06-28追加確認ではこのチェックはデフォルトOFFで、ON送信も可能でしたが、検証用 `#IO-1029` は `作業中` タブへ移らず `出荷待ち` タブに残りました。条件にヒットした実データで作業中へ変わる成功例は未確認です。

---

### ステップ 2: 出力結果を確認する

エクスポートは非同期処理で実行される。完了後は、CSVエクスポートまたは出荷管理側の関連履歴で出力結果を確認する。

- **ヤマトB2クラウド取り込み用CSV**

ダウンロードリンクが表示されたら、CSVファイルをダウンロードする。

2026-07-02の実機確認では、同日23:34作成のヤマトB2 CSVエクスポート履歴が `成功 完了` になり、23:35の完了メールからCSVをダウンロードできました。メール本文にはダウンロードリンクの有効期限が表示されます。このメールに含まれていたリンクはCSV 1件のみで、同梱納品書PDFリンクは確認できませんでした。取得CSVのヘッダーは `送り状種類`、`お客様管理番号`、`お届け先名`、`お届け先電話番号`、`お届け先郵便番号`、`お届け先住所`、`お届け先住所（アパートマンション名）`、`出荷予定日`、`お届け予定（指定）日`、`配達時間帯`、`コレクト代金引換額（税込）`、`コレクト内消費税額等`、`備考欄` の13列で、データ行に `#IO-1047` が出力されていました。

一方、2026-07-02に条件指定エクスポートを新規実行した際は、トースト `ヤマトB2クラウドのエクスポートを開始しました` は表示されましたが、確認時点では新しい履歴行・新しいメールは確認できませんでした。

出荷一覧で対象行をチェックして `アクション > エクスポート > ヤマトB2クラウド（CSV）` からもCSV出力を実行できます。この導線では確認モーダル `ヤマトB2クラウドのCSVをエクスポートする` が開き、`エクスポートした出荷指示のステータスを作業中に更新する` チェックボックスが表示されます。2026-07-02に `#IO-1050` を1件選択してチェックOFFで実行したところ、23:47にCSV出力履歴が作成されましたが、約20秒後に `重大 / 失敗` になり、`ダウンロード` はdisabled、完了/失敗メールも確認できませんでした。

> 納品書PDFについては、2026-06-28の実機確認でヤマトB2クラウド条件指定エクスポートフォームに「同梱する納品書PDF」と表示されることを確認しました。さらに2026-07-02には、出荷管理一覧で対象出荷指示をチェックし、`アクション > 納品書（PDF）` からPDF出力を実行できることを確認しました。確認モーダル後に `/admin/pdf_export/pdf_export_operation_packing_slips` の履歴へ遷移します。ただし `#IO-1047` / `#IO-1046` / `#IO-1048` / `#IO-1020` と2026-07-02追加実行の `#IO-1048`、2026-07-03追加実行の `#IO-1045`（キャンセル済み）/ `#IO-1051`（出荷待ち）はいずれも `重大 / 失敗` となり、ダウンロードはdisabledでした。メールにもPDF/納品書通知は見つかっていません。ステータスを変えても今回のstagingでは成功しておらず、実PDFファイルの内容・印字は未確認のため、「PDFが必ず生成される」とは断定しないでください。

---

### ステップ 3: ヤマトB2クラウドで発送処理を行う

ヤマトB2クラウドにダウンロードしたCSVを取り込み、発送処理を行う。

<!-- 残課題: ヤマトB2クラウド側の操作手順はSQ外のシステムのため記録なし -->

発送完了後、ヤマトB2クラウドから出荷実績CSVをダウンロードしておく（次のステップで使用する）。

---

### ステップ 4: 出荷実績CSVをSQへインポートする

1. 左メニューの「出荷管理」をクリックし、出荷管理の一覧画面を開く。
2. 画面右上の「インポート」ボタン（右側に下向き矢印がある）をクリックする。ドロップダウンが開く。
3. 「ヤマトB2クラウド（出荷実績）」を選択する。ヤマトB2クラウドの出荷実績インポート画面へ遷移する。

![出荷管理の「インポート」ドロップダウン（ヤマトB2クラウド（出荷実績）・DHL（出荷実績））](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/gap-purchase-ops-15-outbound-import-dropdown.png)
4. 「テンプレート」リンクをクリックすると、CSVフォーマットの定義書（Googleスプレッドシート）を確認できる。

   ヤマトB2クラウドCSVのフォーマット仕様（必須列と任意列）:

   | 列 | 項目名 | 必須/任意 |
   |:--|:--|:--|
   | A | 伝票番号 | 必須 |
   | B | お届け先コード | 任意 |
   | C | お届け先名 | 任意 |
   | D | 荷物状況 | 任意 |
   | E | 日付 | 任意 |
   | F | 時刻 | 任意 |
   | G | 出荷日 | 必須 |
   | H | サイズ品目 | 任意 |
   | I | 運賃 | 任意 |
   | J | お客様管理番号 | 必須 |

   必須列は A（伝票番号）・G（出荷日）・J（お客様管理番号）の3列のみ。任意列はインポート処理で読み捨てられる。

5. 「新規インポート」ボタンをクリックする。インポート実行フォームへ遷移する。

![「ヤマトB2クラウドの出荷実績をCSVでインポートする」フォーム（ファイル選択と「保存する」ボタン）](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/gap-purchase-ops-16-yamato-import-form.png)
6. 「ファイルを選択する」ボタンをクリックしてCSVファイルを選ぶか、ファイルをドラッグ&ドロップでアップロードエリアに置く。
7. 「保存する」ボタンをクリックする。2026-07-02再確認では、ファイル未選択のまま保存しても同じ作成画面に留まり、画面上に赤いエラーメッセージは出なかった。2026-06-28の `#IO-1032`、2026-07-01の `IO-1048`、2026-07-02の存在しないお客様管理番号 `TEST_NO_SUCH_IO_20260702_2358`、2026-07-03の既存出荷指示 `#IO-1050` の検証用CSVでは、保存後にCSVインポート詳細は作成されたが、検証成功0件/検証失敗0件のまま、画面上は「CSVのバリデーションを実行しています」と表示され続けた。`実行する` はdisabledで、実行・出荷完了遷移は未確認。

<!-- 残課題: ヤマトB2クラウドから出力した実CSVサンプルでの検証完了、実行、出荷完了遷移、配送キャリア/追跡コード反映。検証用CSVでは画面上「CSVのバリデーションを実行しています」のまま進まない。 -->

> インポート後の実行履歴はインポート一覧画面で確認できる。各履歴行をクリックすると詳細（検証ステータス・実行ステータス・成功件数・失敗件数）を確認できる。

## DHL の出荷実績をインポートする場合

ヤマトB2クラウドと同様の手順で行う。手順 4 のステップ 3 で、ドロップダウンの「DHL（出荷実績）」を選択する。

- DHL のインポート画面URL: `/admin/csv_import/csv_import_operation_fulfillment_by_dhls`
- DHL のインポートフォームはCSVファイルのアップロードのみ。2026-07-03再確認では、DHL一覧に「テンプレート」リンクは表示されませんでした。CSVをアップロードすると詳細作成までは進みましたが、画面上は「CSVのバリデーションを実行しています」、`実行ステータス 未実行`、検証成功0件/検証失敗0件のままで、`実行する` はdisabledでした。ファイル未選択保存時は同じ作成画面に留まり、画面上に赤いエラーメッセージは出ません。

## うまくいかないとき

**「条件指定でエクスポート」ドロップダウンに「ヤマトB2クラウド」が表示されない**
- ドロップダウンの選択肢は「ヤマトB2クラウド」のみです。DHL のエクスポートは存在しません。DHL は出荷実績インポートのみ対応しています。

**エクスポート完了メールが届かない**
- エクスポートは非同期処理のため、完了まで時間がかかる場合があります。しばらく待ってからメールを確認してください。
- 2026-07-02の確認では、成功済みの23:34履歴は23:35にメールが届きましたが、同日に新規実行した条件指定エクスポートと、一覧一括アクションの失敗ジョブでは新しいメールは確認できませんでした。

**「保存する」ボタンを押してもインポートが進まない**
- CSVファイルが選択されていない場合はアップロードできません。「ファイルを選択する」でファイルを選んでから「保存する」をクリックしてください。

**インポート後に出荷実績が反映されない**
- インポート一覧画面で該当履歴の詳細を開き、検証ステータスと実行ステータスを確認してください。検証失敗がある場合は失敗件数のリンクから内容を確認できます。

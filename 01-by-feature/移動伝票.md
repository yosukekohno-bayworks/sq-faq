# 移動伝票

> 対象画面: 在庫管理 > 移動伝票 / `/admin/inventory_movement_orders`　|　最終確認: 2026-06-28

## この機能でできること

- ロケーション間（例: 物流倉庫 → 店舗）の在庫移動を伝票として管理する
- 移動伝票を手動で新規作成する（単体作成）
- 取り寄せ販売フロー（在庫依頼 → 確保済み → 移動伝票作成）で自動生成された伝票を確認する
- 伝票の進捗を一覧のタブで把握する（出荷作業 / 一部受領済み / 受領済み / キャンセル）

## 画面・項目の説明

### 一覧タブ

| タブ名 | 説明 |
|:--|:--|
| すべて | 全ステータスの伝票を表示する |
| 出荷作業 | 出荷待ちの伝票 |
| 一部受領済み | 一部の商品が入荷済みの伝票 |
| 受領済み | すべての商品が入荷済みの伝票 |
| キャンセル | キャンセルされた伝票 |

### 作成フォーム（`/admin/inventory_movement_orders/create`）

| 項目（UIラベル） | 説明 | 必須 | 制約・補足 |
|:--|:--|:--|:--|
| 配送元 | 在庫の移動元ロケーション（「配送元を選択してください」） | 必須 | 「選択」ボタンからロケーション選択モーダルを使う。配送元を先に選択しないと商品欄がグレーアウトする |
| 配送先 | 在庫の移動先ロケーション（「配送先を選択してください」） | 必須 | 「選択」ボタンからロケーション選択モーダルを使う。配送元と同じロケーションも選択でき、保存時も拒否されない |
| 商品を追加 | 移動する商品の指定 | 必須（1件以上） | 配送元を選択するまでdisabled。テキスト検索または「参照」ボタンで商品を選ぶ。数量0は保存不可 |

> 注意: 配送元と配送先に同じロケーションを選択してもUIレベル・保存時ともにエラーにならず、移動伝票・出荷指示・入荷指示が作成されます。誤作成した場合は詳細の「その他の操作」>「キャンセル」で取り消してください。

> 注意: 配送元の表示在庫が0のSKUでも、商品候補に出て移動伝票を作成できます。出荷実績登録も在庫不足でブロックされないため、マイナス在庫になり得ます。

### 伝票詳細のステータスバッジ

単体作成した移動伝票でも、出荷実績と入荷実績を最後まで登録すると「**成功 完了**」と「**入荷完了**」の2つのバッジが並んで表示されます。

## 移動伝票の2つの作られ方

### 1. 取り寄せ販売フロー経由（自動作成）

在庫依頼 → 引当て → 確保済み一覧（`/admin/inventory_allocation_request_confirmations`）の「**移動伝票を作成する**」ボタンから自動作成されます。

この場合、伝票詳細に次の注意文が表示され、商品の追加・削除などの編集ができません。

> 「この移動伝票は外部システムから連携されているため、編集（商品の追加や削除など）は制限されています。連携元システムの管理画面より操作を行ってください。」

![取り寄せ販売フロー経由で自動作成された移動伝票の詳細。編集制限の注意文と、関連欄の出荷指示・入荷指示が表示される](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/flow-03-movement-detail.png)

### 2. 単体作成

上記フロー以外に、「**移動伝票を作成する**」リンクボタンから単体で作成できます。

## 補足・注意点

### ステータスを進める方法

移動伝票の詳細画面にはステータスを直接進めるボタンがありません。出荷指示（出荷管理）と入荷指示（入荷管理）の実績登録に連動して、移動伝票のステータスが自動的に変わります。

- 出荷実績を登録する → 出荷管理（`/admin/inventory_outbound_orders`）で「出荷実績を登録する」を実行
- 入荷実績を登録する → 入荷管理（`/admin/inventory_inbound_orders`）で「入荷実績を登録する」を実行

移動伝票作成と同時に、出荷指示と入荷指示が自動生成されます。

2026-06-18の単体作成と、2026-06-27の在庫依頼由来 `#IM-1025` で次の流れを確認しています。

| 操作 | 移動伝票 / 関連伝票の状態 | 在庫への影響 |
|:--|:--|:--|
| 移動伝票を保存/作成 | 移動伝票は「未完了 / 出荷待ち」。関連欄に出荷指示と入荷指示が表示される | 移動元の販売可能が減って取置中が増える。移動先には入荷予定が立つ |
| 出荷実績を登録 | 出荷指示は「成功 完了 / 出荷完了」。出荷完了タブに表示される | 移動元の取置中と手持ちが減る。移動先は積送中状態になる |
| 入荷実績を登録 | 入荷指示は「成功 完了 / 入荷完了」。入荷完了タブに表示される | 移動先の販売可能・手持ちが増える |
| 移動伝票を再確認 | 移動伝票は「成功 完了 / 入荷完了」 | 移動が完了する |

2026-06-27の `#IM-1025` では、移動元 `TEST_E2E_20260622_GU倉庫_ON_1905` の対象SKUが、作成後に販売可能9→8・取置中0→1・手持ち9のまま、出荷後に取置中1→0・手持ち9→8へ変化しました。移動先 `TEST_E2E_20260622_GU店舗_OFF_1905` は一覧上で `入荷予定` → `積送中` を経て、入荷完了後に販売可能1→2・手持ち1→2になりました。

2026-06-28の `#IM-1026` でも、関連する出荷指示 `#IO-1026` を配送キャリア・追跡コード入力ありで出荷完了し、入荷指示 `#II-1028` を受領数1で入荷完了すると、親移動伝票は `成功 完了 / 入荷完了` になりました。入力した配送キャリア・追跡コードは出荷完了後の出荷指示詳細・一覧には表示されません。

2026-06-28の `#IM-1027` / `#II-1029` では、出荷完了後に同一入荷指示へ1回目1個、2回目2個を登録し、予定数2に対して入荷済み3個の累計過入荷にできました。この状態でも `#II-1029` と親 `#IM-1027` は自動完了せず `入荷待ち` のままで、入荷指示詳細の「完了に更新する」→確認ダイアログ「完了にする」を実行すると、入荷指示と親移動伝票が `成功 完了 / 入荷完了` になりました。

入荷と出荷の順序は固定ではありません。出荷前に入荷を一括受領できますが、その時点では親の移動伝票は完了せず、出荷実績登録後に完了します。予定数より少ない入荷、予定数より多い入荷、同一入荷指示への複数回登録も可能です。予定数と入荷済み数が一致しない場合や累計過入荷の場合は、入荷指示側の「完了に更新する」で手動完了します。

作成時のバリデーション:

| 条件 | エラー |
|:--|:--|
| 配送元未選択 | 配送元を選択してください |
| 配送先未選択 | 配送先を選択してください |
| 商品0件 | 商品を1つ以上追加してください |
| 数量0 | quantityは1以上で入力してください |

### キャンセルについて

伝票詳細の「**その他の操作**」メニュー内に「**キャンセル**」があります。キャンセルを実行すると次のダイアログが表示されます。

> 見出し「移動伝票をキャンセルしますか？」
> 本文「移動伝票をキャンセルすると、この伝票に紐づく入荷予定および出荷指示もすべてキャンセルされます。」

キャンセル後の状態:

- 移動伝票のステータス: 「キャンセル」
- 入荷指示: 「入荷キャンセル」バッジで入荷管理のキャンセルタブに表示される
- 出荷指示: キャンセル専用タブはなく、ステータス別タブには表示されない。2026-06-27再確認では、キャンセル済み行は出荷管理の「すべて」タブに `キャンセル済み` として表示された。関連する移動伝票詳細のリンクや直接URLから詳細を開ける場合がある。条件指定エクスポートにはキャンセル済みステータスの選択肢がある
- 確保していた在庫: 「販売可能」区分に戻る

2026-06-18の実機確認では、出荷実績を登録した後は親移動伝票のキャンセル項目がdisabledになりました。2026-06-27の `#IM-1025` でも、完了後の「その他の操作」メニューには「キャンセル」が残りますが disabled で実行できません。キャンセルできる前提で案内するのは、出荷前の移動伝票に限定してください。

### 重要な注意（取り寄せ販売の行き止まり）

取り寄せ販売フロー経由で作成した移動伝票をキャンセルした場合、**元の在庫依頼は「確保済み」のまま残ります**。この状態では確保済み一覧から移動伝票を再作成することも、在庫依頼をクローズすることもできません（行き止まり）。

同じ取り寄せをやり直すには、在庫依頼を新規作成してください。

2026-06-18の実機確認では、在庫依頼の確保済み一覧から作成した移動伝票 `#IM-1017` に「外部システムから連携されている移動伝票です」と表示されました。2026-06-27の `#IM-1025` でも同じ表示を確認し、関連する出荷指示 `#IO-1025` と入荷指示 `#II-1025` が自動生成されました。この経路の伝票は商品編集が制限されますが、出荷実績登録・一括受領・親移動伝票の完了までは管理画面で実行できます。完了後も元の在庫依頼ステータスは「確保済み」のままです。

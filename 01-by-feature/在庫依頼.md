# 在庫依頼

> 対象画面: 在庫依頼 / `/admin/inventory_allocation_requests`　|　最終確認: 2026-06-27

## この機能でできること

取り寄せ販売の起点となる機能。倉庫や他店舗にある在庫を、指定したロケーションへ確保・移動するよう依頼できる。

- 商品バリエーションと希望数を指定して在庫確保を依頼する
- 引当元にするロケーション（リクエスト先）を絞り込んで依頼する
- 依頼ごとのステータス（確認待ち／確保済み／終了）を一覧で管理する
- 確保済み後、移動伝票の作成へ進む

---

## 画面構成

### 在庫依頼一覧（/admin/inventory_allocation_requests）

画面上部に3つのタブがある。タブを切り替えると、それぞれのステータスにある依頼だけが表示される。

| タブ名 | 表示される依頼 |
|:--|:--|
| 確認待ち | 作成済みで未引当の依頼 |
| 確保済み | 在庫の引当てが完了した依頼 |
| 終了 | クローズ操作により終了した依頼 |

各行に表示される列は次のとおり。

| 列名 | 内容 |
|:--|:--|
| 商品 | 商品名・バリエーション情報 |
| SKU | SKUコード |
| ステータス | 現在のステータス（確認待ち／確保済み／終了） |
| チャネル | 依頼の発生元チャネル。管理画面から手動作成した在庫依頼は `web-admin` |
| 希望数 | 依頼した数量 |
| 確保済み | 引当て済みの数量 |
| 作成日時 | 依頼が作成された日時 |

### サブメニュー「確保済み」（/admin/inventory_allocation_request_confirmations）

左サイドバーの「在庫依頼」配下に「確保済み」サブメニューがある。引当て完了済みの在庫をまとめて確認し、移動伝票を一括作成する専用画面。

| 列名 | 内容 |
|:--|:--|
| 商品 | 商品名・バリエーション情報 |
| SKU | SKUコード |
| 確保数 | 引当て済みの数量 |
| 移動元 | 引当て元ロケーション |
| 移動先 | 在庫を届けるロケーション |
| 確保日時 | 引当てが完了した日時 |

![「確保済み」一覧画面。商品・SKU・確保数・移動元・移動先・確保日時の列と「移動伝票を作成する」ボタンが表示される](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/flow-02-kakuho-list.png)

既存の確保済み行が残っている場合、「移動伝票を作成する」は対象外の行も巻き込む可能性があります。2026-06-28再確認では、この一覧にキーワード検索・絞り込み追加・ロケーションフィルタは表示されません。対象外行がある状態では実行せず、対象行だけが一覧に残っていることを確認してから進めます。2026-06-27の `#IM-1025` 作成後は、対象行が確保済み一覧から消え、在庫依頼詳細に `#IM-1025` のリンクが追加されました。

## 作成フォームの項目（/admin/inventory_allocation_requests/create）

右上の「在庫依頼を作成する」から遷移する。

### 商品セクション

| 項目 | 必須 | 入力方法 |
|:--|:--|:--|
| 商品バリエーション* | 必須 | テキスト検索 + 「選択」ボタン |

### リクエスト内容セクション

| 項目 | 必須 | 入力方法 | 補足 |
|:--|:--|:--|:--|
| 希望数* | 必須 | 数値入力（初期値: 0） | — |
| 移動先ロケーション* | 必須 | テキスト検索 + 「選択」ボタン | 在庫の届け先（引当先） |
| リクエスト先ロケーション | 任意（※引当には実質必須） | テキスト検索 + 「選択」ボタン | 在庫の引当元になるロケーション。指定すると、後の引当てダイアログでこのロケーションのみが選択肢に表示される。**未指定だと引当ダイアログの引当元ロケーション選択肢が空になり引当不可**（2026-06-20実機確認。エラー「引当元ロケーションを選択してください」） |

入力後、「保存する」ボタンをクリックすると依頼が作成され、詳細画面へ遷移する。

2026-06-18の実機確認では、空保存時に商品・希望数・移動先ロケーションが必須エラーになりました。リクエスト先ロケーションは未指定でも保存可能ですが、**引当するには指定が実質必須**（未指定だと引当ダイアログの引当元ロケーション選択肢が空）。希望数0は「1以上の数量を入力してください」で保存できません。

## 詳細画面の表示と用語

詳細画面では、作成フォームとは異なるラベルが使われる箇所がある。

| 作成フォームのラベル | 詳細画面でのラベル |
|:--|:--|
| 移動先ロケーション | 引当先ロケーション |
| リクエスト先ロケーション | 依頼先 |

依頼直後の詳細画面には次の情報が表示される。

| 項目 | 内容 |
|:--|:--|
| 引当先ロケーション | 在庫の届け先 |
| 確保済み数量 | 引当て済み数 / 希望数（例: 0 / 2 点） |
| 残り確保数 | まだ引当てが必要な数量 |
| チャネル | 依頼の発生元。管理画面から手動作成した在庫依頼は `web-admin` |
| 依頼先 | リクエスト先として指定したロケーション名 |

## ステータス遷移

```
在庫依頼を作成
       ↓
  【確認待ち】  ← 「在庫依頼をクローズする」→ 【終了】
       ↓  「在庫を引当てる」
  【確保済み】
       ↓  確保済み一覧から「移動伝票を作成する」
  移動伝票・出荷指示・入荷指示が自動生成（→ 後続の出荷／入荷フローへ）
```

- **確認待ち**: 依頼が作成された初期ステータス。引当てを待っている状態。
- **確保済み**: 「在庫を引当てる」を実行して希望数**全量**を引当て、ロケーションと数量を確定した状態。部分引当（希望数に満たない）では「確認待ち」のまま確保済み数量のみ増え、確保済み一覧には引当てた分だけ表示される。
- **終了**: 「在庫依頼をクローズする」を実行した結果。確認待ちのときのみ実行可能。

## 引当てダイアログの仕様

詳細画面右上の「在庫を引当てる」をクリックするとダイアログが開く。

| 項目 | 内容 |
|:--|:--|
| ロケーション | 選択肢はリクエスト先として指定したロケーションのみ表示される |
| 数量 | 数値入力。上限は希望数まで |

「引当てる」ボタンで確定すると確保済み数量が増えます。希望数全量を引当てるとステータスが「確保済み」に変わります（部分引当では「確認待ち」のまま。詳細は補足を参照）。

![「在庫を引当てる」ダイアログ。ロケーションと数量を指定して「引当てる」で確定する](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/flow-02-hikiate-dialog.png)

## クローズ操作（在庫依頼をクローズする）

詳細画面の「その他の操作」メニューに「在庫依頼をクローズする」がある。

| ステータス | 実行可否 |
|:--|:--|
| 確認待ち | 実行可能 → ステータスが「終了」になる |
| 確保済み | グレーアウトで実行不可 |
| 終了 | 再開・再利用はできない。「在庫依頼をクローズする」は残るがグレーアウト |

確保済みになった後は、移動伝票をキャンセルしても依頼は「確保済み」のまま残り、クローズもできない。同じ取り寄せをやり直す場合は、在庫依頼を新規作成する必要がある。

2026-06-18の実機確認では、未引当の確認待ち依頼は「在庫依頼をクローズする」で終了にできました。確認ダイアログには「在庫依頼をクローズします。注文明細に修正が必要な場合はその対応を行なってください。」と表示されます。

2026-06-28の実機再確認では、確認待ち依頼をクローズすると詳細ステータスは「終了」になり、「在庫を引当てる」は消えます。「その他の操作」内の「在庫依頼をクローズする」はdisabledで残りますが、再開・再利用に相当する操作は表示されません。

## 補足・注意点

- 「移動先ロケーション」と「引当先ロケーション」は同じ意味で、画面によって表記が異なる。
- 「リクエスト先ロケーション」は引当元になるロケーションを指す。保存上は任意ですが**引当には実質必須**です（未指定だと引当ダイアログのロケーション選択肢が空になり引当不可）。詳細は作成フォームの項目（84行）を参照。
- 在庫依頼の「確保済み数量」は、「在庫を引当てる」を実行した時点で増える。ただし、この時点ではSKU在庫詳細の販売可能・取置中・手持ちは変化しない。
- 在庫が実際に移動するのは、確保済み一覧から移動伝票を作成し、出荷/入荷の実績登録を進めた後。確保済みステータスと在庫区分の「取置中」は別物として説明する。
- 確保済み一覧から移動伝票を作成すると、対象行は確保済み一覧から消え、元の在庫依頼詳細に移動伝票番号リンクが追加される。
- 確保済み一覧から作成された移動伝票は「外部システムから連携されている移動伝票です」と表示され、商品編集が制限される。ただし出荷実績登録・入荷一括受領・移動完了までは管理画面で実行できる。
- 移動伝票を完了しても、元の在庫依頼ステータスは「確保済み」のまま残る。
- **2026-06-27一気通貫確認（`#IM-1025`）**: 在庫依頼作成直後は `確認待ち`、引当後は `確保済み`。この2操作では移動元・移動先の在庫数は変わりません。移動伝票作成後に移動元は販売可能9→8・取置中0→1、出荷後に取置中1→0・手持ち9→8、入荷完了後に移動先は販売可能1→2・手持ち1→2になりました。
- **部分引当の挙動（2026-06-20実機確認）**: 希望数に対して引当可能数が足りない場合、確保済み一覧には引当てた分だけ表示され移動伝票を作成できます。ただし依頼のステータスは希望数全量を引当てるまで「確認待ち」のまま推移します（部分引当では「確保済み」ステータスに遷移しない）。例: 希望2・引当元在庫1 → 確保1/2で残り1、ステータスは確認待ちのまま。
- サイドバーには未完了の依頼件数がバッジで表示される（0件のときは非表示。100件超は「100+」と表示）。

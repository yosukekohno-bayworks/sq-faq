# ポイント（注文ポイント）

> 対象画面: 注文ポイント / `/admin/point_calculation_rules` | 最終確認: 2026-07-16

## この機能でできること

- 注文ポイントの付与ルールを作成・管理する（購入金額○円ごとに○ポイント）
- ポイントの有効日数を設定する
- オンライン注文と店舗注文でポイントが付与されるまでの遅延日数を個別に設定する
- ポイント利用分を付与対象に含めるか、セール価格商品を付与対象から除外するかを設定する
- 会員ランク算出ルールをポイント付与ルールに紐付ける
- ポイントキャンペーンの競合時の優先順位（新しい開始日時 / 古い開始日時）を設定する
- ポイント付与の対象外店舗を設定する
- キャンペーン・誕生日ポイント・利用外商品・失効通知を子メニューから個別に管理する

## 画面・項目の説明

| 項目（UIラベル） | 説明 | 必須 | 制約・選択肢 |

| 項目（UIラベル） | 説明 | 必須 | 制約・選択肢 |
|:--|:--|:--|:--|

| セール価格の商品を除外する | セール価格の商品には通常ポイント・キャンペーンポイントを付与しない | 任意 | チェックボックス |

| 付与までの日数（店舗） | 店舗での決済完了からポイントを利用可能にするまでの日数 | 必須（★） | 数値（日後）。空欄で保存すると「値を入力してください」エラーが表示される。実付与は未確認 |

#### 会員ランク算出ルール

|:--|:--|:--|:--|
| 会員ランク算出ルール | このポイント付与ルールに紐付ける会員ランク算出ルール | 任意 | コンボボックス。初期表示: 「未設定」 |

#### ポイントキャンペーン優先ルール

UIラベルは「キャンペーン適用優先度」です。

複数のポイントキャンペーンが同一注文に適用できる条件を満たす場合に、どちらを優先するかを設定する。

| 選択肢（UIラベル） | 説明 |
|:--|:--|
| 開始日時が新しいキャンペーンを優先する | より新しく設定されたキャンペーンを優先（デフォルト） |
| 開始日時が古いキャンペーンを優先する | より古く設定されたキャンペーンを優先 |

ヘルプ文（原文）: 「注文に対して条件を満たすポイントキャンペーンが複数個ある場合に、どのキャンペーンを優先的に適用するかを設定します。」

### 注文ポイント付与ルール詳細画面（保存後）

付与ルールを保存すると詳細画面が開きます。2026-07-16時点では「基本設定 / 会員ランク倍率 / 商品倍率 / 対象外店舗」の4タブです。基本設定は作成フォームと同じ項目を直接編集して保存する構成で、「ルールを削除」も表示されます。

![注文ポイント付与ルール詳細の「対象外店舗」タブ](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/crm-point-detail-target-stores.png)

| セクション / 項目 | 内容 |
|:--|:--|
| 利用テナント数 | このルールを使用しているテナントの数 |
| 付与率 | 設定した付与率（例: 100円ごとに1ポイント） |
| 付与対象 | `ポイント利用分を含める`、`セール価格の商品を除外する` の設定 |
| 付与までの日数（オンライン） | オンライン注文での遅延日数 |
| 付与までの日数（店舗） | 店舗注文での遅延日数 |
| 利用可能になってから有効な日数 | 設定したポイント有効日数 |
| 会員ランク算出ルール | 紐付けた会員ランク算出ルール |
| ポイントキャンペーン優先設定 | 設定したキャンペーン優先方式 |

#### 会員ランク倍率セクション

詳細画面には「**会員ランク倍率**」タブがあります。2026-06-27の実機確認では、URLは `/admin/point_calculation_rules/{id}/rank_multipliers` でした。

「追加する」を押すと `会員ランク倍率を追加する` ダイアログが開き、`会員ランク*` と `倍率*` を入力します。`既に設定されている会員ランクは選択できません` という注意文も表示されます。2026-06-28実機確認では、`Bronze` を `2倍` として保存でき、一覧に `Bronze` / `2倍` と表示されました。行選択後の `削除する` で確認ダイアログを経て削除でき、検証用の追加行は削除済みです。実注文への計算は未確認です。

#### 商品倍率セクション

詳細画面には「**商品倍率**」タブがあります。URLは `/admin/point_calculation_rules/{id}/point_multiplier_products` です。

「追加する」を押すと `商品倍率を追加する` ダイアログが開き、`商品`（選択ボタン付き）と `倍率*` を入力します。`/product_multipliers` という推測URLは404でした。2026-06-28実機確認では、商品 `TEST_20260622_OPTION` を倍率0で保存でき、一覧に `0倍` と表示されました。検証用の追加行は削除済みです。実注文への計算は未確認です。

#### 対象外店舗セクション

「**対象外店舗**」タブのURLは `/admin/point_calculation_rules/{id}/retail_location_exclusions` です。「追加する」で `ロケーションを選択する` ダイアログが開き、場所コード検索と列 `名前` / `場所コード` から、このポイント付与ルールを適用しない店舗を選択します。

現行の作成フォームには「ポイントキャンペーンは以下の条件に該当する注文にのみ適用されます。未入力の場合は適用されません。」と表示されます。対象店舗は作成時に選択でき、保存後も「対象店舗」タブから追加・削除できます。

![ポイントキャンペーン作成フォームの適用条件](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/crm2-campaign-conditions.png)

2026-07-01に `stack-ps-yosuke.myshopify.com` を選択した検証用ポイントキャンペーンを作成したところ、保存後の詳細画面に `チャネル stack-ps-yosuke.myshopify.com` と表示されました。一覧で行選択すると `ルールを削除する` が表示され、確認ダイアログ「ポイントキャンペーンを削除しますか？」から削除できました。検証用キャンペーンは削除済みです。

種別 `なし` で空保存した場合は、`タイトルを入力してください`、`開始日時を入力してください`、`終了日時を入力してください`、`ポイントを入力してください`、`対象の注文ポイント付与ルールを選択してください` が表示されます。

#### ポイントキャンペーン種別による動作の違い

| 種別 | 概要 |
|:--|:--|
| なし | 会員ランク・購入金額の追加条件なし。倍率または追加ポイントを入力する。チャネルまたは対象店舗の指定は必要 |
| 会員ランク | 会員ランクごとに異なるポイントを設定できる。キャンペーン保存後、詳細ページの「会員ランク」セクションからランク別に設定する |
| 購入金額 | 購入金額ごとに倍率またはポイントを設定する。保存後、詳細ページの「購入金額を追加する」から金額条件を追加する |

#### 種別ごとの保存後設定

| 種別 | 保存後の追加導線 | 追加フォーム | 削除操作 |
|:--|:--|:--|:--|
| 会員ランク | 「会員ランクを追加する」 | 会員ランク算出ルール、会員ランク、ポイント倍率 | 行選択 > 「会員ランクを削除」。確認「会員ランクを削除しますか？」 |
| 購入金額 | 「購入金額を追加する」 | 購入金額（円）、ポイント倍率 | 行選択 > 「購入金額を削除」。確認「購入金額を削除しますか？」 |

> 旧UIにあった「商品」種別と商品/ブランド/CSV追加導線は、2026-07-16時点の新規作成フォームでは利用できません。既存の商品種別キャンペーンや直接URLの扱いは未確認です。

#### 倍率の計算例（アプリ画面の説明文原文）

> ポイントキャンペーン倍率は「通常ポイントに加えた合計倍率」として適用されます。
> 例）ポイントキャンペーン倍率に「3」と入力すると、通常倍率が2倍の会員ランクなら、通常2倍 + ポイントキャンペーン3倍 = 合計5倍 となります。

#### キャンペーン作成後に変更できない項目

- ポイント付与方法（編集フォームに項目が表示されない）
- 対象の注文ポイント付与ルール（編集フォームに項目が表示されない）

### 利用外商品（`/admin/point_application_excluded_products`）

ポイント利用を止めたい商品を登録する。「追加する」ボタンから `利用外商品を追加する` 画面へ遷移し、`商品を選択する*` の `選択` ボタンで商品を選ぶ。

商品選択ダイアログの列は「商品」「商品コード」。SKU/バリエーション列ではなく、商品コード単位で選択する画面。

ポイント付与を止めたい場合は、この画面ではなく、注文ポイント付与ルール詳細の「商品倍率」で対象商品の倍率を0倍に設定する。2026-06-28実機確認では、利用外商品も追加・一覧反映・削除まで確認済みです。

## 補足・注意点

- 注文ポイント付与ルール作成時、「付与までの日数（オンライン）」「付与までの日数（店舗）」「利用可能になってから有効な日数」はいずれも必須
- ポイント付与ルールと会員ランク算出ルールの紐付けは任意。紐付けを行うと当該ポイントルールが適用される顧客を指定の会員ランク算出ルールで管理できる
- 会員ランクごとに異なるポイントキャンペーン倍率を設定したい場合は、キャンペーン種別「会員ランク」を使い、保存後の詳細ページから設定する
- 購入金額・ポイント数の入力欄に表示されている数値（100・1など）はプレースホルダ（参考値）であり、保存時に実際の値入力が必要
- 付与ルール詳細は「基本設定」「会員ランク倍率」「商品倍率」「対象外店舗」の4タブ。2026-07-16に対象外店舗のロケーション選択ダイアログまで確認済み
- ポイントキャンペーンは「チャネル」「対象店舗」がともに未入力だと適用されない。新規作成種別は3択で、旧「商品」は選べない
- 2026-07-01実機確認では、支払い済み・発送済み内部GraphQL注文 `FAQ-ORDER-DECREMENT-SHIP-202607010132` は注文詳細で `付与対象に含む` と表示されるが、`ポイント付与 0ポイント` で、顧客ポイント履歴にも注文由来行は表示されなかった
- ポイントキャンペーンの付与方法で「通常付与ポイントに加えて指定のポイントを追加で付与する」を選ぶと、種別「なし」では固定ポイント数の入力欄に切り替わる

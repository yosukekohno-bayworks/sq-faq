# 注文管理

> 対象画面: 注文管理 / `/admin/orders`　|　最終確認: 2026-07-02

## この機能でできること

- 販売チャネルや連携経路から流入した注文の一覧表示・確認・追跡
- 下書き注文の一覧確認（現行空状態では作成ボタンはdisabled相当）
- 返品の一覧確認（返品の受領・完了は注文詳細内で扱う）
- 注文・返品から顧客詳細と売上実績へ反映される状態を確認する

> **注文の変更はどこで行う？（開発元の説明）**: Shopify経由の注文は、受注内容の変更などを**Shopify側の管理画面で行い、変更内容がSQへ自動反映**される運用です。SQ側から操作するのは、ECモール（楽天など）経由の注文の返品処理などが中心です。

<!-- 再検証: Shopify未連携商品での注文流入・返金・出荷/返品未反映は確認済み。正式連携SKUでの在庫引当・出荷/返品反映は商品アクセストークン設定後に再検証。 -->

## 画面・項目の説明

### 注文管理 一覧（/admin/orders）

2026-06-28に空状態と内部GraphQLで投入した検証注文のデータあり状態、2026-07-02にShopify経由の未連携商品注文を確認しました。

| 確認項目 | 確認した表示 |
|:--|:--|
| 見出し | `注文管理` |
| タブ | `すべて` |
| 空状態メッセージ | `アイテムが見つかりませんでした` / `絞り込みや検索ワードを変更してみてください` |
| データあり状態の列 | `注文 / 制限 / 注文日時 / 顧客 / チャネル / 店舗 / アイテム / 合計 / 決済状況 / 出荷状況 / 返品状況 / タグ` |
| 検証注文の表示 | `FAQ-ORDER-BYPASS-20260628100919`、顧客、チャネル `FAQ_INTERNAL_GRAPHQL`、`￥1,990`、`支払い済み`、`発送済み` |
| 検索・絞り込みUI | 空状態では表示なし。データあり状態では `注文番号で検索する`、`絞り込みを追加`、`名前を付けて保存` が表示。絞り込み条件は `テナント`、`タグ付けされている`、`タグ付けされていません`、`チャネル` |
| `注文を作成する` ボタン | 表示なし |
| 直URL `/admin/orders/create` | `予期せぬエラーが発生しました` |

Shopify経由注文では、`#1008` / `#1009` が注文一覧に表示され、チャネルは `stack-ps-yosuke.myshopify.com` でした。注文詳細には `外部システムから連携されている注文です` と表示されます。ただし、今回の商品 `Selling Plans Ski Wax` はSQの正式連携SKUではなく、商品行は `新規` として取り込まれました。

### 注文詳細

検証用に用意した注文では、注文詳細に `返金`、`返品・交換を作成`、`返金履歴`、`出荷指示`、`在庫依頼`、ポイント/会員ランク算出対象、タグが表示されました。在庫を動かさない検証注文（内部値: `inventoryBehavior=BYPASS`）では、画面に `この注文による在庫変動は無視されます` と表示され、在庫引当・手持ち変動は発生しません。

一方、在庫を動かす検証注文（内部値: `inventoryBehavior=DECREMENT`）では、注文作成時に対象SKUの `販売可能 1→0`、`引当済み 0→1`、`手持ち 1→1`、`売可 1→0` を確認しました。つまり、注文時点では商品はまだ倉庫にありますが、注文用に1点確保されたため他の注文には売れない状態になります。さらに出荷対象の商品だったため、出荷管理に注文由来の出荷指示 `#IO-1034` が自動作成され、「出荷待ち」として表示されました。2026-07-01の `FAQ-ORDER-DECREMENT-SHIP-202607010132` / `#IO-1046` では、出荷完了後に約8秒で `販売可能17 / 引当済み0 / 取置中0 / 手持ち17` へ変わり、確保していた1点が倉庫から出たことを確認しました。

Shopify未連携商品の注文 `#1008` / `#1009` では、Shopify側で発送済みにしてもSQ注文は `未出荷` のままで、注文内 `出荷指示` タブと `在庫依頼` タブは空でした。`#1008` をShopify側で全額返金すると、SQ注文の決済状況は `返金済み` になり、注文情報内の決済履歴に `返金 / 成功 / $24.95` が追加されました。ただし、注文内 `返金履歴` タブは `返金はありません` のままでした。`#1009` でShopify返品 `#1009-R1` を作成しても、SQ返品一覧には表示されませんでした。

顧客詳細には直近の注文、初回/最終購入日、初回/最終購入場所が反映されました。売上実績には注文作成時に `数量 1 / 売上 ￥1,990` の明細が作成されました。

### 返品（/admin/order_returns）

注文に紐づく返品処理を一覧で確認するための画面です。

2026-06-28に空状態と検証用返品データあり状態、2026-07-03に注文詳細からの返品・交換作成画面を確認しました。

| 確認項目 | 確認した表示 |
|:--|:--|
| 見出し | `返品` |
| 空状態メッセージ | `アイテムが見つかりませんでした` / `絞り込みや検索ワードを変更してみてください` |
| データあり状態の列 | `返品 / 返品状況 / 交換出荷状況 / 作成日時` |
| 返品ステータス | 作成直後 `情報 返送待ち`、受領後 `注意 返品受取済み`、完了後 `成功 返品処理済み` |
| 検索・絞り込みUI | 空状態では `検索と絞り込みの結果` / `絞り込みを追加` / `キャンセル` / `交換出荷` は表示なし。データあり状態では `絞り込みを追加` が表示され、条件は `キャンセル` / `交換出荷`。検索ボックスと保存ビューはなし |
| 新規作成ボタン | 表示なし |
| 直URL `/admin/order_returns/create` | `このページは存在しないようです` |
| 直URL `/admin/order_returns/{id}` | `このページは存在しないようです` |

返品単体の詳細ページはなく、返品の作成・受領・完了は注文詳細内で扱います。注文詳細の `返品・交換を作成` を押すと `/admin/orders/{order_id}/return` の `返品を作成する` 画面が開きます。数量を入れるだけなら通常返品、数量入力後に出る `交換先のバリエーションを選択する` から交換先SKUを選び、さらに `出荷倉庫` を指定すると交換返品になります。画面上も `交換アイテムは、返品された各アイテムごとに同じ数量で作成されます。` と表示されます。

通常返品は、注文詳細の返品ブロックで `返送待ち` → `返品受取済み` → `返品処理済み` と進み、交換品の出荷指示は作られません。返品完了後、売上実績には `数量 -1 / 売上 -￥1,990` の明細が追加されました。

交換返品では、作成直後と商品受取後は交換出荷指示がまだ生成されず、返品完了後に交換品用の出荷指示 `#IO-1040` が生成されました。2026-07-03時点の `#IO-1040` はキャンセル済みですが、出荷管理に残っており、作成元は `FAQ-REMAINING-EXCHANGE-20260628140117`、出荷明細は `TEST_E2E_20260622_GU_1905_NAVY_M` 1個です。

<!-- 再検証: Shopify未連携商品での返品未反映は確認済み。正式連携SKUやRecustomer起点の返品反映は再検証。 -->

直接 `/admin/order_returns/create` にアクセスすると404画面になります。返品は少なくとも返品一覧画面や直URLからは新規起票できません。

## サイドバーの未完了バッジ

注文管理は左サイドバーのメニュー項目に「未完了件数」を示すバッジが表示されます。

| 仕様 | 内容 |
|:--|:--|
| 表示タイミング | 未完了の出荷指示を持つ注文が1件以上ある場合に表示 |
| 0件のとき | バッジは表示されない |
| 100件超のとき | 「100+」と表示（正確な件数は表示されない） |

2026-06-28追加検証では、`OrderCountFilter` が `inventoryOutboundOrderWorkingStatuses` / `retailLocationID` / `isCancelled` のみを受け付け、`paymentStatus` 条件を持たないことを確認しました。在庫を動かす注文で出荷指示 `#IO-1034` ができると未完了件数が増え、交換返品完了で交換品用の出荷指示 `#IO-1040` ができた後はサイドバーにも `未完了の注文 2件` / `未完了の出荷指示 2件` が表示されました。

## 補足・注意点

- 注文は販売チャネル（Shopify等）から自動的に流入する設計です。内部GraphQL注文、Shopify未連携商品注文、正式連携SKUのShopify注文は条件が違うため、在庫・出荷・売上実績の説明では分けて扱ってください。
- `/admin/orders/create` に直接アクセスすると「予期せぬエラーが発生しました」画面になります。注文管理一覧からの手動作成入口も確認できていません。
- 返品の起票は注文管理一覧や返品一覧画面からは行えません（作成ボタンが存在しない）。返品データがある場合、受領・完了は注文詳細内の返品ブロックで確認します。
- 内部GraphQLのDECREMENT注文では、在庫引当・販売可能数減少・出荷指示生成・出荷完了後の引当済み/手持ち減少まで確認済みです。Shopify未連携商品注文ではSQへ注文流入と返金反映は確認済みですが、出荷/返品/売上実績化は反映されませんでした。正式連携SKUでのShopify注文在庫引当、出荷/返品反映、実ポイント付与・返品時減算は未確認です。
- 下書き注文の「注文を作成する」ボタンに `href` がないまま変わらない場合、購入顧客データの有無だけでは判断できません。会社データ・担当者データ・検証用購入顧客が存在しても有効化されないことを確認済みです。OmnibusCore/リテールポータル/権限/チャネル接続など別条件は未確認です。

# 注文管理

> 対象画面: 注文管理 / `/admin/orders`　|　最終確認: 2026-06-16

## この機能でできること

- 販売チャネルから流入した注文の一覧表示・確認・追跡
- テナント・タグ・チャネルによる絞り込みと、よく使う絞り込み条件の保存
- 下書き注文の一覧確認・注文番号での検索（作成はチャネル接続後に有効化）
- 返品の一覧確認とキャンセル・交換出荷による絞り込み

> **注文の変更はどこで行う？（開発元の説明）**: Shopify経由の注文は、受注内容の変更などを**Shopify側の管理画面で行い、変更内容がSQへ自動反映**される運用です。SQ側から操作するのは、ECモール（楽天など）経由の注文の返品処理などが中心です。

## サブメニュー

注文管理を展開すると以下のサブページにアクセスできます。

| ラベル | URL | 目的 |
|:--|:--|:--|
| 下書き | `/admin/draft_orders` | 管理画面から手動で起票する注文の管理 |
| 返品 | `/admin/order_returns` | 返品処理の一覧確認と絞り込み |

## 画面・項目の説明

### 注文管理 一覧（/admin/orders）

| 項目（UIラベル） | 説明 | 備考 |
|:--|:--|:--|
| 検索と絞り込みの結果 | クリックすると検索バーと「絞り込みを追加」ボタンを展開する | フィルター非表示時はこのボタンのみ表示 |
| 注文番号で検索する | 注文番号で一覧を絞り込むテキストボックス | フィルター展開時に表示 |
| 絞り込みを追加 | 絞り込み条件の選択肢を展開するドロップダウンボタン | 選択肢は下表のとおり |

### 下書き（/admin/draft_orders）

![下書き一覧画面。右上の「注文を作成する」ボタンから遷移できない](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/draft-orders-list.png)

管理画面から手動で注文を起票するための画面です。

| 項目（UIラベル） | 説明 | 備考 |
|:--|:--|:--|
| 注文を作成する | 下書き注文の作成フォームへ進むボタン | 現在は `href` がなく、クリックしても遷移しない。詳細は下記の注意点を参照 |
| 検索と絞り込みの結果 | クリックすると「注文番号で検索する」テキストボックスを展開する | |

| キャンセル | フィルター入力を閉じる（変更を破棄） | フィルター展開時に表示 |
| 前へ／次へ | ページネーションボタン。データがない場合はクリック不可 | |

**注文管理一覧との主な違い**

| 項目 | 注文管理一覧 | 下書き一覧 |
|:--|:--|:--|
| 絞り込みの種類 | テナント/タグ/チャネルの4種 | 注文番号検索のみ |
| 「絞り込みを追加」ボタン | あり | なし |
| 「名前を付けて保存」ボタン | あり | なし |

**タブ構成**

- 「すべて」タブのみ固定表示。

**空状態のメッセージ**

- 「注文が見つかりませんでした」「絞り込みや検索ワードを変更してみてください」

#### 「注文を作成する」が押せない場合

「注文を作成する」ボタンは現在 `href` がない状態です。クリックしても画面遷移・ダイアログ表示は起きません。直接 `/admin/draft_orders/create` にアクセスすると「予期せぬエラーが発生しました」画面になります。

実機確認では、会社データ・担当者データが存在する状態でも有効化されませんでした。販売チャネル（Shopify等）との接続によって生成される「購入顧客」データが必要と推測されますが、現環境（チャネル未接続）では確認できていません。

## サイドバーの未完了バッジ

注文管理は左サイドバーのメニュー項目に「未完了件数」を示すバッジが表示されます。

| 仕様 | 内容 |
|:--|:--|
| 表示タイミング | 未完了の注文が1件以上ある場合に表示 |
| 0件のとき | バッジは表示されない |
| 100件超のとき | 「100+」と表示（正確な件数は表示されない） |

## 補足・注意点

- 注文は販売チャネル（Shopify等）から自動的に流入する設計です。現環境ではチャネルが未接続のため注文データが存在せず、注文詳細画面・ステータス遷移・出荷との連動は確認できていません。**チャネル接続後にこのガイドを追補する予定です。**
- `/admin/orders/create` に直接アクセスすると「予期せぬエラーが発生しました」画面になります。注文管理一覧からの手動作成入口も確認できていません。
- 返品の起票は注文管理一覧や返品一覧画面からは行えません（作成ボタンが存在しない）。返品フローの詳細はチャネル接続後の注文詳細画面確認後に追補します。
- 下書き注文の「注文を作成する」ボタンに `href` がないまま変わらない場合、購入顧客データの有無を確認してください。会社データや担当者データを作成しても有効化されないことが確認されています。

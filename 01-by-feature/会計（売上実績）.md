# 会計（売上実績）

> 対象画面: 売上実績 / `/admin/sale_change_line_items`　|　最終確認: 2026-06-28

## この機能でできること

- 売上実績の一覧を確認する
- 売上実績をCSVでエクスポートする（「売上実績（注文軸）」「売上実績（明細軸）」）
- 注文作成時の正の売上明細、返品完了時の負の売上明細を確認する

### 「売上実績を作成する」ボタン

- 画面上に「売上実績を作成する」と表示されますが、2026-06-27再確認ではリンクではなく、クリックしても操作できません。2026-06-28のデータあり状態でも、操作ボタンとして確認できたのは `エクスポート` のみです。
- `/admin/sale_change_line_items/create` に直接アクセスすると「このページは存在しないようです」画面になります（2026-06-28再確認）。
- 手動での売上実績作成は現時点ではできません。

### CSVエクスポート

- 売上実績一覧の「エクスポート」ボタンを押すと、「注文軸」「明細軸」のメニューが表示されます。
- 「注文軸」は `/admin/csv_export/csv_export_operation_sale_changes`、「明細軸」は `/admin/csv_export/csv_export_operation_sale_change_line_items` に遷移します。
- CSVエクスポート画面（`/admin/csv_export`）の「実績」グループに「**売上実績（注文軸）**」「**売上実績（明細軸）**」カテゴリがあります。
- 注文軸・明細軸とも、履歴一覧の列は `作成日 / テナント / 対象期間 / ステータス / ダウンロード` です。2026-06-27再確認では既存履歴に `成功` / `完了` と `ダウンロード` リンクが表示されました。2026-06-28再確認でも同じ列構成でした。
- 実行フォームはどちらも `テナント*`, `開始日時*`, `終了日時*` で、空保存時は `テナントを選択してください`, `開始日時を入力してください`, `終了日時を入力してください` が表示されます（2026-06-28再確認）。
- 2026-06-28にテナント `ユニクロ`、対象期間 `2026年06月28日 00:00 〜 2026年06月29日 00:00` で注文軸・明細軸を実行し、成功完了後のダウンロードリンクから実CSVを保存しました。どちらも `utf-8-sig`、ヘッダー含め6行でした。
- 注文軸のヘッダーは `ID / イベント / 作成元 / 作成元オブジェクトID / 店舗コード / 処理日時 / 税込フラグ / 通貨コード / 売上 / 税額 / 小計 / 小計に対する税額 / 送料 / 送料に対する税額 / 注文に対する割引額 / 注文明細に対する割引額 / 送料に対する割引額 / 利用ポイント` です。
- 明細軸のヘッダーは `SaleChangeID / SaleChangeLineItemID / 処理日時 / SKU / 数量 / 通貨コード / 売上 / 税額 / 割引合計 / ポイント配賦額 / 注文割引配賦額` です。

## 補足・注意点

- 内部GraphQL検証注文では、注文作成で正の売上明細、返品完了で負の売上明細が作成されることを確認済みです。
- 「売上実績を作成する」表示が、他の条件で手動作成導線として有効になるかは未確認です。現時点では手動作成は案内しません。
- 一覧画面に検索・絞り込みUIは表示されません。CSV実ファイルは注文軸・明細軸とも確認済みですが、メール通知の有無・通知文面・リンク有効期限は未確認です。
- CSVダウンロード用の署名付きURLは秘密情報として扱い、FAQや問い合わせ文面には貼り付けないでください。

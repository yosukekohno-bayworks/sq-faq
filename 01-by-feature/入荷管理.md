# 入荷管理

> 対象画面: 入荷管理 / /admin/inventory_inbound_orders　|　最終確認: 2026-07-01

## この機能でできること

- 入荷指示の一覧をステータス別タブで確認する
- 入荷指示の詳細から入荷実績を登録し、ステータスを入荷完了にする
- 移動伝票がキャンセルされた入荷指示を「キャンセル」タブで確認する

## 画面・項目の説明

### ステータスタブ

![入荷管理のステータスタブ（入荷待ち・入荷依頼済み・入荷作業中・要対応・入荷完了・キャンセル）](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/audit-purchase-ops-09-inbound-tabs.png)

入荷指示はステータスによって 6 つのタブに分類されます。

| タブ名 | 説明 |
|:--|:--|
| 入荷待ち | 入荷作業を待っている入荷指示 |
| 入荷依頼済み | 入荷依頼を済ませた入荷指示 |
| 入荷作業中 | 入荷作業中の入荷指示 |
| 要対応 | 対応が必要な入荷指示 |
| 入荷完了 | 入荷実績が登録された入荷指示 |
| キャンセル | キャンセルされた入荷指示 |

### 一覧の「種別」列

入荷指示一覧には「種別」列があります。移動伝票から生成された入荷指示は「移動」と表示されます。詳細画面を開くと「在庫移動」と表示されます。

### 検索・絞り込み

検索パネルは初期表示では出ず、`F` キーまたは検索アイコン操作で開きます。2026-06-28再確認では、検索欄は「管理番号で検索する」、絞り込み条件は `作業ステータス` / `ロケーション` でした。管理番号検索は `#II-1029` のように `#` 付きで1件に絞り込まれ、`II-1029` やステータス名 `入荷完了` は0件でした。

### PDF/印刷/メール導線

2026-07-01追加確認では、入荷管理一覧に出荷一覧のような行チェックボックス・一括アクションバーは表示されませんでした。入荷完了済み `#II-1038` と入荷キャンセル済み `#II-1039` の詳細、および読み込まれている入荷管理JSでも、PDF/印刷/メール送信/エクスポート導線は確認できません。

### サイドバーの未完了バッジ

サイドナビの「入荷管理」横に未完了の入荷指示件数がバッジ表示されます。0 件のときはバッジが非表示になります。件数が 100 を超える場合は「100+」と表示されます。この仕様は出荷管理と同じです。

### 「入荷実績を登録する」ボタン（入荷指示詳細）

![入荷指示詳細画面。右上に「入荷実績を登録する」ボタンが表示される](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/flow-05-inbound-detail.png)

入荷指示の詳細画面右上に表示されます。実機では、入荷実績を登録するフォームへ遷移して数量を入力するパターンと、一括受領・完了更新の確認画面へ遷移するパターンを確認しています。リンク先が `/complete` でも空状態になり完了しない場合があるため、その場合は詳細へ戻り「入荷実績を登録する」（`/receive`）から受領数を入力して登録してください。

確認画面には「この処理は巻き戻すことができません。」という注意文が明記されています。「登録する」または完了系の実行ボタンを押すと入荷実績が記録されます。

![「入荷実績を登録する」確認ダイアログ。「この処理は巻き戻すことができません。」の注意文が表示される](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/flow-05-inbound-dialog.png)

登録後の変化は次のとおりです。

- 予定数どおりに受領した場合、入荷指示のステータスが「入荷完了」になる
- 一部だけ受領した場合、入荷指示は「入荷待ち」のまま残る
- 同じ入荷指示に複数回の入荷実績を登録できる
- 予定数より多い数量も登録できる。累計で予定数を超えても自動完了せず、必要に応じて「完了に更新する」→「完了にする」を実行する
- 紐づく移動伝票は、出荷指示と入荷指示の両方が完了した時点で完了になる

2026-06-18の実機確認では、次の挙動を確認しています。

| ケース | 確認結果 |
|:--|:--|
| 出荷前に入荷を完了 | 可能。入荷完了後も、出荷が未完了の間は親移動伝票は完了しない |
| 予定2に対して入荷1 | 可能。ステータスは入荷待ちのまま。「完了に更新する」で部分入荷のまま完了にできる |
| 予定1に対して入荷2 | 可能。過入荷でも保存できるが、自動完了はしない |
| 完了後の再登録 | 入荷実績登録ボタンはdisabledになり、再登録できない |

2026-06-27の `#II-1025` では、入荷前詳細に `入荷待ち` と表示され、`入荷指示を一括受領で完了する` から `/complete` の確認画面へ遷移しました。確認画面には「未受領分をすべて受け取って入荷実績を完了します。この処理は巻き戻すことができません。」と表示され、実行後は `成功 完了 / 入荷完了` になり、入荷完了タブ（`?tab=5`）に表示されました。親の移動伝票 `#IM-1025` も同時に `成功 完了 / 入荷完了` になりました。

2026-06-28の `#II-1028` では、詳細に「入荷指示を一括受領で完了する」と「入荷実績を登録する」の両方が表示されましたが、`/complete` は空状態（`アイテムが見つかりませんでした`）になり入荷完了しませんでした。`/receive` の入荷実績登録フォームで受領数1を入力し「登録する」を実行すると、`成功 完了 / 入荷完了` になり、親 `#IM-1026` も `成功 完了 / 入荷完了` になりました。

2026-06-28の `#II-1029` では、移動予定数2に対して1回目に1個、2回目に2個を登録でき、累計入荷済みは3個になりました。この時点でも入荷指示は `情報 入荷待ち` のままで自動完了せず、「完了に更新する」→確認ダイアログ「完了にする」で手動完了すると `成功 完了 / 入荷完了` になりました。出荷完了済みの親移動伝票 `#IM-1027` も同時に `成功 完了 / 入荷完了` になりました。

### 入荷完了後の「入荷実績を登録する」ボタン

入荷実績の登録が完了すると、詳細画面の「入荷実績を登録する」ボタンは**無効化**（クリックできない状態）になります。2026-06-27の `#II-1025` と2026-06-30の `#II-1035` では、「入荷指示を一括受領で完了する」「入荷実績を登録する」がどちらも `href` なしの disabled リンクになっていました。2026-07-01の `#II-1038`（入荷完了）と `#II-1039`（入荷キャンセル）でも同じく、両ボタンは表示されるが `Polaris-Button--disabled` かつ `href` なしでした。入荷完了後・キャンセル後は、画面から実績の再登録・編集・削除を行うことはできません。

## 補足・注意点

- 入荷実績を登録すると**ステータスを巻き戻すことはできません**。登録前に内容を確認してください。
- 入荷完了後は「入荷実績を登録する」ボタンが無効化され、画面からの再登録・編集・削除は行えません。<!-- 開発元確認: 入荷実績の修正が必要な場合の対処方法。API経由の操作可否を開発元に確認 -->
- 入荷管理画面から入荷指示を新規作成するボタンはありません。入荷指示は移動伝票の作成と同時に自動生成されます。
- 移動伝票をキャンセルすると、紐づく入荷指示は「キャンセル」タブに「入荷キャンセル」として残ります。
- 入荷と出荷の順序は固定ではありません。出荷前に入荷を完了できますが、移動伝票全体の完了には出荷と入荷の両方が必要です。

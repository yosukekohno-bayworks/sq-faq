# 顧客管理

> 対象画面: 顧客管理 / /admin/purchasing_customers　|　最終確認: 2026-06-28

## この機能でできること

- 購入顧客の一覧表示
- メールアドレスによる顧客検索
- テナント・メタフィールド・会員証バーコードによる絞り込み
- ポイントの一括付与（CSVインポート経由）

---

## 顧客一覧（/admin/purchasing_customers）

2026-06-28時点のstaging実測では、管理画面UIから顧客を直接新規作成する導線はありません。検証用に内部GraphQLで購入顧客を作成した状態では、一覧に顧客行が表示されることを確認しました。

空状態では「アイテムが見つかりませんでした」「絞り込みや検索ワードを変更してみてください」と表示されます。

|:--|:--|

## 顧客の作成方法

管理画面から顧客を直接作成するボタン・フォームはありません。顧客は販売チャネル（Shopify など）との連携を通じて自動的に登録される設計です。<!-- 連携実接続対象外: チャネル連携後に実機確認 -->

直接 `/admin/purchasing_customers/create` にアクセスすると、「予期せぬエラーが発生しました」「エラーが発生しました。しばらくしてから再度お試しください」が表示されます。「再実行」を押しても復旧しません。

会社詳細で作成する「担当者」は、顧客管理の顧客とは別データです。担当者を作成しても顧客一覧には表示されず、ディスカウントの顧客管理タブでメール検索しても候補には出ません。

## 顧客詳細画面

2026-06-28に内部GraphQLで作成した検証用購入顧客で、詳細画面の基本構成を確認しました。さらに同じ顧客で検証注文を作成し、注文後の表示変化を確認しました。

| セクション | 表示内容 |
|:--|:--|
| 直近の注文 | 注文なしでは「このお客様はまだ注文していません」。検証注文後は注文カード、支払い済み/発送済み、作成元、金額が表示 |
| ポイント | 保有ポイント、付与予定、ポイント加算/減算ボタン |
| マイル | 保有マイル、累計獲得、加算/減算ボタン |
| ディスカウント | 顧客に紐づくディスカウント |
| メタフィールド | 性別、会員番号など |
| 顧客情報 | 氏名、メールアドレス、電話番号、SMS認証状態、外部ID、作成日時/更新日時、購買サマリー、メモ、タグ。検証注文後は初回/最終購入日と購入場所が更新 |

内部GraphQL検証注文では、直近の注文と購買サマリーへの反映は確認済みです。ポイント残高の変化、会員ランク反映、チャネル由来注文での反映は未確認です。

## 補足・注意点

- 「インポート」ドロップダウンの選択肢は「ポイント一括付与」のみです。顧客データ自体のCSVインポートはこの画面からは行えません。
- 会員証バーコードによる絞り込みは 2026-06-04 のリリースで追加された機能です。
- 顧客のCSVエクスポートはこの画面ではなく、CSVエクスポート専用画面（/admin/csv_export）から行います。
- ディスカウントの対象顧客追加は、同じテナントの購入顧客が候補になります。2026-06-28に検証用購入顧客で検索、追加、一覧表示、削除まで確認済みです。テナントが一致しない顧客や会社担当者は候補に出ません。

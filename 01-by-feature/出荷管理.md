# 出荷管理

> 対象画面: 出荷管理 / /admin/inventory_outbound_orders　|　最終確認: 2026-06-28

## この機能でできること

- 出荷指示の一覧をステータス別タブで確認する
- 出荷指示の詳細から出荷実績（配送キャリア・追跡コード）を登録し、ステータスを出荷完了にする
- ヤマトB2クラウドまたは DHL の出荷実績 CSV をインポートして出荷実績を一括登録する（ヤマトB2はCSV保存・詳細作成まで実機確認済み。実行後の出荷完了遷移は未確認）
- 条件を指定してヤマトB2クラウド取り込み用 CSV を生成する

## 画面・項目の説明

### ステータスタブ

![出荷管理のステータスタブ（保留中・出荷待ち・依頼済み・作業中・欠品・要対応・出荷完了）](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/audit-purchase-ops-06-outbound-tabs.png)

出荷管理には「すべて」タブと、ステータス別の 6 つのタブがあります。

| タブ名 | URLパラメータ | 説明 |
|:--|:--|:--|
| すべて | なし | ステータス横断の一覧。2026-06-27再確認では `キャンセル済み` の出荷指示もここに表示された |
| 保留中 | `?tab=on_hold` | 出荷作業を保留している出荷指示 |
| 出荷待ち | `?tab=waiting` | 出荷作業を待っている出荷指示 |
| 依頼済み | `?tab=requested` | 出荷依頼を済ませた出荷指示 |
| 作業中 | `?tab=in_progress` | 出荷作業中の出荷指示 |
| 欠品・要対応 | `?tab=rejected` | 欠品など対応が必要な出荷指示 |
| 出荷完了 | `?tab=complete` | 出荷実績が登録された出荷指示 |

キャンセル専用タブは一覧に存在しません。2026-06-27再確認では、キャンセル済みの出荷指示（例: `#IO-1015`, `#IO-1014`, `#IO-1007`）は「すべて」タブに `キャンセル済み` として表示され、ステータス別タブには表示されませんでした。移動伝票詳細などの関連リンクや直接URLからは、キャンセル済みの出荷指示詳細を開ける場合があります。条件指定エクスポートには出荷作業ステータス「キャンセル済み」の選択肢があります。

### 検索・絞り込み

検索パネルは初期表示では出ず、`F` キーまたは検索アイコン操作で開きます。2026-06-28再確認では、検索欄は「管理番号で検索する」、絞り込み条件は `作業ステータス` / `出荷方法` / `代引き` / `配送先(国)` でした。管理番号検索は `#IO-1027` のように `#` 付きで1件に絞り込まれ、`IO-1027` やステータス名 `出荷完了` は0件でした。

### サイドバーの未完了バッジ

サイドナビの「出荷管理」横に未完了の出荷指示件数がバッジ表示されます。0 件のときはバッジが非表示になります。件数が 100 を超える場合は「100+」と表示されます。

### 「インポート」ボタン（ドロップダウン）

![「インポート」ボタンのドロップダウン（ヤマトB2クラウド（出荷実績）・DHL（出荷実績））](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/gap-purchase-ops-15-outbound-import-dropdown.png)

出荷実績 CSV をインポートします。クリックすると 2 種類の選択肢が表示されます。

| 選択肢 | 遷移先 |
|:--|:--|
| ヤマトB2クラウド（出荷実績） | ヤマトB2クラウドの出荷実績インポート画面 |
| DHL（出荷実績） | DHL の出荷実績インポート画面 |

いずれの画面でも「新規インポート」から CSV ファイルをアップロードして出荷実績を一括登録できます。アップロードするファイルはCSV ファイルのみです。2026-06-27再確認では、ファイル未選択のまま保存するとヤマトB2クラウド/DHLともに `ファイルを選択してください` と表示されました。2026-06-28のヤマトB2検証用CSVでは、保存後にCSVインポート詳細は作成されましたが、再確認時も `実行ステータス 未実行`、`検証成功 0件`、`検証失敗 0件` のままで、実行・出荷完了遷移は未確認です。

ヤマトB2クラウドの CSV フォーマットは全 10 列固定で、必須列は「伝票番号」「出荷日」「お客様管理番号」の 3 列のみです。列の詳細は画面内「テンプレート」のリンクから確認できます。DHLのインポート一覧には、2026-06-27再確認時点でテンプレートリンクは表示されませんでした。

### 「条件指定でエクスポート」ボタン（ドロップダウン）

クリックすると「ヤマトB2クラウド」の選択肢のみ表示されます（DHL のエクスポートはありません）。選択するとエクスポートフォームに遷移します。

#### エクスポートフォームの項目

| 項目（UIラベル） | 必須 | 説明・選択肢 |
|:--|:--|:--|
| 開始日時 | 必須（*） | 出荷指示の絞り込みを開始する日時 |
| 終了日時 | 必須（*） | 出荷指示の絞り込みを終了する日時（デフォルト: 現在日時） |
| 配送先 (国) | 任意 | 国を選択して絞り込む（マルチ選択可） |
| 決済方法 | 任意 | すべての決済 / 代引きのみ / 代引き以外の決済 |
| 出荷作業ステータス | 任意 | 指定しない / 出荷待ち / 保留中 / 依頼済み / 作業中 / 欠品・要対応 / キャンセル済み / 出荷完了 |
| 注文タグ（含む） | 任意 | 入力したタグを含む出荷指示のみが対象（例: priority） |
| 注文タグ（除外） | 任意 | 入力したタグを含まない出荷指示のみが対象（例: test） |
| CSVの出力後に出荷指示のステータスを出荷作業中に変更する | 任意 | チェックボックス（デフォルト: 未チェック） |

「実行する」ボタンを押すと非同期処理が開始されます。2026-06-28の実機確認では、フォーム上の出力物として「ヤマトB2クラウド取り込み用CSV」と「同梱する納品書PDF」が表示され、決済方法は `すべての決済 / 代引きのみ / 代引き以外の決済` の3択、ステータス変更チェックはデフォルトOFFでした。ONにするとラベル上は対象出荷指示を「出荷作業中」に変更する設定ですが、2026-06-28追加確認では、移動伝票由来・配送希望日なしの検証用出荷指示 `#IO-1029` はON実行後も `出荷待ち` のままでした。条件にヒットした実データでの成功例は接続環境で検証待ちです。

納品書PDFについては、PDFエクスポート画面・出荷完了済み出荷指示詳細のどちらにも任意新規生成導線を確認できませんでした。ヤマトB2 CSVエクスポート履歴画面も空状態でダウンロードリンクなしです。実際のメールリンク・PDFファイルは接続環境で検証待ちのため、「PDFが必ず生成される」とは断定しないでください。

### 「出荷実績を登録する」ボタン（出荷指示詳細）

![「出荷実績を登録する」ダイアログ（配送キャリア・追跡コード）](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/flow-06-outbound-shipment-dialog.png)

出荷指示の詳細画面右上に表示されます。クリックするとダイアログが開きます。

#### ダイアログの入力項目

| 項目（UIラベル） | 必須 | 説明 |
|:--|:--|:--|
| 配送キャリア | 任意 | 配送業者名（空でも登録可） |
| 追跡コード | 任意 | 荷物追跡番号（空でも登録可） |

「登録する」を押すと出荷実績が記録され、ステータスが「出荷完了」に変わります。2026-06-16の実機確認では、配送キャリア・追跡コードを空欄のまま登録しても成功し、トースト「出荷を完了しました」が表示されました。2026-06-27の `#IO-1025` でも両項目を空欄のまま登録でき、詳細は `成功 完了 / 出荷完了`、出荷予定1/出荷済み1になり、出荷完了タブ（`?tab=complete`）に表示されました。

2026-06-28の `#IO-1026` では、配送キャリア `FAQCarrier-20260628031813` と追跡コード `FAQTRACK-20260628031813` を入力して登録し、出荷完了まで進むことを確認しました。ただし、最終再ロード後の出荷指示詳細に表示される出荷履歴は日時のみで、入力した配送キャリア・追跡コードは詳細画面にも「すべて」一覧/「出荷完了」一覧にも表示されませんでした。画面上で後から値を確認できる前提で案内しないでください。CSV実ファイルや外部通知への反映は未確認です。

2026-06-18の実機確認では、配送元ロケーションの表示在庫が0のSKUでも出荷実績登録はブロックされず、出荷完了まで進みました。SQは在庫不足を出荷時に止めないため、実在庫を超える出荷登録をするとマイナス在庫になり得ます。

### 出荷完了後の「出荷実績を登録する」ボタン

出荷実績の登録が完了すると、詳細画面の「出荷実績を登録する」ボタンは**無効化**（クリックできない状態）になります。2026-06-27の `#IO-1025` ではDOM上は `button.disabled=false` ですが、`aria-disabled=true` と disabled クラスが付いた状態でした。出荷完了後は、画面から実績の再登録・編集・削除を行うことはできません。

## 補足・注意点

- 出荷実績を登録すると**ステータスを巻き戻すことはできません**。登録前に内容を確認してください。
- 出荷完了後は「出荷実績を登録する」ボタンが無効化され、画面からの再登録・編集・削除は行えません。<!-- 開発元確認: 出荷実績の修正が必要な場合の対処方法。API経由の操作可否を開発元に確認 -->
- 出荷管理画面から出荷指示を新規作成するボタンはありません。出荷指示は移動伝票の作成と同時に自動生成されます。
- キャンセル済みの出荷指示は専用タブではなく「すべて」タブに `キャンセル済み` として表示される場合があります。ステータス別タブには表示されません。詳細は関連リンクや直接URLで開ける場合があります。エクスポートで追跡する場合は、出荷作業ステータスを「キャンセル済み」に指定してください。
- 移動伝票経由で生成された出荷指示は、**引当ステータスが「引当待ち」のまま**推移する場合があります（出荷実績登録・出荷完了になっても変わりません）。出荷完了の判断は、出荷実績の登録有無と「出荷完了」ステータスを優先してください（2026-06-20 `#IO-1018`、2026-06-27 `#IO-1025` 実機確認）。
- 出荷実績登録ダイアログの数量は出荷予定数を全量対象（個別数量入力ではなく、配送キャリア・追跡コードのみ任意入力）。「登録する」で確定します（不可逆）。
- 配送キャリア・追跡コードは入力できますが、2026-06-28実機確認では登録後の出荷指示詳細・一覧には値が表示されませんでした。後から画面で確認する運用には向きません。

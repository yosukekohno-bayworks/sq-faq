# 出荷管理

> 対象画面: 出荷管理 / /admin/inventory_outbound_orders　|　最終確認: 2026-07-03

## この機能でできること

- 出荷指示の一覧をステータス別タブで確認する
- 出荷指示の詳細から出荷実績（配送キャリア・追跡コード）を登録し、ステータスを出荷完了にする
- ヤマトB2クラウドまたは DHL の出荷実績 CSV をインポートして出荷実績を一括登録する（ヤマトB2/DHLともCSV保存・詳細作成まで実機確認済み。ただし検証用CSVでは画面上「CSVのバリデーションを実行しています」のまま進まず、実行後の出荷完了遷移は未確認）
- 条件を指定してヤマトB2クラウド取り込み用 CSV を生成する
- 出荷一覧で対象行を選択し、`アクション > 納品書（PDF）` から納品書PDFの非同期出力を実行する（2026-07-01〜2026-07-03確認。ただし検証ジョブは失敗し、実PDFは未確認）

| タブ名 | URLパラメータ | 説明 |
|:--|:--|:--|
| すべて | なし | ステータス横断の一覧 |
| 保留中 | `?tab=on_hold` | 出荷作業を保留している出荷指示 |
| 出荷待ち | `?tab=waiting` | 出荷作業を待っている出荷指示 |

### 一括ステータス変更（行チェック→ステータス別タブ限定）【2026-07-03実測】

一覧の行にチェックを入れると一括操作バーが出ます。**ステータス変更ボタンはステータス別タブでのみ表示**され、「すべて」タブではエクスポート（…メニュー）だけです。ボタンはタブ=現在ステータスごとに固定です。

![出荷待ちタブで行をチェックした状態。保留中/依頼済み/作業中への更新ボタンが表示される](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/status-matrix-04-outbound-waiting-checked.png)

| タブ | 一括ステータス変更ボタン |
|:--|:--|
| すべて / 出荷完了 / キャンセル済み | なし |
| 保留中 | 出荷待ちに更新する |
| 出荷待ち | 保留中に更新する / 依頼済みに更新する / 作業中に更新する |
| 依頼済み | 出荷待ちに更新する / 作業中に更新する |
| 作業中 | 欠品・要対応に更新する |
| 欠品・要対応 | 出荷指示を再依頼する（→出荷待ちに戻る） |

- 一括で**出荷完了・キャンセルにはできません**（出荷完了は「出荷実績を登録する」、キャンセルは親の移動伝票から）
- 保留中⇔出荷待ち、依頼済み→出荷待ちの**逆行が可能**。中間ステータスの行き来で在庫は変化しません
- 欠品・要対応の間は詳細の「出荷実績を登録する」がdisabledになります
- 子のステータスは親の移動伝票に波及し、**依頼済み・作業中の間は親のキャンセル・明細編集がロック**されます（詳細は `01-by-feature/移動伝票.md`）

### 検索・絞り込み

検索パネルは初期表示では出ず、`F` キーまたは検索アイコン操作で開きます。2026-06-28再確認では、検索欄は「管理番号で検索する」、絞り込み条件は `作業ステータス` / `出荷方法` / `代引き` / `配送先(国)` でした。管理番号検索は `#IO-1027` のように `#` 付きで1件に絞り込まれ、`IO-1027` やステータス名 `出荷完了` は0件でした。

### サイドバーの未完了バッジ

サイドナビの「出荷管理」横に未完了の出荷指示件数がバッジ表示されます。0 件のときはバッジが非表示になります。件数が 100 を超える場合は「100+」と表示されます。

### 「インポート」ボタン（ドロップダウン）

![「インポート」ボタンのドロップダウン（ヤマトB2クラウド（出荷実績）・DHL（出荷実績））](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/gap-purchase-ops-15-outbound-import-dropdown.png)

出荷実績 CSV をインポートします。クリックすると 2 種類の選択肢が表示されます。

| 選択肢 | 遷移先 |
|:--|:--|
| ヤマトB2クラウド（出荷実績） | ヤマトB2クラウドの出荷実績インポート画面 |
| DHL（出荷実績） | DHL の出荷実績インポート画面 |

いずれの画面でも「新規インポート」から CSV ファイルをアップロードして出荷実績を一括登録できます。アップロードするファイルはCSV ファイルのみです。2026-07-02再確認では、ファイル未選択のまま `保存する` を押してもヤマトB2クラウド/DHLとも同じ作成画面に留まり、画面上に赤いエラーメッセージは出ませんでした。ジョブも作成されません。2026-06-28の `#IO-1032`、2026-07-01の `IO-1048`、2026-07-02の存在しないお客様管理番号 `TEST_NO_SUCH_IO_20260702_2358`、2026-07-03の既存出荷指示 `#IO-1050` の検証用CSVでは、保存後にCSVインポート詳細は作成されましたが、検証成功0件/検証失敗0件のまま、画面上は「CSVのバリデーションを実行しています」と表示され続けました。DHLも2026-07-03にCSV保存・詳細作成までは進みましたが、同じく検証中0件のまま実行不可でした。現時点では実行・出荷完了遷移は確認できていません。

ヤマトB2クラウドの CSV フォーマットは全 10 列固定で、必須列は「伝票番号」「出荷日」「お客様管理番号」の 3 列のみです。列の詳細は画面内「テンプレート」のリンクから確認できます。2026-07-02にテンプレートの「定義書」「フォーマット」タブをCSV取得し、任意7列は処理で読み捨てる旨と、ヘッダー順 `伝票番号,お届け先コード,お届け先名,荷物状況,日付,時刻,出荷日,ｻｲｽﾞ品目,運賃,お客様管理番号` を確認しました。DHLのインポート一覧には、2026-07-03再確認時点でもテンプレートリンクは表示されませんでした。

### 「条件指定でエクスポート」ボタン（ドロップダウン）

クリックすると「ヤマトB2クラウド」の選択肢のみ表示されます（DHL のエクスポートはありません）。選択するとエクスポートフォームに遷移します。

#### エクスポートフォームの項目

| 項目（UIラベル） | 必須 | 説明・選択肢 |
|:--|:--|:--|
| 開始日時 | 必須（*） | 出荷指示の絞り込みを開始する日時 |
| 終了日時 | 必須（*） | 出荷指示の絞り込みを終了する日時（デフォルト: 現在日時） |
| 配送先 (国) | 任意 | 国を選択して絞り込む（マルチ選択可） |
| 決済方法 | 任意 | すべての決済 / 代引きのみ / 代引き以外の決済 |
| 出荷作業ステータス | 任意 | 指定しない / 出荷待ち / 保留中 / 依頼済み / 作業中 / 欠品・要対応 / キャンセル済み / 出荷完了 |
| 注文タグ（含む） | 任意 | 入力したタグを含む出荷指示のみが対象（例: priority） |
| 注文タグ（除外） | 任意 | 入力したタグを含まない出荷指示のみが対象（例: test） |
| CSVの出力後に出荷指示のステータスを出荷作業中に変更する | 任意 | チェックボックス（デフォルト: 未チェック） |

「実行する」ボタンを押すと非同期処理が開始されます。2026-07-02の実機確認では、フォーム上の出力物として「ヤマトB2クラウド取り込み用CSV」と「同梱する納品書PDF」が表示され、決済方法は `すべての決済 / 代引きのみ / 代引き以外の決済` の3択、ステータス変更チェックはデフォルトOFFでした。ONにするとラベル上は対象出荷指示を「出荷作業中」に変更する設定ですが、2026-06-28追加確認では、移動伝票由来・配送希望日なしの検証用出荷指示 `#IO-1029` はON実行後も `出荷待ち` のままでした。

2026-07-02には、条件指定エクスポートをOFFで実行するとトースト `ヤマトB2クラウドのエクスポートを開始しました` は表示されましたが、確認時点では新しい履歴行・新しいメールは確認できませんでした。一方で、同日23:34作成の既存成功履歴は `/admin/csv_export/csv_export_operation_inventory_outbound_order_yamato_b2_clouds` に `成功 完了` として表示され、23:35の完了メールからCSVをダウンロードできました。メール本文のリンクはCSV 1件のみで、同梱納品書PDFリンクは確認できませんでした。取得したCSVはヘッダー13列（`送り状種類`、`お客様管理番号`、`お届け先名`、`お届け先電話番号`、`お届け先郵便番号`、`お届け先住所`、`お届け先住所（アパートマンション名）`、`出荷予定日`、`お届け予定（指定）日`、`配達時間帯`、`コレクト代金引換額（税込）`、`コレクト内消費税額等`、`備考欄`）で、データ行に `#IO-1047` が出力されていました。

出荷一覧で対象行を選択して `アクション > エクスポート > ヤマトB2クラウド（CSV）` を選ぶ導線もあります。2026-07-02に `#IO-1050` を1件選択して実行したところ、確認モーダル `ヤマトB2クラウドのCSVをエクスポートする` が開き、チェックボックス `エクスポートした出荷指示のステータスを作業中に更新する` が表示されました。チェックOFFで実行すると23:47にCSV出力履歴が作成されましたが、約20秒後に `重大 / 失敗` になり、`ダウンロード` はdisabled、完了/失敗メールも確認できませんでした。

### 一覧一括操作の「納品書（PDF）」

2026-07-02追加確認では、出荷指示詳細にはPDF/印刷/納品書ボタンはありませんが、出荷管理一覧で対象行にチェックを入れると一括操作バーが表示されます。`アクション` メニューの `エクスポート` セクションに `ヤマトB2クラウド（CSV）` と `納品書（PDF）` が表示されます。

`納品書（PDF）` を選ぶと確認モーダル `納品書をPDFでエクスポートする` が開き、「選択されている出荷指示の納品書をPDFでエクスポートします。」と表示されます。`エクスポート` を押すと `/admin/pdf_export/pdf_export_operation_packing_slips` が新しいタブで開き、PDF出力履歴に行が作成されます。

ただし、2026-07-01に `#IO-1047`（店頭販売による商品の受け渡し）、`#IO-1046`（配送）、`#IO-1048`（配送）、`#IO-1020`（配送・引当済み）の4件、2026-07-02に `#IO-1048`（配送）、2026-07-03に `#IO-1045`（キャンセル済み）と `#IO-1051`（出荷待ち）で再確認したところ、いずれも履歴上 `重大 / 失敗` になり、`ダウンロード` ボタンは disabled のままでした。2026-07-02の追加実行では23:49にPDF履歴行が作成され、約20秒後も `重大 / 失敗`、メールにもPDF/納品書の完了通知は見つかりませんでした。GraphQL上も `isFailedExecutionJob=true`、`downloadURL=null` で、公開フィールドに失敗理由はありません。現時点では導線と失敗履歴作成までは確認済みですが、実PDFファイルの内容・印字は未確認です。

### 「出荷実績を登録する」ボタン（出荷指示詳細）

![「出荷実績を登録する」ダイアログ（配送キャリア・追跡コード）](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/flow-06-outbound-shipment-dialog.png)

出荷指示の詳細画面右上に表示されます。クリックするとダイアログが開きます。

#### ダイアログの入力項目

| 項目（UIラベル） | 必須 | 説明 |
|:--|:--|:--|
| 配送キャリア | 任意 | 配送業者名（空でも登録可） |
| 追跡コード | 任意 | 荷物追跡番号（空でも登録可） |

「登録する」を押すと出荷実績が記録され、ステータスが「出荷完了」に変わります。2026-06-16の実機確認では、配送キャリア・追跡コードを空欄のまま登録しても成功し、トースト「出荷を完了しました」が表示されました。2026-06-27の `#IO-1025` と2026-06-30の `#IO-1041` でも両項目を空欄のまま登録でき、詳細は `成功 完了 / 出荷完了`、出荷予定1/出荷済み1になりました。

2026-06-28の `#IO-1026` では、配送キャリア `FAQCarrier-20260628031813` と追跡コード `FAQTRACK-20260628031813` を入力して登録し、出荷完了まで進むことを確認しました。

> **【2026-07-13訂正】登録した配送キャリア・追跡コードは「出荷履歴」から確認できます**: 出荷完了後の出荷指示詳細の下部に **「出荷履歴」** カードがあり、日時行の右端のアコーディオン（下向きシェブロン）を開くと、**配送キャリア・追跡コード・出荷した商品/SKU/数量** が表示されます。2026-07-13に `#IO-1052` で配送キャリア `ヤマト運輸`・追跡コード `TRACK-TEST-20260713-0001` を入力して登録し、出荷履歴を展開して両値と数量1が表示されることを確認しました。旧記述「詳細・一覧に値が表示されない」は、出荷履歴を展開していなかったための誤りです（一覧列・上部の出荷明細には出ませんが、出荷履歴の展開で確認できます）。CSV実ファイルや外部通知への反映は引き続き未確認です。

![出荷履歴を展開すると配送キャリア・追跡コード・数量が表示される](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/shipping-history-03-carrier-tracking.png)

2026-06-18の実機確認では、配送元ロケーションの表示在庫が0のSKUでも出荷実績登録はブロックされず、出荷完了まで進みました。SQは在庫不足を出荷時に止めないため、実在庫を超える出荷登録をするとマイナス在庫になり得ます。

### 出荷完了後の「出荷実績を登録する」ボタン

出荷実績の登録が完了すると、詳細画面の「出荷実績を登録する」ボタンは**無効化**（クリックできない状態）になります。2026-06-27の `#IO-1025` ではDOM上は `button.disabled=false` ですが、`aria-disabled=true` と disabled クラスが付いた状態でした。出荷完了後は、画面から実績の再登録・編集・削除を行うことはできません。

## 補足・注意点

- 出荷実績を登録すると**ステータスを巻き戻すことはできません**。登録前に内容を確認してください。
- 出荷完了後は「出荷実績を登録する」ボタンが無効化され、画面からの再登録・編集・削除は行えません。<!-- 開発元確認: 出荷実績の修正が必要な場合の対処方法。API経由の操作可否を開発元に確認 -->
- 出荷管理画面から出荷指示を新規作成するボタンはありません。出荷指示は移動伝票の作成と同時に自動生成されます。
- キャンセル済みの出荷指示は専用タブではなく「すべて」タブに `キャンセル済み` として表示される場合があります。ステータス別タブには表示されません。詳細は関連リンクや直接URLで開ける場合があります。エクスポートで追跡する場合は、出荷作業ステータスを「キャンセル済み」に指定してください。
- 移動伝票経由で生成された出荷指示は、**引当ステータスが「引当待ち」のまま**推移する場合があります（出荷実績登録・出荷完了になっても変わりません）。出荷完了の判断は、出荷実績の登録有無と「出荷完了」ステータスを優先してください（2026-06-20 `#IO-1018`、2026-06-27 `#IO-1025`、2026-06-30 `#IO-1041` 実機確認）。
- 出荷実績登録ダイアログの数量は出荷予定数を全量対象（個別数量入力ではなく、配送キャリア・追跡コードのみ任意入力）。「登録する」で確定します（不可逆）。
- 配送キャリア・追跡コードは、出荷完了後に**出荷指示詳細の「出荷履歴」カードを展開すると確認できます**（2026-07-13訂正）。一覧列や上部の出荷明細には出ませんが、出荷履歴の日時行を開くと配送キャリア・追跡コード・出荷商品/SKU/数量が表示されます。

# CSVエクスポート・PDFエクスポート

> 対象画面: CSVエクスポート / /admin/csv_export　|　PDFエクスポート / /admin/pdf_export　|　最終確認: 2026-06-28

## この機能でできること

- 在庫・価格・売上実績・ポイント変動履歴などのデータをCSV形式でダウンロードできる（9カテゴリ）
- 納品書PDFのエクスポートカテゴリを確認できる（PDFエクスポート画面からの任意新規生成導線は表示されない）
- CSVエクスポートとPDFエクスポートの実行履歴を一覧で確認できる

##### ヤマトB2クラウドエクスポート

この一覧画面にはエクスポートボタンが存在しない。ヤマトB2クラウド向けのCSVは、出荷管理画面（/admin/inventory_outbound_orders）の「条件指定でエクスポート」→「ヤマトB2クラウド」から実行する。フォームには出力物として「ヤマトB2クラウド取り込み用CSV」と「同梱する納品書PDF」が表示される。2026-06-28再確認では決済方法は `すべての決済 / 代引きのみ / 代引き以外の決済` の3択で、「CSVの出力後に出荷指示のステータスを出荷作業中に変更する」はデフォルトOFF。2026-06-28追加確認では、ステータス変更ONで実行しても、移動伝票由来・配送希望日なしの検証用出荷指示 `#IO-1029` は `出荷待ち` のままだった。実行後はメールでダウンロードリンクが通知される文言があるが、同日の履歴画面は空状態でダウンロードリンクなし。実際のメール受信・リンク内容・PDF生成結果は接続環境で検証待ち。

この画面（/admin/csv_export/csv_export_operation_inventory_outbound_order_yamato_b2_clouds）は実行履歴の確認専用。ただし2026-06-15のstaging確認では、条件指定フォームの「実行する」を押した直後に出荷管理へ戻り、この履歴画面には即時レコードが表示されなかった。ステータス変更チェックをOFFにした場合、対象の出荷指示は出荷待ちのまま変化しない。2026-06-28追加確認ではON実行でも検証用 `#IO-1029` は出荷待ちに残り、履歴画面は空状態でダウンロードリンクも表示されなかった。

#### 納品書PDFの生成方法

PDFエクスポート画面（`/admin/pdf_export`）には「出荷」グループと「納品書」行が表示される。2026-06-28再確認時点では、このトップ画面に新規作成ボタンは表示されない。

納品書カテゴリページ（`/admin/pdf_export/pdf_export_operation_packing_slips`）は表示されるが空状態で、新規作成/任意生成ボタンは確認できない。`/admin/pdf_export/pdf_export_operation_packing_slips/create` に直接アクセスしても「このページは存在しないようです」と表示された。移動/調整/取置/出荷/入荷/発注の各詳細（`#IM-1027` / `#IA-1013` / `#IR-1011` / `#IO-1027` / `#II-1029` / `#IP-1010`）でも、納品書/PDF/印刷ボタンは確認できなかった。一方、ヤマトB2クラウド条件指定エクスポートフォームには「同梱する納品書PDF」という出力物名が表示されるため、PDFは同フォームの非同期出力物として生成される可能性がある。

そのため、現時点のFAQでは「PDFエクスポート画面から任意に納品書PDFを新規作成できる」とは案内しない。ヤマトB2クラウド条件指定エクスポートでPDFファイルが実際に届くか、注文起点の出荷指示など別条件で生成導線が出るかは接続環境で検証待ち。

納品書のテンプレートは設定画面（/admin/settings/pdf_template_package_slip）で管理する。

## 補足・注意点

- 各エクスポートカテゴリの一覧画面には実行履歴が表示される。カテゴリをクリックして専用画面へ進む
- 売上実績（注文軸）は、実行後に一覧上で「処理中」から「成功 完了」へ変わり、ダウンロードリンクが表示される
- ヤマトB2クラウドCSVは非同期処理で実行される。フォーム上は「同梱する納品書PDF」も出力物に含まれるが、今回の実機確認ではPDFメニューからの新規作成導線はなく、ヤマトB2履歴画面も空状態でダウンロードリンクは表示されなかった
- ヤマトB2クラウドの条件指定エクスポートには、「CSVの出力後に出荷指示のステータスを出荷作業中に変更する」チェックがあります。ただし2026-06-28追加確認では、ON実行でも移動伝票由来・配送希望日なしの検証用 `#IO-1029` は `出荷待ち` のままで、作業中への成功例は未確認です。ステータスを変えたくない場合はOFFのまま実行します

# CSVエクスポート・PDFエクスポート

> 対象画面: CSVエクスポート / /admin/csv_export　|　PDFエクスポート / /admin/pdf_export　|　最終確認: 2026-07-03

## この機能でできること

- 在庫・価格・売上実績・ポイント変動履歴などのデータをCSV形式でダウンロードできる（9カテゴリ）
- 納品書PDFのエクスポートカテゴリを確認できる（PDFエクスポート画面からの任意新規生成導線は表示されない）
- CSVエクスポートとPDFエクスポートの実行履歴を一覧で確認できる

##### 在庫エクスポート

![「在庫をCSVでエクスポートする」フォーム](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/15-csv-export-inventory-create.png)

| 項目（UIラベル） | 必須/任意 | 説明 |
|:--|:--|:--|
| ロケーション | 実質必須（* マークなし） | 「選択」ボタンを押してロケーション選択ダイアログから1件選ぶ。未選択のままではエクスポートが実行されない |

**ロケーション選択ダイアログの構成**: タブ「すべて」「店舗」「倉庫」の3種。列はチェックボックス・名前・場所コード。2026-07-01実機確認では、1件チェックすると他ロケーションのチェックボックスがdisabledになったため、在庫CSVエクスポートは実質1ロケーション指定です。

2026-07-01実機確認では、`ユニクロ物流倉庫` を指定して実行すると、履歴一覧に `作成日 / 店舗 / ステータス / ダウンロード` 列が表示され、完了後は `成功 完了` とダウンロードリンクが出ました。CSV本体のヘッダーは `SKU,ProductVariantID,InventoryItemID,Incoming,Available,Committed,Reserved,Damaged,SafetyStock,QualityControl,OnHand` で、ロケーション名・ロケーションコード列はCSV内には含まれません。

##### ロケーションエクスポート

| 項目（UIラベル） | 必須/任意 | 説明 |
|:--|:--|:--|
| ロケーショングループ | 必須（* マーク付き） | 「選択」ボタンを押してロケーショングループ選択ダイアログから選ぶ |

**画面内補足文（原文）**: 「選択したロケーショングループに属するロケーションをCSVでエクスポートします。」

2026-07-01実機確認では、ロケーショングループ選択も実質1件指定でした。`ユニクログループ` を選ぶと他グループのチェックボックスがdisabledになり、履歴一覧に `作成日 / ロケーショングループ / ステータス / ダウンロード` 列が表示されました。完了後のCSVヘッダーは `Name,DisplayName,ExternalID,IsListedPublic,IsArchived,LocationCode,Tags,Email,LocationType,PointApplicationType,IsLocalPickupEnabled,IsInventoryAllocationRequestEnabled,MapURL,Phone,CountryCode,PostalCode,Province,City,Address1,Address2` です。

##### ディスカウントの利用履歴エクスポート

| 項目（UIラベル） | 必須/任意 | 説明 |
|:--|:--|:--|
| ディスカウント | 必須（* マーク付き） | 「選択」ボタンを押してディスカウント選択ダイアログから選ぶ |

**画面内補足文（原文）**: 「選択したディスカウントの使用履歴をCSVでエクスポートします。」

2026-07-01実機確認では、ディスカウント選択も実質1件指定でした。候補は `タイトル / ステータス` 列で表示され、`TEST_FAQ_ディスカウント_対象商品テスト` を選ぶと他候補がdisabledになりました。履歴一覧の列は `作成日 / ディスカウント / ステータス / ダウンロード` で、利用履歴0件でも完了後にヘッダーのみのCSVをダウンロードできました。ヘッダーは `顧客ID,顧客外部ID,店舗名,店舗コード,利用日時` です。

##### 商品バリエーションエクスポート

| 項目（UIラベル） | 必須/任意 | 説明 |
|:--|:--|:--|
| 商品情報を含める | 任意（チェックボックス） | チェックをONにするとCSV先頭に商品本体の列が追加される（デフォルト: 未チェック） |

**チェックボックスの補足文（原文）**: 「CSVに商品情報（商品コード、商品タイトル、ブランド名など）を含めます」

2026-07-01実機確認では、OFF実行の履歴は `商品情報: 含めない`、ON実行の履歴は `商品情報: 含める` と表示され、完了後に各行の `ダウンロード` リンクからCSVを取得できました。OFFのCSVは `SKU,ProductVariantExternalID,ProductVariantTitle...` から始まり、ONのCSVはその前に `ProductCode,ProductExternalID,ProductTitle,ProductStatus,ProductType,ProductVendor,BrandName,BrandCode` が追加されます。

**画面内補足文（原文）**: 「選択した販売価格ルールのセール価格をCSVでエクスポートします。」

2026-07-01実機確認では、候補は通常のセレクトボックスで、`TEST_FAQ_販売価格ルール (JPY)`、`TEST_FAQ_20260624_販売価格_100908 (USD)`、`年末セール価格 (JPY)` などが表示されました。`年末セール価格 (JPY)` を選んで実行すると、履歴一覧に `作成日時 / 販売価格ルール / ステータス / ダウンロード` 列が表示され、完了後にCSVを取得できました。CSVヘッダーは `ProductVariantID,ExternalID,PriceAmount,PriceAmountAfterTaxes,CurrencyCode,StartsAt,EndsAt` です。

##### ヤマトB2クラウドエクスポート

この一覧画面にはエクスポートボタンが存在しない。ヤマトB2クラウド向けのCSVは、出荷管理画面（/admin/inventory_outbound_orders）の「条件指定でエクスポート」→「ヤマトB2クラウド」から実行する。フォームには出力物として「ヤマトB2クラウド取り込み用CSV」と「同梱する納品書PDF」が表示される。2026-07-02再確認では決済方法は `すべての決済 / 代引きのみ / 代引き以外の決済` の3択で、「CSVの出力後に出荷指示のステータスを出荷作業中に変更する」はデフォルトOFF。2026-06-28追加確認では、ステータス変更ONで実行しても、移動伝票由来・配送希望日なしの検証用出荷指示 `#IO-1029` は `出荷待ち` のままだった。

この画面（/admin/csv_export/csv_export_operation_inventory_outbound_order_yamato_b2_clouds）は実行履歴の確認専用。2026-07-02には同日23:34作成の成功履歴が `成功 完了` として表示され、23:35の完了メールからCSVをダウンロードできた。メール本文には有効期限が表示され、リンクはCSV 1件のみで、同梱納品書PDFリンクは確認できなかった。CSVヘッダーは `送り状種類`、`お客様管理番号`、`お届け先名`、`お届け先電話番号`、`お届け先郵便番号`、`お届け先住所`、`お届け先住所（アパートマンション名）`、`出荷予定日`、`お届け予定（指定）日`、`配達時間帯`、`コレクト代金引換額（税込）`、`コレクト内消費税額等`、`備考欄` の13列で、データ行に `#IO-1047` が出力されていた。

同じ2026-07-02に条件指定エクスポートを新規実行した際は、トースト `ヤマトB2クラウドのエクスポートを開始しました` は表示されたが、確認時点では新しい履歴行・新しいメールは確認できなかった。また、出荷一覧の一括アクション `ヤマトB2クラウド（CSV）` で `#IO-1050` を実行したジョブは23:47に履歴作成後、約20秒で `重大 / 失敗` になり、ダウンロードもメールも確認できなかった。

##### 売上実績（注文軸）エクスポート

| 項目（UIラベル） | 必須/任意 | 説明 |
|:--|:--|:--|
| テナント | 必須（* マーク付き） | ドロップダウンから対象のテナントを選ぶ |
| 開始日時 | 必須（* マーク付き） | 絞り込みを開始する日時を入力する |
| 終了日時 | 必須（* マーク付き） | 絞り込みを終了する日時を入力する |

2026-06-16の実機確認では、3項目を入力して「エクスポートを開始する」を押すと一覧画面へ戻り、直近行が「注意 処理中」と表示された。再読み込み後に「成功 完了」になり、ダウンロードリンクが表示された。2026-06-27再確認では既存履歴に `作成日 / テナント / 対象期間 / ステータス / ダウンロード` の列、`成功` / `完了`、`ダウンロード` リンクが表示された。

##### 売上実績（明細軸）エクスポート

売上実績一覧画面の「エクスポート」>「明細軸」から `/admin/csv_export/csv_export_operation_sale_change_line_items` に遷移する。フォーム項目・空保存エラー・履歴列は注文軸と同じ構成。

##### ポイント変動履歴エクスポート

| 項目（UIラベル） | 必須/任意 | 説明 |
|:--|:--|:--|
| テナント | 必須（* マーク付き） | ドロップダウンから対象のテナントを選ぶ |
| 開始日時 | 必須（* マーク付き） | ポイント変動履歴の絞り込みを開始する日時を入力する |
| 終了日時 | 必須（* マーク付き） | ポイント変動履歴の絞り込みを終了する日時を入力する |

2026-07-01実機確認では、`ユニクロ`、対象期間 `2026年07月01日 00:00 〜 2026年07月01日 23:59` で実行すると、履歴一覧に `作成日 / テナント / 対象期間 / ステータス / ダウンロード` 列が表示されました。開始直後は `注意 処理中`、再読み込み後は `成功 完了` と `ダウンロード` リンクが表示されました。CSVヘッダーは `PointChangeID,PurchasingCustomerID,PurchasingCustomerBarcode,Title,Delta,AvailableAt,ExpiresDate,PointChangeCreatedAt,PointChangeActivityType,Note` です。

同日のポイント一括加算・減算後に再エクスポートしたCSVは、ヘッダー + 加算1行 + 減算1行でした。加算行は `FAQ_POINT_PLUS_20260701`, `Delta=5`, `PointChangeActivityType=EARNED`、減算行は `FAQ_POINT_MINUS_20260701`, `Delta=-2`, `PointChangeActivityType=USED` と出力されました。実績0件の場合でも空ファイルではなくヘッダーのみのCSVを取得できます。

#### 納品書PDFの生成方法

PDFエクスポート画面（`/admin/pdf_export`）には「出荷」グループと「納品書」行が表示される。2026-06-28再確認時点では、このトップ画面に新規作成ボタンは表示されない。

納品書カテゴリページ（`/admin/pdf_export/pdf_export_operation_packing_slips`）は表示されるが、このページ自体には新規作成/任意生成ボタンは確認できない。`/admin/pdf_export/pdf_export_operation_packing_slips/create` に直接アクセスしても「このページは存在しないようです」と表示された。移動/調整/取置/出荷/入荷/発注の各詳細（`#IM-1027` / `#IA-1013` / `#IR-1011` / `#IO-1027` / `#II-1029` / `#IP-1010`）でも、納品書/PDF/印刷ボタンは確認できなかった。

2026-07-02追加確認では、出荷管理一覧で対象行をチェックすると一括操作バーが表示され、`アクション > エクスポート > 納品書（PDF）` からPDF出力を実行できた。確認モーダル後に `/admin/pdf_export/pdf_export_operation_packing_slips` が新しいタブで開き、履歴行が作成される。ただし `#IO-1047` / `#IO-1046` / `#IO-1048` / `#IO-1020` と2026-07-02追加実行の `#IO-1048`、2026-07-03追加実行の `#IO-1045`（キャンセル済み）/ `#IO-1051`（出荷待ち）はいずれも `重大 / 失敗` になり、`ダウンロード` は disabled だった。2026-07-02の追加実行ではメールにもPDF/納品書通知は見つからなかった。`#IO-1020` は配送・引当済みでも失敗したため、引当ステータスだけが原因とは断定しない。キャンセル済み・出荷待ちでも失敗しており、ステータス差による成功条件は確認できていない。

そのため、現時点のFAQでは「PDFエクスポート画面から任意に納品書PDFを新規作成できる」とは案内しない。案内できる生成導線は「出荷管理一覧で対象行を選択 → `アクション > 納品書（PDF）`」です。ただし、今回のstaging検証ではPDF生成が失敗しているため、実PDFファイルの内容・印字は未確認です。

納品書のテンプレートは設定画面（/admin/settings/pdf_template_package_slip）で管理する。

## 補足・注意点

- 各エクスポートカテゴリの一覧画面には実行履歴が表示される。カテゴリをクリックして専用画面へ進む
- 売上実績（注文軸）は、実行後に一覧上で「処理中」から「成功 完了」へ変わり、ダウンロードリンクが表示される
- ヤマトB2クラウドCSVは非同期処理で実行される。2026-07-02に成功履歴と完了メールからCSV 1件を確認したが、メール内に同梱納品書PDFリンクはなかった。納品書PDFは出荷管理一覧の一括操作 `アクション > 納品書（PDF）` からも実行できるが、出荷完了・出荷待ち・キャンセル済みのいずれでもPDF出力履歴が `重大 / 失敗` になり、ダウンロードできなかった
- ヤマトB2クラウドの条件指定エクスポートには、「CSVの出力後に出荷指示のステータスを出荷作業中に変更する」チェックがあります。ただし2026-06-28追加確認では、ON実行でも移動伝票由来・配送希望日なしの検証用 `#IO-1029` は `出荷待ち` のままで、作業中への成功例は未確認です。ステータスを変えたくない場合はOFFのまま実行します

# CSVインポート

> 対象画面: CSVインポート / /admin/csv_import　|　最終確認: 2026-06-28

## この機能でできること

- CSVファイルを使って、商品・在庫・価格・ポイントなどのデータを一括で登録・更新できる
- 22カテゴリのデータ種別に対応している
- 各カテゴリにはGoogleスプレッドシート形式のテンプレートが用意されている（一部カテゴリを除く）
- インポート実行後の結果（成功・失敗の件数）を履歴画面で確認できる

### テンプレートの有無

テンプレートがあるカテゴリでは、カテゴリの一覧画面に「テンプレート」ボタンが表示される。テンプレートはGoogleスプレッドシート形式で、別タブで開く。

テンプレートがないカテゴリ（6種）: 販売価格 (通常) / 原価 / 販売閾値 / DHL / キャンペーン対象商品 / 基準ランク

#### 商品一覧の「インポート」5項目

商品一覧（`/admin/products`）の「インポート」メニューから開く5項目は以下です。各一覧画面にテンプレートがあります。

| 項目 | URL | 主な必須列 |
|:--|:--|:--|
| 商品 | `/admin/csv_import/csv_import_operation_products` | `command`, `product_code`, `title`, `product_status`, `option1_name` |
| 商品画像 | `/admin/csv_import/csv_import_operation_product_images` | `product_code`, `image_url` |
| 商品バリエーション | `/admin/csv_import/csv_import_operation_product_variants` | `command`, `sku`, `product_code`, `option1_value`, `price`, `price_currency_code` |
| 商品メタフィールド | `/admin/csv_import/csv_import_operation_product_metafields` | `product_code`, `namespace`, `key`, `value_type`, `value` |
| 商品バリエーションメタフィールド | `/admin/csv_import/csv_import_operation_product_variant_metafields` | `sku`, `namespace`, `key`, `value_type`, `value` |

> CSVインポートトップの「商品」グループは `商品 / 商品画像 / 商品バリエーション / 商品バリエーション画像 / カタログ / 原価`。商品一覧のインポートメニューは、これとは別にメタフィールド2項目を含みます。2026-06-27再確認時点では、原価は「価格」ではなく「商品」グループに表示されます。

> **商品バリエーションテンプレートの在庫系項目**: `inventory_policy` は「在庫切れの場合でも販売を続ける」（CONTINUE/DENY）、`is_tracked` は「在庫追跡」（TRUE/FALSE）です。両者は別項目です。2026-06-21実機では、管理画面のバリエーション作成フォーム初期値はどちらもOFFでした。

> **カタログ商品テンプレート**: 2026-06-28実機確認時点の列は `product_code,command` です。`command` は `NEW` でカタログへ追加、`DELETE` でカタログから削除します。

### インポート実行フォームのパターン

カテゴリによってフォームの構成が異なる。

#### パターンA：ファイルのみ

CSVファイルをアップロードして「保存する」を押す。

該当カテゴリ: 商品 / 商品バリエーション / 商品メタフィールド / 商品バリエーションメタフィールド / 原価 / 在庫移動伝票 / ヤマトB2クラウド / DHL / 管理ユーザー / ロケーション

#### パターンB：ラジオ選択 + ファイル

処理方法をラジオボタンで選んでからCSVファイルをアップロードする。

| カテゴリ | ラジオボタンの選択肢（デフォルト → 変更可） |
|:--|:--|
| 商品画像 | 「画像を追加する」（デフォルト） / 「画像を上書きする」 |
| 商品バリエーション画像 | 「画像を追加する」（デフォルト） / 「画像を上書きする」 |
| 販売可能在庫 | 「絶対値で反映する」（デフォルト） / 「差分値で反映する」 |

**販売可能在庫の補足（画面内の説明原文）**: 「現在の在庫数が10のとき、CSVで20を指定すると、絶対値で反映した場合の在庫数は20になります。差分値で反映した場合の在庫数は30になります。」

![販売可能在庫のインポートフォーム（ラジオ選択 + ファイル）](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/15-csv-import-inventory-create.png)

#### パターンC：ルール選択 + ファイル

対象となるルールをドロップダウンで選んでからCSVファイルをアップロードする。

| カテゴリ | フィールドラベル |
|:--|:--|
| 販売価格 (通常) | 販売価格ルール |
| 販売価格 (セール) | 販売価格ルール |
| 販売閾値 | 販売閾値ルール |
| 予約販売 | 予約販売ルール |
| 基準ランク | テナント |

#### パターンD：検索型選択 + ファイル

対象をテキスト検索＋ダイアログで選択してからCSVファイルをアップロードする。

該当カテゴリ: キャンペーン対象商品（「ポイントキャンペーン」フィールドで「検索」ボタンを押してダイアログから選択する）

#### パターンE：カタログ選択 + ファイル

対象カタログをドロップダウンで選択してからCSVファイルをアップロードする。

該当カテゴリ: カタログ

### ポイント一括加算・一括減算フォームの固有項目

ポイント一括加算には以下の2フィールドが追加されている。

![ポイント一括加算のインポートフォーム（テナント・顧客IDの種別）](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/15-csv-import-point-plus-create.png)

| 項目（UIラベル） | 説明 | 選択肢 |
|:--|:--|:--|
| テナント | 加算対象のテナントを選択する | セレクトボックス（プレースホルダ「選択してください」） |
| 顧客IDの種別 | CSV内の顧客識別子の種類を指定する | 「SQが発番した顧客ID」 / 「外部システムが発番した顧客ID」 |

どちらのフィールドも画面上に必須マーク（*）は表示されていないが、未選択で「保存する」を押すとバリデーションエラーが表示される（実質必須）。

ポイント一括減算フォームの固有項目は「テナント」とCSVファイルのみ。一括加算の「顧客IDの種別」は表示されない。空保存時は「テナントを選択してください」、テナント選択済みでファイル未選択の場合は「ファイルを選択してください」と表示される。

2026-07-01実機確認では、一括加算は `customer_id,title,points,available_date,expires_date,note` 形式で、顧客IDの種別 `SQが発番した顧客ID` を選ぶとSQ内部顧客IDで取込できました。一括減算は `customer_id,title,points,note` 形式で、フォームに顧客IDの種別がないため外部IDで取込します。SQ内部顧客IDを入れた減算CSVは検証失敗し、失敗詳細に `customer_id: 指定された顧客IDの顧客が存在しません` と表示されました。

### 実行履歴

カテゴリの一覧画面に過去のインポート実行履歴が表示される。各履歴行をクリックすると詳細を確認できる。

#### 実行履歴の詳細画面で確認できる項目

![CSVインポート操作の詳細画面（検証・実行ステータス）](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/gap-csv-import-detail-success.png)

| 項目（UIラベル） | 説明 |
|:--|:--|
| 作成日 | インポートを実行した日付 |
| 検証ステータス | CSVの内容を検証した結果（例: 完了） |
| 実行ステータス | 実際のデータ反映の結果（例: 完了） |
| 検証成功 | 検証に成功した件数（例: 6個の商品） |
| 検証失敗 | 検証に失敗した件数。クリックで失敗詳細画面（見出し「検証失敗」）へ移動できる |

インポートは「検証」→「実行」の2段階で処理される。検証で失敗した行がある場合は、失敗詳細画面でエラー内容を確認できる。

2026-06-16に販売可能在庫CSVで確認した流れ:

1. 新規インポートフォームでCSVをドロップし、「保存する」を押す。
2. 詳細画面へ遷移し、検証ステータスが「実行中」、実行ステータスが「未実行」になる。この間、「実行する」ボタンは無効。
3. 検証が「成功 完了」になると「実行する」ボタンが有効になる。
4. 「実行する」を押すと確認ダイアログ「CSVの取り込み処理を実行しますか？」が開く。本文には「1件の商品の在庫数を更新します。この操作は巻き戻すことができません。」と表示される。
5. ダイアログ内の「実行する」を押すと、実行ステータスが「成功 完了」になり、在庫詳細の販売可能数・手持ち数に反映される。
6. 2026-06-27再確認では、実行済み詳細にも「実行する」ボタンは残るがdisabledで再実行できない。検証失敗が0件でも `0個の商品` リンクから見出し「検証失敗」の画面へ遷移できる。

2026-06-28に商品CSVでも同じ2段階フローを確認しました。検証用商品1件のCSVで `検証成功 1個の商品` になり、確認ダイアログ本文は `1件の商品を登録します。この操作は巻き戻すことができません。` でした。実行後は `実行ステータス: 成功 完了` になり、商品一覧/詳細に反映されました。検証用商品は商品詳細から削除済みです。

2026-06-28にカタログ商品CSVの `NEW` / `DELETE` も実行確認しました。`product_code,command` 形式で、`NEW` は対象商品をカタログに追加し、`DELETE` はカタログから外します。どちらも `検証成功 1件の商品`、実行後 `実行ステータス: 成功 完了` になり、確認ダイアログには `1件のカタログ商品を登録します。この操作は巻き戻すことができません。` と表示されました。検証用の商品・カタログは削除済みです。

2026-07-01にポイント一括加算・一括減算も実データで確認しました。加算は確認ダイアログ `1件のポイント付与を実行します。この操作は巻き戻すことができません。`、減算は `1件のポイント減算を実行します。この操作は巻き戻すことができません。` と表示され、どちらも検証成功1件・実行成功1件でした。顧客詳細では保有ポイントが `0 -> 5 -> 3` と変わり、履歴には加算 `5`、減算 `-2` が表示されました。減算CSVの `points` は正数で入力しますが、反映後の履歴・ポイント変動CSVでは負数になります。

## 補足・注意点

- トップ画面（/admin/csv_import）でカテゴリを選ぶと、そのカテゴリ専用の一覧画面に遷移する
- CSVのカラム順・ヘッダー名はテンプレートに準拠する必要がある（列指定UIは存在しない）
- 顧客管理画面の「インポート」ドロップダウン内に表示される「ポイント一括付与」は、CSVインポート画面の「一括加算」と同じ機能（表記がアプリ内で異なる）
- ポイント一括減算は顧客IDの種別選択がなく、2026-07-01実機確認では外部IDで成功、SQ内部顧客IDで検証失敗した
- ヤマトB2クラウドのCSVは固定カラム順・ヘッダー名一致方式で、必須列は「伝票番号」「出荷日」「お客様管理番号」の3列のみ。残り7列は任意（処理で読み捨て）

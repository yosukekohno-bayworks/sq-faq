# 設定

> 対象画面: 設定 / `/admin/settings`　|　最終確認: 2026-06-12

## この機能でできること

- 組織全体のマスタデータ（ロケーション・ブランド・決済方法など）を管理する
- 管理メンバーの追加・権限グループの設定をする
- 外部サービス（ロジザード・Recustomer など）との連携を設定する
- 翻訳・メタフィールド定義・採寸ルールなどのカスタムデータを管理する

## 管理メンバー（/admin/settings/users）

管理画面にアクセスできるメンバーを管理します。

### 一覧画面の主な操作

| 操作 | 説明 |
|:--|:--|
| 追加する | メンバー追加フォームへ遷移 |
| 権限グループ一覧 | 権限グループ管理画面へ遷移 |
| インポート | CSVによる一括追加 |

### メンバー追加フォーム（/admin/settings/users/create）

フォームタイトル: **「ユーザーを追加する」**

| 項目（UIの原文） | 必須 | 補足 |
|:--|:--|:--|
| 姓 * | 必須 | 姓・名は同じ行に横並びで表示 |
| 名 * | 必須 | — |
| メールアドレス * | 必須 | — |
| 権限グループ | — | ラジオボタンで1グループのみ選択可。作成済みのグループが一覧表示される |

- パスワード入力欄はありません。メールアドレスによる認証でログインします。
- 権限グループは1人に1グループのみ割り当て可能です（複数グループへの同時割り当て不可）。

### メンバー詳細画面（保存後）

メンバー詳細は以下の3セクションで構成されます。

| セクション | 内容 |
|:--|:--|
| 基本情報 | 姓・名・メールアドレスの編集 |
| ユーザーテナント | アクセスを許可するテナントのチェックボックス選択 |
| 権限 | 権限グループをコンボボックスで変更。下部に「適用されている権限一覧」（件数表示付き）アコーディオンあり |

- 「その他の操作」ボタン → 「**ユーザーを組織から除外**」（退職・アクセス削除時はこの操作）

## 権限グループ（/admin/settings/permission_groups）

管理メンバーに割り当てる権限グループを管理します。

### アクセス方法

管理メンバー一覧（`/admin/settings/users`）の「**権限グループ一覧**」ボタンから遷移します。

### 権限グループの種類

| グループ名 | 特徴 |
|:--|:--|
| 特権管理者 | グループ名のみ変更可能。権限内容（76件）は変更不可・削除不可の半固定グループ |
| カスタムグループ | 「作成する」から新規作成。76権限から任意に選択できる |

### 権限の体系

権限は **38リソース × 閲覧（:read）/ 編集（:write）の76種**です。

代表的なリソース: お知らせ / アプリ / ブランド / 顧客 / ディスカウント / 在庫 / 発注伝票 / ロケーション / 注文 / 商品 / 店舗スタッフ / テナント / 翻訳 / ユーザー など

すべてのリソース: お知らせ(announcements) / アプリ(apps) / ブランド(brands) / カタログ(catalogs) / 取引先企業(companies) / 顧客マイル(customer_miles) / 顧客ポイント(customer_points) / 顧客会員ランクルール(customer_rank_rules) / 顧客会員ランク(customer_ranks) / 顧客(customers) / ディスカウント(discounts) / 下書き注文(draft_orders) / 在庫(inventory) / 在庫リクエスト(inventory_allocation_requests) / 発注返品伝票(inventory_purchase_order_returns) / 発注伝票(inventory_purchase_orders) / 店舗受取ロケーションルール(local_pickup_location_rules) / ロケーショングループ(location_groups) / ロケーション(locations) / マイル商品(mile_items) / ネクストエンジン連携(next_engine_integrations) / OmnibusCore連携(omnibus_core_integrations) / 注文(orders) / ポイントキャンペーン(point_campaign_order_rules) / ポイントルール(point_rules) / 販売価格ルール(product_price_rules) / 商品(products) / 店舗ポータル連携(retail_portal_integrations) / 店舗スタッフ(retail_staff_members) / 売上データ(sales) / Shopify連携(shopify_integrations) / スマレジ連携(smaregi_integrations) / テナント(tenants) / 翻訳ルール(translation_rules) / 翻訳(translations) / ユーザー権限(user_permissions) / ユーザーテナント(user_tenants) / ユーザー(users)

### カスタム権限グループ作成フォーム（/admin/settings/permission_groups/create）

フォームタイトル: **「権限グループを作成する」**

![「権限グループを作成する」フォーム（76権限のチェックボックス）](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/bfg-F-01-permission-group-create-form.png)

| 項目（UIの原文） | 必須 | 補足 |
|:--|:--|:--|
| グループ名 * | 必須 | テキストボックス |
| 権限（チェックボックス群） | — | 76権限から個別に選択。デフォルトは全オフ |

作成済みのカスタム権限グループは、権限グループ一覧で行を選択して「権限グループを削除」から削除できます。確認ダイアログは「権限グループを削除しますか？」で、本文は「選択された1件の権限グループを削除します。この処理は巻き戻すことができません。」です。削除後はトースト「権限グループを削除しました」が表示されます。

| 項目（UIの原文） | 必須 | 補足 |

| 項目（UIの原文） | 必須 | 選択肢・補足 |

| 販売設定 | 店舗受取（有効・無効）/ 在庫依頼（有効・無効）/ ポイント利用種別 |
| 備考欄 | フリーテキスト |

## 販売員（/admin/settings/retail_staff_members）

リテールポータルで、どの販売員が顧客情報を閲覧したか、どの販売員が客注注文を作成したかを記録するための販売員を管理します。

### 販売員作成フォーム（/admin/settings/retail_staff_members/create）

| 項目（UIの原文） | 必須 | 補足 |
|:--|:--|:--|
| 姓 * | 必須 | 姓・名は同じ行に横並びで表示 |
| 名 * | 必須 | — |
| コード * | 必須 | — |
| ロケーション | 任意 | 「検索する」ボタンをクリックするとロケーション選択モーダルが開く。作成フォームでは1件のみ選択可 |

## メタフィールド定義（/admin/settings/metafield_definitions）

各オブジェクトにカスタムフィールドを追加するための定義を管理します。

### カスタムフィールドを追加できるオブジェクト（16種）

組織 / 商品 / バリエーション / 顧客 / 注文 / 下書き注文 / 会社 / ロケーション / ディスカウント / 在庫移動伝票 / 在庫調整伝票 / 在庫取置伝票 / 発注伝票 / 入荷指示 / 出荷指示 / 仕入れ先

### メタフィールド定義 作成フォーム

| 項目（UIの原文） | 必須 | 制約・補足 |
|:--|:--|:--|
| 名前 * | 必須 | 最大50文字 |
| 説明 | 任意 | 最大100文字 |
| ネームスペース * | 必須 | 最大50文字・最低4文字 |
| キー * | 必須 | 最大50文字・最低4文字 |
| メタフィールドのタイプ * | 必須 | コンボボックス（下記12種から選択） |

**タイプの選択肢（12種）:**
単一行のテキスト / 単一行のテキスト(リスト) / 複数行のテキスト / 日付と時刻 / 日付 / trueまたはfalse / バーコード(NW7) / 整数 / 小数 / JSON / ID / リッチテキスト

- 識別子は「ネームスペース.キー」形式で管理されます（例: `custom.note`）。
- 定義を保存すると、対象オブジェクトの詳細画面に「メタフィールド」セクションが即座に表示されます。

## アプリ（/admin/settings/apps）

SQ Admin APIにアクセスするためのAPIキーを管理します。

- 画面の説明文（UIの原文）: 「APIアクセスを管理する / アプリを作成して、SQのアドミンAPIにアクセスしましょう。」

### アプリ作成フォーム（/admin/settings/apps/create）

| 項目（UIの原文） | 必須 | 補足 |
|:--|:--|:--|

### アプリ詳細画面（保存後）

アプリを作成すると詳細画面に遷移し、以下の情報とセクションが表示されます。

#### Admin API

作成と同時にアクセストークンとシークレットが自動発行されます。

| 項目 | 内容 |
|:--|:--|

「**Playgroundを開く**」ボタンから GraphQL Playground を開いてAdmin APIを試すことができます。

#### リクエストログ

「**リクエストログを見る**」からこのアプリによるAPIリクエストの履歴を確認できます。

#### Storefront API

「**トークンを発行する**」ボタンからStorefront API用のトークンを発行できます。

「**Webhookを作成する**」ボタンからWebhookを追加できます。

**Webhook作成ダイアログの入力項目:**

| 項目（UIの原文） | 説明 | 選択肢・補足 |
|:--|:--|:--|
| イベント | Webhookを送信するトリガーとなるイベント | 「注文の作成」 / 「注文の更新」 / 「在庫の更新」の3種 |
| エンドポイント | Webhookの送信先URL | URLを入力する |

## ロジザード連携（/admin/logizard_integrations）

ロジザードZERO（倉庫管理システム）との連携設定を管理します。

### ロジザード連携作成フォーム（/admin/logizard_integrations/create）

フォームは以下の6セクションで構成されます。

| セクション | 主な入力項目（必須） |
|:--|:--|
| 設定名 | 設定名 * |
| 接続情報 | グループ番号 * / 接続番号 * |
| 認証情報 | オーナーID * / ユーザーID * / パスワード * / アプリケーションキー * |
| 入荷設定 | 入荷予定登録ファイルID * / 入荷予定登録パターンID * |
| 出荷設定 | 出荷予定登録(通販)ファイルID * / 出荷予定登録(通販)パターンID * / 出荷予定登録(卸)ファイルID * / 出荷予定登録(卸)パターンID * |
| 出荷箱明細実績エクスポート設定 | 出荷箱明細実績エクスポートファイルID * / 出荷箱明細実績エクスポートパターンID * |

- 「商品バリエーションを特定するキー *」: コンボボックス（SKUコード / JANコード / EANコード / UPCコード、デフォルト: SKUコード）

---

## Recustomer連携（/admin/recustomer_integrations）

返品・交換プラットフォーム「Recustomer」との連携設定を管理します。

### Recustomer接続フォーム（/admin/recustomer_integrations/create）

| 項目（UIの原文） | 必須 | 補足 |
|:--|:--|:--|
| ストアID * | 必須 | Recustomer側のストアIDを入力 |
| シークレット * | 必須 | Recustomer側のAPIシークレットキーを入力 |

- 「アカウントを接続」リンクから接続フォームへ遷移します。
- Recustomer側のストアIDとシークレットを事前に取得する必要があります。

---

## 補足・注意点

- 特権管理者グループの権限内容は変更できません。グループ名のみ変更可能です。
- ブランドの「外部ID」は作成時のみ設定できます。保存後に変更することはできません。
- 採寸単位は「なし」か「センチメートル」の2種のみです。
- ロケーションの「場所種別」は「倉庫」「店舗」の2種のみです。
- アプリのアクセストークンとシークレットは作成時に自動発行されます。再発行の操作については未確認です。<!-- TODO: 要確認（アクセストークン・シークレットの再発行操作） -->

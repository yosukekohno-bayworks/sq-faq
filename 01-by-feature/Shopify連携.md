# Shopify連携

> 対象画面: Shopify連携 / `/admin/shopify_integrations`　|　最終確認: 2026-07-16

## この機能でできること

[公式ガイド](https://docs.sqstack.com/docs/guide/shopify-integration) によると、ShopifyとSQを連携することで以下が可能になります:

- **商品マスタの自動同期** — Shopifyの商品情報をSQと連携し一元管理
- **ロケーションの統合管理** — 店舗/倉庫の住所・電話・Google Mapsリンクを一元管理（SDKでストアフロント表示も可能）
- **在庫のリアルタイム管理** — 複数チャネル跨ぎの在庫を一括管理（SDKで店舗在庫をオンライン表示も可能）
- **顧客データ・ポイント・会員ランクの同期** — ストアフロント由来の顧客情報を基にSQでポイント/ランク管理（SDKでShopifyに表示）
- **Shopify Flowの利用** — SQ提供のトリガー/アクションで自動化（詳細は後述「Shopify Flow連携」）

加えて、SQ側で設定できる接続後の項目:

- 在庫引当ロケーション・販売価格ルール・会員番号の生成方法・税込設定などを設定する

## 画面・項目の説明

### 接続前の前提

Shopify連携を実行するには、先にShopifyストア側へSQコネクターアプリをインストールしておく必要があります。

公開アプリのSQ Connectは本番SQ（`sq.stackservice.com`）向けです。staging環境（`sqstackstaging.com`）へ接続する場合は、staging向けの別アプリとインストールリンクが必要です。2026-06-29の検証環境では、Shopify側に `erp-stg`（表示名 `SQ (エス・キュー)`）がインストールされていました。

「ストアを接続する」ボタンまたはリンクから接続フォーム（`/admin/shopify_integrations/create`）に進みます。

#### リスティングタブ

連携対象カタログは `UNIQLO` です。2026-07-01再確認時点では `バギーカーブジーンズ`、`UVカットペーパーブレイドハット`、`エアリズムコットンT`、`エアリズムコットンクルーネックTシャツ`、`オーバーサイズスウェットシャツ` の5商品が未連携一覧に表示され、連携済み商品数は0、連携済みバリエーション一覧も空でした。

`商品の初回自動連携を有効にする` は、商品連携用トークンが未設定のため有効化できません。画面には「商品連携用のトークンが設定されていない場合は有効にできません」と表示されます。

未連携商品一覧で商品を選択すると、選択時アクションとして `商品情報を連携する` と `商品の在庫を同期する` が表示されます。2026-07-01に `バギーカーブジーンズ` 1件だけを選択して `商品情報を連携する` を押すと、確認ダイアログには `選択されている商品情報をShopifyに連携します。Shopifyに商品が連携された後にShopify側で販売チャネルの設定を行ってください。この処理は時間がかかります。` と表示され、確定ボタンは `作成する` でした。確定後の `ShopifyIntegrationSetProducts` は `ProductSet 用の認証情報が設定されていません` で停止し、SQ側は連携済み商品0件・未連携商品5件のままでした。Shopify商品管理で `バギーカーブジーンズ` を検索しても `商品が見つかりませんでした` と表示され、Shopify側にも商品は作成されていません。

リスティングタブには `一括同期` メニューがあり、選択肢は `商品情報を一括で同期する` と `在庫を一括で同期する` です。商品情報の一括同期を実行すると、UI上は汎用エラーになりますが、内部エラーは `ProductSet 用の認証情報が設定されていません` でした。Shopify側の商品一覧は17件のままで、上記5商品は作成されていませんでした。

在庫の一括同期では、確認ダイアログでロケーションを明示選択する必要があります。候補には `ユニクロEC（SQ） → My Custom Location（Shopify）` が表示されます。未選択のまま実行すると `ロケーションを選択してください` が表示され、API実行前に止まります。ロケーションを選択して実行すると、内部エラーは `InventorySet 用の認証情報が設定されていません` でした。

## 補足・注意点

- テナント・カタログ・ロケーショングループは、Shopifyと接続する前にSQ内であらかじめ作成しておく必要があります。
- Shopifyストア側にSQ Connectが未インストールの場合、SQ側の「連携する」実行時にエラーになります。
- 2026-07-02時点で接続済みストアの詳細画面は確認済みです。ただし、SQ側の「連携する」押下からShopify認可、SQへ戻るまでの新規接続フローは一連の操作としては未確認です。
- 接続済みでも、SQからShopifyへ送る商品・在庫系の同期はアクセストークン未設定だと失敗します。商品一括同期は `ProductSet`、在庫一括同期は `InventorySet` の認証情報未設定で止まることを確認済みです。
- ShopifyからSQへの注文・顧客・返金・キャンセル受信は、アクセストークン未設定でも実機で動作確認済みです。ただし、出荷/返品、売上実績化、注文によるSQ既存SKUの在庫引当は正式連携SKUでの再検証が必要です。2026-07-02時点では連携済み商品数0、連携済みバリエーション0件のため、Shopify注文を作ってもSQ既存SKUの在庫引当検証にはなりません。
- データ連携チェックをOFFにした場合、注文OFF・顧客OFF・キャンセルOFFは反映抑止を確認済みです。一方、返金OFFでも今回の検証ではSQに `返金済み` が反映されたため、返金チェックは「OFFなら必ず返金が入らない」と案内しないでください。

## Shopify Flow連携

SQはShopify Flowの **トリガー** と **アクション** を提供しています（[公式ガイド](https://docs.sqstack.com/docs/guide/shopify-integration/shopifyflow) より）。これにより商品管理やCRMの処理を自動化できます。

> ※Shopify Flow自体の基本的な利用方法はShopifyの公式ドキュメントを参照してください。

2026-06-29の実機確認では、対象ShopifyストアにShopify Flowアプリは未インストールでした。Shopify管理画面の `/apps/flow` には「このアプリはインストールされていません」と `インストール` ボタンが表示されたため、SQ提供トリガー/アクションの表示と実行は未確認です。

### トリガー（7種）

| トリガー名 | 説明 |
|:--|:--|
| Shopify商品タグが更新されました | Shopify上で商品タグが更新されると起動 |
| Shopify商品バリエーション価格が更新されました | 商品バリエーションの価格変更時に起動 |
| ポイント失効のN日前になりました | 顧客のポイント失効日からN日前に起動（設定は失効通知画面） |
| 会員ランクがアップしました | 顧客の会員ランクがアップすると起動（2段階以上でも1回のみ発火） |
| 会員ランクがダウンしました | 顧客の会員ランクがダウンすると起動（2段階以上でも1回のみ発火） |
| 誕生日ポイントが付与されました | 誕生日ポイント付与時に起動 |
| 顧客の保有ポイントが更新されました | 顧客のポイントに変動があると起動（加算・減算問わず） |

### アクション（6種）

|:--|:--|
| ディスカウントに利用可能な顧客を追加する | 指定の顧客IDをディスカウントの利用可能顧客一覧に追加 |
| ポイントを付与する | 指定のポイント数を顧客に付与 |

| マイルを加算する | 指定の顧客にマイルを加算 |
| マイルを減算する | 指定の顧客からマイルを減算 |
| 商品バリエーションの連携を解除する | 指定のバリエーションのSQ→Shopify連携を解除（再連携可能） |

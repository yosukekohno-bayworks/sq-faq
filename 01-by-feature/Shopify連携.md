# Shopify連携

> 対象画面: Shopify連携 / `/admin/shopify_integrations`　|　最終確認: 2026-06-16

## この機能でできること

[公式ガイド](https://docs.sqstack.com/docs/guide/shopify-integration) によると、ShopifyとSQを連携することで以下が可能になります:

- **商品マスタの自動同期** — Shopifyの商品情報をSQと連携し一元管理
- **ロケーションの統合管理** — 店舗/倉庫の住所・電話・Google Mapsリンクを一元管理（SDKでストアフロント表示も可能）
- **在庫のリアルタイム管理** — 複数チャネル跨ぎの在庫を一括管理（SDKで店舗在庫をオンライン表示も可能）
- **顧客データ・ポイント・会員ランクの同期** — ストアフロント由来の顧客情報を基にSQでポイント/ランク管理（SDKでShopifyに表示）
- **Shopify Flowの利用** — SQ提供のトリガー/アクションで自動化（詳細は後述「Shopify Flow連携」）

加えて、SQ側で設定できる接続後の項目:

- 在庫引当ロケーション・販売価格ルール・会員証バーコード生成方法・税込設定などを設定する

## 画面・項目の説明

### 接続前の前提

Shopify連携を実行するには、先にShopifyストア側へSQコネクターアプリ（SQ Connect）をインストールしておく必要があります。

公開アプリのSQ Connectは本番SQ（`sq.stackservice.com`）向けです。staging環境（`sqstackstaging.com`）へ接続する場合は、staging向けの別アプリとインストールリンクが必要です。

### 接続前の画面（空状態）

Shopifyストアが未接続の場合、次のメッセージが表示されます。

> 「Shopifyストアを接続する / Shopifyと連携して、商品や注文データの連携を行います。」

「ストアを接続する」ボタンまたはリンクから接続フォーム（`/admin/shopify_integrations/create`）に進みます。

![Shopify連携の接続前の画面（空状態）](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/gap-channels-shopify-list.png)

### 接続フォーム「ストアを連携する」の項目

![「ストアを連携する」フォーム](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/gap-channels-shopify-create-form.png)

| 項目（UIラベル） | 説明 | 必須 | 制約・選択肢 |
|:--|:--|:--|:--|
| ストア名 | SQ内でのストア識別名 | 必須 | テキスト入力 |
| ショップドメイン | ShopifyストアのURL | 必須 | `shop.myshopify.com` の形式で入力 |
| テナント | 連携するSQ内テナント | 必須 | コンボボックス（SQ内マスタから選択） |
| カタログ | 連携する商品カタログ | 必須 | コンボボックス（SQ内マスタから選択） |
| ロケーショングループ | 注文在庫の引当元グループ | 必須 | コンボボックス（SQ内マスタから選択）。ヒント: 「Shopifyから連携される注文の在庫は、ロケーションの指定がない限り設定されたロケーショングループのデフォルトのロケーションから引き当てられます」 |
| 販売価格ルール | Shopify側の販売価格算出ルール | 任意 | コンボボックス。ヒント: 「選択した販売価格ルールに基づいて、Shopifyの販売価格を設定します」 |
| 会員証バーコードのフォーマット | 購買顧客の会員証バーコード生成方法 | 任意 | 「Shopify ID（数値部分）」（デフォルト）/ 「JAN-13コード（モジュラス10/ウェイト3方式）」。ヒント: 「Shopifyから連携される購買顧客の会員証バーコードの生成方法を指定します」 |
| 商品価格は税込価格を連携する | 商品価格を税込でShopifyに送るかどうか | 任意 | チェックボックス（デフォルト: オフ） |
| 0円の商品バリエーションを連携する | 価格が0円のバリエーションも連携対象にするかどうか | 任意 | チェックボックス（デフォルト: オフ） |
| 送料は税込として処理する | 送料を税込で処理するかどうか | 任意 | チェックボックス（デフォルト: オフ） |
| 注文による在庫変動を起こさない | Shopify経由の注文で在庫数を変動させないかどうか | 任意 | チェックボックス（デフォルト: オフ） |

**送信ボタン**: 「連携する」

### バリデーション

必須項目が未入力・未選択の場合、フィールド直下にエラーが表示されます。

![必須項目未入力時のバリデーションエラー表示](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/gap-channels-shopify-create-validation.png)

| 条件 | エラー文言 |
|:--|:--|
| ストア名が空 | 「ストア名を入力してください」 |
| ショップドメインが空 | 「ショップドメインを入力してください」 |
| テナント未選択 | 「テナントを選択してください」 |
| カタログ未選択 | 「カタログを選択してください」 |
| ロケーショングループ未選択 | 「ロケーショングループを選択してください」 |

## 補足・注意点

- テナント・カタログ・ロケーショングループは、Shopifyと接続する前にSQ内であらかじめ作成しておく必要があります。
- Shopifyストア側にSQ Connectが未インストールの場合、SQ側の「連携する」実行時にエラーになります。
- 「連携する」ボタン押下後にShopify側のOAuth認証フローが起動すると推測されますが、接続後の実際の遷移先・認証手順は未接続環境のため未確認です。

## Shopify Flow連携

SQはShopify Flowの **トリガー** と **アクション** を提供しています（[公式ガイド](https://docs.sqstack.com/docs/guide/shopify-integration/shopifyflow) より）。これにより商品管理やCRMの処理を自動化できます。

> ※Shopify Flow自体の基本的な利用方法はShopifyの公式ドキュメントを参照してください。

### トリガー（7種）

| トリガー名 | 説明 |
|:--|:--|
| Shopify商品タグが更新されました | Shopify上で商品タグが更新されると起動 |
| Shopify商品バリエーション価格が更新されました | 商品バリエーションの価格変更時に起動 |
| ポイント失効のN日前になりました | 顧客のポイント失効日からN日前に起動（設定は失効通知画面） |
| 会員ランクがアップしました | 顧客の会員ランクがアップすると起動（2段階以上でも1回のみ発火） |
| 会員ランクがダウンしました | 顧客の会員ランクがダウンすると起動（2段階以上でも1回のみ発火） |
| 誕生日ポイントが付与されました | 誕生日ポイント付与時に起動 |
| 顧客の保有ポイントが更新されました | 顧客のポイントに変動があると起動（加算・減算問わず） |

### アクション（6種）

| アクション名 | 機能概要 |
|:--|:--|
| ディスカウントに利用可能な顧客を追加する | 指定の顧客IDをディスカウントの利用可能顧客一覧に追加 |
| ポイントを付与する | 指定のポイント数を顧客に付与 |
| ポイントを減算する | 指定のポイント数を顧客から減算 |
| マイルを加算する | 指定の顧客にマイルを加算 |
| マイルを減算する | 指定の顧客からマイルを減算 |
| 商品バリエーションの連携を解除する | 指定のバリエーションのSQ→Shopify連携を解除（再連携可能） |

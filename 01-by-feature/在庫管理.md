# 在庫管理

> 対象画面: 在庫管理 / `/admin/inventory_items` 配下　|　最終確認: 2026-06-21

## この機能でできること

- SKU（商品バリエーション）×ロケーション単位で在庫数を確認する
- ロケーションを切り替えて、倉庫・店舗ごとの在庫状況を見る
- SKU詳細で在庫区分（販売可能・引当済み・取置中・破損・検品・予備・手持ち・積送中・入荷予定）の内訳を確認する
- 「販売可能在庫数を編集する」ダイアログで販売可能数を直接変更する
- 「在庫振替を行う」で同一商品内の別バリエーションへ在庫を振り替える
- 調整履歴でロケーションごとの変更履歴を確認する
- CSVでの一括インポート・エクスポートを行う

### 在庫詳細画面（/admin/inventory_items/[id]）

特定のSKUの在庫数をロケーション別に確認・操作する画面です。

#### ヘッダーボタン

| ボタン（UIラベル） | 動作 |
|:--|:--|
| 調整履歴 | 変更履歴ページ（`/admin/inventory_items/[id]/history`）へ遷移する |
| 商品管理 | バリエーション詳細画面へ遷移する |
| その他のアクション ▼ | 「在庫振替を行う」メニューが展開する |

タブは「**すべて / 店舗 / 倉庫**」の3種で、ロケーション種別を絞り込めます。

#### 在庫数量の区分（SKU詳細・9列）

> **2026-06-14更新**: 区分名が「確定済み→引当済み」「確保済み→取置中」に変わり、「積送中」「入荷予定」が新設されました。

在庫詳細では以下の9列でロケーション別の内訳を確認できます。

| 列名 | 内容 |
|:--|:--|
| ロケーション | ロケーション名 |
| 販売可能 | 販売できる在庫数（鉛筆アイコン付き） |
| 引当済み | 注文に引き当て済みの在庫数（旧「確定済み」） |
| 取置中 | 取置・移動用に確保された在庫数（旧「確保済み」） |
| 破損 | 破損品の在庫数 |
| 検品 | 検品中の在庫数 |
| 予備 | 予備在庫の数 |
| 手持ち | 実在庫合計（= 販売可能 + 引当済み + 取置中 + 破損 + 検品 + 予備） |
| 積送中 | 出荷済み・入荷前の**輸送中の数**（移動先で計上・手持ちに含まれない） |
| 入荷予定 | 移動伝票作成〜出荷前の**入荷予定数**（移動先で計上・手持ちに含まれない） |

**手持ちの計算式**（積送中・入荷予定は含まない）

```
手持ち = 販売可能 + 引当済み + 取置中 + 破損 + 検品 + 予備
```

![在庫詳細画面。ロケーション別に販売可能・引当済み・取置中・破損・検品・予備・手持ち・積送中・入荷予定が表示される](https://raw.githubusercontent.com/yosukekohno-bayworks/sq-faq/main/_analysis/screenshots/verify-inventory-detail-2026-06-19.png)

例（物流倉庫、SKU: 486125-09-XL）:
- 手持ち 7 = 販売可能 7（他区分はすべて 0。2026-06-15時点の検証環境データ）

### 在庫数の反映タイミング

操作によって在庫の各区分がどのタイミングで変わるかを示します。

| 操作 | 在庫の変化 |
|:--|:--|
| 在庫依頼を作成する（確認待ち状態） | 変化なし |
| 在庫を引当てる（在庫依頼が確保済み状態になる） | 変化なし（在庫の区分は動かない。※在庫依頼ステータスの「確保済み」は名称変更なし） |
| 移動伝票を作成する | 移動元の**販売可能が減り、取置中が増える**（手持ちは変わらない）。移動先には**入荷予定**が立つ |
| 出荷実績を登録する | 移動元の取置中と手持ちが減り、移動先の入荷予定が**積送中**へ移る |
| 入荷実績を登録する | 移動先の積送中または入荷予定が**販売可能・手持ち**へ移る（到着）。出荷前でも入荷登録は可能 |
| 移動伝票をキャンセルする | 販売可能・取置中が元の数に戻る |

**注意: マイナス在庫について**

手持ち在庫が 0 のロケーションから出荷登録を行うと、在庫数がマイナスになることがあります。2026-06-18の実機確認では、配送元の表示在庫が0のSKUでも移動伝票作成・出荷実績登録が完了しました。管理画面はマイナス在庫になっても出荷登録をブロックしません。在庫数がマイナスになっている場合は、調整伝票や移動伝票で在庫を補充してください。

### 「販売可能在庫数を編集する」ダイアログ

在庫詳細の「販売可能」列にある鉛筆アイコン（aria-label: 「販売可能数を編集」）をクリックすると開きます。

| 項目（UIラベル） | 必須/任意 | 内容 |
|:--|:--|:--|
| 理由 | **必須** | テキスト自由入力。空のまま「保存する」を押すと「理由を入力してください」と表示されて保存できない |
| 数量 | — | 数値入力（初期値 0、単位「個」） |
| キャンセル / 保存する | — | 「保存する」で確定 |

**保存したときの挙動:**

- 「数量」に入れた値が**そのまま新しい「販売可能」数になる**（絶対値の上書き。加算ではない。例: 販売可能1の状態で数量9を入力 → 販売可能は9になる）。「手持ち」は差分だけ増減する（例: 1→9なら手持ちも+8）
- 負数も保存できる（例: 数量 -1 を入力すると販売可能・手持ちが -1 になる）。通常運用では実在庫と合わなくなるため、負数登録は補正目的に限定する
- 調整伝票（`/admin/inventory_adjustment_orders`）には新しい伝票は作成されない（調整伝票とは別経路の在庫変更）
- 変更は「調整履歴」（`/admin/inventory_items/[id]/history`）に記録される

---

### 「在庫振替を行う」ダイアログ

「**その他のアクション**」>「**在庫振替を行う**」で開きます。同一商品内の別バリエーションへ在庫を振り替えます。

| 項目（UIラベル） | 必須/任意 | 内容 |
|:--|:--|:--|
| ロケーション | 必須 | テキスト入力 + 「選択」ボタン。ロケーション選択モーダルが開く |
| 振替先のバリエーション | 必須 | ドロップダウン「選択してください」。同一商品内の他バリエーションのみ表示され、振替元の自身は除外される。別商品のバリエーションは表示されない |
| 理由 | 任意 | テキストエリア（プレースホルダ:「在庫振替の理由を入力してください」） |
| キャンセル / 保存する | — | 「保存する」で振替を確定 |

---

### 調整履歴画面（/admin/inventory_items/[id]/history）

「**調整履歴**」ボタンを押すと変更履歴ページへ遷移します。ロケーションを選択して履歴を絞り込めます。データがない場合は「**在庫の変更履歴はありません**」と表示されます。

**履歴テーブルの列:**

| 列名 | 内容 |
|:--|:--|
| 日時 | 変更が行われた日時 |
| アクティビティ | 変更の理由・操作名。「販売可能在庫数を編集する」ダイアログで入力した「理由」がここに表示される |
| 販売可能 | 変更後の数量（差分表示付き。例: 5 (+5)） |
| 引当済み | 変更後の数量（差分表示付き。旧「確定済み」） |
| 取置中 | 変更後の数量（差分表示付き。旧「確保済み」） |
| 破損 | 変更後の数量（差分表示付き） |
| 検品 | 変更後の数量（差分表示付き） |
| 予備 | 変更後の数量（差分表示付き） |
| 手持ち | 変更後の数量（差分表示付き） |

## 補足・注意点

- **破損・検品・予備区分への振替経路**: 管理画面の「その他のアクション」には「在庫振替を行う」の1メニューのみで、破損・検品・予備専用の操作メニューはありません。調整伝票の「理由」選択肢（廃棄・紛失・在庫差異・その他）にもこれらの区分は存在しません。<!-- 連携実接続対象外: WMS連携〈ロジザード等〉経由での設定なのか、開発元への確認が必要 -->

- **手持ちはマイナスになり得る**: 在庫不足のまま出荷登録を行うと手持ちがマイナスになります。在庫依頼・移動伝票・調整伝票で補充または修正してください。

- **一括操作は提供されていない**: 一覧でチェックボックスを使って全件選択できますが、一括操作ボタンは表示されません。

# 発注管理

> 対象画面: 発注管理 / `/admin/inventory_purchase_orders`　|　最終確認: 2026-07-01

## この機能でできること

- 取引先（仕入先）への発注伝票を作成する
- 発注伝票に仕入れる商品・数量・単価・税率を記載し、金額を管理する
- 発注伝票を「下書き」から「発注済み」に進める
- 発注済み後に入荷指示を作成し、入荷予定・入荷完了・在庫反映まで確認する
- 発注済みの伝票をキャンセルする
- 発注に使用する取引先を「設定 > 取引先」で管理する

> **実機確認メモ（2026-06-27追加確認）**: 発注伝票 `#IP-1007` で、下書き → 発注済み → 発注起点の入荷指示 `#II-1026` 作成 → 入荷完了 → 在庫反映まで確認しました。発注だけでは入荷指示や入荷予定は増えません。発注済み後の「入荷指示を作成する」から入荷先ロケーションを選ぶと、入荷管理に `種別: 発注 / 情報 入荷待ち` の入荷指示が作成され、入荷先ロケーションの「入荷予定」が増えます。`#II-1026` を受領すると `成功 完了 / 入荷完了` になり、販売可能・手持ちが増えました。別伝票 `#IP-1008` では、入荷指示未作成の発注済み伝票をキャンセル済みにでき、在庫は変わりませんでした。
>
> **2026-06-28追加確認**: 新規発注伝票 `#IP-1009` で、発注済み後に発注起点の入荷指示 `#II-1027` を作成し、入荷完了前に発注伝票をキャンセルできることを確認しました。キャンセル後、`#IP-1009` は `キャンセル済み`、`#II-1027` は入荷管理一覧で `種別: 発注 / 警告 入荷キャンセル` になり、入荷指示詳細の入荷予定・入荷済みはいずれも0になりました。
>
> **2026-06-28差し戻し確認**: 発注済み未入荷の `#IP-1013` と入荷完了済みの `#IP-1007` で、詳細画面・`その他の操作`・直接URL `/update` / `/edit` を確認しました。編集・再編集・差し戻し・下書きに戻す導線はなく、`/update` / `/edit` は「このページは存在しないようです」でした。
>
> **2026-07-01帳票/メール確認**: 発注済み未入荷の `#IP-1014` と入荷完了済みの `#IP-1007` で、詳細画面・`その他の操作`・読み込まれている発注詳細JSを確認しました。管理画面UI上はPDF/印刷、発注確認メールの手動送信、編集/保存/更新/差し戻しの導線はいずれもありません。`#IP-1014` は `その他の操作 > キャンセルする` が有効、`#IP-1007` は入荷完了後のため同項目がdisabledです。

### 発注伝票 詳細とステータス

実機では次の流れを確認しました。

```
作成する
  ↓
下書き
  ↓ 発注する
発注済み
  ↓ キャンセルする
キャンセル済み
```

| 状態 | 実機で確認した表示/操作 | 在庫・入荷への影響 |
|:--|:--|:--|
| 下書き | `注意 / 下書き`。主要操作は「発注する」。 | 在庫は変わらない。 |
| 発注する | 確認ダイアログに「発注後は伝票を編集できません。発注後に入荷指示を作成できます。」と表示される。 | 発注だけでは入荷管理一覧に新規行は増えず、対象SKUの入荷予定も増えない。 |
| 発注済み | `情報 / 発注済み`。発注者/発注日時が表示され、商品セクションは「発注済み商品」になる。主要導線は「入荷指示を作成する」。 | 編集・再編集・差し戻し・下書きに戻す導線は見えない。入荷完了前ならキャンセル可能。直接 `/update` / `/edit` も存在しないページ。 |
| 入荷指示作成 | 発注済み伝票から入荷先ロケーションを選び、発注明細を引き継いだ入荷指示を保存する。未選択なら「入荷先のロケーションを選択してください」。 | `#II-1026` が `種別: 発注 / 情報 入荷待ち` で作成され、入荷先の入荷予定が0→+1。販売可能・手持ちはまだ増えない。 |
| 入荷完了 | 入荷指示の「入荷実績を登録する」で今回受領数を登録する。 | `#II-1026` は `成功 完了 / 入荷完了` になり、入荷完了タブへ移る。入荷先倉庫の販売可能8→9、手持ち8→9、入荷予定+1→0。 |
| キャンセル | 発注済み伝票で「その他の操作 > キャンセルする」を実行する。確認ダイアログは「発注伝票をキャンセルします。この処理は巻き戻すことができません。」。 | 入荷指示未作成の `#IP-1008` はキャンセル済みになり、在庫・入荷指示は変わらない。入荷指示作成済み・入荷完了前の `#IP-1009` もキャンセルでき、紐づく `#II-1027` は `警告 入荷キャンセル`、入荷予定0/入荷済み0になった。入荷完了済みの `#IP-1007` ではキャンセルがdisabled。 |
| キャンセル済み | ステータスが `キャンセル済み`。キャンセル者/キャンセル日時が表示される。`#IP-1009` では「入荷指示を作成する」の文言は残るが `href` はなく、遷移リンクではない。 | 再キャンセル・入荷指示作成はできない。発注起点の未完了入荷指示がある場合は入荷管理の「キャンセル」側に `入荷キャンセル` として残る。 |

### 取引先マスタ（設定 > 取引先 / /admin/settings/suppliers）

発注伝票の「取引先」に表示される仕入先の一覧・登録・管理を行う画面です。

**タブ**: 「すべて」 / 「アーカイブ」

**テーブル列**: 名前 / コード

**右上のボタン**

| ボタン | 動作 |
|:--|:--|
| インポート | クリックしても画面遷移・ダイアログが表示されない（未実装） |
| 取引先を作成 | 取引先の作成フォームへ遷移する |

**取引先 作成フォーム（/admin/settings/suppliers/create）**

| 項目（UIラベル） | 説明 | 必須 | 制約・選択肢 |
|:--|:--|:--|:--|
| 取引先名* | 取引先の名前 | 必須 | テキスト入力（プレースホルダー: NIKE Japan） |
| 取引先コード * | 取引先の識別コード | 必須 | テキスト入力（プレースホルダー: SUP-001）。空保存では `取引先コードを入力してください` |

| ボタン | 動作 |
|:--|:--|
| 保存する | 取引先を保存し、取引先一覧（/admin/settings/suppliers）へ遷移する |

取引先名を入力せずに「保存する」を押すと「取引先名を入力してください」というエラーが表示されます。
保存が完了すると「取引先を作成しました」というメッセージが表示されます。

## 補足・注意点

- 取引先マスタに登録したデータは、発注伝票作成フォームの「取引先」選択肢にすぐ反映されます。
- 2026-06-28の実機確認では、未使用の検証用取引先は一覧で行選択後 `アーカイブする` → 確認ダイアログ `取引先を削除しますか？` → `削除する` で、取引先一覧・アーカイブタブ・発注伝票作成フォーム候補から消えました。発注伝票で使用済みの取引先は削除されず、すべて/アーカイブ両タブと発注フォーム候補に残ります（`#IP-1013` で確認）。
- 通貨は発注伝票ごとに設定できます（国際仕入れに対応）。下書き状態では詳細画面で変更保存できます（2026-06-28 `#IP-1010`: `USD` → `EUR`、`$100.00` → `€100.00`）。
- 金額欄は単価・数量・税率から計算されます。2026-06-27の `#IP-1007` / `#IP-1008` では、作成フォーム入力中は `￥100`、発注済み詳細・一覧では `￥110` と表示されました。
- 取引先一覧の「インポート」ボタンは現時点では動作しません（未実装）。
- 発注だけでは入荷指示・入荷予定は自動作成されません。発注済み後の「入荷指示を作成する」で入荷先ロケーションを選ぶと、発注起点の入荷指示が作成されます。
- 発注起点の入荷指示を作成した後でも、入荷完了前であれば発注伝票をキャンセルできます。この場合、作成済みの入荷指示は `入荷キャンセル` へ移り、入荷予定は0になります。
- 発注伝票詳細には、現行管理画面UI上のPDF/印刷出力ボタンや発注確認メールの手動送信ボタンはありません（2026-07-01 `#IP-1014` / `#IP-1007` で確認）。自動通知や外部帳票/APIの有無は開発元確認事項です。

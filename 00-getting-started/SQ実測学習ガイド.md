# SQ実測学習ガイド

> この資料は、2026-06-19までにstaging管理画面で実際にボタンを押して確認した事実を中心に、SQを1から理解するためにまとめたものです。
> 外部連携の実接続、実注文、メール送信、APIキー発行、本番書き込みが必要な操作は対象外です。

## 1. SQを一文で理解する

SQは、商品・在庫・注文・顧客・CRM設定を1つの管理画面で扱うバックオフィスシステムです。

お客様が買う画面ではなく、運営者が「何を売るか」「どこに何個あるか」「注文や出荷をどう処理するか」「顧客へどんな割引やポイントを適用するか」を管理する場所です。

## 2. 最初に覚えるデータ

| データ | 意味 | 重要ポイント |
|:--|:--|:--|
| 商品 | 商品名・商品コード・ブランドなどを持つ商品台帳 | 商品コードは作成後に変更できない |
| バリエーション / SKU | 色・サイズなどの組み合わせ。SKUが在庫管理の最小単位 | 在庫・価格・出荷・入荷はSKU単位で動く |
| ロケーション | 在庫の置き場所。店舗または倉庫 | 在庫は「SKU × ロケーション」で見る |
| テナント | 事業部・販売単位のような区分 | ディスカウント、発注、連携設定などの必須選択肢 |
| 会社 | B2Bの法人顧客 | 管理画面から作成できる |
| 顧客 | 購入顧客 | 管理画面から直接作成できない。チャネル流入が前提 |
| 注文ポイント付与ルール | 通常ポイントの基本ルール | 会員ランク倍率・商品倍率を配下に持つ |
| ポイントキャンペーン | 通常ポイントに倍率や追加ポイントを加える期間ルール | 種別により保存後の設定画面が変わる |
| 会員ランク算出ルール | 購入金額に応じたランクを決めるルール | 初期ランクBronzeは条件変更・削除不可 |

## 3. 画面の大分類

| 領域 | 画面 | 役割 |
|:--|:--|:--|
| 商品 | 商品管理 / カタログ / 店舗受取 | 売る商品、SKU、表示・連携対象を管理する |
| 在庫 | 在庫管理 / 調整伝票 / 取置伝票 / 移動伝票 | SKUごとの在庫数を確認し、増減や移動を記録する |
| 注文 | 注文管理 / 下書き / 返品 | チャネルから流入した注文・返品を見る |
| 顧客 | 顧客管理 / 会社 | 購入顧客と法人顧客を扱う |
| 販売設定 | 販売価格 / 予約販売 / 販売上限 / 販売閾値 | 売価、在庫0販売、上限、閾値を制御する |
| CRM | ディスカウント / ポイント / 会員ランク | 割引、ポイント付与、ランク制度を設定する |
| 連携 | Shopify / OmnibusCore / スマレジ / リテールポータル | 外部サービスとの接続設定 |
| 設定 | テナント、ロケーション、権限、メタフィールドなど | 他画面の選択肢になる基本データを作る |

## 4. 在庫の基本

SQの在庫は「どのSKUが、どのロケーションに、何個あるか」で管理されます。

| 区分 | 意味 |
|:--|:--|
| 販売可能 | 売ってよい数 |
| 引当済み | 注文などに引き当てられた数 |
| 取置中 | 取置・移動用に確保された数 |
| 破損 / 検品 / 予備 | 販売不可の内訳 |
| 手持ち | 販売可能 + 引当済み + 取置中 + 破損 + 検品 + 予備 |
| 積送中 / 入荷予定 | 移動中・入荷予定の数。手持ちには含まれない |

在庫操作で重要なのは、どの操作が在庫数を動かすかです。

| 操作 | 在庫への影響 |
|:--|:--|
| 調整伝票を実行 | 指定ロケーションの販売可能・手持ちが増減する |
| 取置伝票を処理済みにする | 取置中が解放される |
| 移動伝票を作成 | 移動元の販売可能が減り、取置中が増える。移動先に入荷予定が立つ |
| 出荷実績を登録 | 移動元の手持ちが減る。移動先は積送中になる |
| 入荷実績を登録 | 移動先の販売可能・手持ちが増える |

出荷実績・入荷実績・CSV実行などは巻き戻せません。FAQでは必ず「対象SKU、ロケーション、数量を確認してから実行」と書く必要があります。

## 5. 取り寄せ販売の流れ

取り寄せ販売は、SQの中心的な業務フローです。

1. 在庫依頼を作成する
2. 在庫依頼詳細で在庫を引当てる
3. 確保済み一覧から移動伝票を作成する
4. 出荷管理で出荷実績を登録する
5. 入荷管理で入荷実績を登録する

実測で確認できた重要挙動は次のとおりです。

| 挙動 | 内容 |
|:--|:--|
| 在庫依頼は確認待ちならクローズできる | 確保済み後はクローズ不可 |
| 移動伝票作成で出荷指示と入荷指示が自動生成される | 伝票が在庫の移動をつなぐ |
| 移動伝票をキャンセルすると関連する出荷指示・入荷指示もキャンセルされる | 元の在庫依頼は確保済みのまま残る |
| 配送元と配送先が同じ移動伝票も保存できる | 誤作成した場合は移動伝票詳細の「その他の操作」>「キャンセル」で取り消す |
| 出荷実績・入荷実績は取り消せない | 誤りは調整伝票などで補正する |

## 6. 顧客と会社を混同しない

SQには「顧客」と「会社」と「会社担当者」があります。

| データ | 管理画面で作れるか | 何に使うか |
|:--|:--|:--|
| 顧客（購入顧客） | 作れない | 注文、ポイント、会員ランク、ディスカウント対象顧客 |
| 会社 | 作れる | B2B法人顧客 |
| 会社ロケーション | 作れる | 法人顧客の納品先 |
| 会社担当者 | 作れる・削除できる | 発注確認メールの宛先 |

2026-06-15の実測では、顧客一覧は0件でした。`/admin/purchasing_customers/create` は「予期せぬエラーが発生しました」になり、再実行しても作成フォームは開きません。2026-06-28には検証用購入顧客を内部GraphQLで作成し、一覧・詳細UIを確認しましたが、管理画面UIから顧客を新規作成できない点は変わりません。

会社担当者を作っても顧客一覧には出ません。ディスカウントの顧客管理タブで会社担当者メールを検索しても候補には出ません。

## 7. ディスカウントの理解

ディスカウントは、クーポンコード付きの注文割引ルールです。

| 項目 | 実測で確認したこと |
|:--|:--|
| クーポンコード | 必須。作成後は詳細では「割引コード」と表示される |
| テナント | 必須。作成後変更不可 |
| 割引方法 | 割引率 / 割引額 |
| 顧客管理タブ | 顧客データがないと追加を最後まで実行できない |
| 店舗管理タブ | ロケーション追加・適用種別変更・削除を実行済み |
| 商品管理タブ | 商品バリエーション追加・削除を実行済み |
| すべての商品を割引対象に設定する | ONで作成すると基本情報タブの適用条件 > 対象商品が「成功 / すべての商品」と表示される |
| Shopify連携 | 接続済み環境が必要なため実連携は対象外 |

店舗管理タブの適用種別は「値引き」と「金種」が表示されますが、実測では「金種」はdisabledで、「値引き」のみ選択できました。

## 8. ポイントの理解

ポイントは大きく5つに分かれます。

| 種類 | 役割 |
|:--|:--|
| 注文ポイント付与ルール | 通常のポイント付与率を決める |
| ポイントキャンペーン | 期間限定で倍率や追加ポイントを設定する |
| 誕生日ポイント | 誕生日に付与するポイントを決める |
| 利用外商品 | ポイント利用対象外の商品を登録する。付与対象外は商品倍率0倍で制御する |
| 失効通知 | ポイント失効前の通知ルールを作る |

注文ポイント付与ルールでは、購入金額あたりのポイント、ポイント有効日数、オンライン注文/店舗注文の有効化日数、キャンペーン優先設定を管理します。

詳細画面では次の追加設定があります。

| 設定 | 実測済み挙動 |
|:--|:--|
| 会員ランク倍率 | 詳細画面にタブと「追加する」ボタンがあることは確認済み。ただし2026-06-19の再確認では、ボタン押下後のフォーム遷移・モーダル表示は確認できていない |
| 商品倍率 | 詳細画面にタブと「追加する」ボタンがあることは確認済み。ただし2026-06-19の再確認では、ボタン押下後のフォーム遷移・モーダル表示は確認できていない |

## 9. ポイントキャンペーンの種別

ポイントキャンペーンは、作成時の種別で保存後の操作が変わります。

| 種別 | 作成フォーム内 | 保存後に設定すること |
|:--|:--|:--|
| なし | 付与方法に応じて倍率または固定ポイント数の入力欄が出る | 全体キャンペーンとして使う |
| 会員ランク | 具体的なランク入力欄は出ない | 会員ランク算出ルール、会員ランク、ポイント倍率を追加する |
| 購入金額 | 具体的な金額入力欄は出ない | 購入金額（円）とポイント倍率を追加する |
| 商品 | 具体的な商品入力欄は出ない | 商品SKUとポイント倍率を追加する |

商品種別の「商品を追加する」はポップオーバーです。選択肢は「商品を選択して追加する」「ブランドを指定して追加する」「CSVでインポートする」です。ブランド指定は大量追加になる可能性があるため、フォームとバリデーション確認までに留めています。

ポイントキャンペーン一覧で複数選択して削除すると、「現在有効なキャンペーンを削除すると、注文の作成や返品時のポイント計算に影響が出る可能性があります。」という警告が出ます。

「通常付与ポイントに加えて指定のポイントを追加で付与する」を選ぶと、種別「なし」では倍率欄ではなく固定ポイント数の入力欄に切り替わります。

## 10. 会員ランクの理解

会員ランク算出ルールは、購入金額に応じたランクを管理します。

| 項目 | 実測で確認したこと |
|:--|:--|
| 算出方法 | 購入金額固定。獲得ポイントは選択不可 |
| Bronze | 初期ランク。達成条件変更・削除不可。表示名のみ編集可 |
| ランク追加 | 表示名と購入金額で追加できる |
| ランク編集・削除 | 追加したランクは編集画面で更新・削除できる |
| 除外商品 | 商品を選択してランク算出から除外できる。追加・削除を確認 |
| 算出期間 | 「直近365日」「無期限」を選ぶと開始月・月初算出・持ち越し設定が非表示になる |

会員ランクは単体で完結するだけでなく、注文ポイント付与ルール、ポイントキャンペーン、誕生日ポイントでも参照されます。

## 11. 販売設定の注意点

| 機能 | 実測で確認したこと |
|:--|:--|
| 販売価格 | 通常価格・セール価格のルールを作る |
| 予約販売 | 対象SKUを追加・削除できる。削除時のトースト文言にズレがある |
| 販売上限 | 作成フォームはルール名中心で、チャネル未接続だと「チャネルを選択してください」で保存不可 |
| 販売閾値 | SKU別閾値追加、閾値削除ダイアログ、ルール削除ダイアログを確認。SKU別閾値は削除確定直後に行が残る場合があるが、詳細を再読み込みすると削除反映を確認できた |

販売上限は、チャネル接続済み環境でないと最後まで作成できない状態です。FAQでは「チャネル未接続時は作成不可」と明記してください。

## 13. 外部連携が必要で未検証の範囲

次は「未確認」ではなく、「外部連携がないと実行できない」範囲です。FAQでは断定しないでください。

| 領域 | 未検証理由 |
|:--|:--|
| Shopifyへの実連携 | 接続済みストアとOAuthが必要 |
| Shopify注文の流入 | 実注文が必要 |
| スマレジ在庫同期 | 接続済みスマレジ環境が必要 |
| OmnibusCore連携後の下書き注文 | 接続済みサイトが必要 |
| ロジザード入出荷連携 | WMS接続が必要 |
| Recustomer返品連携 | Recustomer接続が必要 |
| 実注文へのポイント・ディスカウント適用 | 注文データとチャネル接続が必要 |
| メール通知 | 実メール送信が伴う |

## 14. FAQを書くときの重要な注意

| 注意 | 理由 |
|:--|:--|
| 製品理解を先に置く | 2026-06-12のキックオフで、FAQ化より先に機能・データ・依存関係の理解を優先する方針になった |
| 操作方法だけで終わらせない | 顧客が迷うのは「どう押すか」だけでなく、「どんな時にその設定が必要か」だから |
| 標準機能と顧客別の個別連携を混ぜない | 個別開発のバックエンドロジックは、標準FAQだけでは解決しにくい |
| stagingの未実装/開発中を通常仕様と混同しない | stagingには開発中で画面だけ出ている機能がある |
| プレースホルダを初期値と書かない | 例の数値であり、保存時には入力が必要な項目がある |
| 会社担当者を顧客と書かない | 実測で別データだと確認済み |
| 画面文言のズレはそのまま書く | 例: ブランド削除の確認「ブランドを削除を削除しますか？」、誕生日ポイント会員ランク設定削除のタイトル |
| 削除は対象を明確にする | ルール本体削除と配下設定削除が似た文言で出る |
| 外部連携前提の挙動は断定しない | staging単体では注文流入・同期・実適用まで確認できない |
| 巻き戻せない操作は必ず警告する | 実績登録、CSV実行、削除系は復元不可が多い |

## 15. 最初に読む順番

1. この資料で全体像を読む
2. データ事典①で設定マスタを確認する
3. データ事典②で商品・在庫・CRMのデータ構造を確認する
4. 機能別ガイドで該当画面の項目を確認する
5. 作業別ガイドで実際の操作手順を確認する
6. サポートデスクHTMLでエラー文言・制約を逆引きする

この順番で読むと、画面のどのボタンがどのデータを作り、どの状態を変えるのかが追いやすくなります。

# 残存未確認の例外台帳（2026-06-19）

ユーザー依頼「確認不可なものを除き、未確認を実機で確認」に対する残存例外です。

ここに載せるものは、今回のstaging管理画面・現ログイン権限・外部連携なしの条件では最後まで確認できないものです。FAQ本文では断定しないでください。

## 1. 外部連携が必要

| 領域 | 残る確認事項 | 理由 |
|:--|:--|:--|
| Shopify | OAuth完了、商品同期、注文流入、在庫同期、メタフィールド同期 | 接続済みShopifyストアとOAuthが必要 |
| OmnibusCore | 商品・在庫・注文同期、下書き注文の有効化条件 | 接続済みOmnibusCore環境が必要 |
| スマレジ | 在庫同期の方向・頻度・エラー時挙動 | 接続済みスマレジ環境が必要 |
| リテールポータル | 店舗スタッフ画面、店舗受取の実注文挙動 | 接続済み環境または別権限ユーザーが必要 |
| ロジザード | WMS連携後の入出荷・破損/検品/予備の扱い | WMS接続が必要 |
| Recustomer | 返品連携 | Recustomer接続が必要 |

## 2. 実データが必要

| 領域 | 残る確認事項 | 理由 |
|:--|:--|:--|
| 注文 | 注文詳細、キャンセル、返金、返品、交換出荷 | 現環境の注文一覧が0件 |
| 顧客 | 顧客詳細、注文履歴、ポイント残高、会員ランク表示 | 現環境の顧客一覧が0件 |
| 売上実績 | 実データありのテーブル列、注文からの売上実績生成 | 売上実績一覧が0件 |
| ポイント | 実注文への付与、返品・キャンセル時の増減 | 実注文・顧客・ポイント履歴が必要 |
| 会員ランク | 購入金額による実ランク反映 | 顧客購入実績が必要 |
| PDF出力 | 納品書PDFの実出力内容 | 生成済みPDFまたは生成導線が出る出荷データが必要 |

## 3. メール・別アカウントが必要

| 領域 | 残る確認事項 | 理由 |
|:--|:--|:--|
| 管理メンバー | 招待メール本文、有効期限、追加ユーザーの初回ログイン | メール受信と別アカウント操作が必要 |
| CSV/PDFエクスポート | 完了メールの有無・本文・リンク有効期限 | メール受信が必要。画面上の実行履歴とダウンロードリンクは確認済み |
| 権限差分 | 店舗スタッフなど権限別のメニュー・ボタン差分 | 別権限ユーザーでのログインが必要 |

## 4. 管理画面上に入口がない、または途中実装に見える

| 領域 | 実機で確認したこと | 残る確認事項 |
|:--|:--|:--|
| 破損/検品/予備 | 在庫詳細に列はあるが、管理画面の「在庫振替」は同一商品内の別バリエーション振替のみ | WMS/API経由で区分へ振り替えるのか開発元確認 |
| 発注後の入荷指示 | `#IP-1000` / `#IP-1001` / `#IP-1003` で発注まで実行したが、入荷管理一覧やSKU入荷予定増加は確認できず | 条件不足・非同期・未実装のどれか開発元確認 |
| 注文ポイント倍率 | 会員ランク倍率/商品倍率タブと「追加する」ボタンは確認。押下後のフォーム/モーダルは出ず | 追加フォームが別条件で出るのか開発元確認 |
| 販売上限 | 作成フォームに「販売上限ルール名」「チャネル」は表示。チャネル選択肢なし | チャネル接続済み環境での詳細画面確認 |
| 分析/卸売 | `/admin/analytics`、`/admin/analytics/revenue`、`/admin/analytics/reports`、`/admin/b2b` はTODO表示 | 機能実装後に再確認 |

## 5. 再現条件が未確定

| 領域 | 実機で見えたこと | 扱い |
|:--|:--|:--|
| 取置伝票 | 商品参照モーダルで複数SKU明細を設定可能 | 再現条件は後続確認で確定済み。2026-06-28 `#IR-1011` で作成フォーム12明細・数量入力12件を保存し処理済み化まで確認。詳細の商品明細テーブルは伝票固有表示として扱わない |
| 移動伝票の一部受領済み/受領済みタブ | 部分入荷・過入荷自体は実行済みだが、現在一覧タブは対象データなしでdisabled | 対象データが出た時点で一覧表示条件を再確認 |

## 6. 今回、未確認から確認済みに変えた主な項目

| 領域 | 2026-06-19に確認したこと |
|:--|:--|

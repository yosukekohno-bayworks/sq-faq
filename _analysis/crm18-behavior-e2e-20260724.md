### テナントCRMルール割り当て解除の確定手順（GraphQL remove mutation）

通常画面の各セレクトボックスは「〜を選択してください」（未設定）オプションが引き続き `disabled` で、UIからの解除は不可（前回から不具合変わらず）。今回、ブラウザのGraphQLイントロスペクションが有効だったため、正式なmutation名と引数を特定できた。

- エンドポイント: `POST /api/graphql`（`credentials: include` で管理画面のCookieセッションを利用）
- 4つの専用remove mutation（各`input`は`tenantID: ID!`のみ）:
  - `tenantRemovePointCalculationRule(input: { tenantID })`
  - `tenantRemoveCustomerRankCalculationRule(input: { tenantID })`
  - `tenantRemovePointCalculationBirthdayRule(input: { tenantID })`
  - `tenantRemovePointExpirationNotificationRule(input: { tenantID })`
- 各Payload型が返すフィールドは `tenant { id }` のみ（`userErrors` は存在しないため問い合わせるとスキーマエラーになる点に注意）。
- 実行例:
  ```graphql
  mutation M($input: TenantRemovePointCalculationRuleInput!) {
    tenantRemovePointCalculationRule(input: $input) { tenant { id } }
  }
  ```
  variables: `{ "input": { "tenantID": "da86c1b7-191a-51c2-80e9-f9c3a2a09d9f_Tenant" } }`
- 4件とも実行後、`/admin/settings/tenants/{id}` を再読み込みし、4項目すべてがプレースホルダー表示（未設定）に戻ったことを確認済み。

この手順は次回以降の同種検証（テナントへのCRMルール一時割り当て→復旧）に再利用できる。

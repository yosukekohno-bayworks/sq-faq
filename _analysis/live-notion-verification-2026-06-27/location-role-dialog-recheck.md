# ロケーション種別と用途の再確認（2026-06-27）

## 確認対象

- `/admin/settings/locations/create`
- `/admin/retail_portal_integrations/create`
- `/admin/local_pickup_retail_location_rules/create`

## 確認結果

### ロケーション作成フォーム

- h1: `ロケーションを作成`
- 基本情報: `名前`, `表示名`, `コード`, `場所種別`
- 場所種別の選択肢: `倉庫`, `店舗`
- 販売設定: `店舗受け取りを有効にする`, `在庫依頼を受け付ける`, `ポイント利用種別`

### リテールポータル連携作成フォーム

- h1: `リテールポータル`
- フォーム項目: `店舗ロケーション`, `在庫ロケーション`, `テナント`, `カタログ`, `販売閾値ルール`, 各種許可チェックボックス
- `店舗ロケーション` の選択ダイアログには、店舗種別のロケーションのみ表示された。
  - 例: `ユニクロ - 銀座店 / R0001`, `TEST_E2E_20260622_GU店舗_OFF_1905 / TEST_E2E_20260622_STORE_1905`
  - 倉庫例 `ユニクロ物流倉庫 / W0001`, `TEST_E2E_20260622_GU倉庫_ON_1905` は表示されなかった。
- `在庫ロケーション` の選択ダイアログには、倉庫種別のロケーションのみ表示された。
  - 例: `ユニクロ物流倉庫 / W0001`, `TEST_E2E_20260622_GU倉庫_ON_1905 / TEST_E2E_20260622_WH_1905`
  - 店舗例 `ユニクロ - 銀座店 / R0001`, `TEST_E2E_20260622_GU店舗_OFF_1905` は表示されなかった。

### 店舗受取ロケーションルール作成フォーム

- h1: `店舗受取を作成する`
- フォーム項目: `ロケーション`
- `ロケーション` の選択ダイアログには、店舗種別のロケーションのみ表示された。
  - 例: `ユニクロ - 銀座店 / R0001`, `TEST_E2E_20260622_GU店舗_OFF_1905 / TEST_E2E_20260622_STORE_1905`
  - 倉庫例 `ユニクロ物流倉庫 / W0001`, `TEST_E2E_20260622_GU倉庫_ON_1905` は表示されなかった。

## 判断

- 確定: ロケーションの場所種別は `倉庫` / `店舗` の2種。
- 確定: リテールポータル連携では、店舗種別が `店舗ロケーション`、倉庫種別が `在庫ロケーション` の候補になる。
- 確定: 店舗受取ロケーションルールでは、店舗種別のロケーションのみ候補になる。
- 注意: `店舗受け取りを有効にする` のON/OFFが候補表示やストアフロントでの受取可否にどう影響するかは、接続環境で未確認。
- 注意: 手動の在庫依頼・移動伝票では、移動元/移動先として倉庫・店舗の両方が関与するため、「倉庫=出荷元だけ」「店舗=受け口だけ」と全機能共通に断定しない。

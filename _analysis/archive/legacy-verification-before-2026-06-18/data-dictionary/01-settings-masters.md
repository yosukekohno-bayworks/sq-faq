## 組織ID
- **分類**: システム生成データ
- **何か**: 組織を識別するID。設定トップページの「組織ID」セクションに表示される。
- **作成場所**: システム自動生成（ユーザー作成不可）。設定トップページ（`/admin/settings`）で確認・コピー（クリップボードアイコンのコピーボタンあり）。
- **主な項目**: 組織ID文字列（形式例: `{uuid}_Organization`）
- **作用先**:
  - 設定トップページ: 表示・コピーのみ（他画面での利用先は対象ファイルに記載なし）
- **制約・注意**: —
- **出典**: 設定.md / 14-settings.md

## テナント
- **分類**: マスタ
- **何か**: SQ内でブランド・部門などの運用単位を分ける管理単位。説明文（UIの原文）:「事業部を表すテナントデータを管理することができます。」初期状態で1件登録されている場合がある。商品マスタの「ブランド」とは別データ。
- **作成場所**: 設定 > テナント（`/admin/settings/tenants`）>「テナントを作成」ボタン
- **主な項目**: ⚠️未検証（作成フォームの入力項目は対象ファイルに記載なし。「必要事項を入力して保存する」とのみ記載）
- **作用先**:
  - ロケーション・管理メンバー・発注などの設定: 後続の設定に紐づく（初期設定の手順.md）
  - 管理メンバー詳細「ユーザーテナント」セクション: アクセスを許可するテナントのチェックボックス選択肢になる
  - 注文・CSVエクスポート: テナント選択の選択肢になる（設定と権限のよくある質問.md）
- **制約・注意**: マスタが1件も登録されていない場合、選択肢のドロップダウンには何も表示されない（空のまま）
- **出典**: 設定.md / 初期設定の手順.md / 管理メンバーを追加する.md / 設定と権限のよくある質問.md / 14-settings.md

## ロケーション
- **分類**: マスタ
- **何か**: 在庫を管理する物理的な拠点（倉庫・店舗）。在庫移動・発注・出荷などのすべての在庫操作に必要。
- **作成場所**: 設定 > ロケーション（`/admin/settings/locations`）>「ロケーションを作成」ボタン →（`/admin/settings/locations/create`）。一覧には「インポート」「エクスポート」ボタンもあり
- **主な項目**:
  - 基本情報: 名前 *（注文や商品などの画面で表示される名前）/ 表示名 / コード * / 場所種別 *（セレクトボックス: **倉庫 / 店舗** の2種のみ）
  - 所在地・連絡先: **ロケーションの住所を登録する**（チェックボックス。オンで国/地域・郵便番号・都道府県・市区町村・住所・建物名の入力欄が展開）/ マップ（URL）/ 電話番号 / メールアドレス
  - 販売設定（右サイドバー）: **店舗受け取りを有効にする**（チェックボックス、デフォルト: オフ）/ **在庫依頼を受け付ける**（チェックボックス、デフォルト: オフ）/ ポイント利用種別 *（セレクトボックス: **値引き（デフォルト）/ 金種** の2種のみ）
- **作用先**:
  - 在庫操作全般: 在庫移動・発注・出荷などのすべての在庫操作に必要（初期設定の手順.md）。FAQでは「在庫・発注・リテールポータル設定」の用途と記載
  - 在庫依頼（取り寄せ販売）: 「在庫依頼を受け付ける」をオンにすると在庫依頼の選択肢（リクエスト先・送付先）に表示されるようになる
  - ロケーショングループ: グループに属するロケーションとして選択される（詳細画面に「所属ロケーショングループ」セクション=「n個のグループに属しています」）
  - 販売員: 販売員作成フォームの「ロケーション」欄の選択肢になる（「ロケーションを選択する」モーダル、タブ「すべて / 店舗 / 倉庫」）
  - ロケーショングループ経由のShopify連携: グループ内で「在庫依頼を受け付ける」が有効な店舗に在庫リクエストが送られる（その他のマスタを登録する.md）
- **ステータス**:
  | 表記 | 意味 | 遷移 | 在庫・データへの影響 |
  |:--|:--|:--|:--|
  | 営業中 | 詳細画面の基本情報セクションに表示される営業状態バッジ | ⚠️未検証（遷移操作は対象ファイルに記載なし） | ⚠️未検証 |
  | 公開中 | 詳細画面の「公開設定」セクションに表示される公開状態バッジ | ⚠️未検証 | ⚠️未検証 |
  | （一覧テーブル列「公開」「アーカイブ」） | 一覧の列として存在（14-settings.md テーブル列: 名前 / コード / 場所種別 / 公開 / アーカイブ / ロケーショングループ / タグ） | ⚠️未検証 | ⚠️未検証 |
- **制約・注意**:
  - 場所種別は「倉庫」「店舗」の2種類のみ。それ以外の種別は選択できない（実機確認済み）
  - ポイント利用種別は「値引き」「金種」の2種類のみ。「値引き」と「金種」の具体的な動作の違いは ⚠️未検証（TODO: 要確認）
  - 詳細画面にはほかに「備考欄」（フリーテキスト）と「タグ」（追加・管理）セクションあり
- **出典**: 設定.md / 初期設定の手順.md / 14-settings.md / 設定と権限のよくある質問.md

## ロケーショングループ
- **分類**: マスタ
- **何か**: 複数のロケーションをひとまとまりとして管理するグループ。Shopify連携の設定（在庫の引き当てロケーション指定）などで参照される。
- **作成場所**: 設定 > ロケーショングループ（`/admin/settings/location_groups`）>「グループを作成」ボタン →（`/admin/settings/location_groups/create`）
- **主な項目**:
  - 名前 *（例: 関東全店。プレースホルダー: 「関東圏・全店舗」）
  - **このグループに新しいロケーションを自動的に含める**（チェックボックス。オンにすると今後作成したロケーションを自動でグループに追加する。**作成後は変更不可** ⚠️未検証: 既存グループでのdisabled状態が「変更不可」か「表示のみ」か）
  - ロケーション（作成フォームでは「選択」ボタン＋「ロケーションを選択する」ダイアログ。タブ「すべて」「店舗」「倉庫」、複数選択可。作成後はグループ詳細画面の「編集する」リンク → サブページ（h1「グループに含まれるロケーション」）>「ロケーションを追加」から追加・変更）
  - デフォルトのロケーション（詳細画面の「デフォルトのロケーション」セクションの変更ボタン（鉛筆ボタン）から変更）
- **作用先**:
  - Shopify連携の設定: ロケーショングループを指定すると、グループ内で「在庫依頼を受け付ける」が有効な店舗に在庫リクエストが送られる。また、グループ内のデフォルトロケーションから在庫が引き当てられる（その他のマスタを登録する.md）
  - 取り寄せ販売（EC注文起点）: グループ内で「在庫依頼を受け付ける」が有効な店舗に在庫リクエストが送られる
  - Shopify / OmnibusCore / 在庫エクスポート: 選択肢として参照される（設定と権限のよくある質問.md）
  - 在庫依頼の作成フォーム: 「ロケーショングループ」欄は**存在しない**（実測）。ロケーションは「移動先ロケーション」「リクエスト先ロケーション」を個別指定する形式（14-settings.md）
- **制約・注意**:
  - デフォルトロケーションに設定されたロケーションは、グループのロケーション編集画面でチェックを外せない（グループから除外できない仕様）。外したい場合は先に「デフォルトのロケーション」を別のロケーションに変更する
  - OmnibusCore連携接続環境での在庫振り分けへの具体的な使われ方は ⚠️未検証（OmnibusCore未接続のため）
  - フォームにブランド指定項目はなし（ロケーション集合のみ定義）
- **出典**: その他のマスタを登録する.md / 設定.md / 14-settings.md / 設定と権限のよくある質問.md

## ブランド
- **分類**: マスタ
- **何か**: 商品に紐づける「ブランド情報」のマスタ。商品分類やカタログへの自動追加ルールで参照される。
- **作成場所**: 設定 > ブランド（`/admin/settings/brands`）> 右上「追加する」ボタン →（`/admin/settings/brands/create`）
- **主な項目**: 名前 * / **外部ID**（任意。外部システム連携用の識別子。**作成時のみ設定可。保存後は変更不可**）/ コード *
- **作用先**:
  - 商品詳細の「ブランド」欄: 選択肢になる（設定と権限のよくある質問.md）
  - カタログへの自動追加ルール: 条件として参照される（その他のマスタを登録する.md）
- **ステータス**:
  | 表記 | 意味 | 遷移 | 在庫・データへの影響 |
  |:--|:--|:--|:--|
  | すべて / アーカイブ | 一覧タブの2種類 | ⚠️未検証（アーカイブ操作自体は対象ファイルに記載なし） | ⚠️未検証 |
- **制約・注意**:
  - 外部IDは作成フォームにのみ表示され、保存後の詳細・編集フォームには表示されない（編集フォームは「ブランド名 *」「コード *」のみ。実機確認済み）。変更が必要な場合は削除して再作成
  - 削除: 一覧で行チェックボックスにチェック →「削除する」ボタン。削除確認ダイアログ（タイトル原文: 「ブランドを削除を削除しますか？」/ 本文原文: 「選択された1件のブランドを削除します。この処理は巻き戻すことができません。」）。**削除は取り消しできない**
  - 一覧テーブル列: 名前 / コード / 外部ID / 更新日時
- **出典**: その他のマスタを登録する.md / 設定.md / 14-settings.md / 設定と権限のよくある質問.md

## 取引先
- **分類**: マスタ
- **何か**: 在庫を仕入れる業者。説明文（UIの原文）:「在庫を仕入れる取引先の業者を管理することができます。」発注伝票の作成に必要。
- **作成場所**: 設定 > 取引先（`/admin/settings/suppliers`）>「取引先を作成」ボタン →（`/admin/settings/suppliers/create`）
- **主な項目**: 取引先名 *（必須）/ 取引先コード（任意。例: SUP-001。詳細画面から後で変更可能）
- **作用先**:
  - 発注伝票の「取引先」欄: 選択肢になる（設定と権限のよくある質問.md）
- **ステータス**:
  | 表記 | 意味 | 遷移 | 在庫・データへの影響 |
  |:--|:--|:--|:--|
  | アーカイブ | アーカイブした取引先は一覧の「アーカイブ」タブに移動し、通常の「すべて」タブには表示されなくなる | アーカイブ解除の操作方法は ⚠️未検証 | 発注伝票の取引先選択肢にも表示されなくなる。アーカイブ後の既存発注伝票への影響は ⚠️未検証 |
- **制約・注意**: `/admin/inventory_suppliers` というURLは404（存在しない）。正しくは `/admin/settings/suppliers`
- **出典**: 初期設定の手順.md / 設定.md / 設定と権限のよくある質問.md / 14-settings.md

## 決済方法
- **分類**: マスタ
- **何か**: 顧客が注文時に利用する支払い手段（クレジットカード・代引きなど）の設定。
- **作成場所**: 設定 > 決済方法（`/admin/settings/payment_methods`）>「決済方法を作成」ボタン →（`/admin/settings/payment_methods/create`）
- **主な項目**:
  - 名前 *（プレースホルダー: 「クレジットカード」）
  - コード *（プレースホルダー: 「credit_card」）
  - ゲートウェイ *（プレースホルダー: 「shopify_payments」）
  - **支払い待ちでも注文を出荷する**（チェックボックス、デフォルト: オフ。オンにすると入金確認前でも出荷処理を進められる — 推測・実動作 ⚠️未検証）
  - **代引き**（チェックボックス、デフォルト: オフ。代金引換の場合にチェックする — 効果の実動作は ⚠️未検証）
- **作用先**:
  - 注文の決済方法設定: 選択肢になる（設定と権限のよくある質問.md）
- **制約・注意**: 「支払い待ちでも注文を出荷する」「代引き」チェックボックスの効果はフォームに存在することは確認済みだが実動作は未検証（推測）のまま（14-settings.md [A07]関連・なお未確認の項目3）
- **出典**: 初期設定の手順.md / 設定.md / 14-settings.md / 設定と権限のよくある質問.md

## 販売員
- **分類**: マスタ
- **何か**: リテールポータル（店舗向け画面）で顧客情報を閲覧した人・客注注文を作成した人として記録する担当者情報。
- **作成場所**: 設定 > 販売員（`/admin/settings/retail_staff_members`）>「販売員を作成」ボタン →（`/admin/settings/retail_staff_members/create`）
- **主な項目**:
  - 姓 * / 名 *（同じ行に横並び表示）
  - コード *（販売員を識別するコード）
  - ロケーション（任意。作成フォームでは初期ロケーションを1件選択。作成後の詳細画面で「店舗を追加する」から複数店舗を紐づけ可能）
- **作用先**:
  - リテールポータル: 顧客情報の閲覧者・客注注文の作成者として記録される
  - ロケーション: 担当店舗との紐づけに使われる
- **制約・注意**: 作成フォームで選べる初期ロケーションは1件。販売員詳細では複数店舗を追加できるため、販売員自体が1ロケーション専属とは限らない。
- **出典**: その他のマスタを登録する.md / 設定.md / 14-settings.md

## 管理メンバー（ユーザー）
- **分類**: マスタ
- **何か**: 管理画面にアクセスできるメンバー。説明文（UIの原文）:「管理画面にアクセス可能なメンバーを管理することができます」
- **作成場所**: 設定 > 管理メンバー（`/admin/settings/users`）> 右上「追加する」ボタン →「ユーザーを追加する」フォーム（`/admin/settings/users/create`）。CSVによる一括追加は「インポート」ボタン
- **主な項目**:
  - 姓 * / 名 *（同一行に横並び）
  - メールアドレス *
  - 権限グループ（「権限」セクションのラジオボタンで1グループのみ選択可。作成済みグループが一覧表示される）
  - パスワード入力欄は**ない**（メールアドレスによる認証でログイン）
- **保存後の詳細画面（3セクション、保存ボタンはセクションごとに独立）**:
  | セクション | 内容 |
  |:--|:--|
  | 基本情報 | 姓・名・メールアドレスの編集 |
  | ユーザーテナント | **アクセスを許可するテナントのチェックボックス選択** |
  | 権限 | 権限グループをコンボボックスで変更。下部に「適用されている権限一覧: N件」アコーディオン |
- **作用先**:
  - 管理画面へのアクセス: メンバーとして登録された人だけがアクセスできる
  - テナント: 「ユーザーテナント」チェックボックスでメンバーごとにアクセス可能なテナントを制御（`user_tenants:read/write` 権限の実体）
  - 権限グループ: 割り当てられたグループの権限で画面の閲覧・編集が制御される
- **ステータス**:
  | 表記 | 意味 | 遷移 | 在庫・データへの影響 |
  |:--|:--|:--|:--|
  | ユーザーを組織から除外 | 「その他の操作」ドロップダウンの唯一の選択肢。退職・アクセス削除時の操作（削除ではなく「除外」という表現） | 確認ダイアログの有無・文言は ⚠️未検証 | 除外後にログインがブロックされるかは ⚠️未検証 |
- **制約・注意**:
  - 1人のメンバーに割り当てられる権限グループは1つのみ（複数グループへの同時割り当て不可）。追加フォームはラジオボタン形式、詳細画面はコンボボックス形式だがどちらも単一選択
  - 「保存する」後に招待メールが送信されるか・メール内容・有効期限は ⚠️未検証
  - SQへのログインはメール認証（ワンタイムコード方式）。6桁のコードを手入力。パスワードの設定・変更画面は存在しない
- **出典**: 管理メンバーを追加する.md / 設定.md / 初期設定の手順.md / 14-settings.md / 設定と権限のよくある質問.md

## 権限グループ
- **分類**: ルール（アクセス制御の単位）
- **何か**: 管理メンバーに割り当てるアクセス制御の単位。35種類のリソースに対して「閲覧（read）」「編集（write）」の権限を個別に設定できる（35リソース × 閲覧/編集 = 70権限）。
- **作成場所**: 管理メンバー一覧（`/admin/settings/users`）右上の「権限グループ一覧」ボタン → 権限グループ一覧（`/admin/settings/permission_groups`）>「作成する」ボタン →「権限グループを作成する」フォーム（`/admin/settings/permission_groups/create`）
- **主な項目**:
  - グループ名 *（テキストボックス。プレースホルダー: 「リテールポータル管理者」）
  - **権限（チェックボックス群）**: 70権限から個別に選択。**デフォルトは全オフ（最小権限）**。read/writeは別々に設定可能

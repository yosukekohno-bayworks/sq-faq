## 商品（商品管理）
- **分類**: マスタ
- **何か**: 販売対象の商品データ。マスタ設定（テナント・ロケーション・取引先）を土台に登録され、在庫の前提になる。
- **作成場所**: 商品管理（一覧・作成フォーム・詳細編集画面）
- **主な項目**: 商品一覧の列: 商品 / ステータス / 商品コード / 在庫 / カタログ / 商品タイプ / 製造元。**ステータス**（公開状態を制御）、**オプション種別**（バリエーション作成時: サイズ / カラー / その他 — **作成時のみ設定可。保存後は変更不可**。2026-06-02リリース）
- **作用先**:
  - 在庫: 商品登録 → 在庫が用意される（データの流れ ①〜③）
  - 商品一覧タブ: ステータスにより表示タブが変わる（公開中 / 下書き / アーカイブ済み）
- **ステータス**:
  | 表記 | 意味 | 遷移 | 在庫・データへの影響 |
  |:--|:--|:--|:--|
  | 一覧タブ: すべて / 公開中 / 下書き / アーカイブ済み | — | — | 「アーカイブ済み」はナビ「商品管理 > 全タブ」から確認できる |
  | 作成時ステータス: 公開 / 下書き / 非公開 | — | **「非公開」で作成した商品は編集画面では「アーカイブ済み」と表示される** | — |
  | 編集時ステータス: 公開中 / 下書き | 2択のみ | 作成時の「非公開」はこの画面には表示されない | — |
- **制約・注意**:
  - アーカイブ・削除の操作経路: 商品詳細「その他の操作」>「商品をアーカイブする」「商品を削除する」（全機能ガイドは「下書き/公開中」の2つしか説明しておらず反映漏れと指摘済み）
  - 編集画面のステータス2択（公開中/下書き）からはアーカイブへ直接遷移できない＝「その他の操作」経由のみ
- **出典**: statuses.md（L5-7, L47）、LIVE-RECHECK-2026-06-10.md（A表, C-5）、データの流れ-図解.md

## 移動伝票
- **分類**: 伝票
- **何か**: 在庫をロケーション間で移動する伝票。**作成した瞬間に出荷指示（#IO）と入荷指示（#II）が自動生成される**（取り寄せ販売の中核）。
- **作成場所**: 在庫依頼詳細から「移動伝票を作成する」（確保済み状態の依頼から）。移動伝票作成ダイアログの移動元選択ボタンには「選択」ラベルあり（記録内矛盾を実機で決着）
- **主な項目**: 移動元ロケーション / 移動先 / 商品・数量。詳細画面に「関連」欄（自動生成された出荷指示#IO・入荷指示#IIが表示）
- **作用先**:
  - 在庫（移動元）: **作成した瞬間に 販売可能 −1 / 確保済み +1（手持ちは不変）**。引当時点では在庫数は動かない（実測: 銀座店 販売可能4→3、確保済み0→1）
  - 出荷管理: 出荷指示が自動生成される
  - 入荷管理: 入荷指示が自動生成される
  - キャンセル時: 確保済み → 販売可能に全数戻る（実測: 完全復帰）。紐づく入荷予定・出荷指示もすべてキャンセル。未完了バッジ（出荷/入荷）も消滅
- **ステータス**:
  | 表記 | 意味 | 遷移 | 在庫・データへの影響 |
  |:--|:--|:--|:--|
  | すべて / 出荷作業 / 一部受領済み / 受領済み / キャンセル（一覧タブ） | — | **ステータスを直接進めるボタンはなく、出荷・入荷実績登録に連動して自動変化する** | 出荷実績登録で移動元の手持ちが減り、入荷実績登録で移動先の手持ちが増える |
  | 完了（詳細バッジ） | 完了状態 | 出荷完了＋入荷完了で到達 | **詳細のステータスバッジは「完了」と「入荷完了」の2つが並んで表示される** |
  | キャンセル | 取消済み | 詳細「その他の操作」>「キャンセル」（入荷完了後の伝票でもメニュー項目として表示） | 確認ダイアログ原文: 見出し「移動伝票をキャンセルしますか？」本文「移動伝票をキャンセルすると、この伝票に紐づく入荷予定および出荷指示もすべてキャンセルされます。」 |
- **制約・注意**:
  - 外部システム連携伝票の編集制限文言（原文）: 見出し「外部システムから連携されている移動伝票です」＋本文「この移動伝票は外部システムから連携されているため、編集（商品の追加や削除など）は制限されています。連携元システムの管理画面より操作を行ってください。」
  - **キャンセルしても元の在庫依頼は「確保済み」のまま残り、移動伝票の再作成もクローズもできない（行き止まり）。やり直すには在庫依頼を新規作成する**
- **出典**: statuses.md（L8）、LIVE-RECHECK-2026-06-10.md（C-4, C-6, F-2, F-3, F-4, H-1）、データの流れ-図解.md

## 在庫依頼
- **分類**: 伝票
- **何か**: 取り寄せ販売の起点。店舗が倉庫等へ在庫を依頼し、引当→移動伝票作成へつながる。
- **作成場所**: 在庫依頼（作成フォーム）
- **主な項目**: 商品バリエーション*（必須）/ 希望数*（必須）/ 移動先ロケーション*（必須）/ **リクエスト先ロケーション**（必須マーク（*）なし＝任意項目）。一覧列に「**チャネル**」あり（管理画面起票は `web-admin` と表示）
- **作用先**:
  - 在庫: 作成（確認待ち）では変化なし。「在庫を引当てる」（確保済み）でも**在庫詳細の「確保済み」列は動かない**。数字が動くのは移動伝票作成時
  - 移動伝票: 確保済みから「移動伝票を作成する」で発行
  - 未完了バッジ: サイドナビの表示対象
  - 引当ダイアログ: ロケーション選択肢はリクエスト先のみ表示
- **ステータス**:
  | 表記 | 意味 | 遷移 | 在庫・データへの影響 |
  |:--|:--|:--|:--|
  | 確認待ち | 未引当 | 「在庫を引当てる」→確保済み／「在庫依頼をクローズする」→終了 | 在庫変化なし |
  | 確保済み | 引当後（例: 2 / 2） | 「移動伝票を作成する」へ | 在庫変化なし（確保済み列は動かない） |
  | 終了 | クローズ済み | 詳細の「その他の操作」>「在庫依頼をクローズする」 | — |
- **制約・注意**:
  - **クローズできるのは「確認待ち」のみ。「確保済み（引当後）」では「在庫依頼をクローズする」はdisabled（実行不可）。移動伝票キャンセル後も依然disabled**（依頼は確保済みのまま残る＝行き止まり。確保済み在庫テーブルの行選択でも追加操作は出ない）
  - 用語ゆれ: フォーム「移動先ロケーション」→詳細画面では「引当先ロケーション」、フォーム「リクエスト先ロケーション」→詳細画面では「依頼先」
- **出典**: statuses.md（L12）、LIVE-RECHECK-2026-06-10.md（C-1, C-6, F-1, F-2, F-4, F-5）、データの流れ-図解.md

## 出荷指示（出荷管理）
- **分類**: システム生成データ
- **何か**: 移動伝票作成時に自動生成される出荷の指示（#IO）。倉庫（WMS）への出荷指示として流れる。
- **作成場所**: 直接作成しない（移動伝票から自動生成）
- **主な項目**: 出荷実績の登録操作、条件指定エクスポートの「出荷作業ステータス」絞り込み
- **作用先**:
  - 在庫: 出荷実績登録で移動元の手持ちが減る。**手持ち0のロケーションから出荷実績を登録しても処理はブロックされず、在庫がマイナスになる**（実データ: 486125-31-L 物流倉庫 手持ち -2 / 販売可能 -2）
  - 移動伝票: 実績登録に連動してステータスが自動変化
  - 未完了バッジ: 表示対象
- **ステータス**:
  | 表記 | 意味 | 遷移 | 在庫・データへの影響 |
  |:--|:--|:--|:--|
  | 保留中 / 出荷待ち / 依頼済み / 作業中 / 欠品・要対応 / 出荷完了（一覧タブ6種） | — | 出荷実績登録に連動 | **キャンセルタブなし。移動伝票キャンセル後の出荷指示はどのタブにも表示されない（一覧から消える）** |
  | エクスポートの出荷作業ステータス（8種）: 指定しない / 出荷待ち / 保留中 / 依頼済み / 作業中 / 欠品・要対応 / キャンセル済み / 出荷完了 | 「条件指定でエクスポート」フォームの絞り込み選択肢 | — | キャンセル済み出荷指示はエクスポートでのみ追跡可能（⚠️「と思われる」と記載、確認手段が他にないかは未確認） |
- **制約・注意**:
  - 出荷実績・入荷実績の登録は**巻き戻せない**
  - 完了後の詳細で実績の編集・削除UIなし（公式一覧の「編集・削除」はAdmin API経由または将来機能の可能性。⚠️Stack社確認事項）
- **出典**: statuses.md（L13-14）、LIVE-RECHECK-2026-06-10.md（C-3, F-3, D-4）、FEATURE-SWEEP-2026-06-12.md（5）、データの流れ-図解.md

## 在庫（在庫区分・4区分／7区分）
- **分類**: ステータス群
- **何か**: SKU×ロケーションごとの在庫数。一覧では4区分、SKU詳細のロケーション別テーブルでは7区分で表示される。
- **作成場所**: 直接作成しない（商品登録・伝票・連携・CSVインポートにより変動）
- **主な項目**:
  - 一覧（4区分の列構成）: 販売不可 / 確定済み / 販売可能 / 手持ち。**手持ち = 販売可能 + 確定済み + 販売不可**
  - 詳細（7区分の列構成）: 販売可能 / 確定済み / 確保済み / 破損 / 検品 / 予備 / 手持ち。**手持ち = 販売可能 + 確定済み + 確保済み + 破損 + 検品 + 予備**（一覧の「販売不可」が詳細では「破損/検品/予備」等に分かれる構造）
  - 在庫詳細 > 在庫振替のロケーション種別タブ: すべて / 店舗 / 倉庫（在庫詳細画面・ロケーション選択モーダルで共通）
- **作用先**:
  - 取置伝票: 作成で「確保済み」へ、処理済みマークで解放
  - 移動伝票: 作成で 販売可能→確保済み に振り替わる（在庫依頼の作成・引当の時点では変化なし）
  - 出荷実績/入荷実績: 移動元の手持ちが減る／移動先の手持ちが増える
  - 調整伝票: マイナス在庫の修正に使用
- **ステータス**（反映タイミング・実測 F-2）:
  | 操作 | 移動元の在庫変化 |
  |:--|:--|
  | 在庫依頼を作成（確認待ち） | 変化なし |
  | 在庫を引当てる（確保済み） | 変化なし（「確保済み」列は動かない） |
  | 移動伝票を作成する | 販売可能 4→3、確保済み 0→1（手持ち4は不変） |
  | 移動伝票をキャンセル | 販売可能 3→4、確保済み 1→0（完全に復帰） |
- **制約・注意**:
  - **在庫はマイナスになり得る**（実データ: 486125-31-L 物流倉庫 手持ち -2 / 販売可能 -2。手持ち0のロケーションから出荷実績を登録してもブロックされない）
  - ⚠️未検証: **破損/検品/予備への振替経路は未解明**（管理画面に操作経路が見当たらない。WMS連携（ロジザード）経由の可能性）
  - 恒等式は実データで整合確認済み（手持ち4 = 販売可能3 + 確保済み1 等）
- **出典**: statuses.md（L25-26, L34）、LIVE-RECHECK-2026-06-10.md（A表, C-2, C-3, F-2, G-4）、データの流れ-図解.md

# 商品・バリエーション・カタログ

## 商品
- **分類**: マスタ
- **何か**: 商品名・説明文・画像・バリエーションなどをまとめた管理単位（例: 「エアリズムコットンクルーネックTシャツ」）。商品1件に対して複数のバリエーションを持てる。
- **作成場所**: 商品管理（/admin/products）>「商品を作成する」→ 商品作成フォーム（/admin/products/create）。CSVインポート「商品」カテゴリでも一括登録・更新可。
- **主な項目**:
  - **商品コード\***（必須）: 半角英数字・ハイフン・アンダースコアのみ使用可（例: 7835152003XL180）。**作成後に変更できません**（編集画面では値が表示されコピーボタンあり。⚠️未検証: 編集画面で商品コードを変更できるか）
  - 商品名\*（必須）
  - 説明文（任意、5000字上限、文字数カウンター付き）
  - メディア: 「画像をアップロード」エリア（クリック/ドラッグ&ドロップ）と「追加」ボタン。どちらもローカルファイル選択。URLからの画像追加はできない
  - 検索エンジンリスティング: ページタイトル / メタディスクリプション（5000字上限）— 検索結果に表示されるタイトル・説明文
  - **ステータス**（作成時3択: 下書き〔デフォルト〕/ 公開 / 非公開）
  - 商品分類: 商品タイプ（任意テキスト）/ **製造元**（任意テキスト）/ **ブランド**（「選択」ボタンでブランドマスターから選択）
  - **タグ**（任意。未設定時「タグは付与されていません」）
- **作用先**:
  - 商品一覧（/admin/products）: 列「商品 / ステータス / 商品コード / 在庫 / カタログ / 商品タイプ / 製造元」に表示。「在庫」列は実在庫数量ではなく**バリエーション数**（例: 20個のバリエーション）、「カタログ」列はこの商品が属するカタログの数
  - 商品一覧の検索: 「検索と絞り込みの結果」→「商品コードで検索する」。検索できるのは**商品コードのみ**（商品名・ステータス・商品タイプ・タグでの絞り込みUIなし）
  - カタログ「商品を追加する」画面（/admin/catalogs/{id}/create）: 商品が選択候補として表示され、「商品コードで検索する」で絞り込み
  - 自動追加ルール: 商品の「製造元」「ブランドコード」属性がルールの完全一致判定対象になる
  - カタログ詳細・カタログ内検索: 「商品コードで検索する」で絞り込み
- **ステータス**: 「商品ステータス」の項を参照
- **制約・注意**:
  - 商品コードは作成後に変更できない（事前に決めておくこと）
  - 「保存する」は商品コード*・商品名*・オプション名*・種別*・オプション値*のいずれかが未入力だと保存不可
  - 変更がない場合「保存する」ボタンは無効（グレー）
  - 削除（「商品を削除する」）は取り消し不可。アーカイブは取り消し可能（論理削除）
  - ⚠️未検証: 商品削除時の確認ダイアログのタイトル・説明文・ボタンラベル原文
- **出典**: 商品管理.md, 商品を作成する.md, 商品を編集する.md, 商品をアーカイブ・削除する.md, 商品と商品登録のよくある質問.md, 03-products.md

## 商品ステータス（下書き / 公開中 / アーカイブ済み）
- **分類**: ステータス群
- **何か**: 商品の公開状態。作成フォーム・編集フォーム・一覧タブで表記が異なる（表記ゆれあり）。
- **作成場所**: 商品作成フォームのサイドバー「ステータス」セレクト（3択、デフォルト: 下書き）。編集はサイドバー「ステータス」セレクト（2択）と「その他の操作」（アーカイブ/解除）。
- **主な項目**: 作成時の選択肢（下書き / 公開 / 非公開）、編集時の選択肢（公開中 / 下書き）
- **作用先**:
  - 商品一覧のタブ: 「すべて」「公開中」「下書き」「アーカイブ済み」でステータス別に絞り込み（タブがステータスフィルターの代替。URLは ?tab=N、すべて=0/なし・公開中=1・下書き=2・アーカイブ済み=3。サーバーサイドフィルタリングで毎回 POST /api/graphql 発生）
  - 一覧バッジ: 下書き=DRAFT（toneInfo・青系）、公開中=ACTIVE（toneSuccess・緑系）。DOMの「情報」はバッジ色名のvisuallyHiddenテキストで画面非表示
  - アーカイブ済み商品: 「販売チャネルと管理画面上で非表示」（確認ダイアログ原文）。編集フォームに「アーカイブ済み」固定テキストと説明文「管理画面では、商品リスト以外は非表示です。」が表示される
- **ステータス**:
  | 作成フォーム | 編集フォーム | 一覧タブ | 遷移 | 影響 |
  |:--|:--|:--|:--|:--|
  | 下書き | 下書き | 下書き | 編集画面で「公開中」へ変更可 | — |
  | 公開 | 公開中 | 公開中 | 編集画面で「下書き」へ変更可 | 表記ゆれだが同一ステータス（値: ACTIVE） |
  | 非公開 | アーカイブ済み | アーカイブ済み | 「その他の操作 > 商品のアーカイブを解除する」で**下書き**になる | 管理画面の商品リスト以外で非表示。「非公開」=「アーカイブ済み」は同じ状態 |
- **制約・注意**:
  - 編集画面のステータス選択肢は「公開中」「下書き」の2択のみ。「非公開」は表示されない。アーカイブしたい場合は「その他の操作 > 商品をアーカイブする」を使う
  - アーカイブ解除後のステータスは**必ず「下書き」**（アーカイブ前が公開中でも公開中には戻らない）。解除ダイアログ原文: 「この商品のアーカイブを解除すると、ステータスが下書きに変更され再び作業できるようになります。」
  - アーカイブ確認ダイアログ原文: 「この商品をアーカイブすると、販売チャネルと管理画面上で非表示になります。商品リストにあるステータス絞り込み機能を使用して商品を探します。」
  - アーカイブ済み商品の「その他の操作」は「商品のアーカイブを解除する」「商品を削除する」の2項目（「商品をアーカイブする」は出ない）
- **出典**: 商品管理.md, 商品を作成する.md, 商品を編集する.md, 商品をアーカイブ・削除する.md, 商品と商品登録のよくある質問.md, 03-products.md

## バリエーションオプション（オプション軸）
- **分類**: マスタ（商品の属性設定）
- **何か**: 商品のサイズ違い・カラー違いなど「軸」ごとに管理するための設定。1つの商品に複数の軸を持てる（例: カラー × サイズ）。
- **作成場所**: 商品作成フォーム（/admin/products/create）の「バリエーション」セクション。「追加」ボタンで値の行を追加、「別のオプションを追加する」で軸を追加。
- **主な項目**:
  - オプション名\*（必須、例: サイズ）
  - **種別\***（必須、セレクト: 「サイズ」「カラー」「その他」の3択）
  - オプション値\*（必須、例: S）
  - コード（任意、例: 001）
- **作用先**:
  - バリエーション作成フォーム（/admin/products/{id}/variants/create）: 商品に設定したオプション軸ごとにセレクトが表示され、登録済みのオプション値の中から選択する（例: カラー* = BEIGE/GRAY/NAVY/BLACK、サイズ* = XS/S/M/L/XL）
  - 第1部保存時: 設定したオプション軸の組み合わせ分だけバリエーションが作成される
  - 商品詳細「バリエーション」セクション: 属性グループ（カラー・サイズ等）の折りたたみ表示
- **制約・注意**:
  - オプション値の行が1行のみの場合「削除」ボタンは無効
  - 新しい値をバリエーション追加時に使いたい場合は、先に商品編集側でオプション値を追加する必要がある
  - ⚠️未検証: 既存商品のオプション値を後から追加・編集する操作画面の有無。オプションの軸（カラー・サイズ以外の新ジャンル）を後から追加できるかも未確認（詳細画面に編集UIは見当たらず、作成時のみの可能性が高い。CSV/API経由の可否含めStack社確認事項）
- **出典**: 商品管理.md, 商品を作成する.md, 商品と商品登録のよくある質問.md, 03-products.md

## バリエーション（SKU）
- **分類**: マスタ
- **何か**: 商品の色・サイズなどのオプションを組み合わせた1パターン（例: 「白 / Mサイズ」）。SKU（最小管理単位）はバリエーションに紐づく在庫管理コード（例: 486102-70-S）。
- **作成場所**: 商品詳細「バリエーション」セクション >「バリエーションを追加する」→ バリエーション作成フォーム（/admin/products/{id}/variants/create）。送信ボタンは「作成する」。CSVインポート「商品バリエーション」カテゴリでも一括登録・更新可。
- **主な項目**:
  - オプション（必須）: 商品のオプション軸から値をセレクトで選択
  - メディア: 「画像を選択する」ボタン（商品の既存画像から選択）
  - 価格設定: **上代\***（必須、販売価格・円）/ 仕入価格（任意、円。作成フォームのみ）
  - 在庫: **SKU (最小管理単位)\***（必須、例: 9999990000T001）/ メーカーSKU（任意）/ **在庫を追跡する**（チェックボックス。在庫数の追跡を行う場合はオン）/ **在庫切れの場合でも販売を続ける**（チェックボックス）
  - 販売: バーコード / JAN / EAN / UPC（UPCは編集画面のみ表示）
  - 配送: 重量（任意）/ 単位（グラム / キログラム〔デフォルト〕/ オンス / ポンド）/ **配送を必須にする**（チェックボックス、デフォルトON）
  - 関税情報: 原産国コード（国・地域を選択）/ 統計品目 (HS) コード
  - 原価（編集画面のみ）: 「原価が設定されていません」+「+ 原価を登録する」ボタン
- **作用先**:
  - 商品一覧「在庫」列: バリエーション数として表示される（実在庫数量ではない）
  - 在庫管理: バリエーション編集画面上部に「在庫管理」リンクがあり、該当バリエーションの在庫管理画面へ遷移。実在庫数量はここで確認
  - カタログ「SKU一覧」（/admin/catalogs/{id}/catalog_product_variants): カタログに追加されているSKUが一覧表示される
  - 店舗受取（/admin/local_pickup_product_variants）: 店舗受取可能なバリエーションとして選択・追加する対象
- **制約・注意**:
  - 「作成する」/「更新する」は「上代*」「SKU (最小管理単位)*」が未入力だと押せない
  - 作成フォームと編集フォームの差異: 編集には「仕入価格」欄がなく、代わりに「原価」セクションと「UPC」フィールドがある。送信ボタンは「更新する」
  - ⚠️未検証: 作成フォームで「仕入価格」を入力した場合、編集フォームの「原価」セクションに値が引き継がれるか（「仕入価格」と「原価」が同一概念かはUIラベルが異なるため断定不可）
  - ⚠️未検証: 「+ 原価を登録する」クリック後の入力フォームの構成
  - 編集画面の在庫セクションで「在庫を追跡する」がchecked表示 → デフォルトONと推測（03-products.md記載、推測扱い）
- **出典**: 商品管理.md, 商品を作成する.md, 商品を編集する.md, 商品と商品登録のよくある質問.md, 03-products.md

## カタログ
- **分類**: マスタ
- **何か**: 商品をグループ化する入れ物。外部連携（OmnibusCoreなど）のチャネル（連携先）ごとに1つのカタログを指定し、どの商品をそのチャネルに出すかを制御する。
- **作成場所**: 商品管理 サブメニュー「カタログ」（/admin/catalogs）>「カタログを作成する」→ /admin/catalogs/create。
- **主な項目**: **タイトル\***（必須、例: 渋谷店）のみ。編集も「編集する」ボタン →「カタログを編集する」ダイアログでタイトル変更のみ。
- **作用先**:
  - OmnibusCore連携（/admin/omnibus_core_integrations）: 連携設定の「商品同期設定 > カタログ」セレクトの選択肢になる。チャネルに出品する商品群をカタログ単位で制御
  - カタログ一覧の列: 名前 / 商品（登録商品数、例: 6個の商品）/ 販売先（このカタログを参照しているチャネル連携の数、例: 1つの販売先）
  - カタログ名はチャネル連携でカタログを選ぶときの識別名になる
  - CSVインポート「カタログ」カテゴリ（/admin/csv_import/csv_import_operation_catalog_products）: 「カタログ」ドロップダウン（必須）の選択肢になり、商品を一括追加・削除
- **制約・注意**:
  - 商品の追加方法は3つ: 手動（「商品を追加する」→ チェックボックス選択 →「カタログに追加する」）/ 自動追加ルール / CSVインポート（「その他の操作 > インポート」）
  - チャネル割り当て・ロケーション紐付けは編集ダイアログから設定できない。「販売先」はチャネル連携側の設定から紐付ける（カタログ詳細画面から直接設定するUIはない）
  - 削除は「その他の操作 > カタログを削除する」。削除前に商品との紐付けを確認すること
  - カタログ内の商品検索は「商品コードで検索する」のみ（商品名での検索不可）
  - ⚠️未検証: Shopify連携・スマレジ連携でのカタログ紐付け設定経路（OmnibusCoreのみ実機確認済み。リテールポータルも未確認）
  - ⚠️未検証: 商品が登録されている状態でのSKU一覧（/admin/catalogs/{id}/catalog_product_variants）の列構成（テスト環境0件のため列ヘッダー非表示）
- **出典**: カタログ.md, カタログを作成して商品を追加する.md, 商品をCSVで一括登録する.md, 03-products.md

## 自動追加ルール
- **分類**: ルール
- **何か**: 条件に一致する商品を自動的にカタログへ追加するルール。
- **作成場所**: カタログ詳細 >「自動追加ルール」（/admin/catalogs/{id}/automatic_add_rules）>「追加する」→「ルールを追加する」ダイアログ。
- **主な項目**:
  - 項目: 「製造元」「ブランドコード」の2択
  - 条件: 「一致する」の1択（変更不可）
  - 値: テキスト自由入力（例: UNIQLO）
- **作用先**:
  - カタログ: 設定したルールと**完全一致**する商品が自動的にカタログへ追加される
- **制約・注意**:
  - 完全一致のみ対応。部分一致・正規表現には対応していない
  - 大文字・小文字・スペースの違いも一致判定に影響する
- **出典**: カタログ.md, カタログを作成して商品を追加する.md, 03-products.md

## ブランド（ブランドマスター）
- **分類**: マスタ
- **何か**: 商品に紐づけるブランドのマスター。事前登録が必要。
- **作成場所**: 設定 > ブランド（/admin/settings/brands）>「追加する」→ /admin/settings/brands/create。
- **主な項目**: 名前\*（必須）/ 外部ID（任意）/ **コード\***（必須）
- **作用先**:
  - 商品作成・編集フォームのサイドバー「ブランド」欄: 「選択」ボタン →「ブランドを選択する」ダイアログでブランドマスターから選択（「キャンセル」「選択する」ボタン）
  - カタログの自動追加ルール: 項目「ブランドコード」の一致判定対象
- **制約・注意**: 商品にブランドを設定するには事前にマスター登録が必要（未登録だと選択ダイアログは「アイテムが見つかりませんでした」表示）
- **出典**: 商品管理.md, 商品を作成する.md, 商品を編集する.md, 03-products.md

## メタフィールド定義・メタフィールド値
- **分類**: マスタ（定義）＋商品の属性（値）
- **何か**: 商品に追加するカスタムフィールド。「設定 > メタフィールド定義」で定義を登録し、商品詳細画面で値を設定する。
- **作成場所**: 定義: 設定 > メタフィールド定義（商品オブジェクトの定義は「設定 > メタフィールド定義 > 商品」）。値: 商品詳細画面の「メタフィールド」セクションで編集ボタン（鉛筆アイコン）から入力。
- **主な項目**: ⚠️未検証: メタフィールド値の編集UIの詳細構成（インライン編集か別画面遷移か）
- **作用先**:
  - 商品詳細画面: 商品オブジェクトの定義が登録されている場合のみ「メタフィールド」セクション（見出し）が表示される。定義が存在しない場合セクション自体が表示されない
  - CSVインポート: 商品一覧「インポート」ドロップダウンの「商品メタフィールド」（/admin/csv_import/csv_import_operation_product_metafields）「商品バリエーションメタフィールド」（/admin/csv_import/csv_import_operation_product_variant_metafields）で一括インポート可
- **制約・注意**: 「メタフィールド」セクションが表示されない場合は「設定 > メタフィールド定義 > 商品」で定義を追加する
- **出典**: 商品管理.md, 商品を編集する.md, 03-products.md

## タグ
- **分類**: マスタ（商品の属性）
- **何か**: 商品に付与するタグ（例: ゆったり / クルーネック / 長袖）。
- **作成場所**: 商品作成・編集フォームのサイドバー「タグ」欄の追加ボタン。
- **主な項目**: タグ名（テキスト）。未設定時は「タグは付与されていません」と表示。
- **作用先**:
  - 商品詳細サイドバーに表示される
  - 商品一覧での絞り込み: **タグでの絞り込みUIは存在しない**（検索は商品コードのみ）
- **制約・注意**: 一覧・検索でタグを条件に使う手段は対象ファイル内に記載なし
- **出典**: 商品管理.md, 商品を作成する.md, 商品を編集する.md, 03-products.md

## 店舗受取バリエーション（店舗受取可能バリエーション一覧）
- **分類**: 連携設定（店舗受取対象の登録データ）
- **何か**: 店舗受取可能なバリエーションの一覧管理。画面サブタイトル原文「店舗受取可能バリエーション一覧」。
- **作成場所**: 商品管理 サブメニュー「店舗受取」（/admin/local_pickup_product_variants）>「バリエーションを追加する」→「バリエーションを選択する」ダイアログ。
- **主な項目**: ダイアログ内: 「SKUコードで検索する」テキストボックス /「絞り込みを追加」ボタン（押すと「商品コード」フィルターが追加される）/ テーブル列: バリエーション（商品名＋オプション値）・商品コード・SKU / チェックボックスで複数選択 /「キャンセル」「選択する」ボタン
- **作用先**:
  - 店舗受取: ここに追加したバリエーションが店舗受取可能になる（対象ファイルに記載の範囲では一覧管理のみ確認）

## CSVインポート実行履歴（商品グループ）
- **分類**: システム生成データ（実行履歴）＋ステータス群
- **何か**: CSVインポート（/admin/csv_import）の「商品グループ」5カテゴリ（商品 / 商品バリエーション / 商品画像 / 商品バリエーション画像 / カタログ）の実行記録。「検証」→「実行」の2段階で処理される。
- **作成場所**: 各カテゴリ画面（例: /admin/csv_import/csv_import_operation_products）>「新規インポート」→ CSVファイルをアップロード →「保存する」。入口は2か所: CSVインポートトップの「商品グループ」内各カテゴリ、または商品一覧の「インポート」ドロップダウン。
- **主な項目**:
  - CSVファイル（「ファイルを選択する」またはドラッグ&ドロップ）。「テンプレート」ボタンでGoogleスプレッドシートのテンプレートが開く
  - 「商品画像」「商品バリエーション画像」のみ: 画像の処理方法ラジオボタン — **「画像を追加する」（デフォルト）**=既存の画像を残したまま追加 / **「画像を上書きする」**=既存の画像を削除して置き換え
  - 「カタログ」カテゴリのみ: カタログ（ドロップダウン必須）+ ファイルをアップロード（必須）。列マッピング・プレビューステップはない
  - 詳細画面の項目: 検証ステータス / 実行ステータス / 検証成功（成功件数、例: 6個の商品）/ 検証失敗（件数。クリックで「検証失敗」画面 /{id}/validation_failure へ）
- **作用先**:
  - 商品・バリエーション・画像・カタログのデータ: 「実行する」ボタンをクリックするとシステムに反映される（検証完了・検証失敗0件が前提）
  - 一覧画面: 実行履歴（作成日 / 実行ステータス）が表示される
- **ステータス**:
  | 表記 | 意味 | 遷移 | 影響 |
  |:--|:--|:--|:--|
  | 検証ステータス（例: 完了） | CSVの検証結果 | 「保存する」後に検証が走る | 検証失敗があればCSV修正→「新規インポート」からやり直し |
  | 実行ステータス（例: 完了 / 成功 完了） | データ反映の結果 | 検証成功後「実行する」で反映 | **成功完了後は「実行する」ボタンが無効（disabled）になり、一度実行したインポートは再実行できない** |
- **制約・注意**:
  - CSVのカラム順・ヘッダー名はテンプレートに準拠必須。列の順番・ヘッダー名を変えると取り込みに失敗する
  - 商品グループの5カテゴリはすべてテンプレートが用意されている（テンプレートがないカテゴリでは「テンプレート」ボタンは非表示）
  - ⚠️未検証: 「商品」カテゴリのCSVテンプレートの具体的な列名・必須列の一覧
  - ⚠️未検証: 検証失敗詳細画面に表示される列構成（行番号・エラー理由等）
- **出典**: 商品をCSVで一括登録する.md, 商品管理.md, カタログ.md, 03-products.md

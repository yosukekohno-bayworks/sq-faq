## 12. つまずきポイントと用語集

SQを使い始めた人が「あれ、おかしい？」と感じやすいポイントと、「これってどういう意味？」と迷いやすい用語を一か所にまとめました。操作に行き詰まったときはまずこの章を確認してください。

---

### 12-1. 巻き戻せない操作

次の操作は、実行すると**取り消しができません**。慎重に確認してから進めてください。

| 操作 | 巻き戻せない理由 | 誤ったときの対処 |
|:--|:--|:--|
| 調整伝票の「実行」 | 確認ダイアログに「この処理は巻き戻すことができません。」と明示されている | 逆向きの調整伝票を新規作成して相殺する |
| 出荷実績の登録 | 登録後にボタンが無効化され、再登録・編集・削除が不可になる | 調整伝票で在庫数を修正する |
| 入荷実績の登録 | 登録後にボタンが無効化され、再登録・編集・削除が不可になる | 調整伝票で在庫数を修正する |
| 取置伝票の「処理済みとしてマークする」 | 確認ダイアログ後に実行され、取置中の在庫が解放される | 取置伝票を新規作成して再確保する |
| CSVインポートの「実行する」 | 「成功 完了」になると「実行する」ボタンが無効化されて同じインポートは再実行できない | 修正したCSVで新規インポートを実施する |
| 商品・ブランドの削除 | 削除は取り消し不可（ブランドのダイアログに「この処理は巻き戻すことができません。」と明示）| 商品はアーカイブなら「商品のアーカイブを解除する」で戻せる（ただし「下書き」に戻る）。ブランドは復元不可 |

> **ポイント**: 出荷実績・入荷実績の誤登録は調整伝票で数量を直すことになります。「登録したがキャンセルしたい」という操作はできません。

---

### 12-2. 移動伝票をキャンセルしたときの「行き止まり」

取り寄せ販売のフローで移動伝票をキャンセルすると、元の在庫依頼が「行き止まり」の状態になります。これはSQの仕様上の重要な注意点です。

**何が起きるか**

1. 移動伝票の「その他の操作」から「キャンセル」を選ぶと、確認ダイアログ（「移動伝票をキャンセルすると、この伝票に紐づく入荷予定および出荷指示もすべてキャンセルされます。」）が表示される
2. 実行すると、紐づく出荷指示・入荷指示がすべてキャンセルされる
3. 確保していた在庫区分「取置中」の在庫は「販売可能」に戻る
4. **元の在庫依頼は「確保済み」のまま残る**

**行き止まりになる理由**

在庫依頼のステータスが「確保済み」のとき、以下の操作がいずれもできません。

- 「在庫依頼をクローズする」→ グレーアウトして実行不可
- 移動伝票の再作成 → できない

つまり、移動伝票をキャンセルした後は、その在庫依頼をどの操作でも前に進められない状態になります。

**やり直し方**

同じ取り寄せをやり直す場合は、**在庫依頼を新規作成**してください。既存の確保済み在庫依頼を再利用する手段はありません。

---

### 12-3. 「確保済み」という言葉の二つの意味

SQの中で「確保済み」という言葉が**二つの異なる意味**で使われていました。混同しないよう整理します。

| 「確保済み」の使われ方 | 現在の表記 | 意味 |
|:--|:--|:--|
| 在庫依頼のステータス | **確保済み**（変更なし） | 引当て済みの在庫依頼の状態。在庫依頼画面の「確保済み」タブに表示される |
| 在庫区分（SKU詳細の列） | **取置中**（旧称が「確保済み」） | 取置・移動用に確保された在庫数のこと |

在庫依頼の「確保済み」はステータス名として変わっていません。一方、在庫区分の列名は「確保済み」から「取置中」に改称されました（同様に「確定済み」は「引当済み」に改称）。

「確保済みになったのに在庫数が変わっていない」と感じたら、12-4も合わせて確認してください。

---

### 12-4. 在庫依頼を「確保済み」にしても在庫数が動かない

よくある疑問です。これはSQの仕様どおりの動作です。

在庫依頼で「在庫を引当てる」を実行してステータスが「確保済み」になっても、この時点では**在庫数は一切変化しません**。

在庫数が実際に動くのは**移動伝票を作成した瞬間**です。

| 操作 | 在庫数への影響 |
|:--|:--|
| 在庫依頼を作成（確認待ち） | 変化なし |
| 「在庫を引当てる」を実行（確保済みへ） | 変化なし |
| 移動伝票を作成 | 移動元の「販売可能」が減り、「取置中」が増える。移動先に「入荷予定」が立つ |
| 出荷実績を登録 | 移動元の手持ちが減る |
| 入荷実績を登録 | 移動先の手持ちが増える |

---

### 12-5. 作成後に変更できない項目

以下の項目は、一度保存すると後から変更できません。設定時に慎重に入力してください。

**商品・販売設定**

| データ | 変更できない項目 | 回避策 |
|:--|:--|:--|
| 商品 | 商品コード（作成後変更不可） | 誤入力した場合は商品を削除して作り直す（削除は取り消し不可） |
| バリエーション | オプションの軸と種別（色・サイズなどの分類軸は商品作成時にしか設定できない） | 軸の変更は不可。商品を作り直す必要がある |
| 販売価格ルール | 通貨 | 別の通貨で新規ルールを作成する |
| ディスカウント | クーポンコード・テナント | 新規のディスカウントを作成する |
| ポイントキャンペーン | ポイント付与方法・対象の注文ポイント付与ルール | 新規のポイントキャンペーンを作成する |
| 会員ランクの初期ランク「Bronze」 | 達成条件（削除も不可） | 表示名のみ変更可能 |

**設定マスタ**

| データ | 変更できない項目 | 回避策 |
|:--|:--|:--|
| ブランド | 外部ID | 削除して再作成する（削除は取り消し不可） |
| ロケーショングループ | 「新しいロケーションを自動的に含める」の設定 | 新規グループを作成する |
| 翻訳ルール | 言語 | 別言語で新規ルールを作成する |

### 12-6. 発注伝票が作成できない（現時点の不具合）

発注管理の画面で発注伝票を作成しようとして「作成する」ボタンを押すと、「エラーが発生しました。しばらくしてから再度お試しください」というエラーが表示されて保存できません。

これはアプリ側の不具合とみられており、現時点では発注伝票の作成は行えません。発注を起点とした入荷指示の自動生成フローも、この不具合が解消されるまで確認できない状態です。

### 12-7. 画面はあるが現在使えない機能・未実装の機能

SQには、メニューに表示されているものの現時点で使えない機能があります。

| 機能 | 状態 |
|:--|:--|
| 分析（収益・レポート） | メニューには表示されるが、未実装 |
| 卸売 | メニューには表示されるが、現時点ではTODO表示で未実装 |
| 顧客マイル・マイル商品 | 権限グループの設定リストには「顧客マイル」「マイル商品」というリソースがあるが、対応する管理画面が存在しない（将来機能。ポイントとは別系統のCRM機能として実装予定とみられる） |
| お知らせ | 同様に権限リストにのみ存在し、画面がない |
| 下書き注文の「注文を作成する」 | ボタンが現在無効になっており、手動での注文作成はできない |
| 売上実績の手動作成 | 「作成する」ボタンが現在無効。売上実績は注文から自動生成される設計 |

### 12-8. ログインにパスワードはない

「パスワードを忘れた」「パスワードを設定したい」という操作を探している方へ。

SQにはパスワードによるログイン機能がありません。ログインは**メール認証（ワンタイムコード方式）**です。

ログイン画面でメールアドレスを入力すると、そのアドレス宛に6桁のコードが届きます。届いたコードをログイン画面に手入力してログインします。パスワードの設定・変更・リセット画面は存在しません。

---

### 12-9. フォームの選択肢が空になる

「テナントを選択したいのに何も表示されない」「取引先の選択肢が出ない」といった状況が起きることがあります。

これは、選択肢の元になるマスタが**1件も登録されていないとき**に起きる共通の挙動です。エラーメッセージは表示されず、ドロップダウンが空白のまま開きます。

選択肢が空のときは、対応するマスタを先に登録してください。

| 空になっている選択肢 | 登録が必要なマスタと場所 |
|:--|:--|
| テナント | 設定 > テナント |
| ロケーション | 設定 > ロケーション |
| ロケーショングループ | 設定 > ロケーショングループ |
| ブランド | 設定 > ブランド |
| 取引先 | 設定 > 取引先 |
| 決済方法 | 設定 > 決済方法 |

### 12-10. 商品一覧の「在庫」列はバリエーション数

商品一覧画面の「在庫」列に表示されている数字は、**実際の在庫数量ではなく、バリエーション（SKU）の登録数**です。

例: 「20個のバリエーション」と表示されている場合、その商品には20種類のバリエーションが登録されています。

各バリエーションの実際の在庫数量は、商品名をクリックして商品詳細を開き、バリエーション一覧から個別のバリエーション詳細画面にある「在庫管理」リンクから確認できます。

### 12-11. 主要用語集

SQで頻繁に登場する用語を五十音順で整理しました。初めて目にした用語はここで確認してください。

#### あ行

| 用語 | 説明 |
|:--|:--|
| アーカイブ | 商品や取引先を「見えない状態」にすること。削除と違い、解除すれば元の状態（商品は「下書き」）に戻せる |
| 引当済み（在庫区分） | 注文に引き当て済みの在庫数。旧称は「確定済み」 |
| 入荷指示 | 移動伝票から自動生成される、入荷を管理するデータ。移動先で入荷実績を登録する作業の単位 |
| 入荷実績 | 入荷指示に対して実際に入荷した記録。登録すると移動先の手持ちが増える。巻き戻し不可 |

#### か行

| 用語 | 説明 |
|:--|:--|
| カタログ | 商品をグループ化する入れ物。「Shopifyにどの商品を出すか」など、チャネルに連携する商品の範囲を決める |
| 確保済み（在庫依頼ステータス） | 在庫依頼で「在庫を引当てる」を実行した後の状態。この時点では在庫数は動かない |
| 管理メンバー | SQの管理画面にログインできるスタッフ。メール認証（6桁コード）でログインする |
| 権限グループ | 管理メンバーに割り当てる権限のセット。70種類（35リソース × 閲覧/編集）の権限をまとめたもの |

#### さ行

| 用語 | 説明 |
|:--|:--|
| 在庫依頼 | 「この店舗へ、ここから在庫をn個確保して送ってほしい」という依頼データ。取り寄せ販売の起点になる |
| 在庫区分 | 在庫の状態を細かく分類した種別。「販売可能」「引当済み」「取置中」「破損」「検品」「予備」「手持ち」「積送中」「入荷予定」の9区分がある |
| 在庫振替 | 同一商品内の別バリエーションへ在庫を移す操作。別商品への振替は不可 |
| 調整伝票 | 理由付きで在庫数を増減する伝票。マイナス在庫の修正にも使う。理由は「廃棄/見本/紛失/棚卸差異/その他」の5択 |
| 出荷指示 | 移動伝票・注文から自動生成される、出荷作業の単位。手動で作成する画面はない |
| 出荷実績 | 出荷指示に対して実際に出荷した記録。登録すると移動元の手持ちが減る。巻き戻し不可 |
| 積送中（在庫区分） | 出荷済みで移動先にまだ到着していない輸送中の在庫。移動先で計上され、手持ちには含まれない |
| SKU（最小管理単位） | バリエーションに紐づく在庫管理コード。在庫・各種ルール・CSV処理のキーになる |

#### た行

| 用語 | 説明 |
|:--|:--|
| 取置伝票 | 在庫を「取置中」区分として確保する伝票。保存した瞬間に在庫が動く |
| 取置中（在庫区分） | 取置・移動用に確保された在庫数。旧称は「確保済み」 |
| テナント | ブランド・部門などの運用単位を分ける管理単位。発注伝票・ディスカウント・ポイント/会員ランク・チャネル連携など、ほぼ全機能の紐づけ先になる |

#### な行

| 用語 | 説明 |
|:--|:--|
| 入荷予定（在庫区分） | 移動伝票作成から出荷実績登録前までの、移動先での入荷予定数。手持ちには含まれない |

#### は行

| 用語 | 説明 |
|:--|:--|
| 販売可能（在庫区分） | 実際に販売に使える在庫数。鉛筆アイコンで直接編集できる（入力値は加算ではなく絶対値になる） |
| 破損（在庫区分） | 品質問題により販売不可の在庫数。手持ちには含まれる |
| バリエーション | 商品の色・サイズなどのオプションを組み合わせた1パターン（例: 「白 / Mサイズ」）。SKUが紐づく単位 |
| 発注伝票 | 取引先への仕入れ依頼を管理する伝票 |
| 引当済み（在庫区分） | 注文に引き当て済みの在庫数。旧称は「確定済み」（「引当済み」を参照） |
| 手持ち（在庫区分） | 物理的に手元にある在庫の合計数。「販売可能 + 引当済み + 取置中 + 破損 + 検品 + 予備」の合計（積送中・入荷予定は含まない） |

#### ま行

| 用語 | 説明 |
|:--|:--|
| 移動伝票 | ロケーション間で在庫を移動する伝票。作成と同時に出荷指示・入荷指示が自動生成される |

#### ら行

| 用語 | 説明 |
|:--|:--|
| ロケーション | 在庫を持つ場所（倉庫・店舗）。在庫はすべて「SKU × ロケーション」の組み合わせで管理される |
| ロケーショングループ | ロケーションをまとめた束。Shopify連携などチャネル設定の必須項目 |
| 論理ロケーション（仮想店舗） | 物理的な場所を持たない、管理上の仮想的なロケーション。「店舗在庫のうちECで販売してよい数だけ」を管理する目的で、通常のロケーションと同じ手順（場所種別「店舗」）で作成する |

#### カテゴリ別クイックリファレンス

**在庫区分（9種）**

| 区分 | 手持ちに含む？ | 意味 |
|:--|:--|:--|
| 販売可能 | 含む | 売ってよい数 |
| 引当済み | 含む | 注文に引き当て済み（旧「確定済み」） |
| 取置中 | 含む | 取置・移動用に確保（旧「確保済み」） |
| 破損 | 含む | 販売不可（品質問題） |
| 検品 | 含む | 販売不可（検品中） |
| 予備 | 含む | 販売不可（予備確保） |
| 手持ち | — | 上記6区分の合計 |
| 積送中 | 含まない | 出荷済み・輸送中（移動先で計上） |
| 入荷予定 | 含まない | 移動伝票作成〜出荷前（移動先で計上） |

**伝票の種類**

| 伝票 | 何をする伝票か |
|:--|:--|
| 調整伝票 | 在庫数を理由付きで増減する |
| 取置伝票 | 在庫を「取置中」として確保する |
| 移動伝票 | ロケーション間で在庫を移す |
| 発注伝票 | 取引先への仕入れを管理する（現時点で作成不可の不具合あり） |

**「巻き戻せない」操作まとめ**

調整伝票の「実行」／出荷実績の登録／入荷実績の登録／取置伝票の「処理済みとしてマークする」／CSVインポートの「実行する」／商品・ブランドの削除

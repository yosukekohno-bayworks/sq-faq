## 11. はじめ方 — 何をどの順番で設定するか

この章では、SQをゼロから動かせる状態にするための設定順と、各設定がなぜその順番でなければならないかを説明します。「先に作ったデータが後の設定の選択肢になる」という仕組みを理解すれば、迷わずセットアップを進められます。

---

### SQの設定が「順番通り」でないといけない理由

SQの設定は、**後で設定するものが前に作ったデータを選択肢として参照する**つくりになっています。

たとえば「ロケーション（倉庫・店舗）」を登録せずに在庫を入れようとしても、場所を選べないので何もできません。同様に、「カタログ」を作っていない状態でShopifyを接続しようとしても、フォームの選択肢が空のまま保存できません。

順番を間違えると「選択肢に何も出てこない」状態になります。このページの順番どおりに進めれば、その詰まりを回避できます。

---

### 全体像 — 5つのフェーズ

| フェーズ | やること | 備考 |
|:--|:--|:--|
| 0 | 管理メンバーと権限グループ | 1人で使い始めるならスキップ可 |
| 1 | マスタ登録（テナント・ロケーションなど） | 最重要。ここの漏れが後の全工程に影響する |
| 2 | 商品・バリエーション・カタログ・在庫数 | 商品数による |
| 3 | 販売ルール（価格・販売上限など） | チャネル接続前に価格ルールだけ作っておくと楽 |
| 4 | Shopify・スマレジ・リテールポータルの接続 | フェーズ1〜3が前提 |
| 5 | 注文・出荷・取り寄せの日常運用 | — |

CRM（ポイント・会員ランク・ディスカウント）はフェーズ1の後、任意のタイミングで設定できます。

---

### フェーズ0: メンバーと権限（複数人で使う場合のみ）

1人だけで使い始める場合はこのフェーズをスキップして構いません。複数人で使う場合は、最初に済ませておきます。

**なぜ権限グループを先に作るか:** 管理メンバーの追加フォームで権限グループを選ぶ必要があるため、グループが先に存在していないと選択肢が空になります。

1. 権限を制限したいメンバーがいる場合は「権限グループ」を先に作成する
2. 「管理メンバーを追加する」（追加時に権限グループを選ぶ）

> ログインはパスワードではなく、メールアドレスに届く認証コードで行います。

#### 権限グループについて

権限グループは、35種類のリソースそれぞれに「閲覧」「編集」を設定した70権限のセットです。新しく作ったグループは最初すべてOFF（何も見えない状態）から始まります。グループを作っただけでは機能しません。メンバーへの割り当てが必要です。

初期から用意されている「特権管理者」グループはすべての70権限を持ちます。名前以外は変更・削除できません。

---

### フェーズ1: マスタ登録 — すべての土台

**マスタとは:** 他のすべての画面の「選択肢」になるデータです。必要なマスタが登録されていないと、参照先フォームのドロップダウンには何も表示されません（エラーメッセージも出ません）。次の順番で登録してください。

| 順番 | マスタ | 必須度 | なぜ先に必要か |
|:--|:--|:--|:--|
| 1 | テナント | 必須（通常は開設時に作成済み） | 発注・ディスカウント・チャネル連携のすべてがテナントに紐づく |
| 2 | ロケーション（倉庫・店舗） | 必須 | 在庫はすべて「どのロケーションに何個」で記録される。これがないと在庫を持てない |
| 3 | ロケーショングループ | チャネル接続するなら必須 | Shopify連携・OmnibusCore連携の必須入力項目 |
| 4 | 取引先 | 発注機能を使うなら必須 | 発注伝票の必須入力項目 |
| 5 | 決済方法 | 店舗販売（POSレジ・リテールポータル）を使うなら必須 | 注文の支払い手段 |
| 6 | ブランド / 販売員 / 通知用メールアドレス / 納品書テンプレート | 任意 | ブランド=商品登録時の分類属性（一部の自動追加条件にも利用可）、販売員=リテールポータルでの閲覧者・客注作成者の記録 |

#### ロケーション登録時に注意すべき2つの設定

ロケーションには、後の機能に直結する2つのチェックボックスがあります。設定時に意識して設定しておくと、後から「なぜ動かない？」を防げます。

- 「**在庫依頼を受け付ける**」… 取り寄せ販売でこの店舗にリクエストを送るならON。これがOFFの店舗には在庫リクエストが届きません
- 「**店舗受け取りを有効にする**」… 店舗受取（お客様が注文後に店舗で受け取るBOPIS）の受取店舗にするならON

#### テナントについて

テナントはブランド・部門などの運用単位を分けるための管理単位で、ほぼ全機能の紐づけ先になります。テナントが0件だと、発注伝票・ディスカウント・Shopify/OmnibusCore/スマレジ/リテールポータル連携・CSVエクスポート・ポイント一括加算の各フォームで選択肢が空になり、保存できません。通常は開設時に作成済みですので、まず「設定 > テナント」で確認してください。商品マスタの「ブランド」とは別データです。

---

### フェーズ2: 商品とカタログ

1. **商品を作成する**
   - バリエーション（カラー・サイズなどのオプション軸）と「種別」は**商品の作成時にしか設定できません**。後からオプション軸を増やせないため、色・サイズ展開は最初に決め切ってから作成してください
2. バリエーションごとの詳細（上代・SKU・在庫追跡の設定）を登録する
3. **カタログを作成して商品を入れる**
   - チャネル接続（Shopifyなど）をするなら必須です。Shopify連携は「どのカタログの商品を出すか」で管理するためです
4. **在庫数を設定する** — SKU詳細の「販売可能数を編集」か、CSVで一括更新
5. 商品ステータスを「公開中」にする

> 在庫区分の名称: SQでは在庫の状態を「販売可能 / 引当済み / 取置中 / 破損 / 検品 / 予備 / 手持ち / 積送中 / 入荷予定」と呼びます。

---

### フェーズ3: 販売ルール

販売ルールは4種類あります。チャネル接続の前後で「作れるか否か」が変わるものがあるため、注意が必要です。

| ルール | 接続前に作れるか | いつ使うか |
|:--|:--|:--|
| 販売価格 | 作れる（推奨） | チャネル接続フォームで指定できるため、先に作っておくと楽 |
| 予約販売 | 作れる | 在庫0でも注文を受け続けたい商品があるとき |
| 販売閾値 | 作れる | 在庫が少なくなったら販売を止める運用をするとき |
| 販売上限 | **作れない** | チャネル接続後でないと選択肢が出ない（チャネル選択が必須のため） |

#### 販売価格ルールの注意点

販売価格ルールは作成時に「通貨」を選びます。**通貨は保存後に変更できません**。別の通貨が必要な場合は新しいルールを作成してください。

---

### フェーズ4: チャネル接続

接続フォームの必須項目は、ここまでのフェーズで作ったものです。接続前に揃っているか確認してください。

| 接続するもの | 事前に必要なもの（必須） |
|:--|:--|
| Shopify | テナント・カタログ・ロケーショングループ（＋Shopifyストア側へのコネクターアプリのインストール） |
| OmnibusCore | テナント・メーカーコード |
| スマレジ | テナント・契約ID（＋スマレジ側にSQアプリのインストール） |
| リテールポータル | 店舗ロケーション・在庫ロケーション（倉庫）・テナント・カタログ。接続後に「ユーザーを追加する」を忘れずに |

#### 取り寄せ販売（店舗在庫のEC販売）をやる場合

通常のチャネル接続に加えて、以下の追加設定が必要です。

1. 「店舗在庫EC販売用」のような論理ロケーション（実在しない仮想の場所として使うロケーション）を店舗種別で作成する
2. そのロケーションのSKUにECで販売してよい数を設定する
3. リクエストを受ける実店舗の「在庫依頼を受け付ける」をONにする
4. ロケーショングループにまとめてShopify連携に設定する

（Shopify接続後の実際の引当・在庫リクエスト変換の挙動は、接続後でないと確認できないため、ここでは仕様の説明にとどめます）

#### Shopify連携の主な設定項目

| 設定項目 | 効果 |
|:--|:--|
| カタログ | Shopifyに出す商品の範囲 |
| ロケーショングループ | 在庫の引当元と在庫リクエストの送付範囲 |
| 販売価格ルール（任意） | 指定したルールでShopifyの販売価格が決まる |
| 注文による在庫変動を起こさない = ON | Shopify経由の注文が来てもSQの在庫を動かさない |

#### スマレジ連携の注意

スマレジ連携では「在庫同期の方向」を「スマレジからSQへ」「SQからスマレジへ」「在庫を同期しない」の3択から選べます。在庫同期の実際の反映タイミング・頻度・エラー時の挙動は、スマレジ接続後でないと確認できないため、ここでは仕様の説明にとどめます。

---

### フェーズ5（任意）: CRM — ポイント・会員ランク・ディスカウント

CRMは**設定の参照関係が逆順**になっているため、この順で作るとスムーズです。

1. 「会員ランク算出ルール」を先に作る
2. 「ポイント付与ルール」を作る（フォームで会員ランク算出ルールを選ぶため、1が先）
3. ポイントキャンペーン・誕生日ポイント・失効通知などを必要に応じて設定する
4. 「ディスカウント」はいつでも独立して作成可能

---

### 最短セットアップ・チェックリスト

「とりあえず動く状態」までの最小限はこれだけです。

**基本（SQ単体で在庫を管理するだけ）**

- [ ] テナントを確認した（設定 > テナント）
- [ ] ロケーションを登録した（最低1つ。倉庫または店舗）
- [ ] 商品を1つ作成した（バリエーション・上代・SKUまで）
- [ ] 在庫数を入れた（SKU詳細の「販売可能数を編集」）
- [ ] 商品ステータスを「公開中」にした

**チャネルと繋ぐ場合は、さらに**

- [ ] カタログを作成して商品を入れた
- [ ] ロケーショングループを作成した
- [ ] 販売価格ルールを作成した
- [ ] チャネルを接続した

---

### 目的別ショートカット

やりたいことが決まっている場合は、必要な設定だけを最短で進めます。

| やりたいこと | 必要な設定（上から順に） |
|:--|:--|
| ECサイト（Shopify）と繋ぎたい | マスタ → 商品＋カタログ → ロケーショングループ → Shopify接続 |
| 店舗在庫をECで売りたい（取り寄せ） | 上記 ＋ 論理ロケーション ＋ 店舗の「在庫依頼を受け付ける」ON |
| 店舗間・倉庫間で在庫を動かしたい | ロケーション2つ以上 ＋ 商品・在庫 |
| 店舗スタッフに画面を渡したい | リテールポータル接続 ＋ ユーザー追加（＋権限グループ） |
| ポイント制度を始めたい | 会員ランク算出ルール → ポイント付与ルール |
| 仕入れ（発注）を管理したい | 取引先マスタ → 発注伝票 |
| 商品・在庫をまとめて登録したい | マスタ → CSVインポート |

---

### 設定後に変更できない項目（作成前に確認を）

作り直しが必要になる前に、以下の項目は最初の設定時に確定させてください。

| 設定 | 変更できない項目 | 対処法 |
|:--|:--|:--|
| ブランド | 外部ID | 削除して再作成 |
| ロケーショングループ | 「新しいロケーションを自動的に含める」 | 新しいグループを作成 |
| 翻訳ルール | 言語 | 新規ルールを作成 |
| 販売価格ルール | 通貨 | 新規ルールを作成 |
| 商品 | バリエーションの軸（カラー・サイズなどのオプション）・種別 | 商品を作り直す |

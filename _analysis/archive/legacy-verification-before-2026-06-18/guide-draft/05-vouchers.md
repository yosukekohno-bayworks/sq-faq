## 4. 伝票のしくみ — 調整・取置・移動・発注・入荷・出荷

この章では、SQで扱う6種類の伝票それぞれの役割と、伝票を使ったときに在庫がどう変わるかを説明します。「どの場面でどの伝票を使えばよいか」がわかるようになることを目標にしています。

---

### 4-1. 伝票とは何か

**伝票**とは、在庫の増減・移動・確保・仕入れといった操作を記録する「指示書」のことです。SQでは操作を伝票として残すことで、「いつ・誰が・何を・なぜ動かしたか」を後から確認できます。

SQには次の6種類の伝票があります。

| 伝票の種類 | 何をするか | 在庫への主な影響 |
|:--|:--|:--|
| 調整伝票 | 棚卸し差異・廃棄・紛失などで在庫数を手動で増減する | 「実行」後に増減数がそのまま反映（巻き戻し不可） |
| 取置伝票 | 特定の用途のために在庫を確保しておく | 作成した瞬間に対象在庫が「取置中」になる |
| 移動伝票 | 倉庫から店舗など、ロケーション間で在庫を動かす | 作成した瞬間に移動元が「取置中」、移動先が「入荷予定」に |
| 発注伝票 | 仕入先（取引先）への発注を記録する | ※現在、不具合により新規作成できません（後述） |
| 入荷指示 | 移動伝票から自動生成される。入荷を記録する | 「入荷実績を登録する」で移動先の手持ちが増える（巻き戻し不可） |
| 出荷指示 | 移動伝票から自動生成される。出荷を記録する | 「出荷実績を登録する」で移動元の手持ちが減る（巻き戻し不可） |

---

### 4-2. 調整伝票 — 在庫数を手で増減する

#### どんなときに使うか

- 棚卸しをして実数と帳簿が合わなかった
- 商品を廃棄した・紛失した
- 見本として使ったので在庫から引いておきたい
- マイナス在庫（在庫が0を下回った状態）を修正したい

#### 理由の5種類

調整伝票を作るとき、**なぜ増減するか**の理由を次の5種類から選びます（必須）。

| 理由 | 使う場面の例 |
|:--|:--|
| 廃棄 | 商品を処分した |
| 見本 | 展示・試着用として使った |
| 紛失 | 在庫が見当たらない |
| 棚卸差異 | 棚卸しで実数と帳簿が一致しなかった |
| その他 | 上記に当てはまらないとき |

> 「破損」「検品」「予備」は調整伝票の理由の選択肢にはありません。

#### 作成と実行の流れ

1. 「在庫管理 > 調整伝票」画面で「調整伝票を作成する」を選ぶ
2. **ロケーション**・**理由**・**商品（増減数）** の3項目を入力して「**保存する**」を押す
3. 伝票のステータスが「未実施」になる（この時点ではまだ在庫は変わらない）
4. 内容を確認したら「**実行**」ボタンを押す
5. 確認ダイアログ「在庫調整を実行する」が表示されるので「**実行する**」を押す
6. ステータスが「実施済み」に変わり、在庫に即時反映される

> **注意**: 未保存の変更があると「実行」ボタンがグレーアウトして押せません。先に「**保存する**」を押してから「**実行**」を押してください。

#### ステータス遷移

```
作成・保存 → [未実施] → 「実行」→ 確認ダイアログ「実行する」 → [実施済み]
                 ↓
           「キャンセル」 → [キャンセル]
```

| ステータス | 在庫への影響 | 編集できるか |
|:--|:--|:--|
| 未実施 | まだ変わらない | ロケーション・理由・商品・増減数を変更できる |
| 実施済み | 増減数がそのまま反映。**巻き戻し不可** | すべて読み取り専用 |
| キャンセル | なし | — |

> 実行後の取り消しはできません。「この処理は巻き戻すことができません。」と確認ダイアログに明記されています。やり直したい場合は、反対方向の増減数で新しい調整伝票を作成してください。

### 4-3. 取置伝票 — 在庫を「取置中」に確保する

#### どんなときに使うか

- 店舗での取置（お客さまのために商品をとっておく）
- 特定の用途のために一時的に在庫を止めておきたい

#### 作成と処理の流れ

1. 「在庫管理 > 取置伝票」画面で「取置伝票を作成する」を選ぶ
2. **ロケーション**・取置する**商品**を入力する（メモ欄は任意）
3. 「**保存する**」を押す
4. **この瞬間、対象在庫が「取置中」の区分に移動する**（販売可能から引かれる）
5. 伝票のステータスが「未処理」になる
6. 取置が終わったら詳細画面の「**処理済みとしてマークする**」を押す
7. 確認ダイアログ「処理済みとしてマークする」が表示されるので「**実行する**」を押す
8. 取置中の在庫が解放され、販売可能に戻る。ステータスが「処理済み」になる

#### 在庫の動き

| タイミング | 在庫の変化 |
|:--|:--|
| 取置伝票を保存したとき | 対象商品が「販売可能」から「取置中」へ移る |
| 「処理済みとしてマークする」を実行したとき | 「取置中」が解放され、「販売可能」へ戻る |

#### ステータス遷移

```
作成・保存 → [未処理]（在庫が「取置中」になる）
                 ↓
       「処理済みとしてマークする」 → 確認ダイアログ「実行する」
                 ↓
           [処理済み]（取置中が解放され販売可能へ戻る）
```

> 「処理済みとしてマークする」を実行すると確認ダイアログに「**取置伝票を処理済みにすることで、取置中の在庫が解放されます。**」と表示されます。実行後は元の「未処理」に戻すことはできません。

### 4-4. 移動伝票 — ロケーション間で在庫を移す

#### どんなときに使うか

- 物流倉庫から店舗へ在庫を送る
- 店舗から別の店舗へ在庫を移す
- 取り寄せ販売の注文を受けて、在庫のある拠点から別の拠点へ在庫を移動させる

#### 2つの作られ方

移動伝票には手動で作る方法と、フローから自動で作られる方法があります。

**1. 単体作成（手動）**
「在庫管理 > 移動伝票」画面で「移動伝票を作成する」から作成します。

**2. 取り寄せ販売フロー経由（自動作成）**
在庫依頼を引当てた後、確保済み一覧から「**移動伝票を作成する**」ボタンを押すと自動生成されます。この場合、伝票の詳細画面に次の注意文が表示され、商品の追加・削除などの編集はできません。

> 「この移動伝票は外部システムから連携されているため、編集（商品の追加や削除など）は制限されています。連携元システムの管理画面より操作を行ってください。」

#### 作成時に起きること（重要）

移動伝票を作成した瞬間、次の3つのことが同時に起きます。

1. **移動元の在庫**: 「販売可能」から「取置中」へ移る（手持ち数は変わらない）
2. **移動先の在庫**: 「入荷予定」が増える（まだ手持ちには入らない）
3. **出荷指示と入荷指示が自動生成される**（出荷管理・入荷管理に通知バッジが出る）

#### ステータス遷移

移動伝票自体にステータスを進めるボタンはありません。出荷管理・入荷管理での実績登録に連動して自動的にステータスが変わります。

```
作成直後 → [出荷作業]
               ↓ 出荷実績を登録する
          [一部受領済み]（一部の入荷実績が登録された状態）
               ↓ 残りの入荷実績を登録する
          [受領済み]（すべての入荷実績が登録された状態）
```

<!-- TODO: 要確認（「一部受領済み」「受領済み」への正確なステータス遷移条件） -->

| ステータス | 在庫の状態 |
|:--|:--|
| 出荷作業 | 移動元: 取置中 / 移動先: 入荷予定 |
| （出荷実績登録後） | 移動元: 手持ち減少 / 移動先: 入荷予定→積送中 |
| （入荷実績登録後） | 移動先: 積送中→販売可能・手持ち増加 |

#### キャンセルするとき

伝票詳細の「**その他の操作**」メニュー内に「**キャンセル**」があります。キャンセルすると確認ダイアログが表示されます。

> 「移動伝票をキャンセルすると、この伝票に紐づく入荷予定および出荷指示もすべてキャンセルされます。」

キャンセル後の変化:

- 移動伝票のステータス: 「キャンセル」
- 確保していた在庫: 「取置中」から「販売可能」へ戻る
- 入荷指示: 入荷管理の「キャンセル」タブに「入荷キャンセル」として残る
- 出荷指示: 出荷管理の一覧タブには表示されなくなる

> **注意（取り寄せ販売フロー経由の伝票をキャンセルした場合）**: 元の在庫依頼が「確保済み」のまま残り、そこから移動伝票を再作成することも在庫依頼をクローズすることもできなくなります。同じ取り寄せをやり直す場合は、在庫依頼を新規作成してください。

### 4-5. 発注伝票 — 仕入先への注文を記録する

#### どんなときに使うか

- メーカーや問屋など、仕入先（取引先）に商品を発注するときに使います
- 発注する商品・数量・単価・税率を伝票として残し、金額を管理します

#### 発注伝票の作成フォームの主な項目

| 項目 | 説明 |
|:--|:--|
| 取引先 | 仕入先。「設定 > 取引先」で事前に登録しておく必要がある |
| テナント | 発注を紐付ける事業部 |
| 通貨 | 米ドル・ユーロ・日本円・タイバーツ・シンガポールドルから選べる（初期値: 日本円） |
| 商品 | 発注するバリエーション（SKU）を検索して追加する |
| 単価・数量・税率 | 明細行に入力。金額は単価×数量の税抜きで自動計算 |

#### 現在の状況（重要）

> **現在、発注伝票の新規保存はできません。** 「作成する」ボタンを押すとシステム側のエラーが発生し、保存が失敗します。アプリ側の不具合とみられます。修正されるまでの間、発注伝票の新規作成はご利用いただけません。

### 4-6. 出荷指示と出荷実績 — 在庫を送り出す

#### 出荷指示とは

移動伝票を作成すると、**出荷指示が自動生成**されます（手動での新規作成はできません）。出荷指示は「在庫を出荷する」という作業の指示書です。

出荷管理画面（「在庫管理 > 出荷管理」）でステータス別に一覧確認できます。

#### ステータスタブ

| タブ名 | 説明 |
|:--|:--|
| 保留中 | 出荷作業を保留している出荷指示 |
| 出荷待ち | 出荷作業を待っている出荷指示 |
| 依頼済み | 出荷依頼を済ませた出荷指示 |
| 作業中 | 出荷作業中の出荷指示 |
| 欠品・要対応 | 欠品など対応が必要な出荷指示 |
| 出荷完了 | 出荷実績が登録された出荷指示 |

※ キャンセルタブは出荷管理の一覧には存在しません。移動伝票がキャンセルされた出荷指示は一覧から消えます。確認が必要な場合は「条件指定でエクスポート」で出荷作業ステータスを「キャンセル済み」に指定してエクスポートしてください。

#### 出荷実績を登録する

1. 出荷指示の詳細画面を開く
2. 右上の「**出荷実績を登録する**」ボタンを押す
3. ダイアログで配送キャリア・追跡コードを入力する（どちらも任意。空でも登録できる）
4. 「**登録する**」を押す
5. ステータスが「出荷完了」に変わり、移動元の**手持ち在庫が減る**

> **注意**: 出荷実績の登録後は巻き戻せません。登録後は「出荷実績を登録する」ボタンが無効化され、画面からの再登録・編集・削除はできません。

### 4-7. 入荷指示と入荷実績 — 在庫を受け取る

#### 入荷指示とは

移動伝票を作成すると、出荷指示と同時に**入荷指示も自動生成**されます（手動での新規作成はできません）。入荷指示は「在庫を受け取る」という作業の指示書です。

入荷管理画面（「在庫管理 > 入荷管理」）でステータス別に一覧確認できます。一覧の「種別」列には「移動」と表示されます。

#### ステータスタブ

| タブ名 | 説明 |
|:--|:--|
| 入荷待ち | 入荷作業を待っている入荷指示 |
| 入荷依頼済み | 入荷依頼を済ませた入荷指示 |
| 入荷作業中 | 入荷作業中の入荷指示 |
| 要対応 | 対応が必要な入荷指示 |
| 入荷完了 | 入荷実績が登録された入荷指示 |
| キャンセル | キャンセルされた入荷指示 |

#### 入荷実績を登録する

1. 入荷指示の詳細画面を開く
2. 右上の「**入荷実績を登録する**」ボタンを押す
3. 確認ダイアログに「この処理は巻き戻すことができません。」と表示される
4. 「**登録する**」を押す
5. 入荷指示のステータスが「入荷完了」に変わる
6. 紐づく移動伝票も自動的に「入荷完了」になる
7. 移動先の**手持ち在庫が増える**

> **注意**: 入荷実績の登録後は巻き戻せません。登録後は「入荷実績を登録する」ボタンが無効化され、画面からの再登録・編集・削除はできません。

### 4-8. 在庫の動きを一表で確認する

移動伝票を中心とした在庫の動きを時系列でまとめます。

| 操作 | 移動元の変化 | 移動先の変化 |
|:--|:--|:--|
| 取置伝票を作成 | 販売可能 −n、取置中 +n（手持ち不変） | — |
| 「処理済みとしてマークする」 | 取置中 −n → 販売可能 +n（解放） | — |
| 移動伝票を作成 | 販売可能 −n、取置中 +n（手持ち不変） | 入荷予定 +n |
| 出荷実績を登録 | 取置中 −n、手持ち −n | 入荷予定 −n → 積送中 +n |
| 入荷実績を登録 | — | 積送中 −n → 販売可能 +n、手持ち +n |
| 移動伝票をキャンセル | 取置中 → 販売可能に戻る | — |
| 調整伝票を実行 | 増減数がそのまま反映（手持ちも変わる） | — |

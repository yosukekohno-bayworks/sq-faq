## 10. 道具と土台 — CSV/PDF・権限・メンバー・カスタムデータ

SQを使い始めるとき、最初に「誰が使えるか」「どんなデータをどこへ入れるか」を決める必要があります。この章では、データの一括操作に使うCSV/PDFのエクスポート・インポート機能と、管理メンバーの権限設定、そして商品や注文に独自の項目を追加するカスタムデータの仕組みを説明します。

---

### CSVインポート — 一括登録・更新の入口

#### 対応カテゴリ（22種）

CSVインポート（画面左メニューから「CSVインポート」で開く）では、22種類のデータを一括で登録・更新できます。カテゴリはグループ別に整理されています。

**商品グループ**

| カテゴリ | 何ができるか |
|:--|:--|
| 商品 | 商品を一括で登録・更新できます |
| 商品バリエーション | 商品バリエーション（SKU）を一括で登録・更新できます |
| 商品画像 | 商品画像を一括で追加できます |
| 商品バリエーション画像 | 商品バリエーション画像を一括で追加できます |
| カタログ | カタログへの商品の一括追加・削除ができます |

**価格グループ**

| カテゴリ | 何ができるか |
|:--|:--|
| 販売価格 (通常) | 通常区分の販売価格をSKU単位で一括設定できます |
| 販売価格 (セール) | セール区分の販売価格をSKU単位で一括設定できます |
| 原価 | SKUごとの原価を一括設定できます |

**販売グループ**

| カテゴリ | 何ができるか |
|:--|:--|
| 販売閾値 | 販売閾値ルールごとにSKUと閾値を一括設定できます |

**在庫グループ**

| カテゴリ | 何ができるか |
|:--|:--|
| 販売可能在庫 | SKUとロケーションを指定して在庫数を一括設定できます |
| 予約販売 | SKUごとの予約販売数量を一括設定できます |
| 在庫移動伝票 | 在庫移動伝票を一括で作成できます |

**メタフィールドグループ**

| カテゴリ | 何ができるか |
|:--|:--|
| 商品 | 商品メタフィールドを一括追加できます |
| 商品バリエーション | 商品バリエーションメタフィールドを一括登録・更新できます |

**出荷グループ**

| カテゴリ | 何ができるか |
|:--|:--|
| ヤマトB2クラウド | ヤマトB2クラウドのCSVで出荷実績を一括登録できます |
| DHL | DHLのCSVで出荷実績を一括登録できます |

**ポイントグループ**

| カテゴリ | 何ができるか |
|:--|:--|
| 一括加算 | 顧客へのポイントを一括加算できます |
| 一括減算 | 顧客のポイントを一括減算できます |
| キャンペーン対象商品 | キャンペーンの対象商品を一括設定できます |

**会員ランクグループ**

| カテゴリ | 何ができるか |
|:--|:--|
| 基準ランク | 顧客の基準ランクを一括設定できます |

**各種マスターグループ**

| カテゴリ | 何ができるか |
|:--|:--|
| 管理ユーザー | 管理ユーザーを一括登録・更新できます |
| ロケーション | ロケーション（在庫拠点）を一括登録・更新できます |

#### テンプレートの使い方

多くのカテゴリにはGoogleスプレッドシート形式のテンプレートが用意されています。カテゴリの画面を開くと「テンプレート」ボタンが表示され、クリックすると別タブでテンプレートが開きます。

テンプレートが用意されていないカテゴリは次の6種です。

- 販売価格 (通常)
- 原価
- 販売閾値
- DHL
- キャンペーン対象商品
- 基準ランク

CSVのカラム順やヘッダー名はテンプレートに合わせる必要があります。列を自由に指定するUIはありません。

#### インポートフォームのパターン

カテゴリによってフォームの構成が4パターンに分かれています。

**パターンA — ファイルのみ**

CSVファイルをアップロードして「保存する」を押します。

該当カテゴリ: 商品バリエーション / 商品メタフィールド / 商品バリエーションメタフィールド / ヤマトB2クラウド / DHL / 管理ユーザー / ロケーション

**パターンB — 処理方法の選択 + ファイル**

ラジオボタンで処理方法を選んでからCSVファイルをアップロードします。

| カテゴリ | 選択肢（デフォルト → 変更可） |
|:--|:--|
| 商品画像 | 「画像を追加する」（デフォルト） / 「画像を上書きする」 |
| 商品バリエーション画像 | 「画像を追加する」（デフォルト） / 「画像を上書きする」 |
| 販売可能在庫 | 「絶対値で反映する」（デフォルト） / 「差分値で反映する」 |

「販売可能在庫」の違いは画面内にこう説明されています。「現在の在庫数が10のとき、CSVで20を指定すると、絶対値で反映した場合の在庫数は20になります。差分値で反映した場合の在庫数は30になります。」

**パターンC — ルール選択 + ファイル**

対象のルールをドロップダウンで選んでからCSVファイルをアップロードします。

| カテゴリ | フォームのラベル |
|:--|:--|
| 販売価格 (通常) | 販売価格ルール |
| 販売価格 (セール) | 販売価格ルール |
| 販売閾値 | 販売閾値ルール |
| 予約販売 | 予約販売ルール |
| 基準ランク | テナント |

**パターンD — 検索型の選択 + ファイル**

「ポイントキャンペーン」フィールドの「検索」ボタンを押してダイアログから対象を選んでからCSVファイルをアップロードします。

該当カテゴリ: キャンペーン対象商品

#### ポイント一括加算の固有項目

「一括加算」カテゴリのフォームには、次の2項目が追加されています。

| 項目 | 説明 | 選択肢 |
|:--|:--|:--|
| テナント | 加算対象のテナントを選択する | セレクトボックス（「選択してください」） |
| 顧客IDの種別 | CSV内の顧客識別子の種類を指定する | 「SQが発番した顧客ID」 / 「外部システムが発番した顧客ID」 |

どちらの項目も画面上に必須マーク（*）は表示されていませんが、未選択のまま「保存する」を押すとバリデーションエラーが表示されます（実質必須）。

なお、顧客管理画面の「インポート」ドロップダウン内に「ポイント一括付与」という項目がありますが、これはCSVインポート画面の「一括加算」と同じ機能です（表記がアプリ内で異なっています）。

#### 検証 → 実行の2段階処理

CSVインポートは「検証」と「実行」の2段階で動きます。

1. CSVをアップロードして「保存する」を押すと、まず検証が走ります
2. 検証が完了すると、実際のデータ反映（実行）が行われます
3. 結果は一覧画面の実行履歴から確認できます

実行履歴の行をクリックすると詳細を確認できます。

| 項目 | 説明 |
|:--|:--|
| 作成日 | インポートを実行した日付 |
| 検証ステータス | CSVの内容を検証した結果（例: 完了） |
| 実行ステータス | 実際のデータ反映の結果（例: 完了） |
| 検証成功 | 検証に成功した件数（例: 6個の商品） |
| 検証失敗 | 検証に失敗した件数。クリックで失敗詳細画面へ移動できる |

検証で失敗した行がある場合、失敗詳細画面（見出し「検証失敗」）でエラーの内容を確認できます。成功した行だけ実行に進む仕組みかどうかは、詳細画面の検証結果から判断してください。

### CSVエクスポート — データを取り出す

CSVエクスポート画面では、在庫・価格・売上実績・ポイント変動履歴など9種類のデータをCSV形式でダウンロードできます。

#### エクスポートできるカテゴリ（9種）

**マスターデータグループ**

| カテゴリ | 何をエクスポートするか |
|:--|:--|
| 在庫 | 指定したロケーションの在庫数 |
| ロケーション | 指定したロケーショングループに含まれるすべてのロケーション |
| ディスカウントの利用履歴 | 指定したディスカウントの利用履歴 |
| 商品バリエーション | 商品バリエーション一覧 |

**価格グループ**

| カテゴリ | 何をエクスポートするか |
|:--|:--|
| セール価格 | 指定した販売価格ルールのセール価格 |

**出荷グループ**

| カテゴリ | 何をエクスポートするか |
|:--|:--|
| ヤマトB2クラウド | ヤマトB2クラウドに取り込む出荷指示 |

**実績グループ**

| カテゴリ | 何をエクスポートするか |
|:--|:--|
| 売上実績（注文軸） | 指定した期間の注文軸の売上実績 |

**ポイントグループ**

| カテゴリ | 何をエクスポートするか |
|:--|:--|
| ポイント変動履歴 | 指定した期間のポイント変動履歴 |

#### 主なフォーム項目

**在庫エクスポート**

「選択」ボタンを押してロケーション選択ダイアログから対象ロケーションを選びます（複数選択可）。ダイアログはタブ「すべて」「店舗」「倉庫」の3種に分かれています。ロケーションを選ばないとエクスポートが実行されません。

**ロケーションエクスポート**

「ロケーショングループ」が必須（* マーク付き）です。「選択」ボタンで対象グループを選びます。

**ディスカウントの利用履歴エクスポート**

「ディスカウント」が必須（* マーク付き）です。「選択」ボタンで対象を選びます。

**商品バリエーションエクスポート**

「商品情報を含める」チェックボックス（デフォルト: 未チェック）をONにすると、CSVに商品コード・商品タイトル・ブランド名などの商品情報列が追加されます。

**セール価格エクスポート**

「販売価格ルール」が必須（* マーク付き）です。ドロップダウンから対象ルールを選びます。

**売上実績（注文軸）エクスポート / ポイント変動履歴エクスポート**

どちらも「テナント」「開始日時」「終了日時」が必須（* マーク付き）です。

#### ヤマトB2クラウドCSVについて

CSVエクスポート画面の「ヤマトB2クラウド」は実行履歴の確認専用です。ヤマトB2クラウド向けのCSVは、出荷管理画面の「条件指定でエクスポート」→「ヤマトB2クラウド」から生成します。生成は非同期処理で、完了後にメールでダウンロードリンクが届きます。

### PDFエクスポート — 納品書を出力する

PDFエクスポートでは、出荷に同梱する納品書をPDF形式で出力できます。

PDFエクスポート画面（「納品書」カテゴリ）は実行履歴の確認専用です。納品書PDFの生成は、出荷管理画面の「条件指定でエクスポート」→「ヤマトB2クラウド」から行います。ヤマトB2クラウド向けCSVと同じフォームから同時に生成され、完了後にメールでダウンロードリンクが届きます。

納品書のレイアウトは、設定画面の「納品書」でHTMLテンプレートとして管理します。

### 管理メンバー — 誰が管理画面を使えるか

管理メンバーとは、SQの管理画面にログインできる人のことです。「設定」→「管理メンバー」で追加・管理します。

#### ログイン方式

SQへのログインはメールアドレスによる認証で行います。フォームにパスワード入力欄はありません。

#### メンバーを追加する手順

1. 左メニュー下部の「設定」をクリックする
2. 「管理メンバー」をクリックする
3. 画面右上の「追加する」ボタンをクリックする。「ユーザーを追加する」フォームが開く
4. 次の項目を入力する

| 項目 | 必須/任意 |
|:--|:--|
| 姓 * | 必須 |
| 名 * | 必須 |
| メールアドレス * | 必須 |

5. 「権限」セクションで、割り当てる権限グループをラジオボタンで選択する（1人1グループのみ）
6. 「保存する」ボタンをクリックする

### 権限グループ — 見える画面・押せるボタンを制御する

権限グループとは、管理メンバーに割り当てるアクセス制御の単位です。グループごとに「どのリソースを閲覧できるか」「どのリソースを編集できるか」を設定します。

権限グループは「設定」→「管理メンバー」の画面右上にある「権限グループ一覧」ボタンから管理します（設定トップページには表示されません）。

#### 権限の体系

権限は **35種類のリソース × 閲覧（:read）/ 編集（:write）= 70権限** で構成されています。

35のリソースは次のとおりです。

お知らせ / アプリ / ブランド / カタログ / 取引先企業 / 顧客マイル / 顧客ポイント / 顧客会員ランクルール / 顧客会員ランク / 顧客 / ディスカウント / 下書き注文 / 在庫 / 在庫リクエスト / 店舗受取ロケーションルール / ロケーショングループ / ロケーション / マイル商品 / OmnibusCore連携 / 注文 / ポイントキャンペーン / ポイントルール / 販売価格ルール / 商品 / 店舗ポータル連携 / 店舗スタッフ / 売上データ / Shopify連携 / スマレジ連携 / テナント / 翻訳ルール / 翻訳 / ユーザー権限 / ユーザーテナント / ユーザー

この70権限は、アプリ（APIキー）の権限スコープと同じ体系です。管理画面の権限とAPIの権限が1対1で対応しています。

#### 最初から用意されているグループ

| グループ名 | 権限数 | 変更可否 |
|:--|:--|:--|
| 特権管理者 | 70（全権限） | グループ名のみ変更可。権限内容は変更不可。削除不可 |

#### カスタム権限グループを作成する手順

1. 左メニュー下部の「設定」をクリックする
2. 「管理メンバー」をクリックする
3. 画面右上の「権限グループ一覧」ボタンをクリックする
4. 「作成する」ボタンをクリックする。「権限グループを作成する」フォームが開く
5. 「グループ名 *」にグループ名を入力する（例: 店舗スタッフ）
6. 権限のチェックボックス一覧から、このグループに付与する権限にチェックを入れる。デフォルトはすべてオフ（最小権限の状態）
7. 「作成する」ボタンをクリックする

作成しただけではメンバーには適用されません。メンバーの追加フォームまたはメンバー詳細画面の「権限」セクションで、このグループを割り当ててください。

### カスタムデータ — 商品や注文に独自の項目を追加する

カスタムデータグループには「メタフィールド定義」「翻訳」「採寸ルール」の3つの機能があります。いずれも「設定」からアクセスします。

#### メタフィールド定義 — 独自フィールドを追加する

メタフィールド定義とは、SQの標準フォームにない独自の入力欄（カスタムフィールド）を追加するための設定です。定義を作成すると、対象オブジェクトの詳細画面に「メタフィールド」セクションが即座に現れ、値を入力できるようになります。

**カスタムフィールドを追加できるオブジェクト（16種）**

組織 / 商品 / バリエーション / 顧客 / 注文 / 下書き注文 / 会社 / ロケーション / ディスカウント / 在庫移動伝票 / 在庫調整伝票 / 在庫取置伝票 / 発注伝票 / 入荷指示 / 出荷指示 / 仕入れ先

**メタフィールド定義の作成手順**

1. 「設定」→「メタフィールド定義」をクリックする
2. カスタムフィールドを追加したいオブジェクト名（例: 「商品」）をクリックする
3. 「定義を追加する」ボタンをクリックする
4. 次の項目を入力する

| 項目 | 必須/任意 | 制約 |
|:--|:--|:--|
| 名前 * | 必須 | 最大50文字 |
| 説明 | 任意 | 最大100文字 |
| ネームスペース * | 必須 | 最低4文字・最大50文字 |
| キー * | 必須 | 最低4文字・最大50文字 |
| メタフィールドのタイプ * | 必須 | 下記12種から選択 |

**「メタフィールドのタイプ」の選択肢（12種）**

単一行のテキスト / 単一行のテキスト(リスト) / 複数行のテキスト / 日付と時刻 / 日付 / trueまたはfalse / バーコード(NW7) / 整数 / 小数 / JSON / ID / リッチテキスト

5. 「保存する」ボタンをクリックする

保存が完了すると、対象オブジェクトの詳細画面に「メタフィールド」セクションが即座に表示されます。フィールド右側の編集ボタン（鉛筆アイコン）をクリックして値を入力できます。

**識別子の形式**

定義の識別子は「ネームスペース.キー」形式で管理されます（例: `custom.note`）。定義一覧では「定義名 / ネームスペース.キー」の2行で表示されます。

**他の機能との連携**

- 顧客の定義は顧客一覧の絞り込み条件「メタフィールド」として利用できます
- 商品・バリエーションの定義はCSVインポートの「商品メタフィールド」「商品バリエーションメタフィールド」カテゴリで値を一括投入できます
- 商品・バリエーションの定義は翻訳ルールの翻訳対象にもなります

#### 翻訳ルール — 多言語対応の設定をする

翻訳ルールとは、商品データなどを指定した言語へ翻訳するための設定です。ルールごとに翻訳先の言語と、AI翻訳に使うプロンプトを管理します。

**対応言語（10言語）**

#### 採寸ルール — 採寸項目と単位を定義する

採寸ルールとは、衣料品などの商品に対して「どの項目を・どの単位で測るか」を定義したテンプレートです。

**採寸ルールの作成手順**

1. 「設定」→「採寸ルール」をクリックする
2. 「採寸ルールを作成する」ボタンをクリックする
3. 次の項目を入力する

| 項目 | 必須/任意 | 選択肢・補足 |
|:--|:--|:--|
| ルール名 * | 必須 | テキストボックス（例: トップス） |
| 採寸単位 | 任意 | 「なし」（デフォルト）または「センチメートル」の2種のみ |

4. 「採寸項目を追加」ボタン（+アイコン付き）をクリックして採寸項目の入力欄を追加する（最大5件）
5. 採寸項目名（例: 肩幅、バスト）を入力する
6. 「保存する」ボタンをクリックする

採寸ルール一覧では「ルール名」「単位」「採寸項目」の3列で一覧表示されます。

### この章のまとめ — 最初に準備するもの

SQを使い始めるときの準備順序は次のとおりです。

1. **権限グループを作成する** — どの担当者に何を見せるかを先に決める
2. **管理メンバーを追加する** — 権限グループを割り当てて招待する
3. **メタフィールド定義・翻訳ルール・採寸ルールを設定する** — 商品登録前に追加フィールドの定義を済ませておくと、CSVインポートで一括投入できる
4. **CSVインポートでデータを一括登録する** — 商品・在庫・価格などを21カテゴリのCSVで一括投入する
5. **CSVエクスポート・PDFエクスポートで結果を確認する** — 在庫・売上・納品書を必要に応じてダウンロードする

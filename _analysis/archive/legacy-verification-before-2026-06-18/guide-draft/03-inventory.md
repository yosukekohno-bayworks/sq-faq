## 3. 在庫の読み方 — 9つの区分と手持ちの式

SQの在庫は「どの商品（SKU）が・どの場所（ロケーション）に・何個あるか」を区分ごとに管理しています。この章では、画面に表示される数字の意味・計算式・操作による在庫の動き・編集時の注意点を順に説明します。

---

### 在庫はSKU×ロケーションで持つ

SQでは、在庫を「SKU（最小管理単位——商品のサイズやカラーなど、バリエーションごとの品番）×ロケーション（倉庫・店舗などの保管場所）」の組み合わせで管理します。同じSKUでも、倉庫Aと店舗Bでは在庫数が独立して管理されます。

在庫管理画面を開くと、まず**一覧**が表示されます。ここでは各SKUの在庫数を4列で確認できます。特定のSKUをクリックすると**SKU詳細**に進み、ロケーション別に9つの区分を確認できます。

---

### 一覧画面の4列

在庫管理トップの一覧には、以下の4列が表示されます。

| 列名 | 意味 |
|:--|:--|
| 販売可能 | 実際に販売に使える在庫数 |
| 引当済み | 注文に引き当て済みの在庫数 |
| 取置中 | 取置・移動用に確保された在庫数 |
| 手持ち | 物理的に手元にある在庫の合計数 |

> 「引当済み」は旧称「確定済み」、「取置中」は旧称「確保済み」からそれぞれ名称が変わりました。旧称をご存じの方は読み替えてください。

---

### SKU詳細の9列と手持ちの式

特定のSKUをクリックしてSKU詳細に進むと、ロケーションごとに9つの区分で内訳を確認できます。

| 区分 | 意味 |
|:--|:--|
| 販売可能 | 販売に使える在庫数 |
| 引当済み | 注文に引き当て済みの在庫数（旧称「確定済み」） |
| 取置中 | 取置・移動用に確保された在庫数（旧称「確保済み」） |
| 破損 | 品質上の問題で販売できない在庫数 |
| 検品 | 検品中で販売できない在庫数 |
| 予備 | 予備として確保された在庫数 |
| 手持ち | 上記6区分の合計（下記の式を参照） |
| 積送中 | 出荷済みだがまだ到着していない輸送中の数（移動先で計上） |
| 入荷予定 | 移動伝票を作成してから出荷前までに移動先で計上される予定数 |

**手持ちの計算式**

```
手持ち = 販売可能 + 引当済み + 取置中 + 破損 + 検品 + 予備
```

「積送中」と「入荷予定」は手持ちには含まれません。これらは移動中の荷物が「どこにあるか」を追うための別枠です。移動先のロケーションに入荷が完了して初めて、手持ち（と販売可能）に加算されます。

---

### 各区分の意味——何で増えて、何で減るか

| 区分 | 増える操作 | 減る操作 |
|:--|:--|:--|
| 販売可能 | 鉛筆編集 / CSV「販売可能在庫」 / 移動伝票のキャンセル（元に戻る）/ 入荷実績の登録（移動先に到着） | 移動伝票の作成 / 取置伝票の作成（取置中へ振替） |
| 引当済み | 注文の引き当て | 出荷の進行 |
| 取置中 | 取置伝票の作成 / 移動伝票の作成（販売可能から振替） | 「処理済みとしてマークする」（販売可能へ戻る）/ 出荷実績の登録（手持ちごと減る）/ 移動伝票のキャンセル |
| 破損 / 検品 / 予備 | 管理画面上の振替操作は確認されていません<!-- TODO: 要確認（WMS連携経由での変動の可能性） --> | 同左 |
| 手持ち | 入荷実績の登録（移動先）/ 鉛筆編集（販売可能と同時に増減） | 出荷実績の登録（移動元） |
| 積送中 | 出荷実績の登録（移動先の入荷予定が積送中へ移る） | 入荷実績の登録（販売可能・手持ちへ） |
| 入荷予定 | 移動伝票の作成（移動先） | 出荷実績の登録（積送中へ移る） |

---

### 操作と在庫の動き——早見表

移動伝票を使ったロケーション間の移動を例に、操作ごとに在庫がどう動くかを示します。

| 操作 | 移動元の変化 | 移動先の変化 |
|:--|:--|:--|
| 在庫依頼を作成（確認待ち） | 変化なし | — |
| 在庫を引当てる（在庫依頼が「確保済み」ステータスへ） | 変化なし | — |
| 移動伝票を作成 | 販売可能 −n、取置中 +n（手持ち不変） | 入荷予定 +n |
| 出荷実績を登録 | 取置中 −n、手持ち −n | 入荷予定 −n → 積送中 +n |
| 入荷実績を登録 | — | 積送中 −n → 販売可能 +n、手持ち +n |
| 移動伝票をキャンセル | 取置中が販売可能に全数戻る | — |
| 取置伝票を作成 | 販売可能 −n、取置中 +n | — |
| 「処理済みとしてマークする」（取置伝票） | 取置中 −n → 販売可能 +n（解放） | — |
| 調整伝票を実行 | 増減数がそのまま反映 | — |
| 鉛筆で販売可能数を編集 | 「販売可能」が入力値そのものに設定される（絶対値）。手持ちは差分だけ増減 | — |

> 「在庫依頼」ステータスの「確保済み」は名称変更していません。在庫区分の列名「確保済み」だけが「取置中」に変わった点に注意してください。

---

### 鉛筆編集（販売可能数の直接編集）

在庫詳細の「販売可能」列にある鉛筆アイコン（「販売可能数を編集」）をクリックすると、販売可能数を直接変更するダイアログが開きます。

**重要: 入力値は絶対値です**

「数量」に入れた値が、そのまま**新しい販売可能数**になります。加算ではありません。

例: 販売可能が1の状態で「9」と入力して保存すると、販売可能は9になります（8が足されるのではなく、1が9に置き換わります）。手持ちは差分（この例では+8）だけ増減します。

「理由」は必須入力です。空のまま「保存する」を押すと「理由を入力してください」と表示されて保存できません。

この操作は調整伝票とは別の経路です。調整伝票に新しい伝票は作られません。変更は「調整履歴」に記録されます。

---

### マイナス在庫

手持ちが0のロケーションから出荷実績を登録しても、SQは処理をブロックしません。そのため、実際の手持ちを超えて出荷を登録すると、在庫数がマイナスになります。

マイナスになった在庫を修正するには、「在庫管理 > 調整伝票」から調整伝票を作成します。

---

### 調整履歴——いつ・なぜ在庫が変わったかを確認する

SKU詳細画面の「調整履歴」ボタンをクリックすると、そのSKUの在庫区分の変更記録を時系列で確認できます。

| 列名 | 内容 |
|:--|:--|
| 日時 | 変更が行われた日時 |
| アクティビティ | 変更の理由・操作名（鉛筆編集で入力した「理由」もここに表示される） |
| 販売可能 | 変更後の数量（差分付き。例: 5 (+5)） |
| 引当済み | 変更後の数量（差分付き） |
| 取置中 | 変更後の数量（差分付き） |
| 破損 | 変更後の数量（差分付き） |
| 検品 | 変更後の数量（差分付き） |
| 予備 | 変更後の数量（差分付き） |
| 手持ち | 変更後の数量（差分付き） |

在庫の過去の動きや原因を調べるときは、まずここを確認してください。

> 「積送中」「入荷予定」は調整履歴の表には含まれません。

---

### 区分の名称変更について（旧称との対照）

区分名がいくつか変更され、新しい列も追加されています。旧称を使っている資料や会話と照らし合わせる際の参考として、対照表を示します。

| 現在の名称 | 旧称 | 変更の種別 |
|:--|:--|:--|
| 引当済み | 確定済み | 名称変更 |
| 取置中 | 確保済み | 名称変更 |
| 積送中 | （なし） | 新規追加 |
| 入荷予定 | （なし） | 新規追加 |

また、一覧画面（4列）では旧称「販売不可」という合算列がなくなり、かわりに「引当済み」「取置中」が個別に表示されるようになりました。

## クロールで判明した下位概念（サイドナビ24項目に出ない用語）
- **在庫管理 配下の伝票種別**: 移動伝票 / 調整伝票 / 取置伝票（用途・違いの説明がヘルプに無い）
- **データ管理 配下の8項目**: 管理メンバー / テナント / ロケーション / ロケーショングループ / ブランド / 取引先 / 決済方法 / 販売員
  - ロケーショングループは議事録の「取り寄せ販売（複数ブランド間のロケーショングループ管理）」と直結する重要概念。
- **Shopify連携の接続5段階**: 基本設定 → リスティング → データ連携 → ロケーション → アクセストークン
- **注文管理**: 返品・キャンセル（外部システム連携対応）、交換機能（未実装予定）

## 結論（FAQ戦略への示唆）
1. **既存ヘルプは"骨組み（目次）"のみ**。リンク集はあるが、操作手順・画面項目・スクショが全機能で欠落（Shopify除く）。
2. よって FAQ作成は「穴埋め」ではなく **ほぼ全機能を実機ベースで手順化する新規執筆**。既存ページはトピック構造の参考にはなるが、本文の流用はできない。
3. **Shopify連携だけは既存ページ（medium）を土台に補強**でよい。
4. 工数見積もりは「全機能の手順＋スクショをゼロから作る」前提で算定すべき（提案書の核心）。
5. app-analyzer の実機巡回記録が、ほぼ全ページの一次素材になる（既存ヘルプには素材が無いため）。

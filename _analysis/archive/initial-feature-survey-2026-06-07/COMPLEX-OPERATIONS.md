## A. 取り寄せ在庫販売 ★最重要

**何をする運用か**: ECで売れた注文に対し、別ロケーション（店舗）の在庫を取り寄せて引き当て、移動・入荷・出荷して顧客に届ける。

**関連画面**: 在庫依頼 → 確保済み → 移動伝票 → 入荷管理 → 出荷管理（＋発注管理）
**前提マスタ**: ロケーション（店舗・倉庫）、ロケーショングループ、テナント、商品/在庫

**確定フロー（2026-06-05 社内在庫移動フローは通し実証済み）**:

**実証で判明した核心**:
- **「自動連携」の正体**: 移動伝票を作成した瞬間に、出荷指示・入荷指示が**自動生成**される。これが議事録の「データ連携が自動化」の中身。
- **移動伝票はステータスを手で進めない**（外部システム連携扱いで編集制限）。入荷/出荷の実績登録が在庫移動を確定させる。
- **入荷・出荷は並行処理可能**（推測: 11-operations.md L607「並行処理可能と思われる」。出荷→入荷の順序制約は未実証）。<!-- 訂正(2026-06-07): 断定から推測表記に修正 -->
- **「リクエスト先ロケーション」= 在庫の引当元**（引当てダイアログのロケーション選択肢はリクエスト先のみ表示）。（推測: 実証済み動作からの推論。ラベル定義は未確認）<!-- 注記(2026-06-07): 推測表記を明示 -->
- 各実績登録は「**巻き戻し不可**」。FAQで強く注意喚起すべき点。

## F. リテールポータル（店舗スタッフ権限）

> **実証済み（2026-06-06）**: TEST_FAQ_店舗スタッフ権限グループを作成し、カスタム権限グループの動作を確認。

**何をする仕組みか**: 店舗スタッフ向けに権限を制限したビュー。運営管理側と役割分担。

**関連画面**: 販売チャネル > リテールポータル、設定 > 管理メンバー / 権限グループ / 販売員
**現状**: 権限グループは「特権管理者」（固定）＋ TEST_FAQ_店舗スタッフ（4権限、実証作成）。リテールポータル未接続。

### 確定した手順（実証）

| # | ステップ | 操作詳細 |
|:--|:--|:--|
| 1 | 設定 > 管理メンバー > 「権限グループ一覧」ボタンから遷移 | `/admin/settings/permission_groups` |
| 2 | 「作成する」ボタンをクリック | 作成フォームへ遷移 `/admin/settings/permission_groups/create` |
| 3 | 「グループ名 *」を入力 | 必須項目（プレースホルダー: 「リテールポータル管理者」） |
| 4 | 付与する権限のチェックボックスをONにする | 70権限から必要なものだけ選択。デフォルト全オフ |
| 5 | 「作成する」ボタンをクリック | 詳細ページへ遷移（作成完了）。以後は「保存する」ボタン |

### 確認できた要点

- **カスタムグループでは全70権限が enabled（選択可能）** — 特権管理者（固定・全ON・disabled）と完全に異なる動作
- **権限は read と write が分離**: 閲覧だけ付与して編集不可にすることが可能
- **プレースホルダーが「リテールポータル管理者」**: 店舗スタッフ向けグループを想定した設計
- **作成ボタン名「作成する」 / 編集時「保存する」**: 新規作成と既存編集でボタン名が異なる
- **権限グループはメンバー割り当てとは別管理**: グループ作成後、管理メンバー画面から各メンバーの権限グループを変更する（今回は割り当て操作は実施せず）
- **リテールポータルとの関連**: リテールポータル接続設定は `/admin/retail_portal_integrations` で行う（未接続）。権限グループとの具体的な紐づけ方法は未確認 <!-- TODO: 要確認 -->

## G. メタフィールド拡張

> **実証済み（2026-06-06）**: 商品オブジェクトにTEST_FAQ_テキストフィールドを作成。商品詳細での表示を確認。

**何をする設定か**: 標準項目に加え、16種オブジェクトにカスタムフィールドを追加して商品データ等を柔軟に拡張。

**関連画面**: 設定 > メタフィールド定義（`/admin/settings/metafield_definitions`）、商品詳細
**対象16種**: 組織/商品/バリエーション/顧客/注文/下書き注文/会社/ロケーション/ディスカウント/在庫移動伝票/在庫調整伝票/在庫取置伝票/発注伝票/入荷指示/出荷指示/仕入れ先

### 確定した手順（実証）

| # | ステップ | 操作詳細 |
|:--|:--|:--|
| 1 | 設定 > メタフィールド定義 に移動 | `/admin/settings/metafield_definitions` |
| 2 | 対象オブジェクト（例: 商品）をクリック | オブジェクト別定義一覧へ遷移 |
| 3 | 「定義を追加する」をクリック | 作成フォームへ遷移 |
| 4 | 「名前 *」を入力（最大50文字） | 管理画面での表示名 |
| 5 | 「説明」を入力（任意、最大100文字） | 管理者向け注釈 |
| 6 | 「ネームスペース *」を入力（最低4文字、最大50文字） | APIアクセス用識別子の前半 |
| 7 | 「キー *」を入力（最低4文字、最大50文字） | APIアクセス用識別子の後半 |
| 8 | 「メタフィールドのタイプ *」をコンボボックスで選択 | 12種から選択（単一行テキスト等） |
| 9 | 「保存する」ボタンをクリック | 定義一覧へ遷移（作成完了） |

### 確認できた要点

- **保存直後、商品詳細画面の「メタフィールド」セクションに新フィールドが出現する**（リロード不要）
- 商品詳細での表示: 「メタフィールド」（h2）→ 定義名（h3）→「未設定」テキスト＋編集ボタン
- **識別子フォーマット**: ネームスペース.キー（例: `test_faq.test_faq_text`）。APIで参照する際のキー
- **タイプは12種**: 単一行のテキスト / 単一行のテキスト(リスト) / 複数行のテキスト / 日付と時刻 / 日付 / trueまたはfalse / バーコード(NW7) / 整数 / 小数 / JSON / ID / リッチテキスト
- **オブジェクト別に独立管理**: 商品に定義したメタフィールドは商品詳細にのみ表示される
- 定義一覧の「使用箇所」列で何件の商品に値が設定されているか把握できる

### TEST_FAQ_テキストフィールド（実証）
- URL（一覧）: `/admin/settings/metafield_definitions/product`
- 識別子: `test_faq.test_faq_text` / タイプ: 単一行のテキスト / 権限: 編集可

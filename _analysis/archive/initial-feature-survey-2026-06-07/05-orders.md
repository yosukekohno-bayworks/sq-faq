### [照合1] 「注文を作成する」の DOM 要素種別（L39 vs L123 の矛盾解消）

**照合前の記述**:
- L39: 「リンクボタン（遷移先URL無し・disabled状態）」
- L123: 「テキスト（非ボタン）。クリックしても何も起きない。リンク・ボタン属性なし」
- L132: 「アクセシビリティ上もボタン・リンク属性がなく」

**実機 DOM 確認（2026-06-07）**:
```
要素: <a class="Polaris-Button Polaris-Button--pressable Polaris-Button--variantPrimary
             Polaris-Button--sizeMedium Polaris-Button--textAlignCenter Polaris-Button--disabled">
  <span class="Polaris-Text--root Polaris-Text--bodySm Polaris-Text--medium">注文を作成する</span>
</a>
```
- タグ種別: `<a>` タグ（`role=null`、`href` 属性なし → `href=""` として解決）
- disabled 属性: なし（HTMLの `disabled` 属性は付与されていない）
- CSS クラス: `Polaris-Button--disabled` が付与されている（CSSでdisabled外観を実現）
- `pointer-events: none`（CSS により物理的にクリック不可）
- アクセシビリティ: `role`・`aria-disabled` 属性なし

**訂正（旧→正）**:
- L39「リンクボタン（遷移先URL無し・disabled状態）」→ **正: `<a>` タグに `Polaris-Button--disabled` CSS クラスが付与された擬似disabled状態のリンク。`href` 属性なし、`pointer-events:none` でクリック不可。HTML の `disabled` 属性は使用していない。**
- L123「テキスト（非ボタン）。リンク・ボタン属性なし」→ **正: テキスト `<span>` は内包要素であり、その親が `<a>` タグ（Polaris ボタンスタイル）。アクセシビリティ属性（role/aria）はないが、DOM上は `<a>` リンク要素が存在する。**
- L132「アクセシビリティ上もボタン・リンク属性がなく」→ **正: `<a>` タグは存在するが、`href`・`role`・`aria-disabled` 属性は付与されておらず、支援技術からはリンクとして認識されない可能性が高い（Polaris の disabled link パターン）。**

**FAQ への影響**:
- FAQ記述として正確な表現: 「下書き一覧画面の『注文を作成する』は、表示上はボタン形式だが `href` なし・クリック不可（`pointer-events:none`）の無効状態。購入顧客データが存在しない環境では押せない。」
- 「ボタンが無効化されている」と表現することが実態に最も近い（CSSクラスで disabled 外観）。

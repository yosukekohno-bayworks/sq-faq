#### 基本情報セクション
| 項目 | ラベル原文 | 必須/任意 | 形式 |
|:--|:--|:--|:--|
| タイトル | タイトル | — | テキスト |
| 説明文 | 説明文 | — | テキスト |

| 項目 | ラベル原文 | 形式 |

| 最低購入数量 | 最低購入数量 | 数値 |
| 対象商品 | 対象商品 | 「すべての商品を割引対象に設定する」チェックボックス |

### 注文ポイント付与ルール作成フォーム（/admin/point_calculation_rules/create）

- **目的**: 購入金額に対してポイントを付与するルールを新規作成する。ページタイトルは「注文ポイント付与ルールを作成する」（ナビ名「ポイント」とは異なる）。

- **操作要素と挙動（実測）**:

| 要素（UIラベル原文） | 種別 | 操作すると何が起こるか |
|:--|:--|:--|
| ← 注文ポイント付与ルールを作成する（ページタイトル左の矢印） | リンク | 一覧画面へ戻る |
| 会員ランク算出ルール コンボボックス | セレクト | ポイントルールに適用する会員ランク算出ルールを関連付ける。プレースホルダ: 「選択してください」 |
| 開始日時が新しいキャンペーンを優先する | ラジオボタン | ポイントキャンペーンが複数ある場合、開始日時が新しいものを優先（デフォルト選択） |
| 開始日時が古いキャンペーンを優先する | ラジオボタン | ポイントキャンペーンが複数ある場合、開始日時が古いものを優先 |
| 保存する | ボタン（右下） | 入力内容を保存してポイント付与ルールを作成 |

- **入力項目**:

| 項目（ラベル原文） | 必須/任意 | 形式・選択肢・上限・プレースホルダ |
|:--|:--|:--|
| タイトル | 必須（★） | テキスト。プレースホルダ: 「タイトル」 |
| 購入金額（円ごとに） | 必須（★） | 数値。デフォルト: 100。単位: 「円ごとに」 |
| ポイント | 必須（★） | 数値。デフォルト: 1。単位: 「ポイント」。例: 100円ごとに1ポイント付与 |
| ポイントを付与してから有効な日数 | 必須（★） | 数値（日間）。デフォルト: 365 |
| オンライン注文のポイント有効までの日数 | **必須（★）** | 数値（日後）。**空欄不可（実測: 空欄保存時に「値を入力してください」エラー）**。0を入力すると即時付与。ヘルプ文: 「オンラインでの注文の出荷完了から顧客へポイントを付与するまでの日数です。0日に設定した場合、発送が完了後に即時ポイントが付与されます。」 |
| 店舗注文のポイント有効までの日数 | **必須（★）** | 数値（日後）。**空欄不可（実測: 空欄保存時に「値を入力してください」エラー）**。0を入力すると即時付与。ヘルプ文: 「店舗での決済完了から顧客へポイントを付与するまでの日数です。0日に設定した場合、即時ポイントが付与されます。」 |
| 会員ランク算出ルール | 任意 | コンボボックス。プレースホルダ: 「選択してください」。紐付ける会員ランクルールを選択 |

- **セクション構成（実測）**:
  1. **ルール**: タイトル・購入金額・ポイント数
  2. **有効期限**: ポイントを付与してから有効な日数
  3. **オンライン注文のポイント有効までの日数**: オンライン出荷完了→付与までの遅延日数
  4. **店舗注文のポイント有効までの日数**: 店舗決済完了→付与までの遅延日数
  5. **会員ランク算出ルール**: 関連する会員ランクルールの紐付け
  6. **ポイントキャンペーン**: 複数キャンペーン競合時の優先設定（ラジオ2択）

- **ポイントキャンペーン優先設定（実測）**:
  - 「開始日時が新しいキャンペーンを優先する」（デフォルト）
  - 「開始日時が古いキャンペーンを優先する」
  - ヘルプ文: 「注文に対して条件を満たすポイントキャンペーンが複数個ある場合に、どのキャンペーンを優先的に適用するかを設定します。」

- **ナビゲーション補足（実測）**: ポイントメニューを展開すると子メニューに「キャンペーン」「誕生日」「利用外商品」「失効通知」が表示される

### 会員ランク算出ルール作成フォーム（/admin/customer_rank_calculation_rules/create）

- **目的**: 顧客の会員ランクを算出するルールを新規作成する。ページタイトルは「会員ランク算出ルールを作成する」。

- **操作要素と挙動（実測）**:

| 要素（UIラベル原文） | 種別 | 操作すると何が起こるか |
|:--|:--|:--|
| ← 会員ランク算出ルールを作成する（ページタイトル左の矢印） | リンク | 一覧画面 `/admin/customer_rank_calculation_rules` へ戻る |
| 税抜き価格でランクを算出する | チェックボックス | ON にすると税抜き金額でランクを計算する |
| 会員ランクを月初に算出する | チェックボックス | ON にすると毎月1日に会員ランクを再計算する |
| 会員ランクを次の算出期間に持ち越す | チェックボックス | ON にすると最大2年間同じランクを維持できる（制約あり: 「開始月」が未選択、または「会員ランクの反映を次の算出期間の開始日に行う」が有効の場合は利用不可） |
| 保存する | ボタン（右下） | 入力内容を保存して会員ランク算出ルールを作成 |

- **入力項目**:

| 項目（ラベル原文） | 必須/任意 | 形式・選択肢・上限・プレースホルダ |
|:--|:--|:--|
| タイトル | 必須（★） | テキスト。プレースホルダ: 「タイトル」 |
| 算出方法 | — | ラジオボタン。選択肢: 「購入金額」（デフォルト・現状disabled）/ 「獲得ポイント」（disabled）。現在は両方disabled（変更不可） |
| 税抜き価格でランクを算出する | 任意 | チェックボックス |
| 期間 | — | コンボボックス。選択肢: 「1年間」（デフォルト）/ 「直近365日」/ 「無期限」 |
| 開始月 | 任意（期間=1年間 の場合に表示） | コンボボックス。1月〜12月。ヘルプ文: 「開始月に4月を設定した場合、ランク算出期間の1年間は4月1日から3月31日となります」 |
| 会員ランクを月初に算出する | 任意 | チェックボックス。ヘルプ文: 「この設定を有効にすることで顧客の会員ランクは毎月1日に再計算されます。」 |
| 会員ランクを次の算出期間に持ち越す | 任意 | チェックボックス。ヘルプ文: 「持ち越しを有効にすることで顧客は自分の会員ランクを最大2年間維持することができます。この設定は「開始月」が未選択、もしくは「会員ランクの反映を次の算出期間の開始日に行う」が有効の場合は利用できません。」 |

- **セクション構成（実測）**:
  1. **ルール**: タイトル・算出方法（購入金額 / 獲得ポイント）・税抜き価格でランクを算出する（チェックボックス）
  2. **算出期間**: 期間（1年間 / 直近365日 / 無期限）・開始月（1〜12月）
  3. **詳細**: 会員ランクを月初に算出する / 会員ランクを次の算出期間に持ち越す（各チェックボックス）

- **算出方法の注意（実測）**: 「購入金額」「獲得ポイント」のラジオボタンはどちらも `disabled` 状態で変更不可。現状は購入金額固定。<!-- TODO: 要確認 → 将来的に獲得ポイント算出が追加予定か -->

### [調査4] 会員ランク×ポイントの連携

#### 連携の仕組み（実測で確認できた範囲）

会員ランクとポイントは、2つの方向から連携する。

- 紐付けることで「このポイント付与ルールを適用される顧客が、指定ランク算出ルールで管理される」という関連付けになる
- 具体的な倍率設定はこの紐付けでは行わない（ポイント数はルールに固定される）

### [V4] クーポンコード自動適用ロジック・空欄保存挙動の確認（最優先項目）

**検証結果: 確定（推測を完全に覆す）**

#### 検証手順
- 作成フォームでクーポンコード欄を空欄のまま、タイトル・テナント・有効期間を入力して「保存する」を押した

#### 確認結果（実測）

| 確認事項 | 結果 |
|:--|:--|
| クーポンコード空欄で保存 | **フロントエンドバリデーションエラーが表示・保存失敗** |
| エラーメッセージ | 「クーポンコードを入力してください」（赤いエラーボーダー + 赤文字メッセージ） |
| 空欄=自動適用（旧推測） | **誤り**。クーポンコードは必須項目（✳）であり、空欄では保存できない |
| 「SEHD-DEKH」プレースホルダーの正体 | プレースホルダーテキスト（ダミー表示）。自動生成コードではない。保存時に自動採番もされない |

#### 判明した仕様（確定）

- **クーポンコードは必須入力**（✳）。空欄保存は不可
- 「SEHD-DEKH」はプレースホルダー（入力例）であり、自動生成値・自動採番機能は存在しない
- 従って「空欄=自動適用」という動作は存在しない
- クーポンコードを入力した場合 → 顧客がそのコードを入力することでディスカウントが適用される（クーポンコード必須型）
- **全ディスカウントにクーポンコード入力が必須**であり、「自動適用タイプ」の選択肢は作成フォームに存在しない

#### その他バリデーション確認（同時確認）

| 最低購入額（円） | 1000（UI表示のみ、実データは空） | 「最低購入額を入力してください」 |

- デフォルト値はUIに表示されているが、**フォームデータとしては空**。保存時に明示的な入力が必要

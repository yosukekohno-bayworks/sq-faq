# 在庫管理

> 調査日: 2026-06-05 | URL: `/admin/inventory_items` | 読み取り専用

## 画面目的
SKUごとの在庫数量管理。バリエーション×ロケーション軸で在庫状況を確認する。

## 在庫管理 一覧画面（/admin/inventory_items）

### ヘッダー

- 「インポート」リンク → `/admin/csv_import/csv_import_operation_inventory_logical_available_quantities`
- 「エクスポート」リンク → `/admin/csv_export/csv_export_operation_inventory_logical_quantities`

### タブ
- 「すべて」のみ

### フィルター
- 「検索と絞り込みの結果」ボタン

### テーブル列
| 列名 | 内容 |
|:--|:--|
| （チェックボックス） | 一括選択 |
| 商品 | サムネイル + 商品名 + バリエーション名（カラー/サイズ） |
| 品番(SKU) | SKUコード（例: 487973-64-36） |
| 販売不可 | 数量 |
| 確定済み | 数量 |
| 販売可能 | 数量 |
| 手持ち | 数量 |

### 在庫数量の種類（4区分）
- **販売不可**: 販売できない在庫
- **確定済み**: 注文確定済みの在庫
- **販売可能**: 販売可能な在庫（= 手持ち - 確定済み - 販売不可 と推測。全SKU=0のため実機で未検算 / 要確認。L117参照）
- **手持ち**: 実際に手元にある在庫

### サンプルデータ
- バギーカーブジーンズ BLUE / 36（487973-64-36）: 全0
- エアリズムコットンクルーネックTシャツ GREEN / XL（486102-52-XL）: 全0
- オーバーサイズスウェットシャツ BLACK / XL（486125-09-XL）: 全0

### 調整伝票 作成フォーム（/admin/inventory_adjustment_orders/create）

- **目的**: 在庫数を手動で増減する調整伝票を起票する。

**入力項目（実測）**

| 項目（ラベル原文） | 必須/任意 | 形式・選択肢・補足 |
|:--|:--|:--|
| ロケーション | 必須 | テキスト入力 + 「選択」ボタン → ロケーション選択モーダル（すべて/店舗/倉庫タブ、物流倉庫W0001・ユニクロ銀座店R0001が登録済み） |
| 理由 | 任意（保存後も任意） | ドロップダウン「選択してください」。実測で「棚卸差異」が選択肢にあることを確認 |
| 商品（商品を検索する） | 任意（商品なしでも保存可） | テキスト検索 + 「参照」ボタンで商品選択モーダル |
| 増減数（商品追加後に表示） | 必須 | 数値入力（スピナー付き）。プラスで増加、マイナスで減少。デフォルト0 |

**操作要素と挙動（実測）**

| 要素（UIラベル原文） | 種別 | 操作すると何が起こるか |
|:--|:--|:--|
| 選択（ロケーション横） | ボタン | ロケーション選択モーダルを開く |
| 参照 | ボタン | 商品検索モーダルを開く |
| ✕（商品行右端） | アイコンボタン | その商品行を削除する |
| 保存する | ボタン | 調整伝票を保存し、詳細画面（#IA-XXXX）へ遷移。ステータス「未実施」 |
| 変更を破棄 | ボタン | 未保存変更を破棄して直前の状態に戻す（バナー「保存されていない変更があります」表示中に出現） |

**未保存バナー（実測）**

商品追加や数値変更を行うと、画面上部に「⚠ 保存されていない変更があります」バナーが現れ、右上に「変更を破棄」「保存する」ボタンが表示される。

### 調整伝票 詳細・ステータス遷移（/admin/inventory_adjustment_orders/[id]）

- **目的**: 保存済み調整伝票の確認・修正、および「実行」による在庫数への反映。

**ステータス遷移（実測）**

```
作成・保存 → [未実施] → 実行ボタン押下 → 確認ダイアログ → 実行する → [実施済み]
                ↓
           キャンセルボタン → [キャンセル]
```

**各ステータスの画面状態（実測）**

| ステータス | ラベル表示 | 利用可能ボタン | 編集可否 |
|:--|:--|:--|:--|
| 未実施 | 「○ 未実施」（白丸） | キャンセル / 実行 | 可（ロケーション・商品・増減数・理由を編集できる） |
| 実施済み | 「● 実施済み」（黒丸） | キャンセルのみ（押すと？） | 不可（全フィールドが読み取り専用に変わる） |

**重要な挙動（実測）**

- 未保存変更がある状態（未保存バナー表示中）は「実行」ボタンがグレーアウト（disabled）になる。先に「保存する」を押してから「実行」する必要がある。
- 「実行」ボタン押下で確認ダイアログが開く。

**実行確認ダイアログ（実測）**

タイトル: 「在庫調整を実行する」

本文: 「在庫調整を実行することで明細に登録されている差分値が在庫数に反映されます。この処理は巻き戻すことができません。」

ボタン: キャンセル / **実行する**

**詳細画面の表示項目（実測）**

| 区分 | 項目 | 内容 |
|:--|:--|:--|
| 左メイン | ロケーション | 選択済みロケーション名（実施済みは読み取り専用） |
| 左メイン | 商品テーブル | 商品名・バリエーション / 商品コード / SKU / 増減数（実施済みは「+5」のようにリンク表示） |
| 右サイドバー | 作成者 | スタッフ名（例: 河野陽介） |
| 右サイドバー | 作成日 | 日時（例: 2026年06月05日 18:44） |
| 右サイドバー（実施済みのみ） | 実行者 | スタッフ名 |
| 右サイドバー（実施済みのみ） | 実行日 | 日時（例: 2026年06月05日 18:47） |
| 右サイドバー | 理由（未実施時） | ドロップダウン（選択してください） |

# 08 メタフィールド定義 3文字識別子・対象オブジェクト確認 2026-06-28

## 対象

- URL: `/admin/settings/metafield_definitions/organization/create`
- 対象オブジェクト: 組織

## 確認内容

- 作成フォームの見出しは `組織メタフィールドの定義を追加する`。
- フォーム項目は `名前`、`説明`、`ネームスペース`、`キー`、`メタフィールドのタイプ`。
- 対象オブジェクトを複数選択するUIはフォーム内に存在しない。対象はURLパスと一覧タブで分かれる。
- DOM上は `ネームスペース` / `キー` に `minlength=4` が付いている。
- ただし `namespace=abc` / `key=def` / 型 `単一行のテキスト` で保存でき、一覧に `TEST_MINLEN_20260628` / `abc def` / `単一行のテキスト` / `0個の組織` と表示された。
- 検証用定義は行選択 → `定義を削除` → 確認ダイアログ `削除する` で削除済み。削除後トーストは `メタフィールドの定義を削除しました`。

## 判断

実運用ドキュメントでは「4文字以上必須」とは断定しない。DOM属性と保存実挙動が一致していないため、保存実績を優先して「3文字でも保存できた」と記載する。

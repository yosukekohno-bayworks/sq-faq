# 06/08/15 UI残件再確認

日時: 2026-06-28

## 06 カタログ: SKU一覧

対象: `TEST_FAQ_カタログ001`
URL: `https://www.sqstackstaging.com/admin/catalogs/e2a819ab-3f81-558d-ad76-5fae9e8422d2_Catalog?tab=sku`

- 旧URL形式 `/admin/catalogs/{id}/catalog_product_variants` は、対象カタログIDで `このページは存在しないようです` になった。
- 現行UIではカタログ詳細内に `商品` / `SKU` タブがある。
- `SKU` タブを開くと、行あり状態で列は `商品` / `品番(SKU)` の2列。
- 表示例: `ポケモン UT RED / 120(6-7歳)` / `483674-15-120`。

## 08 カスタムデータ: メタフィールド定義の編集導線

対象: 商品メタフィールド定義
URL: `https://www.sqstackstaging.com/admin/settings/metafield_definitions?ownerType=product`
検証用定義: `TEST_EDIT_20260628` / `tedt.editcheck`

- 商品タブは開始時0件だったため、検証用定義を作成した。
- 作成後一覧には `定義名` / `コンテンツタイプ` / `権限` / `使用箇所` が表示された。
- 行選択後に出る一括操作は `定義を削除` のみ。編集ボタンは表示されない。
- 削除確認ダイアログは `メタフィールドの定義を削除しますか？`。
- 削除後トーストは `メタフィールドの定義を削除しました`。検証用定義は削除済み。

## 04/15 取引先: 詳細項目・アーカイブ導線

対象: `TEST_E2E_20260622_取引先_1905`
URL: `https://www.sqstackstaging.com/admin/settings/suppliers/42bb0e6a-5718-546a-8c3f-a185c116ba5f_InventorySupplier`

- 取引先詳細に表示される入力項目は `仕入れ先ベンダー名` と `取引先コード` の2つ。
- 住所、電話番号、メールアドレスなどの所在地・連絡先項目は表示されない。
- `その他の操作` を開くと `アーカイブ` が表示される。
- `削除` は表示されない。アーカイブ操作自体は未実行。

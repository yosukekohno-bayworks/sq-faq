# Shopify経由注文フロー再検証 2026-07-02

## 対象

- SQ staging: `https://www.sqstackstaging.com`
- Shopify store: `stack-ps-yosuke.myshopify.com`
- SQ Shopify連携レコード: `stack-ps-yosuke`
- Shopify商品: `Selling Plans Ski Wax` / バリエーション `Selling Plans Ski Wax`

## 前提確認

2026-07-02 00:22 JST時点で、SQ Shopify連携詳細には `基本設定` / `リスティング` / `データ連携` / `メタフィールド` / `ロケーション` / `アクセストークン` タブが表示された。

アクセストークンタブは次の9種すべて `未設定` のまま。

- 商品
- 在庫（絶対値）
- 在庫（差分値）
- 翻訳
- 注文
- 出荷指示
- ディスカウント
- マーケット
- リカバリー操作

このため、SQからShopifyへ商品・在庫・価格などを送る成功テストはまだ実施不可。今回の検証はShopifyからSQへ流入する注文・出荷・返金・返品の受信挙動を対象にした。

## 注文 `#1008`

### Shopify側操作

1. Shopify下書き注文 `#D18` を作成。
2. 商品 `Selling Plans Ski Wax` を1点追加。
3. `支払い済みとしてマーク` > `注文を作成` を実行し、Shopify注文 `#1008` を作成。
4. `発送済みとしてマークする` > `発送済みにする` を実行し、Shopify側で `#1008-F1` を作成。
5. 返金画面で数量1、返金額 `$24.95` にして全額返金。

### SQ側結果

- 注文一覧 `/admin/orders?query=%231008` に即時表示された。
- SQ注文詳細ID: `3a188fcd-aaaf-5dd9-a702-9652dd2840b0_Order`
- 注文詳細には `外部システムから連携されている注文です` と表示。
- 商品行は `新規` 表示で、SQ既存SKUには紐づかなかった。
- Shopify側で発送済み後も、SQ注文は `注意 未完了 / 未出荷` のまま。
- 注文内 `出荷指示` タブは `アイテムが見つかりませんでした`。
- 注文内 `在庫依頼` タブも `アイテムが見つかりませんでした`。
- 売上実績 `/admin/sale_change_line_items?query=%231008` には `#1008` 由来の行は表示されなかった。
- Shopify側の全額返金後、SQ注文詳細の決済状況は `返金済み` に更新された。
- 注文情報内の決済履歴に、`2026年07月02日 00:28 / manual / 返金 / 成功 / $24.95` が追加された。
- 一方、注文内 `返金履歴` タブは `返金はありません` のままだった。

## 注文 `#1009`

### Shopify側操作

1. Shopify下書き注文 `#D19` を作成。
2. 商品 `Selling Plans Ski Wax` を1点追加。
3. `支払い済みとしてマーク` > `注文を作成` を実行し、Shopify注文 `#1009` を作成。
4. `発送済みとしてマークする` > `発送済みにする` を実行し、Shopify側で `#1009-F1` を作成。
5. `/orders/6798042071276/return` で返品数量1を指定。
6. 返品の配送オプションは、初期選択の `返品用ラベルをアップロードする` のままだと `追跡情報を入力してください` で止まる。
7. `配送不要` を選択して `返品を作成` を実行し、Shopify返品 `#1009-R1` を作成。

### SQ側結果

- 注文一覧 `/admin/orders?query=%231009` に表示された。
- SQ注文詳細ID: `a606f8d8-221b-5bb0-bed1-60835ebf4723_Order`
- 商品行は `新規` 表示で、SQ既存SKUには紐づかなかった。
- Shopify側で発送済み・返品処理中になっても、SQ注文は `注意 未完了 / 未出荷` のまま。
- SQ注文詳細にShopify側の `返品処理中` / `#1009-R1` は表示されなかった。
- SQ返品一覧 `/admin/order_returns?query=%231009` は `アイテムが見つかりませんでした`。

## 結論

- アクセストークン9種が未設定でも、Shopify注文はSQ注文一覧へ流入する。
- 未連携Shopify商品はSQ注文詳細で `新規` として取り込まれ、SQ既存SKUには紐づかない。
- 未連携商品条件では、Shopify側で発送済みにしてもSQ側の出荷状況・出荷指示は変わらない。
- 未連携商品条件では、Shopify側で返品を作成してもSQ返品一覧・注文詳細には反映されない。
- Shopify返金はSQ注文の決済状況と注文情報内の決済履歴には反映される。ただし、注文内の `返金履歴` タブには表示されなかった。
- 未連携商品条件では、Shopify注文は売上実績一覧には表示されない。
- 正式連携SKUでの注文在庫引当、出荷指示生成、返品反映、売上実績化は引き続き、SQ側の `商品` / `在庫` などのアクセストークン設定後に再検証が必要。
